01

什么是 vector？

从固定数组的困境，到会"呼吸"的动态容器

先说困境

假设你在写一道题：读取 n 个数字，但 n 是运行时才知道的。

用普通数组，你得猜一个上限：

int a[100005]; // 猜上限——太小会越界，太大浪费内存
⚠️
痛点

猜小了，越界崩溃；猜大了，占十几倍内存。而且你根本不该猜——n 是已知的！

vector 就是来解决这个问题的。它的大小可以运行时决定，随时自动增长。

核心比喻

vector 就像一个自动伸缩的书架——放一本书，多出一格；书全取走，自动收缩。永远不需要提前预订空间。

最常见的写法

这是 vector.cpp 里读取 n 个数并存储的核心片段：

CODE — vector.cpp 
int n;
cin >> n;

vector<int> a(n);
for (int i = 0; i < n; i++) {
    cin >> a[i];
}
PLAIN CHINESE

声明整数变量 n

从输入读取 n 的值（比如 5）

创建能存 n 个整数的 vector，大小此刻确定

循环 n 次，i 从 0 到 n-1

每次把一个数字读进第 i 个格子

关键点：vector<int> a(n) 里的 n 可以是运行时变量，普通数组做不到这一点。

push_back、pop_back、front、back——操作 vector 的四把瑞士军刀

想象一叠扑克牌

👁️

看第一张的值，但不拿走它

🔍
back()

看最后一张的值，但不拿走它

03

尺寸管理

size、empty、resize、clear——掌控 vector 的"体积"

像管理仓库一样管理 vector

把 vector 想成一个仓库：你需要知道里面有多少货、判断是否为空、扩容或清仓。

📦
size()

返回当前元素数量——仓库里有几箱货？

🔍
empty()

仓库空了吗？返回 true / false

📐
resize(k)

把格子数调整到 k，新格子自动填 0

🗑️
clear()

清空所有货物，size 变为 0

代码 ↔ 中文对照

CODE — vector2.cpp 
cout << v.size() << "\n";

cout << v.empty() << "\n";

cout << !v.empty() << "\n";

v.resize(10);

cout << v.size() << "\n";

v.clear();
PLAIN CHINESE

打印元素数量：当前是 3，输出 3

判断是否为空：有元素，输出 0（false）

取反判断"不为空"：输出 1（true）——更常用的写法

把大小调整为 10，多出的 7 个位置自动补 0

验证：现在 size() 返回 10

清空所有元素，size 变为 0

💡
!v.empty() 比 v.size() > 0 更 C++ 风格

两者等价，但 !v.empty() 更语义化，也是多数代码审查推荐的写法。

resize 的三种情况

①
扩大（新 > 旧）

多出的位置自动用 0 填充。{1,2,3} resize(6) → {1,2,3,0,0,0}

②
缩小（新 < 旧）

超出部分被丢弃。{1,2,3,4,5} resize(3) → {1,2,3}

③
不变（新 = 旧）

什么都不做，此操作是幂等的

CODE — vector.cpp（完整） 
#include <bits/stdc++.h>
using namespace std;

typedef long long ll;
const int MOD = 1e9 + 7;
int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    vector<int> a(n);
    for (int i = 0; i < n; i++) {
        cin >> a[i];
    }
    sort(a.begin(), a.end());

    for (int x : a) {
        cout << x << " ";
    }
    cout << "\n";
    return 0;
}

sort 为什么要 begin() 和 end()？

sort 需要知道"从哪里排到哪里"。vector 提供了两个迭代器：

▶
a.begin()

指向第一个元素，就像"从第 0 格开始"

⏹
a.end()

指向最后元素之后的位置（哨兵），意思是"排到这里为止"

💡
左闭右开 [begin, end)

C++ 的惯例：区间包含 begin，但不包含 end。这样处理空 vector 时（begin == end）逻辑自然一致，不需要特殊判断。

🗣️
给 AI 的精确指令

想让 AI 对 vector 排序，直接说："用 sort(a.begin(), a.end()) 对 a 升序排序"——AI 会准确理解并生成正确代码。

⚠️
竞赛第一杀手：未检查的越界访问

访问 v[i] 时，若 i ≥ v.size() 或 v 为空，就是运行时错误（RE）。编译器不会警告，只会在评测时爆炸。
