# 先讲"内存"是什么

你可以把电脑内存想象成一排格子，每个格子有一个 **地址** （就是一个数字），每个格子里存一个值。每个程序运行时都会从操作系统那里拿到一块内存区域，你声明的每个变量都会占用其中的若干格子。

```
地址:  1000  1001  1002  1003  1004
值:    [10]  [20]  [30]  [40]  [50]
```

当你写 `int a = 10;` 时，系统会：

1. 找一个空格子（比如地址1000）
2. 把 `10` 存进去
3. 以后你用 `a` 这个名字，系统就去地址1000取值

这就是为什么 C++ 变量必须声明类型——系统需要知道 `int` 占几个格子（4字节）、`double` 占几个格子（8字节），才能给你划定地盘。

# 什么是指针？

**指针就是一个变量，但它存的不是普通的值，而是另一个变量的地址。**

```cpp
int a = 10;    // a 存的是值 10，假设地址是 1000
int *p = &a;   // p 存的是 a 的地址 1000
```

图示：

```
变量 a：  地址1000 → [10]
变量 p：  地址2000 → [1000]  ← p 指向 a 的地址
                      ↑
                    p 里存的是 1000，即 a 的地址
```

两个符号要区分清楚：

| 符号       | 位置       | 含义                            |
| ---------- | ---------- | ------------------------------- |
| `&a`     | 在变量前面 | 取 a 的**地址**           |
| `int *p` | 在类型后面 | 声明 p 是一个**指针**变量 |
| `*p`     | 在指针前面 | 取指针指向地址的**值**    |

```cpp
int a = 10;
int *p = &a;   // p 存的是 a 的地址

cout << a;     // 输出：10（a 的值）
cout << &a;    // 输出：1000（a 的地址，实际是个十六进制数）
cout << p;     // 输出：1000（p 存的就是 a 的地址）
cout << *p;    // 输出：10（去 p 存的地址那里取值）
```

*就像指针和普通变量的区别，就像一本书和一张写了图书馆书架位置的纸条的区别——纸条本身不是书，但通过纸条上的地址，你可以找到那本书。*

> 💡 **老师提醒：** `&` 和 `*` 在不同位置意思完全不同：声明时 `int *p` 里的 `*` 是"声明这是个指针"，使用时 `*p` 里的 `*` 是"解引用，取地址里的值"。初学者最容易在这里搞混，多看几遍对照表就清楚了。

# 引用 vs 指针

你之前学了引用 `&`，它和指针很像但更简单：

```cpp
int a = 10;

int &ref = a;   // ref 是 a 的引用（别名）
int *p = &a;    // p 是指向 a 的指针

ref = 20;   // 直接改，a 变成 20
*p = 30;    // 解引用再改，a 变成 30
```

**直觉区别：**

* **引用** ：就是给变量起了个外号，用起来和原变量完全一样，不需要任何额外符号
* **指针** ：存的是地址，用的时候要先用 `*` "解引用"才能拿到值

*就像引用和指针的区别，就像给朋友起了个外号，和记下他家门牌号的区别——外号叫他就能直接响应，但门牌号你还得先上门才能找到人。*

> **竞赛建议：** 初学阶段优先用引用，指针在竞赛中用得相对少（树和链表的手写实现会用到）。

> ✅ 引用和指针搞清楚了，C++ 里很多"奇怪的行为"你都能解释了，这是迈向高手的一步！

# 然后回到 `const char*`

```cpp
auto s = "hello";   // s 的类型是 const char*
```

这里要先理解C++字符串的历史包袱：

C++有**两种**字符串：

| 类型            | 写法        | 来源                  |
| --------------- | ----------- | --------------------- |
| `const char*` | C风格字符串 | 从C语言继承来的，古老 |
| `string`      | C++字符串   | C++自己的，现代，好用 |

`"hello"` 这个字面量，C++默认把它当成 `const char*`，意思是：

```
const  →  不能修改
char   →  字符类型
*      →  指针（指向字符数组的第一个字符）
```

内存里实际存的是：

```
地址: 2000  2001  2002  2003  2004  2005
值:  [ h ]  [ e ]  [ l ]  [ l ]  [ o ]  [\0]
                                          ↑
                                     结束标志
```

`const char*` 就是指向这个数组第一个字符的指针。C 语言没有"字符串类型"，靠这个约定凑出来的——这是历史遗留，不是什么好设计。

> 💡 **老师提醒：** `auto s = "hello"` 推断出来的类型是 `const char*` 而不是 `string`，这是 `auto` 一个常见的"坑"。如果你想要 `string`，就明确写 `string s = "hello"`，别偷懒用 `auto` 写字符串字面量。

---

# 竞赛中你只需要记住一件事

**永远用 `string`，不要用 `const char*`**

```cpp
// ❌ 不要这样
auto s = "hello";        // 类型是 const char*，功能残缺

// ✅ 要这样
string s = "hello";      // 类型是 string，功能完整
// 或者
string s2 = {"hello"};
```

`string` 可以拼接、比较、取长度，`const char*` 这些都很麻烦。

```cpp
string a = "hello";
string b = "world";

string c = a + " " + b;  // 拼接："hello world"
a.size();                  // 长度：5
a[0];                      // 取字符：'h'
a == b;                    // 比较：false
```

*就像 `const char*` 和 `string` 的区别，就像用一根绳子串起来的珠子和装在盒子里的珠子的区别——绳子串的你能数、能看，但很难加或者减；盒子装的什么操作都方便。*

# 总结

```
指针（*）：存地址的变量，要用 * 解引用才能取值
引用（&）：变量的别名，用起来和原变量一样

const char*：C语言遗留的字符串，避免使用
string：    C++现代字符串，竞赛中统一用这个

实际竞赛优先级：
  引用 >> 指针（初学阶段基本用引用就够了）
```

指针后面在讲树、链表的时候还会遇到，到时候结合具体场景再深入。现在知道它是"存地址的变量"就够了。
