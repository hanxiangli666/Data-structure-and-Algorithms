# 根本原因：全局区域只能放"定义"，不能放"执行"

C++程序分两种东西：

 **定义** ：告诉编译器"有这个东西存在"

```cpp
int a = 10;              // 定义变量
const int MOD = 1e9 + 7; // 定义常量
long long power(...) {}  // 定义函数
struct Student {}        // 定义结构体
```

 **执行** ：真正运行的操作

```cpp
cin >> a;         // 读取输入
cout << a;        // 打印输出
a = a + 1;        // 计算
sort(...);        // 调用函数
```

---

**全局区域（main外面）只能放定义，不能放执行。**

```cpp
int a = 10;          // ✅ 定义变量，可以在全局
cin >> a;            // ❌ 执行操作，不能在全局

long long power(){}  // ✅ 定义函数，可以在全局
sort(v.begin()...)   // ❌ 执行操作，不能在全局
```

---

# 为什么这样设计？

因为 **程序需要一个明确的起点** 。

C++规定：程序从 `main` 函数的第一行开始执行。全局区域的代码在程序启动之前就要处理，那时候很多东西还没准备好（比如标准输入输出还没初始化），所以不允许放执行语句。

类比理解：

```
全局区域 = 舞台布景（开演前搭好的东西）
main函数 = 正式演出（演员开始行动）

你不能在布景阶段让演员开始表演
```

# 什么可以放在全局？

```cpp
// ✅ 这些可以放全局
const int MOD = 1e9 + 7;    // 常量
int n;                        // 全局变量（竞赛中有时用）
vector<int> adj[100005];      // 全局数组（竞赛中很常见）

struct Edge { int u, v, w; }; // 结构体定义
long long power(long long a, long long b) {} // 函数定义
```

```cpp
// ❌ 这些不能放全局
cin >> n;           // 执行语句
cout << "hello";    // 执行语句
sort(v.begin()...)  // 执行语句
```

---

# 你一直犯这个错误的原因

你之前写Python，Python没有这个概念：

```python
# Python：这样完全合法
a = int(input())  # 可以在任何地方执行
print(a)
```

Python从第一行开始顺序执行，没有"全局区域"和"函数区域"的区别。

C++不一样，必须有明确的入口点 `main`，所有执行逻辑都从这里开始。

---

# 记住这个模板，永远不会错

```cpp
// 全局区域：只放定义
#include <iostream>
using namespace std;

const int MOD = 1e9 + 7;  // 常量定义 ✅
int n;                      // 变量定义 ✅

long long power(...) {      // 函数定义 ✅
    // 函数里面可以有执行语句
}

// main：所有执行逻辑从这里开始
int main() {
    cin >> n;    // ✅ 执行语句在main里
    // ...
    return 0;
}
```
