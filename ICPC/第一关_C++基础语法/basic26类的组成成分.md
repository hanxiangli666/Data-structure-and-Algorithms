# 先回答"这些是什么的组成成分"

它们都是 **class（类）** 的组成部分。你可以把一个 class 想象成一张"设计图纸"，图纸上标注了这个东西有哪些零件（成员变量）、能做哪些动作（成员函数）、出厂时怎么初始化（构造函数）、报废时怎么清理（析构函数）。每一个从这张图纸造出来的实物，就是一个"对象"。

```cpp
class MyArrayList {
//  ↑ 一个类，里面包含：

    E *data;        // 成员变量
    int size;       // 成员变量
    int cap;        // 成员变量

    MyArrayList() {}   // 构造函数
    ~MyArrayList() {}  // 析构函数
    void addLast() {}  // 成员函数
    void resize() {}   // 成员函数
};
```

# 逐个解释

## 成员变量：这个对象"有什么"

```cpp
class Student {
    string name;   // 成员变量：学生有名字
    int age;       // 成员变量：学生有年龄
    double score;  // 成员变量：学生有分数
};
```

就是这个对象身上携带的数据。*就像一张身份证上印着姓名、年龄、照片一样，成员变量就是这个对象随身携带的"个人信息"。* 每个对象都有自己独立的一份成员变量，互不干扰——你改了 `s1.age`，`s2.age` 不会跟着变。这也是面向对象编程最核心的思想：数据和行为绑在一起，各管各的。

> 💡 **老师提醒：** class 里的成员变量默认是 `private` 的，外部访问不了。如果你发现"明明有这个变量但编译报错访问不了"，十有八九是忘了写 `public:`。struct 则默认 `public`，这也是竞赛中大家更爱用 struct 的原因之一。
>
> ✅ 能理解"成员变量 = 对象身上的数据"，你就抓住了类最基本的一块拼图。

---

## 成员函数：这个对象"能做什么"

```cpp
class Student {
    string name;
    int age;

    void introduce() {           // 成员函数：学生能自我介绍
        cout << name << endl;    // 成员函数里可以直接用成员变量
    }

    void birthday() {            // 成员函数：学生能过生日
        age++;
    }
};
```

成员函数定义了对象的行为能力。为什么成员函数里可以直接用 `name`、`age` 而不需要传参？因为成员函数"天生知道"自己属于哪个对象，编译器背后其实偷偷传了一个 `this` 指针，指向调用它的那个对象。*就像你自己介绍自己的时候，不需要别人先告诉你"你叫什么"——你本来就知道。*

> 💡 **老师提醒：** 成员函数和普通函数最大的区别就是：成员函数能直接访问成员变量。如果你写了一个普通函数想访问某个对象的数据，你必须把对象传进去；但成员函数不用，因为它已经绑在对象身上了。

> ✅ 成员变量是"有什么"，成员函数是"能做什么"，这两块合在一起就构成了一个完整的对象。

## 构造函数：对象"出生时"自动执行

```cpp
class Student {
    string name;
    int age;

    Student(string n, int a) {  // 构造函数：名字和类名一样，没有返回类型
        name = n;
        age = a;
    }
};

int main() {
    Student s("Alice", 20);  // 创建对象时，构造函数自动执行
}
```

**什么时候触发：** 创建对象的那一刻，自动执行，不需要手动调用。为什么需要构造函数？因为如果成员变量不初始化就使用，它们的值是随机垃圾值，程序行为完全不可预测。构造函数就是确保对象"出生即可用"的保障机制。*就像新生儿出生时护士会做一系列检查登记——构造函数就是那个"出生登记流程"，保证一切就绪。*

> 💡 **老师提醒：** 如果你没写任何构造函数，编译器会自动生成一个"啥也不干"的默认构造函数。但是！一旦你自己写了一个带参数的构造函数，编译器就不再生成默认的了。这时候 `Student s;` 这种无参创建会报错，因为没有匹配的构造函数。这是初学者非常容易踩的坑。

> ✅ 记住构造函数的两个特征——和类同名、没有返回类型——你就不会认错它。

## 析构函数：对象"死亡时"自动执行

```cpp
class Student {
    int *data;

    Student() {
        data = new int[100];  // 构造时申请内存
    }

    ~Student() {              // 析构函数：名字是~加类名
        delete[] data;        // 析构时释放内存
    }
};

int main() {
    Student s;
    // ...
}   // ← s在这里消亡，~Student()自动执行
```

**什么时候触发：** 对象消亡时（函数结束、程序退出），自动执行。你不需要手动调用它，C++ 的生命周期机制会在对象离开作用域时自动帮你调用。这就是 C++ 独特的 RAII（资源获取即初始化）思想：构造时拿资源，析构时还资源，一进一出，配对出现。

**主要用途：** 释放构造函数里申请的内存，防止内存泄漏。*就像退房时要归还房卡、检查有没有落东西一样，析构函数就是对象的"退房手续"。*

> 💡 **老师提醒：** 析构函数只能有一个，不能重载，也不能带参数。如果你在构造函数里用了 `new`，析构函数里就一定要有对应的 `delete`，否则每创建一个对象就泄漏一块内存。虽然竞赛里不太在意，但工程中这是致命问题。

> ✅ 构造和析构是一对CP——构造负责"开场准备"，析构负责"散场清理"。理解了这对组合，你对对象生命周期的认识就到位了。

# 用一个完整例子串起来

```cpp
class Student {
private:
    string name;    // 成员变量
    int age;        // 成员变量

public:
    // 构造函数：创建对象时执行
    Student(string n, int a) {
        name = n;
        age = a;
        cout << name << "出生了" << endl;
    }

    // 成员函数：对象能做的事
    void introduce() {
        cout << "我是" << name << "，" << age << "岁" << endl;
    }

    // 析构函数：对象消亡时执行
    ~Student() {
        cout << name << "消亡了" << endl;
    }
};

int main() {
    Student s("Alice", 20);  // 构造函数执行：输出"Alice出生了"
    s.introduce();            // 成员函数：输出"我是Alice，20岁"
}   // 析构函数执行：输出"Alice消亡了"
```

输出：

```
Alice出生了
我是Alice，20岁
Alice消亡了
```

仔细看这个输出顺序：先构造、再使用、最后析构。这就是一个对象完整的"一生"。你以后看到任何类，都可以用这个视角去理解它——它出生时做了什么准备？活着的时候能干什么？死的时候需要清理什么？

> 💡 **老师提醒：** 注意这里 `private:` 和 `public:` 的用法。成员变量放在 `private` 里是为了不让外部直接乱改，成员函数放在 `public` 里是为了让外部能调用。这叫"封装"。你现在不需要深究，但看到别人代码里有这两个关键字时要知道它们在干什么。

# 什么时候用这些东西？

| 东西     | 用途           | 什么时候需要      |
| -------- | -------------- | ----------------- |
| 成员变量 | 存对象的数据   | 每次定义类都有    |
| 成员函数 | 定义对象的行为 | 每次定义类都有    |
| 构造函数 | 初始化成员变量 | 每次定义类都有    |
| 析构函数 | 释放申请的内存 | 只有用了new才需要 |

# 竞赛中的实际情况

说实话，竞赛中 **很少自己写class** 。

```
竞赛中实际用到这些知识的场景：

✅ struct：存坐标、边、学生信息等（高频）
✅ 成员变量：struct里的各个字段（高频）
⚠️ class：偶尔手写数据结构时用
⚠️ 构造函数：手写数据结构时用
❌ 析构函数：竞赛中基本不写，内存泄漏无所谓
```

竞赛不是工程项目，程序跑完就结束了，内存泄漏不影响结果，所以析构函数基本可以忽略。但你要知道为什么工程中必须写——因为服务器程序可能要跑几个月甚至几年，每次泄漏一点点，积少成多就会把内存吃光崩掉。*就像竞赛是一次性筷子用完就扔，工程是自己家的碗筷，不洗迟早堆满水槽。*

**你现在最需要掌握的是 `struct`，class这些看懂别人代码就行。**
