# break 和 continue

两个都是用来**控制循环流程**的，但效果不同。`break` 是"我要停止这个循环"，`continue` 是"这一次迭代我跳过，继续下一次"。区分清楚这两个，你的循环控制就不会出问题。

---

## break：直接跳出整个循环

```cpp
for (int i = 0; i < 5; i++) {
    if (i == 3) break;  // i等于3时，直接结束循环
    cout << i << " ";
}
// 输出：0 1 2
```

图示：

```
i=0 → 打印0
i=1 → 打印1
i=2 → 打印2
i=3 → 遇到break，整个循环结束
i=4 → 永远不会执行
```

> 💡 **老师提醒：** 一旦执行 `break`，循环就完全结束了，剩下的循环体不会执行，循环次数也不会继续增长。有些初学者会以为 `break` 只是"跳过这一次"，那是 `continue` 的作用。

---

## continue：跳过本次，继续下一次

```cpp
for (int i = 0; i < 5; i++) {
    if (i == 3) continue;  // i等于3时，跳过这次，继续下一次
    cout << i << " ";
}
// 输出：0 1 2 4
```

图示：

```
i=0 → 打印0
i=1 → 打印1
i=2 → 打印2
i=3 → 遇到continue，跳过cout，直接去i=4
i=4 → 打印4
```

> 💡 **老师提醒：** `continue` 不会结束循环，它会直接跳到下一次迭代——`i` 照样会自增，循环照样继续。如果你忘了这个，写出来的代码逻辑会很奇怪。

---

## 一句话区别

```
break    = 我不玩了，整个循环结束
continue = 这把不算，直接下一把
```

> ✅ 这个贴切的比喻一旦理解了，你就永远不会搞混！

---

## while循环里也一样

```cpp
int i = 0;
while (i < 10) {
    i++;
    if (i % 2 == 0) continue;  // 偶数跳过
    if (i > 7) break;           // 大于7结束
    cout << i << " ";
}
// 输出：1 3 5 7
```

**不管是 for 还是 while，`break` 和 `continue` 的行为完全一样**——就是循环的形式不同而已。

---

## 嵌套循环里的break/continue

⚠️ 注意： **break和continue只影响最近的那一层循环** 。如果你在内层循环写 `break`，只会跳出内层，外层循环照样继续。

```cpp
for (int i = 0; i < 3; i++) {
    for (int j = 0; j < 3; j++) {
        if (j == 1) break;  // 只跳出内层循环，外层继续
        cout << i << " " << j << endl;
    }
}
// 输出：
// 0 0
// 1 0
// 2 0
```

> 💡 **老师提醒：** 看这个输出就明白了——当 `j==1` 时，内层 break 了。同一个 `i`, 下一次外层循环继续。很多初学者期望 `break` 能直接跳出所有循环，结果被坑了。

如果你想直接跳出外层循环，竞赛中常用一个flag：

```cpp
bool found = false;
for (int i = 0; i < 3; i++) {
    for (int j = 0; j < 3; j++) {
        if (i == 1 && j == 1) {
            found = true;
            break;  // 跳出内层
        }
    }
    if (found) break;  // 跳出外层
}
```

**这个 flag 的模式很常见**——当内层发现满足条件，就设置 flag，然后内外层都检查这个 flag 来 break。这样就能突破"只影响最近层"的限制了。

---

## 竞赛中最常见的用法

```cpp
// 1. 找到目标就停止
for (int i = 0; i < n; i++) {
    if (a[i] == target) {
        cout << i << endl;
        break;  // 找到了就不用继续找
    }
}

// 2. 跳过不需要处理的情况
for (int i = 0; i < n; i++) {
    if (a[i] < 0) continue;  // 负数跳过
    // 处理非负数
    cout << a[i] << endl;
}
```

**这两个模式你会经常在竞赛代码里看到。** 第一个是"找到就提前退出"的优化，第二个是"筛选有效的数据再处理"的通用做法。

> ✅ Break 和 continue 虽然简单，但是用好了能让你的代码更高效、更清晰！
