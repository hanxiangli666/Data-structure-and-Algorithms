思路完全正确，快速幂函数写得很好！就两个问题：

---

**问题一：`cin >> a >> b` 又跑到main外面了**

这个问题出现四五次了，以后我不会再提醒你了，你自己要养成习惯。每次写题，第一步就是打好框架，让所有执行代码的"家"（main函数）存在，再往里填内容。

**问题二：输出没有用读入的a和b**

```cpp
cout << power(2, 11, mod) << endl;  // ❌ 把输入写死了
cout << power(a, b, mod) << endl;   // ✅ 用读进来的变量
```

把输入值写死是一个容易犯的习惯——调试时你可能先固定参数测试，但提交时忘了改成读变量，结果只有你的测试用例能过，其他用例全错。*就像写死参数和用变量的区别，就像把收件地址写死为自己家，和根据信封上的地址投递的区别——写死的只能送到一个地方，用变量才能送到任何地方。*

> 💡 **老师提醒：** 写完代码后，快速检查输出语句：用的是固定数字，还是用了 `cin` 读进来的变量？这个习惯能避免"本地跑通但提交全错"的尴尬。

# 修正后完整代码

```cpp
#include <iostream>
using namespace std;

const long long MOD = 1e9 + 7;

long long power(long long a, long long b, long long mod) {
    long long result = 1;
    a %= mod;
    while (b > 0) {
        if (b & 1) {
            result = result * a % mod;
        }
        a = a * a % mod;
        b >>= 1;
    }
    return result;
}

int main() {
    long long a, b;
    cin >> a >> b;
    cout << power(a, b, MOD) << endl;
    return 0;
}
```
