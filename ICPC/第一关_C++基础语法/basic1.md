# 第一块：基础语法（ctrl.cpp + std.cpp）

## 1. 程序的基本结构

```cpp
#include <iostream>
using namespace std;

int main() {
    // 你的代码写在这里
    return 0;
}
```

逐行解释：

* `#include <iostream>` — 引入"输入输出"工具包，这样才能用 `cout`（打印）和 `cin`（读取输入）。你可以把它理解成 Python 里的 `import`，只不过 C++ 不导入就完全用不了这些工具，编译时直接报错，不会像 Python 那样运行到那行才报错。
* `using namespace std;` — 告诉编译器我们用标准库里的东西，否则每次都要写 `std::cout`，很麻烦。*就像"北京市海淀区中关村大街"和"中关村大街"的区别——加了 `using namespace std;` 就是告诉系统默认在"std"这个命名空间里找，省去了每次写前缀的麻烦。*
* `int main()` — 程序的入口，C++程序从这里开始执行。不管你写了多少函数、多少类，操作系统拉起程序时只认 `main` 这一个入口，其他函数都是被 `main` 直接或间接调用的。
* `return 0;` — 告诉系统"程序正常结束"。这个数字会被操作系统接收，0 表示成功，非 0 表示出了某种错误，在竞赛评测系统里这个很重要。

> 💡 **老师提醒：** `main` 函数的返回类型是 `int`，所以最好显式写 `return 0;`。虽然现代编译器会自动补上，但养成写的习惯，代码更规范，换了编译器也不会有意外。

---

## 2. 输出：cout

```cpp
int a = 10;
cout << a << endl;          // 输出：10
cout << "Hello, World!" << endl;  // 输出：Hello, World!
```

* `cout` 就是"打印到屏幕"。`cout` 是 "character output" 的缩写，专门负责把内容输出到终端。
* `<<` 是"往输出流里塞东西"，可以串联多个。你可以把 `<<` 想象成一条传送带，把内容一件一件按顺序送到屏幕上，最先塞进去的最先出来。
* `endl` 是换行（相当于Python的 `\n`）。不过 `endl` 比 `\n` 多做一件事：它会强制刷新输出缓冲区，在输出量很大的竞赛题里，频繁用 `endl` 会拖慢程序速度。

> 💡 **老师提醒：** 竞赛中大量输出时，用 `"\n"` 代替 `endl` 会快不少，因为 `endl` 每次都刷新缓冲区，有额外开销。记住这个习惯，遇到输出密集的题目直接换成 `"\n"`。

> ✅ cout 用熟了，写代码调试就方便多了，先把这个练顺再说！

---

## 3. 变量和基本类型

```cpp
int a = 10;       // 整数
double b = 3.14;  // 浮点数
bool c = true;    // 布尔值（true/false）
string s = "abc"; // 字符串（需要 #include <string>）
```

跟Python相比，C++必须 **提前声明类型** ，之后不能改变。这不是 C++ 故意麻烦你——这是因为 C++ 需要在编译时确定每个变量占多少内存：`int` 占4字节，`double` 占8字节，知道了类型才能精确分配空间，运行时速度也更快。*就像订酒店和住民宿的区别：民宿可以随到随住随换房型（Python 动态类型），但酒店预定了就是那个房间大小，不能临时变（C++ 静态类型）。*

> 💡 **老师提醒：** 用 `string` 时别忘了 `#include <string>`，虽然很多编译器会间接包含它，但正式写法还是要显式引入，换了编译器环境也不会出问题。

---

## 4. 条件判断（if/else if/else）

```cpp
int a = 10;

if (a > 5) {
    cout << "a > 5" << endl;
} else if (a == 5) {
    cout << "a == 5" << endl;
} else {
    cout << "a < 5" << endl;
}
```

和Python几乎一样，区别是：

* 条件要加括号 `(a > 5)`
* 代码块用花括号 `{}` 而不是缩进

为什么 C++ 要用花括号？因为 C++ 不依赖缩进来判断代码块的范围，缩进只是给人看的，对编译器没有任何意义。所以必须用花括号明确告诉编译器"这段代码是一个整体"。

> 💡 **老师提醒：** 花括号里只有一句话时，理论上可以省略花括号——但强烈建议不要省。省了之后加代码很容易出逻辑 bug，这是 C++ 程序员踩了几十年的老坑，不要去踩它。

## 5. 循环

**for循环：**

```cpp
for (int i = 0; i < 5; i++) {
    cout << i << " ";
}
// 输出：0 1 2 3 4
```

for循环括号里分三段，用分号隔开：

```
for (初始化 ; 条件 ; 每次循环后做什么) {}
```

* `int i = 0` — 从0开始
* `i < 5` — 只要i小于5就继续
* `i++` — 每次循环i加1（等价于Python的 `i += 1`）

这三段的逻辑一目了然，一眼就能看出循环的起点、终点和步长，这是 for 循环比 while 更适合"固定次数循环"的原因。

**while循环：**

```cpp
int num = 100;
while (num > 0) {
    cout << num << " ";
    num /= 2;
}
// 输出：100 50 25 12 6 3 1
```

只要括号里条件为真就一直循环，和Python完全一样。while 更适合"不知道要跑多少次，只知道停止条件"的场景，比如读到某个特定值为止，或者处理到某个状态满足为止。

> ✅ 基础语法这关你已经过了，接下来才是C++真正有意思的地方！

# 第二块：函数传值 vs 传引用

这是C++和Python差异最大的地方之一，非常重要。

## 先理解"传值"

```cpp
void modifyValue(int x) {
    x = 10;  // 只是修改了副本
}

int main() {
    int num = 5;
    modifyValue(num);
    cout << num << endl;  // 输出：5，原来的num没变！
}
```

 **直觉理解** ：传值就像你把一张纸上的数字**抄了一份**给函数。函数改的是那份抄本，你原来那张纸不变。即使函数里把 `x` 改得天翻地覆，调用结束后也不会影响外面的 `num`，因为两个变量根本不是同一块内存。

---

## 再理解"传引用"

```cpp
void modifyReference(int &x) {  // 注意：int后面有个 &
    x = 10;  // 修改的是原始数据
}

int main() {
    int num = 5;
    modifyReference(num);
    cout << num << endl;  // 输出：10，原来的num变了！
}
```

 **直觉理解** ：传引用就像你把那张纸本身交给了函数（或者给它一把能直接操作你那张纸的钥匙）。函数改的就是原始数据。

`&` 符号的意思就是"引用"，即"不要复制，直接操作原来那个变量"。

*就像传值和传引用的区别，就像给朋友发你照片的复印件，和把原版照片直接寄给他的区别——复印件怎么改都不影响原版，但原版寄出去就直接被改了。*

> 💡 **老师提醒：** 参数里写 `int &x` 和 `int& x` 效果完全一样，只是风格不同。竞赛代码里两种写法都会看到，看到了不要被搞混。

**竞赛中的重要规则（你的文件里也提到了）：**

| 情况                                | 推荐方式                                                |
| ----------------------------------- | ------------------------------------------------------- |
| `int`、`bool`等基本类型         | 传值（开销小）                                          |
| `vector`、`unordered_map`等容器 | **传引用** （避免复制大量数据）                   |
| 递归函数里有容器                    | **必须传引用** ，否则每层递归都复制一份，直接超时 |

> ✅ 传值和传引用搞清楚了，你已经比大多数初学者快了一大步！

# 第三块：STL容器

STL（Standard Template Library）是C++自带的工具箱，装了各种常用数据结构。你的文件里涉及了5种，我逐个讲。

---

## 1. vector（动态数组）← 最常用，优先掌握

 **可以把它理解为Python里的 `list`** ，但功能更多、更快。之所以叫"动态"数组，是因为它会自动扩容——不像 C 语言的原始数组大小固定死了，vector 满了会自动申请更大的空间并把数据搬过去，整个过程对你完全透明。

```cpp
#include <vector>
using namespace std;

vector<int> nums(10);     // 大小为10，全为0
vector<int> nums2{1,3,5}; // 直接初始化
vector<int> nums3(5, 2);  // 大小为5，全为2
```

**常用操作：**

```cpp
vector<int> v{1, 2, 3};

v.push_back(4);    // 尾部加入4 → {1,2,3,4}
v.pop_back();      // 删除尾部  → {1,2,3}
v[0] = 99;         // 直接修改  → {99,2,3}
v.size();          // 长度：3
v.empty();         // 是否为空：false

// 遍历（类似Python的 for x in list）
for (int x : v) {
    cout << x << " ";
}
```

> 💡 **老师提醒：** `v.size()` 返回的是 `size_t` 类型，这是一个**无符号整数**。写 `v.size() - 1` 时如果 v 为空，结果会变成一个巨大的正数而不是 -1，这是竞赛中非常常见的 bug 来源！遇到这种情况先加 `if (!v.empty())` 判断。

**二维数组（矩阵）：**

```cpp
// 创建 3行 4列，全为0的二维数组
vector<vector<int>> grid(3, vector<int>(4, 0));
grid[1][2] = 5;  // 第1行第2列设为5
```

> ✅ vector 是你用得最多的容器，把这部分练熟，后面的题会轻松很多！

---

## 2. stack（栈）

 **先进后出** ，就像一叠盘子，只能从顶部取放。往栈里放元素叫"压栈"，取出元素叫"弹栈"，只有顶部的元素是可见的。

```cpp
#include <stack>

stack<int> s;
s.push(10);   // 放入：{10}
s.push(20);   // 放入：{10, 20}
s.push(30);   // 放入：{10, 20, 30}

s.top();      // 看顶部：30（不删除）
s.pop();      // 删除顶部 → {10, 20}
s.empty();    // 是否为空：false
s.size();     // 大小：2
```

*就像栈和队列的区别，就像洗碗机摞碟子和排队买奶茶的区别——碟子是最后放的最先拿，排队是最先来的最先服务。*

> 💡 **老师提醒：** 调用 `s.top()` 或 `s.pop()` 之前，**一定要先检查 `!s.empty()`**。对空栈操作是未定义行为，程序会直接崩溃，这是竞赛中丢分的常见原因。

---

## 3. queue（队列）

 **先进先出** ，就像排队买奶茶，先来的先服务。元素从尾部进入，从头部取出，两头的接口不一样，这和 stack 不同。

```cpp
#include <queue>

queue<int> q;
q.push(10);  // 入队尾
q.push(20);
q.push(30);

q.front();   // 看队头：10
q.back();    // 看队尾：30
q.pop();     // 删除队头 → {20, 30}
q.size();    // 大小：2
```

> 💡 **老师提醒：** 和 stack 一样，`q.front()` 和 `q.pop()` 之前也要检查 `!q.empty()`。queue 在 BFS（广度优先搜索）里用得极多，到时候你会大量用到这个，现在把接口记熟。

---

## 4. unordered_map（哈希表）

 **可以理解为Python里的 `dict`** ，存键值对，查找是O(1)。之所以叫"unordered"，是因为里面的元素没有固定顺序——它用哈希函数定位元素，所以快，但顺序不可预测，如果你需要按顺序遍历，就得用 `map`（有序，但查找是 O(log n)）。

```cpp
#include <unordered_map>

unordered_map<int, string> mp;
mp[1] = "one";   // 插入
mp[2] = "two";

// 查找前先判断键是否存在！
if (mp.contains(1)) {     // C++20写法
    cout << mp[1] << endl; // 输出：one
}

mp.erase(2);  // 删除键2

// 遍历
for (auto &pair : mp) {
    cout << pair.first << " -> " << pair.second << endl;
}
```

⚠️  **C++特有的坑** （和Python不同）：

```cpp
unordered_map<int, string> mp;
cout << mp[99] << endl;  // 键99不存在，但不会报错！
                          // 而是自动创建了键99，值为""
cout << mp.size() << endl; // 输出：1 （被悄悄插入了）
```

这个坑非常隐蔽。Python 里访问不存在的 key 会直接报 KeyError，你马上知道出错了。但 C++ 的 map 会悄悄插入一个默认值，程序继续跑，但结果已经错了，你还不知道。所以 **访问之前一定要用 `.contains()` 先判断** 。

> 💡 **老师提醒：** `.contains()` 是C++20的写法。如果你的环境是C++17，用 `mp.count(key) > 0` 或者 `mp.find(key) != mp.end()` 来判断。竞赛时注意确认编译器版本。

> ✅ unordered_map 是竞赛中频率极高的容器，把这个坑记住，能省不少调试时间！

---

## 5. unordered_set（哈希集合）

 **可以理解为Python里的 `set`** ，只存值，不存键值对，用于快速判断"某个元素在不在"。查找、插入、删除都是O(1)，比在 vector 里线性搜索快得多，当你需要频繁做"这个数存在吗？"这类查询时，第一个想到它。

```cpp
#include <unordered_set>

unordered_set<int> s{1, 2, 3, 4};

s.insert(5);    // 插入
s.erase(2);     // 删除

if (s.contains(3)) {
    cout << "3在集合里" << endl;
}

s.size();   // 大小
s.empty();  // 是否为空
```

> 💡 **老师提醒：** set 不允许重复元素，`insert` 一个已存在的值不会报错，但也不会有任何效果——这一点和 Python 的 set 行为完全一致，不用担心。
