>
> 提示你：
>
> `n >> 1` 只是计算了 n右移一位的结果，但 **没有把结果存回去** 。
>
> 类比一下：
>
> ```cpp
> int a = 5;
> a + 1;    // ❌ 只是计算了6，但a还是5
> a = a + 1; // ✅ 或者写 a++
> ```
> 所以：
>
> ```cpp
> n >> 1;       // ❌ n没有变化，死循环！
> n = n >> 1;   // ✅
> // 或者简写
> n >>= 1;      // ✅ 和 n = n >> 1 完全等价
> ```

`>>=` 是**复合赋值运算符**，和 `+=`、`-=`、`*=` 是同一类东西——先计算，再把结果存回左边的变量。记住这个规律：位运算的每个运算符都有对应的复合赋值版本（`&=`、`|=`、`^=`、`<<=`、`>>=`），用起来既简洁又不会犯"算了但没存"的错误。

> 💡 **老师提醒：** C++ 里所有二元运算符（`+`、`-`、`*`、`/`、`%`、`&`、`|`、`^`、`<<`、`>>`）都有对应的复合赋值形式（`+=`、`-=` 等），它们的含义都是"先算，再存回左边"。以后凡是写 `x = x op something`，都可以简写成 `x op= something`。

> ✅ 搞清楚"计算"和"赋值"的区别之后，这类死循环 bug 你就不会再犯了！
