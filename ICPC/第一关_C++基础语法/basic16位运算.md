# 第六课：位运算

## 先理解"位"是什么

计算机里所有数字都是用**二进制**存的，比如：

```
十进制  二进制
  1  →  0001
  2  →  0010
  3  →  0011
  4  →  0100
  5  →  0101
```

这不是设计者的偏好，而是硬件的本质——CPU 里的晶体管只有"通"和"断"两种状态，天然对应 1 和 0。所有数字运算在硬件层面都是对这些 0 和 1 的操作。

位运算就是 **直接对二进制的每一位进行操作** ，速度极快。位运算直接映射到 CPU 的单条指令，不需要额外的计算步骤，比普通四则运算要快得多。

---

## 六种位运算

### 1. 按位与 `&`

两个位都是1才是1，否则是0：

```
  5 = 0101
  3 = 0011
  & = 0001 = 1
```

```cpp
cout << (5 & 3) << endl;  // 1
```

**最常用场景：判断奇偶**

```cpp
if (n & 1) {
    cout << "奇数" << endl;
} else {
    cout << "偶数" << endl;
}
// 比 n % 2 更快
```

`n & 1` 之所以能判断奇偶，是因为一个整数的二进制最低位决定了它的奇偶性——最低位是 1 就是奇数，是 0 就是偶数。`n & 1` 只保留最低位，结果要么是 1 要么是 0，比 `n % 2` 少了一次除法运算，速度更快。

> 💡 **老师提醒：** `&` 是位运算，`&&` 是逻辑与（布尔运算），两者完全不同。`5 & 3` 是按位与得到 1，`5 && 3` 是逻辑与得到 `true`（两个非零数逻辑与为真）。写代码时不要混用。

---

### 2. 按位或 `|`

两个位有一个是1就是1：

```
  5 = 0101
  3 = 0011
  | = 0111 = 7
```

```cpp
cout << (5 | 3) << endl;  // 7
```

按位或的用途是"把某一位强制设为 1"——无论那一位原来是 0 还是 1，和 1 做或运算之后都变成 1。这在后面的位操作技巧里会用到。

---

### 3. 按位异或 `^`

两个位**不同**则为1，相同则为0：

```
  5 = 0101
  3 = 0011
  ^ = 0110 = 6
```

```cpp
cout << (5 ^ 3) << endl;  // 6
```

**最常用场景：找出只出现一次的数字**

```cpp
// 数组里只有一个数出现一次，其余都出现两次
// 相同的数异或结果为0，0异或任何数等于那个数
vector<int> v = {1, 2, 3, 2, 1};
int result = 0;
for (int x : v) {
    result ^= x;
}
cout << result << endl;  // 3
```

这是一道经典LeetCode题（136题），记住这个技巧！

> 这段代码利用了位运算中 **异或（XOR，符号为 `^`）** 的几个神奇特性。我们可以把它想象成一种"消消乐"或者"开关游戏"。
>
> 理解这段代码的关键在于掌握异或运算的三个核心性质：
>
> ### 1. 异或的核心性质
>
> * **归零性** ：**$a \oplus a = 0$**（相同的数相抵消，变回 0）。
> * **恒等性** ：**$a \oplus 0 = a$**（任何数和 0 异或，结果还是它自己）。
> * **交换律与结合律** ：**$a \oplus b \oplus a = a \oplus a \oplus b = 0 \oplus b = b$**。这意味着 **运算顺序不重要** ，只要成对出现的数字，最终都会互相抵消。
>
> ---
>
> ### 2. 过程拆解
>
> 拿你给的数组 `{1, 2, 3, 2, 1}` 来演示，程序执行的过程实际上是这样的：
>
> 1. **初始状态** ：`result = 0`
> 2. **遇到 1** ：`0 ^ 1 = 1`
> 3. **遇到 2** ：`1 ^ 2` (此时结果是 3，也就是二进制的 `01 ^ 10 = 11`)
> 4. **遇到 3** ：`3 ^ 3 = 0` (注意！这里的 3 是之前 1 和 2 的结合体，这一步比较抽象)
>
> **但如果我们根据"交换律"重新排列逻辑，就非常直观了：**
>
> $$
> 1 \oplus 2 \oplus 3 \oplus 2 \oplus 1
> $$
>
> $$
> \downarrow
> $$
>
> $$
> (1 \oplus 1) \oplus (2 \oplus 2) \oplus 3
> $$
>
> $$
> \downarrow
> $$
>
> $$
> 0 \oplus 0 \oplus 3 = 3
> $$
>
> 所有的"成对"数字都变成了 0，最后只剩下那个孤独的数字 `3` 和 `0` 做异或，结果依然是 `3`。
>
> ---
>
> ### 3. 为什么这比其他方法好？
>
> 如果你不用异或，你可能需要：
>
> * **先排序** ：时间复杂度 **$O(n \log n)$**。
> * **用哈希表计数** ：需要额外的空间来存储每个数字出现的次数。
>
> **异或法的优势** ：
>
> * **快** ：只需要遍历一遍数组 (**$O(n)$**)。
> * **省** ：不需要额外空间 (**$O(1)$**)，只用一个变量 `result` 就搞定了。
>
>> **一句话总结** ：异或运算就像一个自带记忆的"去重滤镜"，它能自动把所有成对出现的数字过滤掉，最后剩下的就是那个唯一的单身数字。
>>
>
> 你想看看如果数组里有**两个**只出现一次的数字，该怎么用位运算解决吗？

> ✅ 异或消消乐技巧记住了，以后看到"找唯一出现一次的数"直接想到 XOR，O(n) 时间 O(1) 空间，完美！

### 4. 按位取反 `~`

每一位0变1，1变0：

```cpp
cout << (~5) << endl;  // -6（涉及补码，暂时记住结论）
// ~n = -(n+1)
```

竞赛中用得相对少，了解即可。结论 `~n = -(n+1)` 来源于 CPU 用补码表示负数的规则——翻转所有位再加 1 等于取负，所以取反等于取负再减 1。

---

### 5. 左移 `<<`

把二进制整体向左移n位，相当于 **乘以2的n次方** ：

```
1 = 0001
1 << 1 = 0010 = 2   （乘2）
1 << 2 = 0100 = 4   （乘4）
1 << 3 = 1000 = 8   （乘8）
```

```cpp
cout << (1 << 3) << endl;  // 8
cout << (3 << 2) << endl;  // 12（3*4）
```

左移之所以等于乘以 2，是因为二进制进位的规律：把所有位往左推一位，每一位的"权重"（1, 2, 4, 8...）都乘以了 2，整个数自然翻倍。这和十进制末尾加 0 等于乘以 10 是完全相同的道理。

> 💡 **老师提醒：** `1 << n` 是生成 2 的 n 次方最快的写法，竞赛里大量用到（状态压缩、快速幂的模拟等）。但要注意：`1 << 31` 在 int 类型下会溢出，如果 n 可能很大，要用 `1LL << n` 确保是 long long 类型。

---

### 6. 右移 `>>`

把二进制整体向右移n位，相当于 **除以2的n次方** （取整）：

```
8 = 1000
8 >> 1 = 0100 = 4   （除2）
8 >> 2 = 0010 = 2   （除4）
```

```cpp
cout << (8 >> 1) << endl;  // 4
cout << (7 >> 1) << endl;  // 3（7/2=3.5，取整为3）
```

右移是向下取整的整数除法，和 Python 的 `//` 对正数效果一样。*就像左移和右移的关系，就像给数字末尾加 0 和去掉最后一位 0 的关系——一个放大，一个缩小，方向相反，规律对称。*

## 竞赛中最常用的位运算技巧

```cpp
// 1. 判断奇偶
n & 1          // 1是奇数，0是偶数

// 2. 乘以2 / 除以2
n << 1         // n * 2
n >> 1         // n / 2

// 3. 2的n次方
1 << n         // 2^n

// 4. 判断第k位是否为1（k从0开始）
(n >> k) & 1   // 1表示第k位是1，0表示是0

// 5. 把第k位设为1
n | (1 << k)

// 6. 把第k位设为0
n & ~(1 << k)
```

这六个技巧是竞赛里的"位运算工具箱"，不用死记，理解了背后的原理，用的时候自然能推导出来。

> 💡 **老师提醒：** 位运算的优先级比比较运算符（`<`、`>`、`==`）低，但比赋值运算符高。写 `(n >> k) & 1` 时括号是必须的——如果写成 `n >> k & 1`，编译器会先算 `k & 1` 再右移，结果完全不对。位运算里养成加括号的习惯，省得被优先级坑。

## 一个实用例子：枚举子集

竞赛中经常要枚举一个集合的所有子集，位运算可以很优雅地做到：

```cpp
// 集合 {0, 1, 2}，用二进制表示哪些元素被选中
// 000 = {}
// 001 = {0}
// 010 = {1}
// 011 = {0,1}
// 100 = {2}
// 101 = {0,2}
// 110 = {1,2}
// 111 = {0,1,2}

int n = 3;
for (int mask = 0; mask < (1 << n); mask++) {
    cout << "子集：";
    for (int i = 0; i < n; i++) {
        if ((mask >> i) & 1) {  // 第i位是否为1
            cout << i << " ";
        }
    }
    cout << endl;
}
```

输出：

```
子集：
子集：0
子集：1
子集：0 1
子集：2
子集：0 2
子集：1 2
子集：0 1 2
```

这个技巧叫 **状态压缩** ，后面做竞赛题会经常用到。核心思想是：n 个元素的集合有 2^n 个子集，恰好对应 0 到 2^n-1 这些整数的二进制表示——第 i 位是 1 就表示第 i 个元素被选中。*就像状态压缩用整数表示集合，就像用一串灯泡的亮灭状态来记录哪些东西被选中了——每盏灯对应一个元素，亮了就选了，灭了就没选。*
