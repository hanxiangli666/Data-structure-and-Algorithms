# 你最常出现的错误

## 错误一：代码写在main外面（出现5次）

```cpp
// ❌ 你经常这样写
int n;
cin >> n;        // 执行语句跑到外面了
vector<int> a(n);

int main() { ... }

// ✅ 正确
int main() {
    int n;
    cin >> n;
    vector<int> a(n);
}
```

**根本原因：** Python没有main函数的概念，习惯带过来了。

**如何根治：** 每次打开编辑器写新题，第一件事就是把框架敲出来（`#include` → `main` → `return 0`），然后把光标移进 `main` 里，再开始想逻辑。这个"先框架后逻辑"的顺序形成肌肉记忆之后，这个错误就不会再出现。

---

## 错误二：变量没有声明类型（出现3次）

```cpp
// ❌ Python习惯
int maxOfThree(a, b, c) { ... }
cin >> n;

// ✅ C++必须声明类型
int maxOfThree(int a, int b, int c) { ... }
int n;
cin >> n;
```

**根本原因：** Python是动态类型，C++是静态类型，每个变量必须提前声明类型。

**如何根治：** 在 C++ 里，每次写变量名前先问自己"这个变量是什么类型"。整数写 `int`，大整数写 `long long`，浮点数写 `double`，字符串写 `string`。这不是可选的，是必须的——编译器不知道你的意图，你必须明确告诉它。

---

## 错误三：花括号没有配对（出现3次）

```cpp
// ❌ 你写的
for (char c : line) {
if (isupper(c)){
count++;}}   // 乱配对

// ✅ 正确，每个{立刻找到对应的}
for (char c : line) {
    if (isupper(c)) {
        count++;
    }
}
```

**根本原因：** 没有养成"写{立刻写}"的习惯。

**如何根治：** 养成"括号成对出现"的习惯——写完 `{` 立刻换行写 `}`，再把光标移回去填内容。代码缩进也很关键：同一层级的 `{` 和 `}` 对齐，眼睛扫一下就能看出有没有漏掉配对。

---

## 错误四：计数器定义在循环里（出现1次）

```cpp
// ❌ 每次循环都重置为0
for (char c : line) {
    int count = 0;
    if (isupper(c)) count++;
}

// ✅ 定义在循环外
int count = 0;
for (char c : line) {
    if (isupper(c)) count++;
}
```

**根本原因：** 没理解变量作用域，定义在{}里的变量出了{}就消失。

**如何根治：** 每次定义一个需要"跨越循环"保留的变量，先问：这个变量的生命周期是什么？如果它需要在整个循环结束后还能用，就必须定义在循环**外面**。作用域规则一句话：变量活在定义它的那对 `{}` 里。

---

## 错误五：去重后还用旧的n遍历（出现1次）

```cpp
// ❌ 去重后size变了，n还是原来的值
a.erase(unique(a.begin(), a.end()), a.end());
for (int i = 0; i < n; i++) { ... }

// ✅ 用实际大小
for (int x : a) { ... }
```

**根本原因：** 没意识到erase会改变vector的大小。

**如何根治：** 对容器做了增删操作之后，凡是遍历该容器，一律用范围 for 循环（`for (int x : a)`），彻底绕开"手动维护大小"的问题。范围 for 自动跟着容器走，不需要任何计数变量。

---

## 错误六：赋值和计算混淆（出现1次）

```cpp
// ❌ 只计算，没有存回去
n >> 1;

// ✅ 要存回去
n >>= 1;   // 或者 n = n >> 1
```

**根本原因：** 和Python的 `n // 2` 不一样，C++不会自动存回去。

**如何根治：** 每次写表达式时，问自己"这个计算结果有没有被存起来"。如果只是 `n >> 1` 这样孤立一行，结果就直接丢了。需要修改变量本身，必须是赋值语句：`n = n >> 1` 或者复合赋值 `n >>= 1`。

---

# 一句话总结

```
你90%的错误来自两个习惯没改过来：
1. Python不需要main，C++所有执行代码必须在main里
2. Python不需要声明类型，C++每个变量必须声明类型
```

把这两条贴在屏幕旁边，写代码前扫一眼，犯这两类错误的概率会大幅下降。
