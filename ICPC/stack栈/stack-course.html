01

认识栈

你每天都在用它，只是还不知道它叫什么名字

Ctrl+Z 背后的秘密

想象你在用代码编辑器写代码。你打了一段，觉得不对，按下 Ctrl+Z——撤销了一步。再按——又撤销一步。每次撤销，都是把最后一个操作取消掉。

这个"最后进来的最先出去"的规则，有个专业名字：LIFO。而储存这些操作的容器，叫做栈（Stack）。

💡
Aha! 时刻

浏览器的"返回"按钮、函数调用的调用栈、撤销/重做、括号匹配——这些功能背后都有栈。ICPC 里大约 15% 的题目可以用栈解决。

后进先出，动图看懂

想象一叠煎饼：你只能从上面加一张，也只能从上面取一张。

底层：10（最早进）
中间：20
栈顶：30（最晚进）
↑ 栈顶（top）：唯一出入口
⬆️ push s.push(x) → 压入栈顶
⬇️ pop s.pop() → 弹出栈顶
👁 top s.top() → 仅查看，不删除
🔑
关键区别：栈 vs 队列

栈（Stack） 是后进先出——好比弹夹，最后压进去的子弹最先射出。队列（Queue） 是先进先出——像排队买奶茶。

02

五大操作

C++ STL 的 stack 只有五个你需要掌握的接口

认识五个成员函数

⬆️
push(x)
把元素 x 压入栈顶。不返回任何值。
s.push(10)
👁
top()
返回栈顶元素的值，但不删除它。
s.top() → 10
⬇️
pop()
删除栈顶元素。注意：不返回值！
s.pop()
🔢
size()
返回栈中元素的个数（size_t 类型）。
s.size() → 3
❓
empty()
栈空返回 true（1），非空返回 false（0）。
s.empty() → 0
📌
没有 clear() 方法！

C++ 的 stack 没有 clear()。想清空只能用 while 循环一个个 pop，或者直接重新声明一个新栈。

04

避坑手册

两个坑，几乎每个初学者都会踩一次

坑 #1：pop() 不返回值

⚠️
这行代码会编译报错！

在 C++ 里，s.pop() 的返回类型是 void——它什么都不返回，只是默默删掉栈顶。

❌ 错误写法 
// 想取出栈顶值——这样写是错的！
int val = s.pop();  // 编译错误

// ✅ 正确写法：先 top() 再 pop()
int val = s.top();  // 先取值
s.pop();            // 再删除
中文翻译

注释：想取出栈顶——这是错误的！

错误：pop() 不返回值，不能赋值给变量。编译器会直接报错。

（空行）

注释：正确姿势——两步走

第一步：用 top() 把栈顶的值读出来，存进 val

第二步：再用 pop() 把栈顶删掉

💡
为什么要这样设计？

这是 C++ 标准库的异常安全设计：读取（top）和删除（pop）分离，避免异常导致数据丢失。记住口诀：先看顶，再弹出。

坑 #2：对空栈调用 top() 或 pop()

💥
未定义行为 — 程序可能直接崩溃

对空栈调用 top() 或 pop()，是未定义行为——程序可能崩溃，也可能输出奇怪的值。ICPC 里这会直接 RE（运行时错误）。

✅ 安全写法 
// 操作前先检查是否非空
if (!s.empty()) {
    int val = s.top();
    s.pop();
    // 安全地使用 val
}
中文翻译

注释：每次操作前都要先检查

如果栈不为空（!empty() = true）才进入

安全地读取栈顶

安全地删除栈顶

在这里处理取出的值，不怕崩溃

if 块结束

✅
ICPC 金律：调用 top()/pop() 之前，永远先检查 empty()

把这个习惯刻进肌肉记忆。比赛时时间紧张，一个 RE 可能让你损失几十分钟。
