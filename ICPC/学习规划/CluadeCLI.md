好，给你整理一份对你实用的 Claude Code CLI 功能速查表。作为备赛工具，你主要会用到这几类：

---

## 🗂️ 一、对话管理命令（最常用）

| 命令                | 作用                       | 什么时候用                                                        |
| ------------------- | -------------------------- | ----------------------------------------------------------------- |
| `/clear`          | 清空对话历史               | 换新题目时用，避免旧上下文干扰                                    |
| `/compact`        | 压缩对话历史但保留关键信息 | 当 context 使用超过 80% 但还在做同一个任务时用，比 `/clear`温和 |
| `/compact [说明]` | 压缩时指定保留什么         | 例：`/compact 保留错误处理的讨论`                               |

 **类比** ：`/clear` 像把白板全擦掉，`/compact` 像只保留白板上的关键结论、擦掉推导过程。

---

## 🤖 二、模型切换

用 `/model` 命令可以切换模型。大多数任务用 Sonnet，遇到复杂架构或难以定位的 bug 时换 Opus，解决后再换回来。

```
/model sonnet   # 默认，速度快、省 token
/model opus     # 更强，用于复杂推理
```

对你备赛来说：刷题讲解用 Sonnet，碰到复杂算法推导（比如网络流、计算几何）再换 Opus。

---

## 🔧 三、内置 Skills（斜杠命令）

内置 Skills 包括 `/simplify`、`/review`、`/batch`、`/loop`、`/debug`、`/claude-api`。

对你最有用的：

| 命令          | 作用                         |
| ------------- | ---------------------------- |
| `/review`   | 代码审查，检查你写的解题代码 |
| `/debug`    | 帮你调试报错的代码           |
| `/simplify` | 检查代码冗余、优化写法       |

---

## ⌨️ 四、键盘快捷键

按 Escape 可以停止 Claude（按 Ctrl+C 会直接退出整个程序）。按两次 Escape 会显示历史消息列表，可以跳回去查看。

| 快捷键       | 作用                 |
| ------------ | -------------------- |
| `Escape`   | 停止当前生成         |
| `Esc Esc`  | 显示历史消息，可跳回 |
| `↑`方向键 | 翻出上一条你发的消息 |

---

## 📁 五、引用文件

你可以直接在对话里用 `@文件名` 引用你的代码文件。

```
@solution.cpp 这段代码 TLE 了，帮我分析
```

---

## 💡 六、运行 Shell 命令

在对话里用 `!` 开头可以直接执行 shell 命令，绕过 Claude 的对话模式，更省 token。

```
!g++ -O2 -o sol solution.cpp && echo "编译成功"
!./sol < input.txt
```

对你备赛很实用——可以直接在 Claude Code 里编译测试 C++ 代码。

---

## 📋 七、查看所有命令

输入 `/help` 可以看到所有可用的斜杠命令，包括你自定义的命令。

---

 **给你的备赛使用建议** ：

* 每道新题开始前 `/clear` 清空上下文
* 写完解法后 `/review` 让 Claude 审查代码质量
* 编译测试用 `!g++ ...` 直接跑，不用出 CLI
* 长时间讨论一道题时用 `/compact` 而不是 `/clear`
