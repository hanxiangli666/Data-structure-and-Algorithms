# ICPC 备赛学习规划

> **目标**：从零基础出发，系统备战 ICPC，冲击区域赛 / NAC 银奖或铜奖级别。
> **长期目标**：进入苹果或谷歌担任软件工程师。

---

## 一、核心学习理念

### 1.1 构建"框架思维"，而非死记硬背

- **数据结构的本质只有两种**：数组 + 链表，其他所有结构都是它们的衍生。
- **算法的本质只有一件事**：穷举。所有花哨的算法名称，底层逻辑都是穷举的变体。
- 以这两条为基础，任何新题都可以找到切入点，逐步优化，写出最优解。

### 1.2 刷题的正确姿势

1. **量不是目的，内化才是**：刷一道、真正理解框架，胜过盲刷十道。
2. **框架先行**：遇到新题，先套框架，填充细节，再优化。
3. **风格统一**：使用固定的代码模板和变量命名习惯，减少出错率，提升稳定性。
4. **禁止背题**：要理解"为什么用 `<=` 而不是 `<`"，而不是记住某题的答案。

## 二、常见学习误区（需规避）

| 误区             | 说明                                           |
| ---------------- | ---------------------------------------------- |
| 被高难度资料劝退 | 算法不需要深厚数学基础才能入门，直接刷题是正道 |
| 贪多求全         | 几百G资料没有价值，看不完也练不到；专注精练    |
| 追求花哨解法     | 框架思维未成型前，不必追求一题多解             |
| 刷题无章法       | 乱刷没有归纳，等于白刷；按框架分类练习         |
| 代码风格不统一   | 风格混乱导致无法形成稳定套路，发挥不稳定       |

## 四、习题练习与复习方法

### 练习方法

1. **先学框架**，把对应的算法讲解认真过一遍。
2. **不看答案，自己尝试**解至少一道对应框架的习题，验证是否真正理解。
3. **参考答案**，找到自己卡住的地方，复盘思路差距。

### 复习节奏

- 完成一个习题章节后，**间隔约 3 天**回来重做。
- 复习时**不看答案**，独立思考求解。
- 做不出来的题，看答案后记下原因，**隔几天再做一遍**，直到能独立解出。

## 七、心态与节奏建议

- **正反馈很重要**：每掌握一个框架，能秒杀一批题，要享受这种成就感。
- **不要贪快**：框架思维一旦形成，是不会退化的长期资产。
- **保持好奇心**：算法不只是刷题工具，它是一种解决实际问题的思维方式。
- **时间不够也不怕**：在校期间有大块时间，把基础彻底打扎实，未来跳槽都不需要额外准备。
