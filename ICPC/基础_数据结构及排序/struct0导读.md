## 二、时间与空间复杂度入门

> 本节为初学者提供够用的复杂度分析基础，完整的分析方法将在学完几种核心算法框架后专题讲解。

### 2.1 核心概念

#### Big O 表示法

复杂度用 Big O 表示，如 `O(1)`、`O(n)`、`O(n²)`、`O(log n)` 等。

**关键规则**：

- 只保留**最高增长项**，忽略常数和低阶项
- `O(2n² + 3n + 1)` → 等同于 `O(n²)`
- `O(1000n + 1000)` → 等同于 `O(n)`

#### 时间复杂度 vs 空间复杂度

| 类型       | 衡量的是     | 越小越好 |
| ---------- | ------------ | -------- |
| 时间复杂度 | 算法执行效率 | ✅       |
| 空间复杂度 | 算法内存消耗 | ✅       |

例：`O(n)` 的算法比 `O(n²)` 执行效率更高；`O(1)` 的算法比 `O(n)` 内存消耗更小。

### 2.2 分析原则

1. **分析最坏情况**：不考虑"运气好"的情况，以最坏的输入估算。
2. **简化估算方法**：
   - **时间复杂度** ≈ `for` 循环的最大嵌套层数
   - **空间复杂度** ≈ 算法额外申请的存储空间大小
3. **输入数组本身不算**在算法的空间复杂度里。

> ⚠️ 注意：以上是简化方法，有些细节不严谨（如隐藏的初始化循环），完整分析方法见后续专题章节。

---

### 2.3 复杂度案例分析

#### 示例一：`O(n)` 时间，`O(1)` 空间

```cpp
// 输入一个整数数组，返回所有元素的和
int getSum(vector<int>& nums) {
    int sum = 0;
    for (int i = 0; i < nums.size(); i++) {
        sum += nums[i];
    }
    return sum;
}
```

- **时间复杂度**：`O(n)` — 一层 for 循环遍历 n 个元素
- **空间复杂度**：`O(1)` — 只使用了一个 `sum` 变量，常数空间

---

#### 示例二：最坏情况分析，`O(n)` 时间，`O(1)` 空间

```cpp
// 当 n 是 10 的倍数时，计算累加和，否则返回 -1
int sum(int n) {
    if (n % 10 != 0) {
        return -1;
    }
    int sum = 0;
    for (int i = 0; i <= n; i++) {
        sum += i;
    }
    return sum;
}
```

- **时间复杂度**：`O(n)` — 虽然有些情况直接返回 `-1`，但**只考察最坏情况**（n 是 10 的倍数时执行 for 循环）
- **空间复杂度**：`O(1)`

---

#### 示例三：嵌套循环，`O(n²)` 时间，`O(1)` 空间

```cpp
// 数组是否存在两个数，它们的和为 target？
bool hasTargetSum(vector<int>& nums, int target) {
    for (int i = 0; i < nums.size(); i++) {
        for (int j = i + 1; j < nums.size(); j++) {
            if (nums[i] + nums[j] == target) {
                return true;
            }
        }
    }
    return false;
}
```

- **时间复杂度**：`O(n²)` — 两层嵌套 for 循环，每层最坏 `O(n)`
- **空间复杂度**：`O(1)` — 只用了 `i`、`j` 两个变量

> ❓ 疑问：内层循环没有遍历完整数组，还是 `O(n²)` 吗？
> ✅ 是的。复杂度分析只看**最坏情况**，不考虑提前 return 的情况。

---

#### 示例四：隐藏的初始化开销，`O(n)` 时间，`O(n)` 空间

```cpp
void exampleFn(int n) {
    vector<int> nums(n);
}
```

- **时间复杂度**：`O(n)` — 创建大小为 n 的数组时，语言底层会对 n 个元素初始化（隐藏的 for 循环）
- **空间复杂度**：`O(n)` — 申请了大小为 n 的数组

> ⚠️ **重要提醒**：时间复杂度不仅体现在你看得到的 for 循环，每行代码都可能有隐藏开销。了解常用数据结构的底层实现，是准确分析时间复杂度的基础。

---

#### 示例五：返回新数组，`O(n)` 时间，`O(n)` 空间

```cpp
// 输入一个整数数组，返回每个元素的平方组成的新数组
vector<int> squareArray(vector<int>& nums) {
    vector<int> res(nums.size());
    for (int i = 0; i < nums.size(); i++) {
        res[i] = nums[i] * nums[i];
    }
    return res;
}
```

- **时间复杂度**：`O(n)` — 初始化 `res` 数组 `O(n)` + for 循环 `O(n)`，合并仍是 `O(n)`
- **空间复杂度**：`O(n)` — 声明了与输入等长的新数组 `res`

### 2.4 常见复杂度对比（由优到劣）

```
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ) < O(n!)
```

## 三、本章小结

| 知识点       | 核心要点                             |
| ------------ | ------------------------------------ |
| Big O 表示法 | 保留最高项，忽略常数和低阶项         |
| 时间复杂度   | 看 for 循环嵌套层数（简化版）        |
| 空间复杂度   | 看额外申请的存储空间                 |
| 分析原则     | 永远分析**最坏情况**           |
| 隐藏开销     | 初始化数组、语言底层操作也有时间开销 |

> 掌握以上内容，足以支撑本章后续的数据结构和排序算法学习。完整的复杂度分析方法，将在学完几种核心算法框架后专题介绍。
>
> 首先，空间复杂度不考虑输入所占的空间。比如示例一，输入是一个数组，这个数组不在计算范围内。 空间复杂度关注的是你处理输入所需占用的空间。还是拿示例一来说，不管输入的数组有多大，反正你用到的始终就是变量 i 和 变量sum，你自己占用的空间不随输入的变化而变化，也就是说像常数一样，因此就是O(1)。 现在拿示例五来说，输入的数组有多大，就创建一个与之同等大小的数组，也就是说你占用的空间和输入有一种线性关系（你就理解成 y = x 这样的函数，其中x是输入，y是你占用的空间） 因此空间复杂度是O(n)
>
> **时间复杂度** ：是算法的执行效率，通常我们要去依靠循环次数和嵌套次数  
>
> **空间复杂度** ：是算法的内存消耗，通常我们要去依靠算法开辟的空间和数据结构
