最简单的循环：求和

先看最基础的一道题：计算 1 + 2 + 3 + ... + n 的值。

手算的话，你会从 1 开始，一个一个往上加。程序也是这么做的，只不过它可以在一秒内加完一亿次。

C++ 代码 逐行解释
#include <iostream> using namespace std; int main() { int n; cin >> n; int sum = 0; for (int i = 1; i <= n; i++) { sum += i; } cout << sum << "\n"; return 0; }
1引入"输入输出"工具箱
2省略每次都要写的命名空间前缀
4程序的主入口，从这里开始运行
5声明一个整数变量 n，用来装输入
6从键盘读入 n 的值
7sum 从 0 开始，准备累加
8循环：i 从 1 增到 n，每次加 1
9每轮把 i 加进 sum（相当于 sum = sum + i）
11把最终结果输出到屏幕
💡 关键洞察
for 循环的三个部件：初始值（i=1）、终止条件（i≤n）、每步变化（i++）。这三个你掌握了，就掌握了 for 循环的一切。

for 循环的三大模式

积累、追踪、筛选——所有 for 循环本质上都是这三种之一

不只是求和

for 循环能做的远不止累加数字。洛谷这批题里，for 循环被用来做三件不同的事情：

➕
积累模式
每轮把某个值加进"罐子"里，最后罐子里就是答案。
求和、打分（去掉最高最低后求均值）
🎯
追踪模式
边扫描边更新一个"当前最优"，循环结束时它就是答案。
找最小值、找最大值、极差
🔀
筛选模式
根据条件把元素分成不同组，分别处理。
分类平均（能被k整除/不能被k整除）

追踪模式：找最小值

关键思路：先假设一个"很大的数"作为初始最小值，然后扫描每个元素，遇到更小的就更新。

这个"很大的数"叫做 哨兵值，代码里写的是 1e9（即十亿）。

找最小值.cpp 逐行解释
int n; cin >> n; int min_val = 1e9; // 初始设一个很大的数 for (int i = 0; i < n; i++) { int x; cin >> x; if (x < min_val) min_val = x; } cout << min_val << "\n";
1读入要处理的数字个数 n
4把"当前最小"初始化为十亿——比任何输入都大
5遍历 n 个数（i 只是轮次计数器）
6-7每轮读入一个新数 x
8如果 x 比当前记录更小，就更新记录
11所有数扫完，min_val 就是最终最小值
💡 关键洞察
追踪模式的本质是一场"擂台赛"——每个选手都来挑战当前冠军，只有打败冠军才能换人。扫描结束，冠军就是最终赢家。

筛选模式：分类平均

把 1 到 n 里的数字分成两组：能被 k 整除的 和 不能被 k 整除的，分别求平均值。

分类平均.cpp 逐行解释
int sumA = 0, cntA = 0; int sumB = 0, cntB = 0; for (int i = 1; i <= n; i++) { if (i % k == 0) { cntA++; sumA += i; } else { cntB++; sumB += i; } } printf("%.1f %.1f\n", (double)sumA/cntA, (double)sumB/cntB);
1-2A 组（整除）和 B 组（不整除）各自的总和与计数
4遍历 1 到 n 的每个整数
5i % k == 0 判断 i 能否被 k 整除
6-7是：A 组计数+1，A 组总和加 i
9-10否：B 组计数+1，B 组总和加 i
14输出两组的平均值，保留一位小数
📐 筛选模式的通式

同一个循环里，用 if / else 把元素"分拨"到不同的组，每组维护自己的计数器和求和器。这在处理分组统计问题时极为常用。

while 循环：什么时候停？

当你不知道要循环多少次，只知道什么时候应该停下来

for 和 while 的区别

for 循环适合"已知要重复几次"的场景——比如处理 n 个数字，就循环 n 次。

while 循环适合"重复到某个条件满足为止"的场景——循环次数事先不确定。

for 循环

像倒计时闹钟：定好响几次，响完就停。

for (i=1; i<=n; i++)
while 循环

像自动续杯机：杯子不满就一直倒，满了停。

while (sum <= k)

级数求和：倒什么时候够？

问题：计算 1 + 1/2 + 1/3 + 1/4 + ... 最少需要加多少项，才能使和 ≥ k？

这就是著名的 调和级数。你不知道要加几项，只知道什么时候可以停——所以用 while。

级数求和.cpp 逐行解释
int k; cin >> k; double sum = 0; int n = 0; while (sum <= k) { n++; sum += 1.0 / n; } cout << n << "\n";
1-2读入目标值 k
4当前总和，从 0 开始
5当前加到了第 n 项，从 0 开始
6只要总和还没超过 k，就继续
7项数加 1
8把第 n 项（1/n）加入总和
11循环退出时，n 就是所需项数

条件检查者 vs 循环体的对话

想象 while 循环内部有两个角色在对话：负责检查条件的检查者和负责执行任务的循环体。以 k=2 为例：

while 循环内部通信

嵌套循环：循环里的循环

用两层循环画出二维图案——行和列的双重控制

什么是嵌套循环？

嵌套循环就是在一个循环的内部再放一个循环。外层循环控制"行"，内层循环控制"列"——就像打印一张表格。

想象你在填一张 n×n 的座位表：外层循环"换一排"，内层循环"在这排填满所有位置"。

💡 关键规律
外层循环每执行 1 次，内层循环就完整执行一遍。如果外层循环 n 次，内层也循环 n 次，总共执行 n×n 次。
点击"演示"观察嵌套循环的执行顺序

数字直角三角形

输出 n 行：第 1 行 n 个数，第 2 行 n-1 个数……第 n 行 1 个数，全部连续编号。

n=4 时的输出：
01 02 03 04
05 06 07
08 09
10
逻辑拆解
外层 i = 1 到 n（控制行）
内层 j = 0 到 n-i（控制每行个数）
每轮打印 num，然后 num++
数字直角三角形.cpp 逐行解释
int n; cin >> n; int num = 1; // 当前数字 for (int i = 1; i <= n; i++) { for (int j = 0; j < n-i+1; j++) { printf("%02d", num); num++; } cout << "\n"; }
4num 全程保持连续，不在行与行之间重置
5外层：i=1,2,...,n 控制第几行
6内层：每行打 n-i+1 个数（第1行n个，第n行1个）
7%02d = 两位补零格式（01,02,…,09,10,…）
8打完一个数，序号+1
10内层结束，换行（每行打完才换行）

数字手术：拆解整数的循环

用 % 和 / 把数字像剥洋葱一样一层层拆开

整数的"位数手术"

很多问题需要处理一个整数的每一位数字——比如反转数字、统计某个数字出现次数。怎么做？

关键是两个操作：

🔧
num % 10
取出最末尾的一位数字。就像只看快递单号最后一位。
1234 % 10 = 4
✂️
num / 10
去掉最末尾那一位，剩下高位部分。
1234 / 10 = 123

把这两个操作配合 while 循环，就能逐位"剥"掉整个数字：

代码解读：计数问题

统计 1 到 n 的所有数字中，数字 x 总共出现了多少次。

计数问题.cpp 逐行解释
int n, x; cin >> n >> x; int cnt = 0; for (int i = 1; i <= n; i++) { int num = i; while (num > 0) { if (num % 10 == x) cnt++; num /= 10; } } cout << cnt << "\n";
1-3读入 n 和目标数字 x；计数器 cnt 从 0 开始
4外层 for：遍历 1 到 n 的每个数 i
5用 num 临时保存 i（不破坏循环变量）
6内层 while：只要 num 还有位数，就继续剥
7末位等于 x？计数器+1
8去掉末位，继续处理高位
💡 关键洞察
"for + while 嵌套"是处理数字问题的经典结构：外层 for 遍历所有数，内层 while 拆解每个数的位。掌握了这个结构，数字类题目大半就搞定了。

较大质数：一个数 n 只有两个质因数（例如 n = 15 = 3 × 5）。代码只需循环到 √n 就能找到较小的那个，为什么？
如果 n 有一个因数小于 √n，那另一个因数一定大于 √n。两个因数不可能都大于 √n（否则乘积就超过 n 了）
√n 是 n 的中位数

用循环模拟时间流逝，以及 20 道题的完整算法地图

模拟类题目：让程序重演一遍

有一类题目不需要数学公式——你只需要让程序忠实地模拟题目描述的过程，循环就是时间的流逝。

最长连号.cpp 逐行解释
int maxLen = 1, curLen = 1; for (int i = 1; i < n; i++) { if (a[i] == a[i-1] + 1) { curLen++; } else { curLen = 1; } maxLen = max(maxLen, curLen); }
1maxLen：历史最长连号；curLen：当前连号长度
2从第二个数开始（需要和前一个比较）
3当前数恰好比前一个大 1？说明连号延续了
4是：当前连号长度+1
6否：连号断了，从 1 重新计
8每轮更新历史最长（追踪模式！）
💡 双变量追踪技巧
一个变量追踪"当前状态"（curLen），一个变量追踪"历史最优"（maxLen）——这是解决"最长/最大连续......"问题的通用框架。
