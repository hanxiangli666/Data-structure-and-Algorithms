# 二叉树的递归遍历与层序遍历 学习笔记

> **前置知识**：二叉树基础概念（上一篇笔记）  
> **重要程度**：⭐⭐⭐⭐⭐ 所有树上算法的根基  
> **核心认知**：二叉树只有两种遍历方式——递归遍历（DFS）和层序遍历（BFS），所有其他写法都是这两种的变体

---

## 一、全局视角：两种遍历方式

```
二叉树的遍历
 │
 ├── 递归遍历（DFS，深度优先搜索）
 │    ├── 前序遍历 ← 代码写在前序位置
 │    ├── 中序遍历 ← 代码写在中序位置
 │    └── 后序遍历 ← 代码写在后序位置
 │    （三者的区别只是代码放的位置不同，遍历顺序是同一个！）
 │
 └── 层序遍历（BFS，广度优先搜索）
      ├── 写法一：最简单，但不知道层数
      ├── 写法二：最常用，能记录层数
      └── 写法三：最灵活，每个节点自带状态
```

> 记住：**再无其他**。栈模拟递归？本质还是 DFS。递归做层序？本质还是 BFS。

## 二、递归遍历（DFS）

### 2.1 代码模板（务必背下来）

```cpp
void traverse(TreeNode* root) {
    if (root == nullptr) {
        return;
    }
    // 【前序位置】—— 刚进入节点时执行
    traverse(root->left);
    // 【中序位置】—— 左子树遍历完，右子树还没开始时执行
    traverse(root->right);
    // 【后序位置】—— 左右子树都遍历完，即将离开节点时执行
}
```

这段代码就 5 行（不算注释），但它是后面学 DFS、回溯、动态规划的**地基**。

### 2.2 遍历顺序是怎样的？

**类比**：想象你是一只蚂蚁，从树根出发，沿着树的外轮廓爬行。规则是：**能往左就往左，左边走不通才往右，左右都走不通就返回上一层**。

用一个具体例子：

```
        1
       / \
      2   3
     /   / \
    4   5   6
```

蚂蚁（root 指针）的移动路径：

```
1 → 2 → 4 → (4左空,返回) → (4右空,返回) → 回到2 → (2右空,返回) 
→ 回到1 → 3 → 5 → (5左空,返回) → (5右空,返回) → 回到3 → 6 
→ (6左空,返回) → (6右空,返回) → 回到3 → 回到1 → 结束
```

**关键理解**：不管你在代码里加什么逻辑，只要 `traverse(root->left)` 在 `traverse(root->right)` 前面，蚂蚁走的路线就是固定的。

### 2.3 三个特殊位置

前/中/后序遍历**不是三种不同的遍历方式**，而是在**同一种遍历过程中**，把代码放在了不同的位置。

```cpp
void traverse(TreeNode* root) {
    if (root == nullptr) return;
    
    /* 前序位置 */   // ← 刚踏入这个节点，对子节点一无所知
    traverse(root->left);
    /* 中序位置 */   // ← 左子树已经全部遍历完了
    traverse(root->right);
    /* 后序位置 */   // ← 左右子树都遍历完了，即将离开
}
```

**类比**：你去一间教室检查。

- **前序位置** = 你刚推门进去，还没看任何东西（"我到了！"）
- **中序位置** = 你检查完了左半边，正准备检查右半边（"左边看完了"）
- **后序位置** = 你左右都检查完了，正准备离开（"全看完了，走人"）

### 2.4 用具体例子理解三种遍历结果

```
        1
       / \
      2   3
     /   / \
    4   5   6
```

#### 前序遍历（根 → 左 → 右）

在**刚进入节点时**记录：

```
进入 1 → 记录 1
进入 2 → 记录 2
进入 4 → 记录 4
4 没有子节点，返回
2 没有右子节点，返回
进入 3 → 记录 3
进入 5 → 记录 5
5 没有子节点，返回
进入 6 → 记录 6
6 没有子节点，返回

前序结果：[1, 2, 4, 3, 5, 6]
```

```cpp
void preorder(TreeNode* root) {
    if (root == nullptr) return;
    cout << root->val << " ";   // 【前序位置】记录
    preorder(root->left);
    preorder(root->right);
}
```

#### 中序遍历（左 → 根 → 右）

在**左子树遍历完后**记录：

```
进入 1 → 先去左子树
  进入 2 → 先去左子树
    进入 4 → 左子树为空 → 记录 4 → 右子树为空 → 返回
  回到 2 → 左子树完成 → 记录 2 → 右子树为空 → 返回
回到 1 → 左子树完成 → 记录 1 → 去右子树
  进入 3 → 先去左子树
    进入 5 → 左子树为空 → 记录 5 → 右子树为空 → 返回
  回到 3 → 左子树完成 → 记录 3 → 去右子树
    进入 6 → 左子树为空 → 记录 6 → 右子树为空 → 返回

中序结果：[4, 2, 1, 5, 3, 6]
```

```cpp
void inorder(TreeNode* root) {
    if (root == nullptr) return;
    inorder(root->left);
    cout << root->val << " ";   // 【中序位置】记录
    inorder(root->right);
}
```

#### 后序遍历（左 → 右 → 根）

在**即将离开节点时**记录：

```
进入 1 → 先遍历左右子树
  进入 2 → 先遍历左右子树
    进入 4 → 左右都空 → 记录 4 → 返回
  回到 2 → 左右都完成 → 记录 2 → 返回
回到 1 → 去右子树
  进入 3 → 先遍历左右子树
    进入 5 → 左右都空 → 记录 5 → 返回
    进入 6 → 左右都空 → 记录 6 → 返回
  回到 3 → 左右都完成 → 记录 3 → 返回
回到 1 → 左右都完成 → 记录 1 → 返回

后序结果：[4, 2, 5, 6, 3, 1]
```

```cpp
void postorder(TreeNode* root) {
    if (root == nullptr) return;
    postorder(root->left);
    postorder(root->right);
    cout << root->val << " ";   // 【后序位置】记录
}
```

### 2.5 三种位置的本质区别

| 位置 | 执行时机 | 你能获取的信息 | 典型用途 |
|------|---------|--------------|---------|
| **前序** | 刚进入节点 | 只知道当前节点和从根到此的路径信息 | 传递参数给子节点 |
| **中序** | 左子树遍历完 | 知道左子树的所有信息 | BST 的有序遍历 |
| **后序** | 左右子树都遍历完 | 知道左右子树的全部信息 | 汇总子树信息返回给父节点 |

**重要性质**：BST 的中序遍历结果是**有序的**（从小到大排列）。

```
BST:      7
         / \
        4   9
       / \    \
      1   5   10

中序遍历：[1, 4, 5, 7, 9, 10]  ← 自动排好序了！
```

这个性质在做 BST 相关题目时非常有用。

## 三、层序遍历（BFS）

### 3.1 核心思路

递归遍历像蚂蚁一样深入到底再回头（深度优先）。层序遍历则是**一层一层地横扫**（广度优先）。

```
        1           ← 第 1 层
       / \
      2   3         ← 第 2 层
     /   / \
    4   5   6       ← 第 3 层

层序遍历结果：[1, 2, 3, 4, 5, 6]
```

**类比**：想象消防员搜索一栋着火的大楼。他们不会一口气冲到顶楼，而是**先搜完 1 楼所有房间，再上 2 楼搜所有房间**，逐层推进。这就是 BFS——广度优先搜索。

### 3.2 层序遍历需要队列

为什么需要**队列**（先进先出）？

因为遍历某一层时，要按从左到右的顺序把下一层的节点排好队。先发现的先处理，这正是队列的特性。

```
初始：队列 = [1]
处理 1：弹出 1，把 1 的孩子 2、3 入队 → 队列 = [2, 3]
处理 2：弹出 2，把 2 的孩子 4 入队   → 队列 = [3, 4]
处理 3：弹出 3，把 3 的孩子 5、6 入队 → 队列 = [4, 5, 6]
处理 4：弹出 4，无孩子               → 队列 = [5, 6]
处理 5：弹出 5，无孩子               → 队列 = [6]
处理 6：弹出 6，无孩子               → 队列 = []
结束
```

### 3.3 写法一：最简单版

```cpp
void levelOrderTraverse(TreeNode* root) {
    if (root == nullptr) return;
    
    queue<TreeNode*> q;
    q.push(root);
    
    while (!q.empty()) {
        TreeNode* cur = q.front();   // 取出队头
        q.pop();                     // 弹出队头
        
        // 访问当前节点
        cout << cur->val << " ";
        
        // 把左右子节点加入队列
        if (cur->left != nullptr)  q.push(cur->left);
        if (cur->right != nullptr) q.push(cur->right);
    }
}
```

**C++ 知识点**：
- `queue<TreeNode*> q`：声明一个存储指针的队列
- `q.push(x)`：入队（加到队尾）
- `q.front()`：查看队头元素（不弹出）
- `q.pop()`：弹出队头元素（不返回值）
- 注意 C++ 的 `queue` 和 Python 不一样：`front()` 和 `pop()` 是分开的

**优点**：写法简单  
**缺点**：不知道当前节点在第几层

### 3.4 写法二：记录层数（最常用！）

关键改动：**每次处理一整层**。先用 `sz` 记录当前层有多少节点，然后 for 循环处理完这一层，再进入下一层。

```cpp
void levelOrderTraverse(TreeNode* root) {
    if (root == nullptr) return;
    
    queue<TreeNode*> q;
    q.push(root);
    int depth = 1;   // 当前层数（根节点是第 1 层）
    
    while (!q.empty()) {
        int sz = q.size();    // ⚠️ 一定要在循环前保存！
        
        // 处理当前层的所有节点
        for (int i = 0; i < sz; i++) {
            TreeNode* cur = q.front();
            q.pop();
            
            // 访问节点，同时知道它在第几层
            cout << "第 " << depth << " 层: " << cur->val << endl;
            
            if (cur->left != nullptr)  q.push(cur->left);
            if (cur->right != nullptr) q.push(cur->right);
        }
        
        depth++;   // 一整层处理完了，层数 +1
    }
}
```

**为什么 `sz` 必须提前保存？**

因为 for 循环里面会 `push` 新节点进队列，`q.size()` 会变。如果你写 `for (int i = 0; i < q.size(); i++)`，循环次数就不对了。

**图解执行过程**：

```
初始：queue = [1], depth = 1

第 1 层：sz = 1
  弹出 1，入队 2、3 → queue = [2, 3]
  输出：第 1 层: 1
depth++, depth = 2

第 2 层：sz = 2
  弹出 2，入队 4   → queue = [3, 4]
  弹出 3，入队 5、6 → queue = [4, 5, 6]
  输出：第 2 层: 2, 第 2 层: 3
depth++, depth = 3

第 3 层：sz = 3
  弹出 4、5、6，无新节点入队
  输出：第 3 层: 4, 第 3 层: 5, 第 3 层: 6
depth++, depth = 4

queue 为空，结束
```

**等价的简洁写法**（for 循环换成 while）：

```cpp
int sz = q.size();
while (sz-- > 0) {    // sz 自减到 0，效果相同
    TreeNode* cur = q.front();
    q.pop();
    // ...
}
```

### 3.5 写法三：每个节点自带状态（最灵活）

当每条边的权重不一样时（比如图的 BFS），写法二那样全局维护一个 `depth` 就不够用了。解决办法是让每个节点**自己记住自己的状态**。

```cpp
// 定义一个状态类，把节点和它的深度绑在一起
struct State {
    TreeNode* node;
    int depth;
    State(TreeNode* n, int d) : node(n), depth(d) {}
};

void levelOrderTraverse(TreeNode* root) {
    if (root == nullptr) return;
    
    queue<State> q;
    q.push(State(root, 1));   // 根节点深度为 1
    
    while (!q.empty()) {
        State cur = q.front();
        q.pop();
        
        cout << "depth = " << cur.depth 
             << ", val = " << cur.node->val << endl;
        
        if (cur.node->left != nullptr)
            q.push(State(cur.node->left, cur.depth + 1));
        if (cur.node->right != nullptr)
            q.push(State(cur.node->right, cur.depth + 1));
    }
}
```

**C++ 知识点**：
- `struct State`：定义一个结构体，把节点指针和深度打包在一起
- `queue<State>`：队列里存的不再是裸指针，而是 State 对象
- 每个子节点入队时，深度 = 父节点深度 + 1（或 + 边的权重）

**使用场景**：后面学图的 BFS 和 Dijkstra 算法时会用到。现在阶段用写法二就够了。

## 四、三种写法对比

| 写法 | 能力 | 复杂度 | 适用场景 |
|------|------|--------|---------|
| 写法一 | 只遍历，不知层数 | 代码最少 | 简单的 BFS 遍历 |
| **写法二** | **能记录层数** | **代码适中** | **绝大多数 BFS 题目** |
| 写法三 | 每个节点带自定义状态 | 需要定义额外结构体 | 带权重的图 BFS、Dijkstra |

## 五、递归遍历 vs 层序遍历

| 对比维度 | 递归遍历（DFS） | 层序遍历（BFS） |
|---------|---------------|---------------|
| **遍历方向** | 先深入到底，再回头 | 一层一层横向推进 |
| **使用的数据结构** | 函数调用栈（隐式） | 队列（显式） |
| **代码形式** | 递归函数 | while + queue |
| **衍生算法** | DFS、回溯、动态规划 | BFS、最短路径 |
| **典型应用** | 路径搜索、树形 DP | 层数计算、最短路 |

**类比**：
- DFS 像是探险家：一条路走到黑，碰壁了再回来换条路
- BFS 像是水波扩散：从中心向外一圈一圈地扩展

## 六、关键概念速查表

| 概念 | 说明 |
|------|------|
| **前序位置** | `traverse(left)` 之前，刚进入节点 |
| **中序位置** | `traverse(left)` 之后、`traverse(right)` 之前 |
| **后序位置** | `traverse(right)` 之后，即将离开节点 |
| **BST 中序遍历** | 结果是有序的（从小到大） |
| **queue** | C++ 队列，`push` 入队，`front` 看队头，`pop` 弹出 |
| **sz = q.size()** | BFS 写法二中，必须在循环前保存当前层的节点数 |
| **State** | BFS 写法三中，把节点和额外信息打包的结构体 |
