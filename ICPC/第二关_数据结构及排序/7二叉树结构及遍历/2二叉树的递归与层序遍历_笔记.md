# 二叉树的递归遍历与层序遍历 学习笔记

> **前置知识**：二叉树基础概念（上一篇笔记）
> **重要程度**：⭐⭐⭐⭐⭐ 所有树上算法的根基
> **核心认知**：二叉树只有两种遍历方式——递归遍历（DFS）和层序遍历（BFS），所有其他写法都是这两种的变体

---

## 一、全局视角：两种遍历方式

```
二叉树的遍历
 │
 ├── 递归遍历（DFS，深度优先搜索）
 │    ├── 前序遍历 ← 代码写在前序位置
 │    ├── 中序遍历 ← 代码写在中序位置
 │    └── 后序遍历 ← 代码写在后序位置
 │    （三者的区别只是代码放的位置不同，遍历顺序是同一个！）
 │
 └── 层序遍历（BFS，广度优先搜索）
      ├── 写法一：最简单，但不知道层数
      ├── 写法二：最常用，能记录层数
      └── 写法三：最灵活，每个节点自带状态
```

> 记住：**再无其他**。栈模拟递归？本质还是 DFS。递归做层序？本质还是 BFS。

不管你在网上看到多少种花里胡哨的写法——用栈模拟的、用 Morris 遍历的、用迭代器的——剥掉外壳，内核都是 DFS 或 BFS 二选一。所以你只需要把这两种彻底搞懂，其他变体都是在此基础上的小改动。

---

## 二、递归遍历（DFS）

### 2.1 代码模板（务必背下来）

**先看一个最简单的例子**——函数自己调用自己（递归）：

```cpp
// 想象你在拆一个俄罗斯套娃，每次打开一层就喊一声
void openDoll(int layer) {     // 参数 layer 表示第几层套娃
    if (layer == 0) return;    // 如果没有更小的套娃了，停下来
    cout << "打开第" << layer << "层" << endl;  // 打开当前层
    openDoll(layer - 1);       // 递归：继续打开里面更小的套娃
}
```

> 下面的 traverse 做的事一样——"打开套娃"变成了"访问节点"，"里面的小套娃"变成了"左右子树"。区别是套娃只有一个方向（一层层往里），而二叉树有两个方向（左和右）。

```cpp
void traverse(TreeNode* root) {           // ⚠️ TreeNode* 是指针，指向当前要访问的节点
    if (root == nullptr) {                // 如果节点为空（走到了树的尽头），停下来
        return;
    }
    // 【前序位置】—— 刚进入节点时执行
    traverse(root->left);                 // ⚠️ root->left：通过指针访问左孩子，然后递归处理整棵左子树
    // 【中序位置】—— 左子树遍历完，右子树还没开始时执行
    traverse(root->right);                // 递归处理整棵右子树
    // 【后序位置】—— 左右子树都遍历完，即将离开节点时执行
}
```

这段代码就 5 行（不算注释），但它是后面学 DFS、回溯、动态规划的**地基**。为什么说是地基？因为回溯就是在这个框架上加"选择/撤销选择"，动态规划就是在后序位置做"汇总子问题的结果"。你把这 5 行刻进肌肉记忆里，后面那些复杂算法就都有了着力点。

> 💡 **老师提醒：** 这个模板的 `if (root == nullptr) return;` 不能省！省掉它，当你访问空节点的 left/right 时程序会直接崩溃（空指针解引用）。这是最常见的 Runtime Error 来源。

> ✅ 这 5 行代码是整个树上算法的骨架，背到闭眼能默写为止！

### 2.2 遍历顺序是怎样的？

**类比**：想象你是一只蚂蚁，从树根出发，沿着树的外轮廓爬行。规则是：**能往左就往左，左边走不通才往右，左右都走不通就返回上一层**。

这个规则其实就是递归调用的自然顺序。`traverse(root->left)` 写在前面，所以蚂蚁总是先尝试往左走；左边走完了才执行 `traverse(root->right)` 往右走；左右都走完了，函数返回，蚂蚁就回到了上一层。

用一个具体例子：

```
        1
       / \
      2   3
     /   / \
    4   5   6
```

蚂蚁（root 指针）的移动路径：

```
1 → 2 → 4 → (4左空,返回) → (4右空,返回) → 回到2 → (2右空,返回)
→ 回到1 → 3 → 5 → (5左空,返回) → (5右空,返回) → 回到3 → 6
→ (6左空,返回) → (6右空,返回) → 回到3 → 回到1 → 结束
```

**关键理解**：不管你在代码里加什么逻辑，只要 `traverse(root->left)` 在 `traverse(root->right)` 前面，蚂蚁走的路线就是固定的。前/中/后序遍历只是蚂蚁在路上不同的"记录时机"——经过同样的路线，在不同的地方拍照，得到不同的相册。

### 2.3 三个特殊位置

前/中/后序遍历**不是三种不同的遍历方式**，而是在**同一种遍历过程中**，把代码放在了不同的位置。

这一点极其重要。很多同学以为前序和后序是两种不同的遍历策略，其实不是。蚂蚁走的路线始终是同一条，只是你选择在路线上的哪个点做记录。*就像同一条旅行路线，有人在出发时拍照（前序），有人在中途拍照（中序），有人在返程时拍照（后序）——行程一样，照片不同。*

**先看一个最简单的例子**——一个函数在不同位置做不同的事：

```cpp
// 想象你进一间房间检查
void checkRoom() {
    cout << "进门了！" << endl;    // 进门时做的事（类似前序）
    // ... 检查左半边 ...
    cout << "左边查完了" << endl;  // 中间做的事（类似中序）
    // ... 检查右半边 ...
    cout << "全查完了，走！" << endl; // 出门前做的事（类似后序）
}
```

> 下面的代码和上面的"检查房间"一样，只是"检查左半边"变成了递归调用 `traverse(root->left)`，"检查右半边"变成了 `traverse(root->right)`。

```cpp
void traverse(TreeNode* root) {
    if (root == nullptr) return;        // 空房间，不用检查，直接回头

    /* 前序位置 */   // ← 刚踏入这个节点，对子节点一无所知
    traverse(root->left);               // 递归检查整棵左子树
    /* 中序位置 */   // ← 左子树已经全部遍历完了
    traverse(root->right);              // 递归检查整棵右子树
    /* 后序位置 */   // ← 左右子树都遍历完了，即将离开
}
```

**类比**：你去一间教室检查。

- **前序位置** = 你刚推门进去，还没看任何东西（"我到了！"）
- **中序位置** = 你检查完了左半边，正准备检查右半边（"左边看完了"）
- **后序位置** = 你左右都检查完了，正准备离开（"全看完了，走人"）

> 💡 **老师提醒：** 很多题目的解法区别就在于把核心逻辑放在前序、中序还是后序位置。拿到一道树的题，先想清楚"我在哪个位置能拿到我需要的信息"，答案就呼之欲出了。

### 2.4 用具体例子理解三种遍历结果

```
        1
       / \
      2   3
     /   / \
    4   5   6
```

#### 前序遍历（根 → 左 → 右）

在**刚进入节点时**记录：

```
进入 1 → 记录 1
进入 2 → 记录 2
进入 4 → 记录 4
4 没有子节点，返回
2 没有右子节点，返回
进入 3 → 记录 3
进入 5 → 记录 5
5 没有子节点，返回
进入 6 → 记录 6
6 没有子节点，返回

前序结果：[1, 2, 4, 3, 5, 6]
```

前序遍历的特点是**根节点永远排第一**。因为蚂蚁一到这个节点就立刻记录，还没去看任何子树。所以前序遍历的第一个元素一定是整棵树的根。

**先看一个最简单的例子**——在函数开头打印：

```cpp
void sayHello(string name) {       // 一个打招呼的函数
    cout << "Hi, " << name << endl; // 一进来就打招呼（前序思维）
}
```

> 下面的 preorder 做的事一样——"进来就打招呼"变成了"进入节点就记录"，然后再去看左右孩子。

```cpp
void preorder(TreeNode* root) {
    if (root == nullptr) return;            // 空节点，停
    cout << root->val << " ";               // 【前序位置】一到就记录——根排最前
    preorder(root->left);                   // 递归处理左子树
    preorder(root->right);                  // 递归处理右子树
}
```

#### 中序遍历（左 → 根 → 右）

在**左子树遍历完后**记录：

```
进入 1 → 先去左子树
  进入 2 → 先去左子树
    进入 4 → 左子树为空 → 记录 4 → 右子树为空 → 返回
  回到 2 → 左子树完成 → 记录 2 → 右子树为空 → 返回
回到 1 → 左子树完成 → 记录 1 → 去右子树
  进入 3 → 先去左子树
    进入 5 → 左子树为空 → 记录 5 → 右子树为空 → 返回
  回到 3 → 左子树完成 → 记录 3 → 去右子树
    进入 6 → 左子树为空 → 记录 6 → 右子树为空 → 返回

中序结果：[4, 2, 1, 5, 3, 6]
```

中序遍历的特点是**左子树的所有节点排在当前节点之前**。这个特点在 BST 上有惊人的效果——因为左子树所有值都比根小，右子树所有值都比根大，中序遍历自然就得到了从小到大的有序序列。

**先看一个最简单的例子**——在两件事中间做记录：

```cpp
void doWork() {
    washDishes();                          // 先洗碗（类似先处理左子树）
    cout << "碗洗完了，开始拖地" << endl;   // 中间记录（中序位置）
    mopFloor();                            // 再拖地（类似再处理右子树）
}
```

> 下面的 inorder 做的事一样——先递归处理左子树，在"左完右未开始"的间隙记录当前节点，再递归处理右子树。

```cpp
void inorder(TreeNode* root) {
    if (root == nullptr) return;            // 空节点，停
    inorder(root->left);                    // 先递归处理完整棵左子树
    cout << root->val << " ";               // 【中序位置】左边全处理完了，记录自己
    inorder(root->right);                   // 再递归处理整棵右子树
}
```

#### 后序遍历（左 → 右 → 根）

在**即将离开节点时**记录：

```
进入 1 → 先遍历左右子树
  进入 2 → 先遍历左右子树
    进入 4 → 左右都空 → 记录 4 → 返回
  回到 2 → 左右都完成 → 记录 2 → 返回
回到 1 → 去右子树
  进入 3 → 先遍历左右子树
    进入 5 → 左右都空 → 记录 5 → 返回
    进入 6 → 左右都空 → 记录 6 → 返回
  回到 3 → 左右都完成 → 记录 3 → 返回
回到 1 → 左右都完成 → 记录 1 → 返回

后序结果：[4, 2, 5, 6, 3, 1]
```

后序遍历的特点是**根节点永远排最后**。因为要等左右子树全部遍历完，才记录自己。这意味着在后序位置，你已经拥有了左右子树的全部信息——所以后序位置特别适合做"汇总"操作，比如"算出整棵子树的节点总数"、"算出子树的最大深度"。

**先看一个最简单的例子**——出门前做总结：

```cpp
void visitMuseum() {
    visitHallA();                           // 先看 A 展厅
    visitHallB();                           // 再看 B 展厅
    cout << "全看完了，写观后感" << endl;    // 都看完了才写总结（后序思维）
}
```

> 下面的 postorder 做的事一样——先递归处理左右子树，等它们都完成了，最后才记录自己。

```cpp
void postorder(TreeNode* root) {
    if (root == nullptr) return;            // 空节点，停
    postorder(root->left);                  // 先递归处理完整棵左子树
    postorder(root->right);                 // 再递归处理完整棵右子树
    cout << root->val << " ";               // 【后序位置】左右都处理完了，最后记录自己
}
```

> ✅ 三种遍历的代码结构完全一样，只有 `cout` 那一行放的位置不同。理解了位置，就理解了一切！

### 2.5 三种位置的本质区别

| 位置 | 执行时机 | 你能获取的信息 | 典型用途 |
|------|---------|--------------|---------|
| **前序** | 刚进入节点 | 只知道当前节点和从根到此的路径信息 | 传递参数给子节点 |
| **中序** | 左子树遍历完 | 知道左子树的所有信息 | BST 的有序遍历 |
| **后序** | 左右子树都遍历完 | 知道左右子树的全部信息 | 汇总子树信息返回给父节点 |

选哪个位置，取决于你解题时需要什么信息。需要"从上往下传"的信息（比如路径、深度），用前序；需要"从下往上收"的信息（比如子树高度、子树节点数），用后序；需要 BST 的有序性，用中序。

**重要性质**：BST 的中序遍历结果是**有序的**（从小到大排列）。

```
BST:      7
         / \
        4   9
       / \    \
      1   5   10

中序遍历：[1, 4, 5, 7, 9, 10]  ← 自动排好序了！
```

这个性质在做 BST 相关题目时非常有用。为什么会这样？因为中序遍历是"左→根→右"，而 BST 恰好保证左 < 根 < 右。所以中序遍历就是从最小的值开始，按顺序输出每一个节点。

> 💡 **老师提醒：** 遇到 BST 的题，第一反应应该是"中序遍历能不能帮上忙"。大量 BST 题目的解法核心就是利用中序遍历的有序性。

## 三、层序遍历（BFS）

### 3.1 核心思路

递归遍历像蚂蚁一样深入到底再回头（深度优先）。层序遍历则是**一层一层地横扫**（广度优先）。

这两种方式就像两种完全不同的搜索策略。*就像打游戏清地图——DFS 是一条路走到底再回头探另一条路，BFS 是从出生点开始，先把周围一圈全探了，再探外面一圈。*两者都能遍历整棵树，但擅长的场景不同（下一篇会详细讲）。

```
        1           ← 第 1 层
       / \
      2   3         ← 第 2 层
     /   / \
    4   5   6       ← 第 3 层

层序遍历结果：[1, 2, 3, 4, 5, 6]
```

**类比**：想象消防员搜索一栋着火的大楼。他们不会一口气冲到顶楼，而是**先搜完 1 楼所有房间，再上 2 楼搜所有房间**，逐层推进。这就是 BFS——广度优先搜索。

### 3.2 层序遍历需要队列

为什么需要**队列**（先进先出）？

因为遍历某一层时，要按从左到右的顺序把下一层的节点排好队。先发现的先处理，这正是队列的特性。如果你用栈（后进先出），顺序就全乱了。*就像食堂排队打饭——先来的先打，后来的排后面。队列保证了"先发现的节点先被处理"。*

```
初始：队列 = [1]
处理 1：弹出 1，把 1 的孩子 2、3 入队 → 队列 = [2, 3]
处理 2：弹出 2，把 2 的孩子 4 入队   → 队列 = [3, 4]
处理 3：弹出 3，把 3 的孩子 5、6 入队 → 队列 = [4, 5, 6]
处理 4：弹出 4，无孩子               → 队列 = [5, 6]
处理 5：弹出 5，无孩子               → 队列 = [6]
处理 6：弹出 6，无孩子               → 队列 = []
结束
```

> ✅ DFS 用的是栈（递归调用栈），BFS 用的是队列。这个对应关系要牢牢记住！

### 3.3 写法一：最简单版

**先看一个最简单的例子**——用队列处理排队叫号：

```cpp
#include <queue>
queue<int> q;        // 创建一个整数队列，就像医院的叫号器
q.push(1);           // 1 号病人挂号（入队）
q.push(2);           // 2 号病人挂号
int first = q.front(); // 看一眼当前叫到谁（1 号），但还没叫走
q.pop();             // 正式叫走 1 号（弹出队头）
// 现在队头变成了 2 号
```

> 下面的代码做的事一样——"病人"换成了"树节点"，"叫号处理"换成了"访问节点并把它的孩子加入队列"。

```cpp
void levelOrderTraverse(TreeNode* root) {
    if (root == nullptr) return;                  // 空树，无需遍历

    queue<TreeNode*> q;                           // ⚠️ queue<TreeNode*>：存储节点指针的队列
    q.push(root);                                 // 根节点入队，作为遍历的起点

    while (!q.empty()) {                          // 只要队列不空，就继续处理
        TreeNode* cur = q.front();                // 取出队头节点（只是看一眼，还没弹出）
        q.pop();                                  // 弹出队头（注意：C++ 的 pop 不返回值！）

        // 访问当前节点
        cout << cur->val << " ";                  // 打印当前节点的值

        // 把左右子节点加入队列（下一轮会被处理）
        if (cur->left != nullptr)  q.push(cur->left);   // 左孩子存在就入队
        if (cur->right != nullptr) q.push(cur->right);   // 右孩子存在就入队
    }
}
```

**C++ 知识点**：
- `queue<TreeNode*> q`：声明一个存储指针的队列。`queue` 在 `<queue>` 头文件中。
- `q.push(x)`：入队（加到队尾）
- `q.front()`：查看队头元素（不弹出）
- `q.pop()`：弹出队头元素（不返回值）
- 注意 C++ 的 `queue` 和 Python 不一样：`front()` 和 `pop()` 是分开的

> 💡 **老师提醒：** C++ 的 `q.pop()` 只弹出不返回值！如果你写 `TreeNode* cur = q.pop()`，编译会报错。必须先 `q.front()` 取值，再 `q.pop()` 弹出。这是 C++ 初学者最常犯的错误之一。

**优点**：写法简单
**缺点**：不知道当前节点在第几层

### 3.4 写法二：记录层数（最常用！）

关键改动：**每次处理一整层**。先用 `sz` 记录当前层有多少节点，然后 for 循环处理完这一层，再进入下一层。

为什么需要一层一层处理？因为很多题目需要你知道"当前节点在第几层"，或者"这一层有哪些节点"。写法一虽然简单，但所有节点混在一起，分不清谁属于哪一层。写法二通过一个巧妙的 `sz` 变量，在不额外使用数据结构的情况下把层和层分开了。

**先看一个最简单的例子**——分批处理一堆任务：

```cpp
int total = 10;                     // 总共 10 个任务
int batch = 3;                      // 第一批处理 3 个
for (int i = 0; i < batch; i++) {   // 只处理这一批
    // 处理第 i 个任务
}
// 这一批处理完了，开始下一批
```

> 下面的代码做的事一样——`batch` 换成了 `sz`（当前层的节点数），每一批就是一层的所有节点。处理完一层，depth +1，开始下一层。

```cpp
void levelOrderTraverse(TreeNode* root) {
    if (root == nullptr) return;                  // 空树，无需遍历

    queue<TreeNode*> q;                           // 创建队列
    q.push(root);                                 // 根节点入队
    int depth = 1;                                // 当前层数（根节点是第 1 层）

    while (!q.empty()) {                          // 只要队列不空
        int sz = q.size();                        // ⚠️ 关键！在循环前保存当前层的节点数

        // 处理当前层的所有节点（恰好 sz 个）
        for (int i = 0; i < sz; i++) {            // 只处理这一层的 sz 个节点
            TreeNode* cur = q.front();            // 取出队头
            q.pop();                              // 弹出队头

            // 访问节点，同时知道它在第几层
            cout << "第 " << depth << " 层: " << cur->val << endl;

            if (cur->left != nullptr)  q.push(cur->left);    // 左孩子入队（属于下一层）
            if (cur->right != nullptr) q.push(cur->right);    // 右孩子入队（属于下一层）
        }

        depth++;                                  // 一整层处理完了，层数 +1
    }
}
```

**为什么 `sz` 必须提前保存？**

因为 for 循环里面会 `push` 新节点进队列，`q.size()` 会变。如果你写 `for (int i = 0; i < q.size(); i++)`，循环次数就不对了。比如当前层有 2 个节点，你处理第 1 个时 push 了 2 个孩子，队列从 2 变成了 3，循环就会多执行一次，把下一层的节点也处理了——层的边界就乱了。

> 💡 **老师提醒：** `int sz = q.size()` 必须写在 for 循环之前！这是写法二最容易出 bug 的地方。每次写 BFS 时先把这一行写好，再写 for 循环，养成习惯。

**图解执行过程**：

```
初始：queue = [1], depth = 1

第 1 层：sz = 1
  弹出 1，入队 2、3 → queue = [2, 3]
  输出：第 1 层: 1
depth++, depth = 2

第 2 层：sz = 2
  弹出 2，入队 4   → queue = [3, 4]
  弹出 3，入队 5、6 → queue = [4, 5, 6]
  输出：第 2 层: 2, 第 2 层: 3
depth++, depth = 3

第 3 层：sz = 3
  弹出 4、5、6，无新节点入队
  输出：第 3 层: 4, 第 3 层: 5, 第 3 层: 6
depth++, depth = 4

queue 为空，结束
```

**等价的简洁写法**（for 循环换成 while）：

```cpp
int sz = q.size();                        // 同样先保存当前层节点数
while (sz-- > 0) {                        // sz 自减到 0，效果和 for 循环相同
    TreeNode* cur = q.front();            // 取出队头
    q.pop();                              // 弹出队头
    // ...
}
```

> ✅ 写法二是 BFS 题目的标准模板，90% 的 BFS 题用它就够了！

### 3.5 写法三：每个节点自带状态（最灵活）

当每条边的权重不一样时（比如图的 BFS），写法二那样全局维护一个 `depth` 就不够用了。解决办法是让每个节点**自己记住自己的状态**。

为什么全局 depth 不够用？因为写法二假设"同一层的所有节点深度相同"，在树上这是对的，但在图上，从起点到不同节点的距离可能不同（因为边的权重不同）。所以我们需要让每个节点自己记住"我的深度/距离是多少"。

**先看一个最简单的例子**——把"人"和"信息"打包在一起：

```cpp
// 想象给每个快递包裹贴上标签
struct Package {                       // 定义一个包裹结构体
    string item;                       // 物品名称
    int weight;                        // 重量
    Package(string i, int w) : item(i), weight(w) {} // 创建包裹时指定名称和重量
};
Package p("书", 2);                    // 创建一个包裹：书，2 斤
```

> 下面的 State 做的事一样——"物品"换成了"节点指针"，"重量"换成了"深度"。每个节点入队时都带着自己的深度信息。

```cpp
// 定义一个状态类，把节点和它的深度绑在一起
struct State {
    TreeNode* node;                                    // ⚠️ 节点指针
    int depth;                                         // 这个节点的深度
    State(TreeNode* n, int d) : node(n), depth(d) {}   // ⚠️ 构造函数 + 初始化列表
};

void levelOrderTraverse(TreeNode* root) {
    if (root == nullptr) return;                       // 空树，无需遍历

    queue<State> q;                                    // ⚠️ 队列存的是 State 对象，不是裸指针
    q.push(State(root, 1));                            // 根节点深度为 1，打包入队

    while (!q.empty()) {                               // 只要队列不空
        State cur = q.front();                         // 取出队头的 State 对象
        q.pop();                                       // 弹出队头

        cout << "depth = " << cur.depth                // 从 State 中读取深度
             << ", val = " << cur.node->val << endl;   // 从 State 中读取节点值

        if (cur.node->left != nullptr)                 // 左孩子存在
            q.push(State(cur.node->left, cur.depth + 1));   // 打包入队，深度 = 父深度 + 1
        if (cur.node->right != nullptr)                // 右孩子存在
            q.push(State(cur.node->right, cur.depth + 1));  // 打包入队，深度 = 父深度 + 1
    }
}
```

**C++ 知识点**：
- `struct State`：定义一个结构体，把节点指针和深度打包在一起。struct 和 class 的区别很小，在竞赛中 struct 更常用（默认 public）。
- `queue<State>`：队列里存的不再是裸指针，而是 State 对象
- 每个子节点入队时，深度 = 父节点深度 + 1（或 + 边的权重）

**使用场景**：后面学图的 BFS 和 Dijkstra 算法时会用到。现在阶段用写法二就够了。

## 四、三种写法对比

| 写法 | 能力 | 复杂度 | 适用场景 |
|------|------|--------|---------|
| 写法一 | 只遍历，不知层数 | 代码最少 | 简单的 BFS 遍历 |
| **写法二** | **能记录层数** | **代码适中** | **绝大多数 BFS 题目** |
| 写法三 | 每个节点带自定义状态 | 需要定义额外结构体 | 带权重的图 BFS、Dijkstra |

*就像开车档位——写法一是自动挡（省事但控制少），写法二是手动挡（最常用），写法三是赛车挡（最灵活但操作复杂）。*

---

## 五、递归遍历 vs 层序遍历

| 对比维度 | 递归遍历（DFS） | 层序遍历（BFS） |
|---------|---------------|---------------|
| **遍历方向** | 先深入到底，再回头 | 一层一层横向推进 |
| **使用的数据结构** | 函数调用栈（隐式） | 队列（显式） |
| **代码形式** | 递归函数 | while + queue |
| **衍生算法** | DFS、回溯、动态规划 | BFS、最短路径 |
| **典型应用** | 路径搜索、树形 DP | 层数计算、最短路 |

**类比**：
- DFS 像是探险家：一条路走到黑，碰壁了再回来换条路
- BFS 像是水波扩散：从中心向外一圈一圈地扩展

DFS 和 BFS 没有优劣之分。选哪个取决于问题的性质——下一篇笔记会专门讲什么时候该选谁。

> ✅ 递归遍历和层序遍历是二叉树的两大武器，掌握了它们，你就能应对绝大多数树的题目！

## 六、关键概念速查表

| 概念 | 说明 |
|------|------|
| **前序位置** | `traverse(left)` 之前，刚进入节点 |
| **中序位置** | `traverse(left)` 之后、`traverse(right)` 之前 |
| **后序位置** | `traverse(right)` 之后，即将离开节点 |
| **BST 中序遍历** | 结果是有序的（从小到大） |
| **queue** | C++ 队列，`push` 入队，`front` 看队头，`pop` 弹出 |
| **sz = q.size()** | BFS 写法二中，必须在循环前保存当前层的节点数 |
| **State** | BFS 写法三中，把节点和额外信息打包的结构体 |
