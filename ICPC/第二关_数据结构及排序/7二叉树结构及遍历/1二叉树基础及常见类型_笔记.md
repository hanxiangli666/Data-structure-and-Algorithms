# 二叉树基础及常见类型 学习笔记

> **前置知识**：链表（链式存储）
> **重要程度**：⭐⭐⭐⭐⭐ 最重要的基础数据结构，没有之一
> **核心认知**：二叉树不仅是数据结构，更是一种**算法思维**——后面学的回溯、BFS、动态规划本质上都是在树上做操作

---

## 一、为什么二叉树如此重要？

### 1.1 向上：高级数据结构的地基

```
二叉树
 ├── 二叉搜索树（BST）
 │    └── 红黑树（自平衡 BST）
 ├── 二叉堆（优先队列）
 ├── 字典树（Trie）
 ├── 线段树
 ├── 并查集
 └── 图（可以看作多叉树的扩展）
```

这些高级数据结构在 ICPC 中**全部都会用到**。搞懂二叉树 = 打好地基。你可以把二叉树理解为一栋大楼的地基——上面盖什么楼（红黑树、堆、线段树），全都要建在这个地基上。地基不牢，上面的楼层学起来就会摇摇晃晃。所以这一篇的内容，值得你反复回来看，直到完全内化。

> 💡 **老师提醒：** 如果后面学红黑树或线段树感觉吃力，大概率是二叉树基础没打牢。回来重读这篇笔记往往比硬啃更有效。

### 1.2 向下：算法思维的核心

**类比**：二叉树就像是一副「X 光眼镜」。戴上它看算法题，很多问题的本质结构就显现出来了。

- **回溯算法** → 在决策树上做深度优先遍历
- **BFS 算法** → 在树上做层序遍历
- **动态规划** → 在子问题树上做自底向上的计算

为什么一棵树能串起这么多算法？因为树天然具备**递归结构**——每棵子树自己也是一棵完整的树。这意味着你可以把一个大问题拆成小问题，小问题再拆成更小的问题，直到不能再拆。这种"大化小"的思路，正是递归、分治、动态规划的本质。

学到后面你会发现，别人看到的是一串代码，你看到的是一棵树。

## 二、二叉树基础概念

### 2.1 什么是二叉树？

每个节点**最多有两个子节点**（左子节点和右子节点）的树形结构。注意是"最多"两个——一个节点可以只有左孩子没有右孩子，也可以一个孩子都没有（那就是叶子节点）。为什么限制在"两个"？因为两个分叉刚好够用，既能高效存储和查找数据，又不至于太复杂。*就像十字路口最常见的是丁字路口和十字路口，而不是八岔路口——分叉太多反而难以管理。*

```
        1          ← 根节点（root）：最顶上的，没有父节点
       / \
      2   3        ← 节点 2 是 1 的左子节点，3 是右子节点
     /   / \
    4   5   6
   /       /
  8       7
```

### 2.2 核心术语

| 术语 | 含义 | 上图中的例子 |
|------|------|-------------|
| **根节点（Root）** | 最顶上的节点，没有父节点 | 节点 1 |
| **叶子节点（Leaf）** | 最底下的节点，没有子节点 | 节点 8、5、7 |
| **父节点（Parent）** | 某节点上方直接相连的节点 | 节点 3 的父节点是 1 |
| **子节点（Child）** | 某节点下方直接相连的节点 | 节点 1 的子节点是 2 和 3 |
| **子树（Subtree）** | 以某节点为根的整棵树 | 节点 3 的左子树 = {5}，右子树 = {6, 7} |
| **深度/高度** | 从根到最深叶子经过的节点个数 | 这棵树的高度 = 4（路径 1→2→4→8） |

**类比**：把二叉树想象成一个家族族谱。根节点是老祖宗，叶子节点是最小的后代，每个人最多生两个孩子。

"子树"这个概念特别重要，后面反复会用到。你可以这样理解：从树上任意选一个节点，把它和它下面的所有后代"剪"下来，就得到了一棵子树。子树本身也是一棵完整的二叉树——这正是递归思维的起点。

> 💡 **老师提醒：** "深度"和"高度"在不同教材中定义可能不同——有的从 0 开始数，有的从 1 开始数。做题时先看清题目的定义，别想当然。

> ✅ 术语搞清楚了，接下来看看二叉树有哪些常见的"品种"！

## 三、常见的二叉树类型

### 3.1 满二叉树（Perfect Binary Tree）

**定义**：每一层都被填满，整棵树像一个正三角形。换句话说，除了最后一层的叶子节点，每个节点都恰好有两个孩子，而且所有叶子节点都在同一层上。满二叉树是最"完美"的二叉树形态，在理论分析中经常作为理想情况出现。

```
         1
       /   \
      2     3
     / \   / \
    4   5 6   7
```

**性质**：如果高度为 h，总节点数 = 2^h - 1

```
高度 1 → 1 个节点
高度 2 → 3 个节点
高度 3 → 7 个节点
高度 4 → 15 个节点
```

这个公式的来源其实很直觉：第 1 层有 1 个节点，第 2 层有 2 个，第 3 层有 4 个……每层节点数是上一层的 2 倍。加起来就是 1 + 2 + 4 + … + 2^(h-1) = 2^h - 1（等比数列求和）。

**类比**：满二叉树就像一个完美对称的金字塔，每层都被砖块铺满。

### 3.2 完全二叉树（Complete Binary Tree）

**定义**：除了最后一层，其他每层都是满的；最后一层的节点**从左到右紧凑排列**。这个"紧凑排列"是关键——意味着最后一层不能有"空洞"，所有节点必须尽可能靠左。如果最后一层只有 3 个节点，它们必须占据最左边的 3 个位置，不能跳过中间留空。

```
         1
       /   \
      2     3
     / \   /
    4   5 6       ← 最后一层从左到右排列，中间不能有空洞
```

**关键性质**：

1. **满二叉树是特殊的完全二叉树**（最后一层也满了）。*就像正方形是特殊的长方形一样——满二叉树满足完全二叉树的所有条件，而且更严格。*
2. **可以用数组存储**——因为节点紧凑排列，父子节点的索引有规律：
   ```
   父节点索引 i
   左子节点索引 = 2*i + 1
   右子节点索引 = 2*i + 2
   ```
   这个性质在学**二叉堆**和**线段树**时非常重要。为什么紧凑排列就能用数组？因为没有空洞，数组里就不会有浪费的空位。每个数组下标都对应一个真实的节点，下标之间的数学关系直接代替了指针的作用。

3. **左右子树中至少一棵是满二叉树**——这个性质可以用来巧妙地计算节点数。利用它，你可以在 O(log²N) 的时间内数出完全二叉树的节点总数，而不用逐个遍历。

**类比**：完全二叉树像是在一个剧场里坐人——从第一排开始坐，每排从左到右坐满才坐下一排，最后一排可以没坐满，但必须从左边开始坐，中间不能空座。

> 💡 **老师提醒：** 完全二叉树 ≠ 满二叉树！很多同学会混淆。记住：满二叉树每层都满，完全二叉树最后一层可以不满但必须靠左。

### 3.3 二叉搜索树（BST - Binary Search Tree）

**定义**：对于**每个节点**，它的左子树**所有**节点的值都小于它，右子树**所有**节点的值都大于它。

口诀：**「左小右大」**

这个定义看起来简单，但有一个非常容易掉的坑——请注意是"所有"，不是"直接子节点"。

```
       7            ✅ 是 BST
      / \
     4   9          4 的所有子节点 < 7，9 的所有子节点 > 7
    / \    \
   1   5   10       1,5 都 < 4 ？不对，5 > 4 → 5 是 4 的右子节点 ✓
```

**易错点**：不是只看左右子节点！要看**整棵子树**！

```
       7            ❌ 不是 BST
      / \
     4   9
    / \    \
   1   8   10       8 是 4 的右子节点 → 8 > 4 ✓
                    但 8 在 7 的左子树中 → 8 > 7 ✗ ← 违反了！
```

节点 8 虽然确实比它的父节点 4 大（放在右子节点没问题），但它在节点 7 的**左子树**中，而 8 > 7，违反了「左子树所有节点 < 根」的规则。

> 💡 **老师提醒：** 验证 BST 时，不能只看"孩子比父亲小/大"，必须看"整棵子树的所有节点都比祖先小/大"。这是 LeetCode 98 题的核心考点，很多人第一次做都会犯这个错。

**BST 的核心优势**：快速查找

```
在普通二叉树中找值为 x 的节点 → 只能遍历所有节点 → O(N)
在 BST 中找值为 x 的节点 → 每次排除一半 → O(log N)（平衡时）
```

为什么能每次排除一半？因为 BST 的左小右大性质天然帮你做了分类。当你要找的值比当前节点小，你就知道它一定在左子树里，右子树的所有节点都可以不看了。这和二分查找的思路一模一样。

**类比**：BST 就像一本按字母排序的字典。你要找"M"开头的单词，翻到中间一看是"N"，太大了，往前翻；看到是"L"，太小了，往后翻。每次都能排除一半，很快就找到了。

> ✅ BST 是后面红黑树、TreeMap 的基础，「左小右大」四个字一定要刻在脑子里！

### 3.4 高度平衡二叉树（Height-Balanced Binary Tree）

**定义**：**每个节点**的左右子树高度差不超过 1。

注意"每个节点"三个字。不是只看根节点，而是树上所有节点都必须满足这个条件。只要有一个节点的左右子树高度差超过 1，整棵树就不平衡。

```
       1            ✅ 高度平衡
      / \
     2   3          节点 1：左高 2，右高 3，差 1 ✓
    /   / \         节点 2：左高 1，右高 0，差 1 ✓
   4   5   6        节点 3：左高 2，右高 1，差 1 ✓
      /             所有节点都满足 ✓
     7
```

```
       1            ❌ 不平衡
      / \
     2   3          节点 2：左高 2，右高 0，差 2 ✗
    /   / \
   4   5   6
  /
 8
```

**关键性质**：

> 平衡二叉树有 N 个节点时，高度为 O(log N)。

这意味着在平衡的 BST 中，增删查改都是 O(log N)。为什么平衡这么重要？因为树上操作的时间复杂度通常和树的高度成正比。高度越矮，操作越快。O(log N) 意味着 100 万个节点只需要约 20 步，而不平衡的树可能需要 100 万步。*就像坐电梯（O(log N)）和爬楼梯（O(N)）的区别——楼层越多，差距越大。*

**反面教材**——极度不平衡的树退化成链表：

```
1
 \
  2
   \
    3
     \
      4
       \
        5
```

这棵树有 5 个节点，但高度也是 5。查找任何元素都要遍历，复杂度变成 O(N)，和链表一样慢。

**类比**：平衡二叉树就像一棵修剪得当的园艺树——左右对称、高度适中。不平衡的树就像一根歪脖子树，偏向一边长，失去了树形结构的优势。

### 3.5 自平衡二叉树（Self-Balanced Binary Tree）

**定义**：在插入/删除节点时，**自动调整结构**来保持平衡。

为什么需要自平衡？因为普通 BST 在最坏情况下会退化成链表（比如按顺序插入 1、2、3、4、5）。自平衡 BST 通过在每次插入/删除后自动调整，保证树永远不会"歪"太厉害，从而保证 O(log N) 的性能。

最经典的实现是**红黑树**——C++ 中 `map` 和 `set` 的底层实现。你每次用 `map` 或 `set`，背后其实就是一棵红黑树在帮你干活。

自平衡的核心操作是**旋转**（rotation），通过局部调整节点的父子关系来恢复平衡。具体原理后面会专门学，现在知道有这么回事就行。

## 四、中英文术语对照（重要！）

看英文资料或 LeetCode 英文题目时会用到，**有坑**：

| 中文 | 英文 | 说明 |
|------|------|------|
| 满二叉树 | **Perfect** Binary Tree | ⚠️ 不是 Full！ |
| 完全二叉树 | **Complete** Binary Tree | 一致 |
| — | **Full** Binary Tree | 中文没有对应名词。指每个节点要么 0 个要么 2 个孩子 |

```
Full Binary Tree（英文定义）：         Perfect Binary Tree（我们说的满二叉树）：
       1                                    1
      / \                                 /   \
     2   3        ← 每个节点              2     3
    / \              0 或 2 个孩子       / \   / \
   4   5                                4   5 6   7
```

Full Binary Tree 允许不对称，只要求「要么没孩子、要么两个孩子」。*就像"满勤"和"全优"的区别——Full 只要求"每个人要么不来、要么全天来"（0 或 2 个孩子），Perfect 要求"每个人都来了而且每天都来"（每层全满）。*

## 五、二叉树的实现方式

### 5.1 链式存储（最常见）

就像链表一样，每个节点是一个对象，通过指针连接。链表用一个 `next` 指针串成一条线，二叉树用 `left` 和 `right` 两个指针分出两条线。这是最直观、最常用的实现方式，LeetCode 上几乎所有二叉树题目都用它。

**先看一个最简单的例子**——用 class 定义一个"盒子"：

```cpp
// 想象一个快递盒子，上面写着编号，里面可以放两个更小的盒子
class Box {                  // 定义一种叫 Box 的盒子
    int number;              // 盒子上的编号
    Box* smallBox1;          // 里面的第一个小盒子（用指针指向它）
    Box* smallBox2;          // 里面的第二个小盒子
};
```

> 下面的 TreeNode 和上面的 Box 做的事一样，只是盒子里装的是"树节点"，两个小盒子分别叫 left（左孩子）和 right（右孩子）。

```cpp
class TreeNode {                                                  // 定义二叉树节点类
public:                                                           // 公开访问权限，外部可以直接使用
    int val;                                                      // 节点存储的值（就像盒子上的编号）
    TreeNode* left;                                               // ⚠️ 指针类型，指向左子节点（不是节点本身，是节点的"地址"）
    TreeNode* right;                                              // ⚠️ 指针类型，指向右子节点
    TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}   // ⚠️ 构造函数 + 初始化列表：创建节点时设置值，左右孩子初始为空
};
```

**C++ 知识点**：
- `TreeNode*`：指针类型，指向一个 TreeNode 对象。指针存的不是节点本身，而是节点在内存中的地址——就像你手里拿的是一张写着地址的纸条，而不是房子本身。
- `left(nullptr)`：初始化时左右子节点都设为空（没有孩子）。`nullptr` 是 C++11 引入的空指针，比老式的 `NULL` 更安全。
- `: val(x), left(nullptr), right(nullptr)` 这种写法叫**初始化列表**，是 C++ 构造函数特有的语法，效率比在函数体内赋值更高。
- 和链表节点对比——链表节点只有一个 `next`，二叉树节点有 `left` 和 `right`

> 💡 **老师提醒：** `TreeNode*` 后面的星号 `*` 表示这是指针。初学者经常忘记写星号，写成 `TreeNode left`，那就变成在节点里直接嵌入另一个完整的节点对象了——编译器会报错（无限递归定义）。

**构建一棵树**：

**先看一个最简单的例子**——用 new 创建对象并用指针串起来：

```cpp
// 想象你在组装一串珠子，每颗珠子有编号，用线（指针）穿起来
int* p = new int(42);    // new 在内存里创建一个值为 42 的整数，返回它的地址
// p 现在指向那个 42
```

> 下面的代码做的事一样，只是 new 创建的不是普通整数，而是 TreeNode 节点，然后用 left/right 指针把它们"穿"成一棵树。

```cpp
TreeNode* root = new TreeNode(1);           // 创建根节点，值为 1，root 指向它
root->left = new TreeNode(2);               // ⚠️ -> 是通过指针访问成员的语法（root 是指针，不能用 . ）
root->right = new TreeNode(3);              // 根节点的右孩子，值为 3
root->left->left = new TreeNode(4);         // 节点 2 的左孩子，值为 4
root->right->left = new TreeNode(5);        // 节点 3 的左孩子，值为 5
root->right->right = new TreeNode(6);       // 节点 3 的右孩子，值为 6

//     1
//    / \
//   2   3
//  /   / \
// 4   5   6
```

LeetCode 上的二叉树题目基本都用这种方式。你不需要自己写 TreeNode 的定义——题目已经帮你定义好了，你只需要操作 `root->left`、`root->right` 和 `root->val`。

> ✅ 链式存储是最核心的实现方式，后面所有二叉树题目都基于它，一定要熟练！

### 5.2 数组存储

利用完全二叉树的索引规律，直接用数组表示：

```
数组:    [1, 2, 3, 4, 5, 6]
索引:     0  1  2  3  4  5

对应的树:
         1 (索引0)
        / \
  (索引1) 2   3 (索引2)
      / \   /
     4   5 6
  (3) (4) (5)
```

索引关系：
```
节点 i 的左子节点 = 2*i + 1
节点 i 的右子节点 = 2*i + 2
节点 i 的父节点   = (i-1) / 2
```

为什么是 `2*i + 1` 和 `2*i + 2`？因为完全二叉树每层的节点数是上一层的 2 倍，而且紧凑排列不留空洞。你可以用上面的图自己验证：节点 1 在索引 0，它的左孩子 2 在索引 `2*0+1=1`，右孩子 3 在索引 `2*0+2=2`。再看节点 2（索引 1），它的左孩子 4 在索引 `2*1+1=3`——完全正确。

**使用场景**：二叉堆、线段树。后面学到时会详细讲。数组存储的优势是**省去了指针的内存开销**，而且数组在内存中是连续的，CPU 缓存命中率更高，实际运行速度更快。

> 💡 **老师提醒：** 数组存储只适用于完全二叉树。如果树不是完全的（中间有空洞），数组里就会有大量浪费的空位。

### 5.3 哈希表存储（邻接表）

用哈希表模拟树结构，key 是节点，value 是子节点列表：

**先看一个最简单的例子**——用 map 存储"谁的孩子是谁"：

```cpp
// 想象一个通讯录：查某人的名字，就能找到他孩子的名字
map<string, string> parent_child;       // key: 父亲名字, value: 孩子名字
parent_child["张三"] = "张小明";         // 张三的孩子是张小明
```

> 下面的代码做的事一样，只是通讯录里一个父亲可以有多个孩子（用 vector 存），而且名字换成了节点编号。

```cpp
unordered_map<int, vector<int>> tree;     // ⚠️ unordered_map: 哈希表，key 是节点编号，value 是子节点编号的列表
tree[1] = {2, 3};                         // 节点 1 的子节点是 2 和 3（用花括号初始化 vector）
tree[2] = {4};                            // 节点 2 的子节点是 4
tree[3] = {5, 6};                         // 节点 3 的子节点是 5 和 6
```

这种存储方式不需要定义 TreeNode 类，完全靠哈希表的 key-value 关系来表达"谁是谁的孩子"。灵活性最高，但查询效率不如指针直连。

**使用场景**：图论题目中，把图当成树来处理时经常用这种方式。这个结构后面学图论时会叫它「邻接表」。*就像链式存储是"每个人自己记住自己的孩子"，邻接表是"有一本公共花名册记录所有人的亲子关系"。*

> ✅ 三种存储方式各有用武之地：刷题用链式，堆/线段树用数组，图论用邻接表！

## 六、各类型二叉树的关系图

```
所有二叉树
 │
 ├── 完全二叉树（每层紧凑靠左）
 │    │
 │    └── 满二叉树（每层都满）  ← 满是完全的特例
 │
 ├── 二叉搜索树 BST（左小右大）
 │    │
 │    └── 自平衡 BST（红黑树等）
 │         │
 │         └── 同时也是高度平衡的
 │
 └── 高度平衡二叉树（每个节点左右高度差 ≤ 1）
```

注意：这些类型之间可以交叉。比如一棵树可以**既是 BST，又是完全二叉树**。这些分类维度是独立的——形状（完全/满）、值的排列（BST）、高度约束（平衡）是三个不同的视角。一棵树可以同时满足多个条件，也可以一个都不满足。

## 七、关键概念速查表

| 概念 | 说明 |
|------|------|
| **根节点** | 树最顶端的节点，没有父节点 |
| **叶子节点** | 树最底端的节点，没有子节点 |
| **子树** | 以某节点为根的整棵树 |
| **高度/深度** | 根到最深叶子的节点个数 |
| **满二叉树** | 每层都满，节点数 = 2^h - 1 |
| **完全二叉树** | 除最后一层外每层满，最后一层靠左排列 |
| **BST** | 左子树所有值 < 根 < 右子树所有值 |
| **平衡二叉树** | 每个节点左右子树高度差 ≤ 1，高度 O(log N) |
| **自平衡** | 插入/删除时自动调整结构保持平衡（如红黑树） |
| **TreeNode** | 二叉树节点类，包含 val、left、right |

## 八、接下来的学习路线
