# 二叉树基础及常见类型 学习笔记

> **前置知识**：链表（链式存储）  
> **重要程度**：⭐⭐⭐⭐⭐ 最重要的基础数据结构，没有之一  
> **核心认知**：二叉树不仅是数据结构，更是一种**算法思维**——后面学的回溯、BFS、动态规划本质上都是在树上做操作

---

## 一、为什么二叉树如此重要？

### 1.1 向上：高级数据结构的地基

```
二叉树
 ├── 二叉搜索树（BST）
 │    └── 红黑树（自平衡 BST）
 ├── 二叉堆（优先队列）
 ├── 字典树（Trie）
 ├── 线段树
 ├── 并查集
 └── 图（可以看作多叉树的扩展）
```

这些高级数据结构在 ICPC 中**全部都会用到**。搞懂二叉树 = 打好地基。

### 1.2 向下：算法思维的核心

**类比**：二叉树就像是一副「X 光眼镜」。戴上它看算法题，很多问题的本质结构就显现出来了。

- **回溯算法** → 在决策树上做深度优先遍历
- **BFS 算法** → 在树上做层序遍历
- **动态规划** → 在子问题树上做自底向上的计算

学到后面你会发现，别人看到的是一串代码，你看到的是一棵树。

## 二、二叉树基础概念

### 2.1 什么是二叉树？

每个节点**最多有两个子节点**（左子节点和右子节点）的树形结构。

```
        1          ← 根节点（root）：最顶上的，没有父节点
       / \
      2   3        ← 节点 2 是 1 的左子节点，3 是右子节点
     /   / \
    4   5   6
   /       / 
  8       7
```

### 2.2 核心术语

| 术语 | 含义 | 上图中的例子 |
|------|------|-------------|
| **根节点（Root）** | 最顶上的节点，没有父节点 | 节点 1 |
| **叶子节点（Leaf）** | 最底下的节点，没有子节点 | 节点 8、5、7 |
| **父节点（Parent）** | 某节点上方直接相连的节点 | 节点 3 的父节点是 1 |
| **子节点（Child）** | 某节点下方直接相连的节点 | 节点 1 的子节点是 2 和 3 |
| **子树（Subtree）** | 以某节点为根的整棵树 | 节点 3 的左子树 = {5}，右子树 = {6, 7} |
| **深度/高度** | 从根到最深叶子经过的节点个数 | 这棵树的高度 = 4（路径 1→2→4→8） |

**类比**：把二叉树想象成一个家族族谱。根节点是老祖宗，叶子节点是最小的后代，每个人最多生两个孩子。

## 三、常见的二叉树类型

### 3.1 满二叉树（Perfect Binary Tree）

**定义**：每一层都被填满，整棵树像一个正三角形。

```
         1
       /   \
      2     3
     / \   / \
    4   5 6   7
```

**性质**：如果高度为 h，总节点数 = 2^h - 1

```
高度 1 → 1 个节点
高度 2 → 3 个节点
高度 3 → 7 个节点
高度 4 → 15 个节点
```

**类比**：满二叉树就像一个完美对称的金字塔，每层都被砖块铺满。

### 3.2 完全二叉树（Complete Binary Tree）

**定义**：除了最后一层，其他每层都是满的；最后一层的节点**从左到右紧凑排列**。

```
         1
       /   \
      2     3
     / \   /
    4   5 6       ← 最后一层从左到右排列，中间不能有空洞
```

**关键性质**：

1. **满二叉树是特殊的完全二叉树**（最后一层也满了）
2. **可以用数组存储**——因为节点紧凑排列，父子节点的索引有规律：
   ```
   父节点索引 i
   左子节点索引 = 2*i + 1
   右子节点索引 = 2*i + 2
   ```
   这个性质在学**二叉堆**和**线段树**时非常重要。

3. **左右子树中至少一棵是满二叉树**——这个性质可以用来巧妙地计算节点数。

**类比**：完全二叉树像是在一个剧场里坐人——从第一排开始坐，每排从左到右坐满才坐下一排，最后一排可以没坐满，但必须从左边开始坐，中间不能空座。

### 3.3 二叉搜索树（BST - Binary Search Tree）

**定义**：对于**每个节点**，它的左子树**所有**节点的值都小于它，右子树**所有**节点的值都大于它。

口诀：**「左小右大」**

```
       7            ✅ 是 BST
      / \
     4   9          4 的所有子节点 < 7，9 的所有子节点 > 7
    / \    \
   1   5   10       1,5 都 < 4 ？不对，5 > 4 → 5 是 4 的右子节点 ✓
```

**易错点**：不是只看左右子节点！要看**整棵子树**！

```
       7            ❌ 不是 BST
      / \
     4   9
    / \    \
   1   8   10       8 是 4 的右子节点 → 8 > 4 ✓
                    但 8 在 7 的左子树中 → 8 > 7 ✗ ← 违反了！
```

节点 8 虽然确实比它的父节点 4 大（放在右子节点没问题），但它在节点 7 的**左子树**中，而 8 > 7，违反了「左子树所有节点 < 根」的规则。

**BST 的核心优势**：快速查找

```
在普通二叉树中找值为 x 的节点 → 只能遍历所有节点 → O(N)
在 BST 中找值为 x 的节点 → 每次排除一半 → O(log N)（平衡时）
```

**类比**：BST 就像一本按字母排序的字典。你要找"M"开头的单词，翻到中间一看是"N"，太大了，往前翻；看到是"L"，太小了，往后翻。每次都能排除一半，很快就找到了。

### 3.4 高度平衡二叉树（Height-Balanced Binary Tree）

**定义**：**每个节点**的左右子树高度差不超过 1。

```
       1            ✅ 高度平衡
      / \
     2   3          节点 1：左高 2，右高 3，差 1 ✓
    /   / \         节点 2：左高 1，右高 0，差 1 ✓
   4   5   6        节点 3：左高 2，右高 1，差 1 ✓
      /             所有节点都满足 ✓
     7
```

```
       1            ❌ 不平衡
      / \
     2   3          节点 2：左高 2，右高 0，差 2 ✗
    /   / \
   4   5   6
  /
 8
```

**关键性质**：

> 平衡二叉树有 N 个节点时，高度为 O(log N)。

这意味着在平衡的 BST 中，增删查改都是 O(log N)。

**反面教材**——极度不平衡的树退化成链表：

```
1
 \
  2
   \
    3
     \
      4
       \
        5
```

这棵树有 5 个节点，但高度也是 5。查找任何元素都要遍历，复杂度变成 O(N)，和链表一样慢。

**类比**：平衡二叉树就像一棵修剪得当的园艺树——左右对称、高度适中。不平衡的树就像一根歪脖子树，偏向一边长，失去了树形结构的优势。

### 3.5 自平衡二叉树（Self-Balanced Binary Tree）

**定义**：在插入/删除节点时，**自动调整结构**来保持平衡。

最经典的实现是**红黑树**——C++ 中 `map` 和 `set` 的底层实现。

自平衡的核心操作是**旋转**（rotation），通过局部调整节点的父子关系来恢复平衡。具体原理后面会专门学，现在知道有这么回事就行。

## 四、中英文术语对照（重要！）

看英文资料或 LeetCode 英文题目时会用到，**有坑**：

| 中文 | 英文 | 说明 |
|------|------|------|
| 满二叉树 | **Perfect** Binary Tree | ⚠️ 不是 Full！ |
| 完全二叉树 | **Complete** Binary Tree | 一致 |
| — | **Full** Binary Tree | 中文没有对应名词。指每个节点要么 0 个要么 2 个孩子 |

```
Full Binary Tree（英文定义）：         Perfect Binary Tree（我们说的满二叉树）：
       1                                    1
      / \                                 /   \
     2   3        ← 每个节点              2     3
    / \              0 或 2 个孩子       / \   / \
   4   5                                4   5 6   7
```

Full Binary Tree 允许不对称，只要求「要么没孩子、要么两个孩子」。

## 五、二叉树的实现方式

### 5.1 链式存储（最常见）

就像链表一样，每个节点是一个对象，通过指针连接。

```cpp
class TreeNode {
public:
    int val;
    TreeNode* left;     // 指向左子节点
    TreeNode* right;    // 指向右子节点
    TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
};
```

**C++ 知识点**：
- `TreeNode*`：指针类型，指向一个 TreeNode 对象
- `left(nullptr)`：初始化时左右子节点都设为空（没有孩子）
- 和链表节点对比——链表节点只有一个 `next`，二叉树节点有 `left` 和 `right`

**构建一棵树**：

```cpp
TreeNode* root = new TreeNode(1);
root->left = new TreeNode(2);
root->right = new TreeNode(3);
root->left->left = new TreeNode(4);
root->right->left = new TreeNode(5);
root->right->right = new TreeNode(6);

//     1
//    / \
//   2   3
//  /   / \
// 4   5   6
```

LeetCode 上的二叉树题目基本都用这种方式。

### 5.2 数组存储

利用完全二叉树的索引规律，直接用数组表示：

```
数组:    [1, 2, 3, 4, 5, 6]
索引:     0  1  2  3  4  5

对应的树:
         1 (索引0)
        / \
  (索引1) 2   3 (索引2)
      / \   /
     4   5 6
  (3) (4) (5)
```

索引关系：
```
节点 i 的左子节点 = 2*i + 1
节点 i 的右子节点 = 2*i + 2
节点 i 的父节点   = (i-1) / 2
```

**使用场景**：二叉堆、线段树。后面学到时会详细讲。

### 5.3 哈希表存储（邻接表）

用哈希表模拟树结构，key 是节点，value 是子节点列表：

```cpp
unordered_map<int, vector<int>> tree;
tree[1] = {2, 3};     // 节点 1 的子节点是 2 和 3
tree[2] = {4};         // 节点 2 的子节点是 4
tree[3] = {5, 6};      // 节点 3 的子节点是 5 和 6
```

**使用场景**：图论题目中，把图当成树来处理时经常用这种方式。这个结构后面学图论时会叫它「邻接表」。

## 六、各类型二叉树的关系图

```
所有二叉树
 │
 ├── 完全二叉树（每层紧凑靠左）
 │    │
 │    └── 满二叉树（每层都满）  ← 满是完全的特例
 │
 ├── 二叉搜索树 BST（左小右大）
 │    │
 │    └── 自平衡 BST（红黑树等）
 │         │
 │         └── 同时也是高度平衡的
 │
 └── 高度平衡二叉树（每个节点左右高度差 ≤ 1）
```

注意：这些类型之间可以交叉。比如一棵树可以**既是 BST，又是完全二叉树**。

## 七、关键概念速查表

| 概念 | 说明 |
|------|------|
| **根节点** | 树最顶端的节点，没有父节点 |
| **叶子节点** | 树最底端的节点，没有子节点 |
| **子树** | 以某节点为根的整棵树 |
| **高度/深度** | 根到最深叶子的节点个数 |
| **满二叉树** | 每层都满，节点数 = 2^h - 1 |
| **完全二叉树** | 除最后一层外每层满，最后一层靠左排列 |
| **BST** | 左子树所有值 < 根 < 右子树所有值 |
| **平衡二叉树** | 每个节点左右子树高度差 ≤ 1，高度 O(log N) |
| **自平衡** | 插入/删除时自动调整结构保持平衡（如红黑树） |
| **TreeNode** | 二叉树节点类，包含 val、left、right |

## 八、接下来的学习路线

学完这篇基础概念后，二叉树的学习路线大致是：

1. **二叉树的遍历**（前序、中序、后序、层序）← 下一步重点
2. **递归思维训练**（在二叉树上练递归是最好的方式）
3. **BST 的操作**（插入、删除、查找）
4. **二叉堆**（优先队列，用数组存储的完全二叉树）
5. **高级树结构**（红黑树、字典树、线段树等）

现阶段的目标：**把基础概念记牢，能在纸上画出各种类型的二叉树，搞清楚它们的区别**。
