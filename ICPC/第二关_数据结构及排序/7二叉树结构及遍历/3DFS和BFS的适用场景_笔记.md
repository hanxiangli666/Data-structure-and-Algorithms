# DFS 和 BFS 的适用场景 学习笔记

> **前置知识**：二叉树的递归遍历和层序遍历（上一篇笔记）  
> **核心结论**：**找最短路径用 BFS，找所有路径用 DFS**  
> **配套例题**：LeetCode 111 - 二叉树的最小深度

---

## 一、一句话总结

| 问题类型 | 首选算法 | 原因 |
|---------|---------|------|
| **找最短路径** | BFS | 逐层推进，第一次到达目标就是最短路径，可以提前结束 |
| **找所有路径** | DFS | 天然一条树枝一条树枝遍历，代码简洁 |

这两条结论在整个算法学习中会反复用到，先记住，下面用例题帮你理解为什么。

## 二、例题：二叉树的最小深度（LeetCode 111）

### 2.1 题目

> 给定一棵二叉树，找出其**最小深度**。最小深度是从根节点到**最近的叶子节点**的最短路径上的节点数量。

```
示例 1：
        3
       / \
      9  20
        /  \
       15   7

最小深度 = 2（路径 3 → 9）

示例 2：
    2
     \
      3
       \
        4
         \
          5
           \
            6

最小深度 = 5（只有一条路径，必须走到底）
```

本质：找根节点到最近叶子节点的**最短距离**。

---

## 三、DFS 解法：遍历所有树枝，找最短的那条

### 3.1 思路

DFS 就像一个探险家，**把每条路都走一遍**，记录每条路的长度，最后挑最短的。

```
        3
       / \
      9  20
        /  \
       15   7

DFS 的遍历路线（先左后右）：
  路径 1: 3 → 9       深度 = 2  ← 记录
  路径 2: 3 → 20 → 15  深度 = 3  ← 记录
  路径 3: 3 → 20 → 7   深度 = 3  ← 记录

最小深度 = min(2, 3, 3) = 2
```

**必须走完所有路才能确定最小值**——这是 DFS 找最短路径的缺点。

### 3.2 代码

```cpp
class Solution {
private:
    int minDepthValue = INT_MAX;  // 记录最小深度，初始化为最大值
    int currentDepth = 0;         // 记录当前走到第几层

public:
    int minDepth(TreeNode* root) {
        if (root == nullptr) return 0;
        traverse(root);
        return minDepthValue;
    }

    void traverse(TreeNode* root) {
        if (root == nullptr) return;

        // 【前序位置】进入节点，深度 +1
        currentDepth++;

        // 判断是否是叶子节点（左右孩子都为空）
        if (root->left == nullptr && root->right == nullptr) {
            minDepthValue = min(minDepthValue, currentDepth);
        }

        traverse(root->left);
        traverse(root->right);

        // 【后序位置】离开节点，深度 -1（回溯）
        currentDepth--;
    }
};
```

### 3.3 代码解读

**`currentDepth` 的变化过程**（以示例 1 为例）：

```
进入 3:  currentDepth = 1
  进入 9:  currentDepth = 2
    9 是叶子节点！minDepthValue = min(INT_MAX, 2) = 2
  离开 9:  currentDepth = 1     ← 回溯！深度减回来
  进入 20: currentDepth = 2
    进入 15: currentDepth = 3
      15 是叶子节点！minDepthValue = min(2, 3) = 2（没变）
    离开 15: currentDepth = 2
    进入 7:  currentDepth = 3
      7 是叶子节点！minDepthValue = min(2, 3) = 2（没变）
    离开 7:  currentDepth = 2
  离开 20: currentDepth = 1
离开 3:  currentDepth = 0
```

**关键模式——前序位置 +1，后序位置 -1**：

这个「进入时 +1，离开时 -1」的操作叫做**回溯**。它保证了 `currentDepth` 始终准确反映当前节点的深度。

**类比**：你在爬楼梯探索一栋楼。每上一层楼（进入子节点）就在计数器上 +1，每下一层楼（返回父节点）就 -1。这样计数器永远显示你当前在第几层。

**C++ 知识点**：
- `INT_MAX`：`<climits>` 中定义的 int 最大值（约 21 亿），用来初始化最小值
- `min(a, b)`：返回两者中较小的那个

### 3.4 判断叶子节点

```cpp
if (root->left == nullptr && root->right == nullptr)
```

叶子节点 = **左右孩子都为空**的节点。这是一个常见的判断条件，要记住。

⚠️ 常见错误：只判断 `root == nullptr` 就认为到底了。但 `nullptr` 不是叶子节点！`nullptr` 只是表示「这条路走不通」，叶子节点是一个**真实存在但没有孩子**的节点。

## 四、BFS 解法：逐层搜索，找到就停

### 4.1 思路

BFS 就像水波扩散，**一层一层向外推进**。第一次碰到叶子节点时，它一定是距离根节点最近的那个，立刻返回就行。

```
        3          ← 第 1 层：只有 3，不是叶子，继续
       / \
      9  20        ← 第 2 层：9 是叶子！立刻返回 depth = 2
        /  \
       15   7      ← 第 3 层：根本不需要遍历到这里
```

**不需要遍历整棵树**——这就是 BFS 找最短路径的优势。

### 4.2 代码

```cpp
class Solution {
public:
    int minDepth(TreeNode* root) {
        if (root == nullptr) return 0;
        
        queue<TreeNode*> q;
        q.push(root);
        int depth = 1;   // root 本身就是第 1 层
        
        while (!q.empty()) {
            int sz = q.size();
            
            for (int i = 0; i < sz; i++) {
                TreeNode* cur = q.front();
                q.pop();
                
                // 碰到叶子节点，立刻返回！
                if (cur->left == nullptr && cur->right == nullptr) {
                    return depth;   // ← 提前结束，不用遍历完
                }
                
                if (cur->left != nullptr)  q.push(cur->left);
                if (cur->right != nullptr) q.push(cur->right);
            }
            
            depth++;
        }
        return depth;
    }
};
```

### 4.3 代码解读

这就是上一篇学的 BFS **写法二**的直接应用。唯一的区别是：在遍历每个节点时多了一个判断——如果当前节点是叶子，直接 `return depth`。

**图解执行过程**（示例 1）：

```
初始：queue = [3], depth = 1

第 1 层：sz = 1
  弹出 3 → 不是叶子（有左右孩子）
  入队 9、20 → queue = [9, 20]
depth = 2

第 2 层：sz = 2
  弹出 9 → 是叶子（左右都空）！
  return depth = 2  ← 直接返回，20 都没处理就结束了！
```

对比 DFS：DFS 需要遍历完 3 条路径（6 个节点）才能确定答案。BFS 只遍历了 2 个节点就结束了。

## 五、核心对比：为什么 BFS 更适合找最短路径？

### 5.1 直觉理解

**类比**：你在一个迷宫里找出口。

- **DFS（深度优先）**：你选一条路一直走到底。如果是死胡同就回来换一条路。你必须把所有路都试完，才知道哪条最短。
- **BFS（广度优先）**：你像一滴水一样向四周扩散。先探索距离 1 步的所有位置，再探索距离 2 步的所有位置……第一次碰到出口时，当前的步数就一定是最短距离。

### 5.2 数学解释

BFS 逐层遍历，保证了：

> 当你第一次遇到目标时，经过的层数就是最少的。

因为如果存在更短的路径，那个叶子节点会在更早的层被发现。BFS 是按层数从小到大遍历的，所以第一个被发现的叶子一定最浅。

### 5.3 复杂度对比

| 情况 | DFS | BFS |
|------|-----|-----|
| **最坏情况** | 遍历所有 N 个节点，O(N) | 遍历所有 N 个节点，O(N) |
| **一般情况（找最短路径）** | 必须遍历完才能确定 | 找到就停，通常更快 |
| **空间复杂度** | O(h)，h 是树的高度（递归栈深度） | O(w)，w 是树的最大宽度（队列大小） |

## 六、为什么 DFS 更适合找所有路径？

### 6.1 DFS 天然适合

DFS 的遍历方式就是**一条树枝一条树枝**地走：

```
        1
       / \
      2   3
     /   / \
    4   5   6

DFS 遍历路线：
  树枝 1: 1 → 2 → 4  ← 到叶子了，记录这条路径
  树枝 2: 1 → 3 → 5  ← 到叶子了，记录这条路径
  树枝 3: 1 → 3 → 6  ← 到叶子了，记录这条路径

所有路径收集完毕！
```

代码只需要在递归遍历框架的基础上，维护一个「当前路径」变量，到叶子节点时记录一下就行了。

### 6.2 BFS 找所有路径很麻烦

如果用 BFS 找所有路径，问题在于：BFS 是一层一层遍历的，当你处理某个节点时，你已经不记得「怎么走到这个节点的」了。

解决办法是用之前学的 BFS **写法三**，给每个节点附带一个 State，把完整的路径信息存进去：

```cpp
struct State {
    TreeNode* node;
    vector<int> path;  // 从根到当前节点的完整路径
};
```

但这样每个节点都要存一份完整路径的拷贝，既费内存又复杂。

### 6.3 总结

| 找所有路径 | DFS | BFS |
|-----------|-----|-----|
| **实现难度** | 简单，天然适合 | 复杂，需要额外维护路径 |
| **空间开销** | 只需维护一条当前路径 | 每个队列元素都要带完整路径 |
| **代码量** | 少 | 多 |

## 七、总结口诀

```
找最短 → BFS（水波扩散，碰到就停）
找所有 → DFS（逐条探索，走完全程）
```

这个结论不仅适用于二叉树，后面学到**图论**、**迷宫问题**、**状态搜索**时，同样适用。

## 八、关键概念速查表

| 概念 | 说明 |
|------|------|
| **叶子节点** | `left == nullptr && right == nullptr` |
| **回溯** | 前序位置 +1，后序位置 -1，保证状态正确 |
| **INT_MAX** | C++ 中 int 的最大值，常用于初始化最小值变量 |
| **BFS 提前终止** | 找到目标就 `return`，不用遍历完 |
| **DFS 必须遍历完** | 找最短路径时，DFS 无法提前结束 |
