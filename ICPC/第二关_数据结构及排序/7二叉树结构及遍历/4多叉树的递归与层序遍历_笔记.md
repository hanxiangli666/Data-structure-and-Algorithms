# 多叉树的递归/层序遍历 学习笔记

> **前置知识**：二叉树的递归遍历和层序遍历  
> **核心认知**：多叉树就是二叉树的自然延伸——把「左右两个孩子」变成「任意多个孩子」，其他一模一样  
> **新概念**：森林 = 多棵多叉树的集合

---

## 一、从二叉树到多叉树：只有一处改动

### 1.1 节点结构对比

二叉树到多叉树，真正变的只有“孩子的存法”，不变的是“父子关系”和“遍历思想”。你把这个核心抓住，后面的 DFS、BFS 都只是小改动。*就像从两格书包换成多格书包，装法变了，背包这件事没变。*

> 💡 **老师提醒：** 不要被“多叉树”这个名字吓到，它不是新体系，而是二叉树模板的扩展版。

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
int leftBox = 2;                        // 二叉树像固定两个盒子之一：左盒子
int rightBox = 3;                       // 另一个是右盒子
vector<int> boxes = {2, 3, 4, 5};      // 多叉树像一个大收纳盒，可放任意多个编号
```

> 这个小例子说明“固定两个成员”和“用数组装多个成员”的区别；下面的节点定义就是这个思路在树结构里的版本。

**第三步：给原始代码加注释**

```cpp
// 二叉树节点：固定 2 个子节点
class TreeNode {
public:
    int val;             // 节点值
    TreeNode* left;      // 左孩子指针
    TreeNode* right;     // 右孩子指针
};

// 多叉树节点：任意个子节点
class Node {
public:
    int val;                       // 节点值
    vector<Node*> children;       // ⚠️ 用 vector 存任意多个孩子指针
};
```

**就这点区别，其他全部一样。**

**类比**：二叉树像一个「每个人最多生两个孩子」的家庭；多叉树像「每个人可以生任意多个孩子」的家庭。家族的遍历方式（挨个拜访）并没有本质变化，只是每家要拜访的孩子数量不同。

你学这一节的目标，不是背结构体代码，而是形成迁移能力：以后看到“固定个数”就写成员，看到“可变个数”就写容器。这个判断会在图、Trie、并查集扩展里反复出现。

> ✅ 你已经跨过了一个关键门槛：会把旧知识迁移到新结构。

### 1.2 图示对比

```
二叉树：                    多叉树：
      1                         1
     / \                     /  |  \
    2   3                   2   3   4
   /   / \                 /|\     / \
  4   5   6               5 6 7   8   9
```

## 二、递归遍历（DFS）

### 2.1 框架对比

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
for (int i = 0; i < 2; i++) { }            // 二叉树像固定拜访 2 个房间
vector<int> rooms = {101, 102, 103};       // 多叉树像一层有多个房间
for (int room : rooms) { }                 // 逐个房间拜访
```

> 上面的最小例子对应下面 DFS 的变化：从“写死两个递归调用”变成“循环递归所有孩子”。

**第三步：给原始代码加注释**

```cpp
// 二叉树 DFS
void traverse(TreeNode* root) {
    if (root == nullptr) return;      // 空节点直接返回
    // 【前序位置】                      // 进入节点时可处理逻辑
    traverse(root->left);             // 递归左子树
    // 【中序位置】                      // 左右子树之间的位置（仅二叉树有明确含义）
    traverse(root->right);            // 递归右子树
    // 【后序位置】                      // 子树都处理完后再处理当前节点
}

// 多叉树 DFS
void traverse(Node* root) {
    if (root == nullptr) return;      // 空节点直接返回
    // 【前序位置】                      // 进入节点时可处理逻辑
    for (Node* child : root->children) {
        traverse(child);              // ⚠️ 逐个孩子递归，而不是只写 left/right
    }
    // 【后序位置】                      // 所有孩子处理完后的收尾位置
}
```

**两个变化**：

| 变化 | 二叉树 | 多叉树 |
|------|--------|--------|
| 递归调用 | 写死两行：`traverse(left)` + `traverse(right)` | 一个 for 循环遍历所有孩子 |
| 中序位置 | 有（左子树和右子树之间） | **没有**（孩子有多个，"中间"没有意义） |

**C++ 知识点**：
- `for (Node* child : root->children)`：范围 for 循环（range-based for），遍历 vector 中的每个元素
- 相当于 Python 的 `for child in root.children`

为什么多叉树没有中序位置？因为“中序”依赖于“左边一个、右边一个”的二分结构，多叉场景没有唯一的“中间”。*就像两扇门中间有走廊，但五扇门就没有唯一中间门。*

> 💡 **老师提醒：** 迁移二叉树模板时，不要硬套“中序”，多叉树通常只谈前序和后序。

### 2.2 前序遍历示例

```
多叉树：
        1
      / | \
     2  3  4
    /|     |
   5 6     7
```

```cpp
void traverse(Node* root) {
    if (root == nullptr) return;         // 空节点直接返回
    cout << root->val << " ";           // 前序位置：进入节点时记录当前值
    for (Node* child : root->children) {
        traverse(child);                 // 依次递归每个孩子
    }
}
```

遍历过程（和二叉树一样，先深入到底再回头）：

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
vector<int> line = {1, 2, 3};          // 三个同学排成一列
for (int x : line) {                   // 从前到后点名
    cout << x << " ";                  // 先报到，再看下一个
}
```

> 这个“先报到再往下走”的顺序，就是前序遍历的直觉版本；下面的过程图只是把“同学”换成了树节点。

```
进入 1 → 记录 1
  进入 2 → 记录 2
    进入 5 → 记录 5（叶子，返回）
    进入 6 → 记录 6（叶子，返回）
  返回到 1
  进入 3 → 记录 3（叶子，返回）
  进入 4 → 记录 4
    进入 7 → 记录 7（叶子，返回）
  返回到 1

前序结果：[1, 2, 5, 6, 3, 4, 7]
```

### 2.3 后序遍历示例

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
vector<int> homework = {1, 2, 3};      // 三份作业按顺序做
for (int i = 2; i >= 0; i--) {         // 都做完后，按完成顺序回看
    cout << homework[i] << " ";         // 最后处理最先安排的那份
}
```

> 这个例子表达的是“先处理子任务，最后处理总任务”；下面代码里总任务就是当前节点。

**第三步：给原始代码加注释**

```cpp
void traverse(Node* root) {
    if (root == nullptr) return;         // 空节点直接返回
    for (Node* child : root->children) {
        traverse(child);                 // 先把所有孩子都处理完
    }
    cout << root->val << " ";           // 后序位置：所有孩子遍历完才记录当前节点
}
```

```
后序结果：[5, 6, 2, 3, 7, 4, 1]
```

每个节点都是在它所有子节点遍历完之后才被记录，所以根节点 1 最后出现。

这个顺序特别适合“先算子问题，再合并答案”的题型，比如统计子树信息。因为你处理父节点时，孩子的信息已经齐了。*就像家长开家长会前，先收齐每个孩子成绩单。*

> ✅ 你已经理解了 DFS 两种记录时机的差异：前序先记，后序后记。

## 三、层序遍历（BFS）

和二叉树完全一样，只是把「加入左右孩子」换成「遍历所有孩子加入队列」。

### 3.1 写法一：最简单

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
queue<int> q;                       // 排队取号
q.push(1001);                       // 1001 号入队
int id = q.front(); q.pop();        // 叫到队首并出队
```

> 这个最小例子就是 BFS 的骨架；下面代码只是把号码换成了节点指针，并补上了“把孩子继续入队”。

**第三步：给原始代码加注释**

```cpp
void levelOrderTraverse(Node* root) {
    if (root == nullptr) return;     // 空树直接返回
    
    queue<Node*> q;                  // 队列用于层序遍历
    q.push(root);                    // 根节点先入队
    
    while (!q.empty()) {
        Node* cur = q.front();       // 取出当前队首节点
        q.pop();                     // 队首出队
        
        cout << cur->val << " ";    // 访问当前节点
        
        // 二叉树：加入左右孩子
        // 多叉树：遍历所有孩子加入
        for (Node* child : cur->children) {
            q.push(child);           // 所有孩子按顺序入队
        }
    }
}
```

改动只有一处：把 `if (left) push(left); if (right) push(right);` 换成了 `for (child : children) push(child);`

你可以记成一句话：BFS 的流程不变，变的只是“找邻居”的方式。为什么这个记忆法有用？因为做题时你常常换图、树、状态图，但 BFS 框架都能直接复用。*就像排队规则不变，只是每次来的队伍类型不同。*

> 💡 **老师提醒：** BFS 最常见 bug 是漏入队某个孩子，导致答案偏小或节点丢失。

### 3.2 写法二：记录层数（最常用）

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
int floor = 1;                         // 当前在第 1 层
int people = 3;                        // 这一层有 3 个人
for (int i = 0; i < people; i++) { }  // 先把这一层都处理完
floor++;                               // 再进入下一层
```

> 上面的结构就是“按层处理”的本质；下面代码用 `q.size()` 来得到每层人数。

**第三步：给原始代码加注释**

```cpp
void levelOrderTraverse(Node* root) {
    if (root == nullptr) return;      // 空树直接返回
    
    queue<Node*> q;                   // 队列用于层序遍历
    q.push(root);                     // 根节点入队
    int depth = 1;                    // 根节点所在层是第 1 层
    
    while (!q.empty()) {
        int sz = q.size();            // 当前层的节点数（先固定，防止层混乱）
        for (int i = 0; i < sz; i++) {
            Node* cur = q.front();    // 取当前层的一个节点
            q.pop();                  // 出队
            
            cout << "depth=" << depth << " val=" << cur->val << endl; // 输出当前节点和层数
            
            for (Node* child : cur->children) {
                q.push(child);        // 下一层节点入队
            }
        }
        depth++;                      // 一整层处理完，层数加一
    }
}
```

和二叉树的写法二一模一样的模式。

你会发现，写法二是竞赛和面试里最稳的模板：既能遍历，又能直接拿到层数。遇到“最小步数”“第几层第一次出现”这类题，它很顺手。

> ✅ 你的 BFS 模板已经可以直接迁移到图搜索题了。

### 3.3 写法三：每个节点自带状态

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
struct Item { int id; int depth; };   // 每个排队的人都带上编号和所在楼层
queue<Item> q;                         // 队列里放“带信息的人”
q.push(Item{10, 1});                   // 10 号在第 1 层
```

> 这个小例子和下面完全同一个思路：把节点和它的额外信息打包进队列。

**第三步：给原始代码加注释**

```cpp
struct State {
    Node* node;                                 // 当前节点指针
    int depth;                                  // 当前节点所在层数
    State(Node* n, int d) : node(n), depth(d) {} // 构造函数，初始化两个字段
};

void levelOrderTraverse(Node* root) {
    if (root == nullptr) return;                // 空树直接返回
    
    queue<State> q;                              // 队列中保存带状态的元素
    q.push(State(root, 1));                      // 根节点和层数 1 一起入队
    
    while (!q.empty()) {
        State cur = q.front();                   // 取出当前状态
        q.pop();                                 // 出队
        
        cout << "depth=" << cur.depth << " val=" << cur.node->val << endl; // 使用状态中的层数和值
        
        for (Node* child : cur.node->children) {
            q.push(State(child, cur.depth + 1)); // 孩子入队，层数在父层基础上 +1
        }
    }
}
```

## 四、新概念：森林

### 4.1 什么是森林？

**森林 = 多棵树的集合**。

```
  树1       树2       树3
   1         5         8
  / \       /|\        |
 2   3     6 7 9      10
 |
 4
```

用代码表示就是一个根节点的列表：

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
vector<int> classes = {1, 2, 3};      // 一所学校有 3 个班
int first = classes[0];                // 每个班都有自己的班长（根）
```

> 这个例子想表达“集合里每个元素都是一个独立整体”；下面的 `forest` 里每个元素就是一棵树的根。

**第三步：给原始代码加注释**

```cpp
vector<Node*> forest;   // ⚠️ 森林：每个元素都是一棵独立树的根节点
```

**单独一棵树也是一个特殊的森林**（只有一个根节点的森林）。

这个定义很重要，因为它让很多算法边界条件更统一。你把“单棵树”看成“森林大小为 1”，很多代码就不用分情况写。*就像一个班级也可以看成“只有一个班级的学校”。*

> 💡 **老师提醒：** 建模时优先追求统一表达，后面写代码会少很多 `if` 分支。

### 4.2 遍历森林

对每棵树分别遍历即可：

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
vector<int> groups = {101, 102};       // 两个小组
for (int g : groups) {                 // 依次通知每个小组
    (void)g;                            // 这里只演示遍历动作
}
```

> 上面的循环和下面完全一致：区别只是元素类型从 `int` 变成了树根指针。

**第三步：给原始代码加注释**

```cpp
// DFS 遍历整个森林
for (Node* root : forest) {
    traverse(root);               // 对当前这棵树做 DFS
}

// BFS 遍历整个森林
for (Node* root : forest) {
    levelOrderTraverse(root);     // 对当前这棵树做 BFS
}
```

### 4.3 森林在哪里用到？

后面学**并查集（Union-Find）**时会用到。并查集维护的就是一个森林——多棵树，每棵树的根节点代表一个集合。现在先知道这个概念就行。

这句话你先记住，后面会突然“连起来”：并查集的 `find` 找根，根就是集合代表。为什么提前埋这个概念？因为你到并查集那一章时，会更快理解路径压缩在优化什么。*就像提前看地图，到了现场就不会迷路。*

> ✅ 你已经把“树”和“森林”这两个层级区分清楚了，后面会越学越顺。

## 五、二叉树 → 多叉树 改动速查表

| 操作 | 二叉树写法 | 多叉树写法 |
|------|-----------|-----------|
| 节点定义 | `left`, `right` 两个指针 | `vector<Node*> children` |
| DFS 递归调用 | `traverse(left); traverse(right);` | `for (child : children) traverse(child);` |
| 中序位置 | 有 | **无** |
| BFS 入队 | `if(left) push(left); if(right) push(right);` | `for (child : children) push(child);` |
| 其他逻辑 | — | **完全一样** |
