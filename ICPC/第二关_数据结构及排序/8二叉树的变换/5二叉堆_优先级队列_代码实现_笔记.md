## 一、为什么用数组而不用 TreeNode？

### 1.1 两个原因

| 原因 | 说明 |
|------|------|
| **省空间** | TreeNode 需要额外的 left/right/parent 指针，数组不需要 |
| **省时间（关键！）** | push 和 pop 都需要「找到树最底层最右边的元素」。用 TreeNode 要遍历整棵树 O(N)；用数组直接取末尾元素 O(1) |

这两点不是写法偏好，而是复杂度层面的关键差异。因为完全二叉树和数组下标天然匹配，很多本来要靠指针遍历的动作就能 O(1) 定位。你理解了这一层，后面写堆题会更稳。

> 💡 **老师提醒：** 看到“完全二叉树”时，先想数组表示。

### 1.2 数组怎么模拟二叉树？

因为二叉堆是**完全二叉树**（节点紧凑排列），所以可以用数组严格对应：

```
数组（0-based）：[9, 7, 8, 3, 5, 2]
索引：            0  1  2  3  4  5

对应的树：
         9 (0)
        / \
   (1) 7   8 (2)
      / \ /
  (3)3  5 2(5)
      (4)
```

**索引公式（0-based，本文使用的方式）**：

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
int room = 5;                          // 像 5 号房间
int parentRoom = (room - 1) / 2;       // 找它的上级房间
cout << parentRoom;                    // 输出上级编号
```

> 上面是“编号找父节点”的最小动作，下面代码就是堆里常用的 0-based 下标公式。

```cpp
int parent(int i) { return (i - 1) / 2; } // 父节点下标
int left(int i)   { return 2 * i + 1; }   // 左子节点下标
int right(int i)  { return 2 * i + 2; }   // 右子节点下标
```

**索引公式（1-based，索引 0 空着不用）**：

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
int seat = 6;                          // 1-based 编号
int parentSeat = seat / 2;             // 找父节点
cout << parentSeat;                    // 输出父节点编号
```

> 这个小例子说明 1-based 公式更直观，下面是完整对应关系。

```cpp
int parent(int i) { return i / 2; }     // 父节点下标
int left(int i)   { return 2 * i; }     // 左子节点下标
int right(int i)  { return 2 * i + 1; } // 右子节点下标
```

两种都行，效果一样。很多教材用 1-based（公式更简洁），本文和 C++ 标准库用 0-based。

*就像“门牌从 0 编号”和“门牌从 1 编号”的区别。规则不同，但都能找到同一间房。*

> ✅ 固定一种下标体系写到底，就是好习惯。

**类比**：把完全二叉树的节点按层序遍历的顺序编号，就是数组的索引。层序遍历的第 i 个节点就是数组的第 i 个元素。

### 1.3 数组末尾 = 树的最底层最右侧

```
数组：[9, 7, 8, 3, 5, 2]
                         ↑ 最后一个元素 = 树的最底层最右侧节点

push 新元素：直接 append 到数组末尾 = 添加到树的最底层最右侧
pop 堆顶：把末尾元素搬到开头 = 把最底层最右侧元素搬到根节点
```

这就是数组的妙处——O(1) 就能定位到「树的最底层最右侧」。

## 二、核心操作：swim（上浮）

### 2.1 什么时候用？

**push（插入新元素）时使用。**

新元素先追加到数组末尾（树的最底层），然后上浮到正确位置。

### 2.2 小顶堆的 swim 逻辑

```
不断和父节点比较：
  如果自己 < 父节点 → 交换，继续上浮
  如果自己 ≥ 父节点 → 停下，位置正确
  如果到达根节点（索引 0）→ 停下
```

### 2.3 代码

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
int box = 3;                            // 当前值
int parentBox = 1;                      // 父节点值
if (box < parentBox) swap(box, parentBox); // 更小就上浮
```

> 上面是上浮的一次交换，下面的 `swim` 把这个动作循环执行。

```cpp
// 小顶堆的上浮
void swim(int node) {
    // 当还没到根节点，且自己比父节点小
    while (node > 0 && heap[parent(node)] > heap[node]) {
        swap(heap[parent(node)], heap[node]);  // 交换，把更小值往上送
        node = parent(node);                    // 更新位置，继续向上检查
    }
}
```

### 2.4 图解

```
小顶堆，插入 1：

步骤 1: 追加到末尾
    [3, 5, 4, 7, 6, 1]
          3
         / \
        5   4
       / \ /
      7  6 1  ← 新元素，比父节点 4 小，上浮！

步骤 2: swap(1, 4)
    [3, 5, 1, 7, 6, 4]
          3
         / \
        5   1  ← 比父节点 3 小，继续上浮！
       / \ /
      7  6 4

步骤 3: swap(1, 3)
    [1, 5, 3, 7, 6, 4]
          1  ← 到达根节点，停止
         / \
        5   3
       / \ /
      7  6 4

完成！堆序性恢复。
```

## 三、核心操作：sink（下沉）

### 3.1 什么时候用？

**pop（删除堆顶）时使用。**

先把堆顶和末尾交换，删掉末尾（原堆顶），然后把新堆顶下沉到正确位置。

### 3.2 小顶堆的 sink 逻辑

```
不断和子节点中较小的那个比较：
  如果自己 > 较小的子节点 → 和它交换，继续下沉
  如果自己 ≤ 所有子节点 → 停下，位置正确
  如果到达叶子节点（没有子节点）→ 停下
```

**为什么要和「较小的」子节点交换？** 因为交换上去的子节点要做新的父节点，必须比另一个子节点小，才能维持小顶堆性质。

这里的思路是“每次修好一个局部父子关系”。局部一直正确，整条路径就会恢复正确。你不需要一次考虑整棵树，只要按循环规则稳定推进就行。

> 💡 **老师提醒：** 访问左右孩子前，一定先做边界判断，避免数组越界。

### 3.3 代码

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
int me = 8;                               // 当前值
int child = 3;                            // 更小的孩子
if (me > child) swap(me, child);          // 比孩子大就下沉
```

> 上面是下沉的一次交换，下面代码是在树上循环执行这个动作。

```cpp
// 小顶堆的下沉
void sink(int node) {
    while (left(node) < size) {   // 至少有左子节点
        // 找出自己和左右子节点中最小的
        int min = node;                  // 先假设当前节点最小
        if (left(node) < size && heap[left(node)] < heap[min]) {
            min = left(node);
        }
        if (right(node) < size && heap[right(node)] < heap[min]) {
            min = right(node);
        }
        if (min == node) {
            break;   // 自己就是最小的，不用下沉了
        }
        swap(heap[node], heap[min]);  // 和更小的子节点交换
        node = min;                   // 更新位置，继续下沉
    }
}
```

### 3.4 图解

```
小顶堆，pop 堆顶：

原始堆：[1, 3, 2, 7, 5, 4]
          1
         / \
        3   2
       / \ /
      7  5 4

步骤 1: 把末尾 4 放到堆顶，删除末尾
    [4, 3, 2, 7, 5]
          4  ← 比子节点大，下沉！
         / \
        3   2  ← 左=3，右=2，较小的是 2

步骤 2: swap(4, 2)
    [2, 3, 4, 7, 5]
          2
         / \
        3   4  ← 4 没有子节点了，停止
       / \
      7   5

完成！返回 1，堆序性恢复。
```

## 四、push 和 pop 的完整流程

### 4.1 push（插入）

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
vector<int> q = {2, 4};                  // 先有两个元素
q.push_back(1);                           // 新元素先放到末尾
// 再执行上浮，把它调到正确位置
```

> 这个小例子就是 push 的两段动作：追加 + 上浮。下面代码是函数实现。

```cpp
void push(int x) {
    heap[size] = x;    // 第一步：追加到数组末尾
    swim(size);        // 第二步：上浮到正确位置
    size++;            // 第三步：元素计数 +1
}
```

```
push 流程：
  数组末尾追加 → swim 上浮 → 完成
  
  [3, 5, 4]  push(1)  →  [3, 5, 4, 1]  swim  →  [1, 3, 4, 5]
```

### 4.2 pop（删除堆顶）

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
int topGift = 1;                          // 先拿走顶部礼物
int tailGift = 5;                         // 末尾礼物补到顶部
// 再执行下沉恢复顺序
```

> 上面是 pop 的核心节奏：取顶、补位、下沉。下面代码按这个节奏实现。

```cpp
int pop() {
    int res = heap[0];         // 第一步：记录堆顶值（要返回的）
    heap[0] = heap[size - 1];  // 第二步：把末尾元素搬到堆顶
    size--;                    // 第三步：元素计数 -1
    sink(0);                   // 第四步：新堆顶下沉到正确位置
    return res;                // 第五步：返回原堆顶值
}
```

```
pop 流程：
  记录堆顶 → 末尾搬到堆顶 → sink 下沉 → 返回原堆顶

  [1, 3, 4, 5]  pop()  →  堆顶=1, [5, 3, 4]  sink  →  [3, 5, 4]  返回 1
```

### 4.3 peek（查看堆顶）

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
int first = 9;                            // 当前第一名
cout << first;                            // 只看，不删除
```

> 这就是 peek 的语义，下面函数是最直接实现。

```cpp
int peek() {
    return heap[0];    // 直接返回数组第一个元素
}
```

## 五、完整的简化版代码

这一段是你要重点消化的模板。先把 `int + 小顶堆` 版本吃透，再考虑泛型和比较器。学习顺序对了，后面升级会很轻松。短句。先会跑，再会扩展。

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
vector<int> heap = {1, 3, 2};            // 三个按规则放好的盒子
swap(heap[0], heap[2]);                  // 末尾元素先顶到开头
// 再下沉修复顺序
```

> 上面是完整实现里的关键一步，下面代码把上浮、下沉、边界都组合成可运行版本。

```cpp
#include <iostream>
#include <vector>
using namespace std; // 教学示例里简化写法

class SimpleMinPQ {
    vector<int> heap; // 底层数组
    int size;         // 当前元素个数

    int parent(int i) { return (i - 1) / 2; }
    int left(int i)   { return 2 * i + 1; }
    int right(int i)  { return 2 * i + 2; }

    void swim(int node) {
        while (node > 0 && heap[parent(node)] > heap[node]) {
            swap(heap[parent(node)], heap[node]);
            node = parent(node);
        }
    }

    void sink(int node) {
        while (left(node) < size) {
            int min = node;
            if (left(node) < size && heap[left(node)] < heap[min])
                min = left(node);
            if (right(node) < size && heap[right(node)] < heap[min])
                min = right(node);
            if (min == node) break;
            swap(heap[node], heap[min]);
            node = min;
        }
    }

public:
    SimpleMinPQ(int capacity) : heap(capacity), size(0) {} // ⚠️ 初始化列表

    int getSize() { return size; }

    int peek() { return heap[0]; }

    void push(int x) {
        heap[size] = x;
        swim(size);
        size++;
    }

    int pop() {
        int res = heap[0];
        heap[0] = heap[size - 1];
        size--;
        sink(0);
        return res;
    }
};

int main() {
    SimpleMinPQ pq(5); // 容量为 5
    pq.push(3);
    pq.push(2);
    pq.push(1);
    pq.push(5);
    pq.push(4);

    // 输出：1 2 3 4 5（从小到大弹出）
    while (pq.getSize() > 0) {
        cout << pq.pop() << " ";
    }
    return 0;
}
```

> ✅ 这份代码你能独立默写出来，就已经具备手写堆的能力了。

**如何改成大顶堆？** 把 swim 和 sink 中的比较符号反过来：
- swim：`heap[parent(node)] > heap[node]` → `heap[parent(node)] < heap[node]`
- sink：`heap[...] < heap[min]` → `heap[...] > heap[max]`

## 六、完善版：泛型 + 自定义比较器 + 扩缩容

简化版有三个限制：只能存 int、容量固定、只能小顶堆。完善版解决了这些问题：

这一步是工程化升级。你会发现不是“推倒重来”，而是在原骨架上增加抽象能力。模板负责类型通用，比较器负责规则通用，扩缩容负责数据规模通用。这样一来，同一套代码能覆盖更多题型。

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
auto lowerFirst = [](int a, int b) { return a > b; }; // ⚠️ lambda 比较规则
cout << lowerFirst(5, 3);                              // true 表示 5 优先级更低
```

> 上面演示了“比较器决定先后顺序”，下面代码把这个思想做成泛型优先级队列。

```cpp
template <typename T>
class MyPriorityQueue {
private:
    vector<T> heap;                                  // 存储元素
    int size;                                        // 当前大小
    function<bool(const T&, const T&)> comparator;  // ⚠️ function 保存比较器

    // ... parent/left/right/swim/sink 和简化版逻辑一样 ...
    // 唯一区别：用 comparator(a, b) 代替 a > b

    void resize(int capacity) {
        heap.resize(capacity);
    }

public:
    MyPriorityQueue(int cap, function<bool(const T&, const T&)> comp)
        : heap(cap), size(0), comparator(comp) {}

    void push(const T& x) {
        if (size == heap.size()) resize(2 * heap.size());  // 自动扩容
        heap[size] = x;
        swim(size);
        size++;
    }

    T pop() {
        T res = heap[0];
        swap(heap[0], heap[size - 1]);
        size--;
        sink(0);
        if (size > 0 && size == heap.size() / 4) resize(heap.size() / 2);  // 自动缩容
        return res;
    }

    // ...
};

// 使用示例
// 小顶堆：a > b 表示「a 的优先级低于 b」，所以 a 应该下沉
MyPriorityQueue<int> minPQ(3, [](const int& a, const int& b) { return a > b; }); // ⚠️ lambda

// 大顶堆：a < b 表示「a 的优先级低于 b」
MyPriorityQueue<int> maxPQ(3, [](const int& a, const int& b) { return a < b; });
```

**C++ 知识点**：
- `template <typename T>`：泛型，支持任意类型
- `function<bool(const T&, const T&)>`：函数对象，存储比较器
- `[](const int& a, const int& b) { return a > b; }`：lambda 表达式，C++ 的匿名函数
- `resize()`：动态调整 vector 大小

**比较器的理解**：比较器返回 `true` 表示「第一个参数的优先级比第二个低」，应该被下沉/后处理。所以小顶堆用 `a > b`（大的优先级低），大顶堆用 `a < b`（小的优先级低）。

*就像“排队规则由口令决定”的区别。口令变了，先后顺序就变了。*

> 💡 **老师提醒：** 比较器先用中文写清语义，再写代码，能避免方向写反。

## 七、复杂度总结

| 操作 | 做了什么 | 复杂度 |
|------|---------|--------|
| `push` | 追加末尾 O(1) + swim O(log N) | **O(log N)** |
| `pop` | 交换首尾 O(1) + 删末尾 O(1) + sink O(log N) | **O(log N)** |
| `peek` | 直接返回 heap[0] | **O(1)** |
| `size` | 返回变量 | **O(1)** |

swim 和 sink 最多走树的高度那么多步。完全二叉树有 N 个节点时高度为 log N，所以是 O(log N)。

## 八、关键概念速查表

| 概念 | 说明 |
|------|------|
| **数组模拟完全二叉树** | 索引公式代替 left/right 指针 |
| **parent(i)** | `(i-1)/2`（0-based） |
| **left(i)** | `2*i+1`（0-based） |
| **right(i)** | `2*i+2`（0-based） |
| **swim（上浮）** | push 时用，新元素向上找正确位置 |
| **sink（下沉）** | pop 时用，替补元素向下找正确位置 |
| **push 流程** | 追加末尾 → swim |
| **pop 流程** | 首尾交换 → 删末尾 → sink → 返回原堆顶 |
| **小顶堆 ↔ 大顶堆** | 只需反转比较符号 |
| **comparator** | 返回 true = 第一个参数优先级更低 |
