> **前置知识**：二叉树基础、二叉树遍历  
> **核心能力**：对数组进行 **O(log N) 区间查询** + **O(log N) 动态修改**  
> **本篇定位**：理解原理和使用场景，具体代码实现后面再学  
> **竞赛重要性**：⭐⭐⭐⭐⭐ ICPC 中高频考点

## 一、为什么需要线段树？

### 1.1 问题引入

假设有一个数组 `nums = [3, 1, 4, 2, 5, 7]`，你需要频繁做两种操作：

- **区间查询**：求 `nums[i..j]` 的元素和（或最小值、最大值等）
- **单点修改**：把 `nums[i]` 改成新的值

### 1.2 暴力做法的问题

| 方案 | 区间查询 | 单点修改 | 问题 |
|------|---------|---------|------|
| 直接遍历 | O(N) 暴力求和 | O(1) 直接改 | 查询太慢 |
| 前缀和数组 | O(1) 直接算 | O(N) 重建前缀和 | 修改太慢 |

**类比**：

- 暴力遍历就像每次要算账都把所有收据重新加一遍
- 前缀和就像一本写死的总账，查起来很快，但每次有一笔账变了，整本账都得重写

两种方案总有一头是 O(N)，当操作次数很多、数组很大时都不够用。

你要看到这里的矛盾本质：查询快和修改快，传统方案往往只能二选一。线段树的价值就在于把这两个操作都压到 O(log N)。这不是魔法，而是通过分层维护区间信息实现的。理解这个动机，后面代码就不再抽象。

> 💡 **老师提醒：** 题目里如果“查询很多 + 修改也很多”，就要警惕 O(N) 做法会超时。

### 1.3 线段树的解决方案

| 操作 | 复杂度 |
|------|--------|
| 区间查询 `query(i, j)` | **O(log N)** |
| 单点修改 `update(i, val)` | **O(log N)** |
| 构建 | O(N)（一次性） |

> **两头都是 O(log N)**，完美平衡了查询和修改的效率。

*就像“收银台和盘点台都开了快速通道”的区别。*

> ✅ 你已经抓住线段树最核心的使用场景了。

## 二、线段树长什么样？

### 2.1 结构概览

线段树是一棵**二叉树**，其中：

- **叶子节点** = 数组中的每个元素
- **非叶子节点** = 某个区间的**汇总信息**（如区间和、区间最小值等）

```
数组 nums = [3, 1, 4, 2, 5, 7]

线段树（存区间和）：

              [0,5] 22        ← 整个数组的和 = 22
             /        \
       [0,2] 8      [3,5] 14   ← 左半部分和=8，右半部分和=14
       /    \        /    \
   [0,1] 4  [2] 4  [3,4] 7  [5] 7
   /    \          /    \
 [0] 3  [1] 1   [3] 2  [4] 5

每个节点标注了它负责的索引区间 [i,j] 和区间和
```

### 2.2 构建过程

递归地从正中间切分数组：

```
[3, 1, 4, 2, 5, 7]
        ↙          ↘
[3, 1, 4]        [2, 5, 7]
  ↙    ↘           ↙    ↘
[3, 1]  [4]     [2, 5]  [7]
 ↙  ↘            ↙  ↘
[3]  [1]        [2]  [5]
```

每次从中间切分 → 左右子树大小基本相同 → 树是**平衡的** → 高度 O(log N)。

### 2.3 「线段树」名字的由来

每个节点代表数组上的一个**连续区间（线段）**，所以叫线段树。根节点代表整个数组 [0, N-1]，每个子节点代表父节点区间的一半。

这个命名很重要，因为它提示你“节点不是单值，而是区间语义”。一旦你这样理解，query 和 update 的递归过程都会顺起来。线段树本质是区间信息的层级缓存。不是普通二叉树换个名字。

> 💡 **老师提醒：** 写代码时把每个节点标注成 `[l,r]`，调试会轻松很多。

## 三、核心 API

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
int left = 1, right = 3;               // 像要查询第 1 到第 3 号货架
int sum = 0;                           // 准备累加结果
cout << left << right;                 // 先明确区间边界
```

> 上面的例子强调“先确定区间再操作”，下面 API 就是把区间查询和区间相关更新封装成固定接口。

```cpp
class SegmentTree {
public:
    // 构造函数：根据数组构建线段树，O(N)
    // merge 函数决定 query 返回什么（求和、求最小值、求最大值等）
    SegmentTree(vector<int>& nums, function<int(int,int)> merge); // ⚠️ function 参数决定聚合规则

    // 区间查询：返回 [i, j] 的聚合值，O(log N)
    int query(int i, int j);   // 查询区间 [i,j]

    // 单点修改：将 nums[i] 改为 val，O(log N)
    void update(int i, int val); // 单点更新
};
```

**merge 函数的作用**：

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
auto add = [](int a, int b) { return a + b; }; // ⚠️ lambda：定义“怎么合并”
cout << add(2, 3);                             // 输出合并结果 5
```

> 这个小例子就是 merge 的本质：把左右区间结果合成父区间结果。下面给出三种常见合并规则。

```cpp
// 区间求和
merge = [](int a, int b) { return a + b; }; // 合并成区间和

// 区间最小值
merge = [](int a, int b) { return min(a, b); }; // 合并成区间最小值

// 区间最大值
merge = [](int a, int b) { return max(a, b); }; // 合并成区间最大值
```

换一个 merge 函数就能改变线段树的功能，非常灵活。

## 四、query 为什么是 O(log N)？

### 4.1 最好情况：直接命中

```
查询 query(3, 5)：

              [0,5] 22
             /        \
       [0,2] 8      [3,5] 14  ← 直接命中！返回 14
       /    \        /    \
   [0,1] 4  [2] 4  [3,4] 7  [5] 7
```

恰好有一个节点负责 [3,5] 区间，直接返回，只走了 2 步。

### 4.2 一般情况：区间分裂

```
查询 query(1, 4)：没有节点恰好负责 [1,4]

              [0,5] 22
             /        \
       [0,2] 8      [3,5] 14
       /    \        /    \
   [0,1] 4  [2] 4  [3,4] 7  [5] 7
   /    \          /    \
 [0] 3  [1] 1   [3] 2  [4] 5

[1,4] 被分裂成 [1,2] 和 [3,4]：
  [1,2] 又分裂成 [1] 和 [2]
  [3,4] 直接命中

结果 = 1 + 4 + 7 = 12（即 nums[1]+nums[2]+nums[3]+nums[4]）
```

**关键洞察**：无论怎么分裂，每一层最多访问**常数个节点**（最多 4 个）。树高 O(log N)，所以总访问节点数是 O(log N)。

这段推理是考试高频表达。你不需要把所有节点都走一遍，只会沿着和查询区间有交集的路径走。每层访问数量受控，层数又是 log N，于是总复杂度就压下来了。逻辑很干净。

> ✅ 你现在已经能自己解释 query 为什么是 O(log N) 了。

### 4.3 直觉理解

**类比**：想象你管理一个公司，公司有层级结构。你想知道「2 号到 4 号员工的总业绩」。

- 你不需要亲自问每个员工（暴力 O(N)）
- 你问几个部门经理，每个经理报告自己部门的汇总数据
- 最多问 log N 层的管理者就够了

## 五、update 为什么是 O(log N)？

修改 `nums[i]` 时，需要更新：叶子节点 + 从叶子到根的所有祖先节点。

```
update(1, 9)：把 nums[1] 从 1 改成 9

修改前：                    修改后：
      [0,5] 22                   [0,5] 30
     /        \                 /        \
[0,2] 8    [3,5] 14       [0,2] 16    [3,5] 14
 /   \                     /   \
[0,1] 4  [2] 4         [0,1] 12  [2] 4
 / \                     / \
[0]3 [1]1             [0]3 [1]9  ← 改了这里

从叶子到根的路径上，所有节点都要更新：
  [1]: 1 → 9
  [0,1]: 4 → 12（3+9）
  [0,2]: 8 → 16（12+4）
  [0,5]: 22 → 30（16+14）
```

从叶子到根的路径长度 = 树高 = O(log N)，所以 update 是 O(log N)。

update 的思路和 query 对照着记最省力。query 是“自顶向下拆区间”，update 是“自底向上回推父节点”。一上一下，正好互补。把这对关系记住，代码结构会更清晰。

*就像“从总部下发任务”和“从一线回传报表”的区别。路径方向不同，但都走同一套层级。*

> 💡 **老师提醒：** 更新后别忘了回溯更新父节点，否则根节点信息会过期。

## 六、三种变体（了解）

| 变体 | 解决什么问题 | 说明 |
|------|-------------|------|
| **基本线段树** | 区间查询 + 单点修改 | 最基础版本 |
| **动态开点线段树** | 区间范围极大（如 [0, 10⁹]）但数据稀疏 | 不预先建树，用到哪个节点才创建 |
| **懒更新线段树** | 区间查询 + **区间修改** | 批量修改 [i,j] 的元素，延迟传播更新 |

### 6.1 基本线段树

就是上面讲的，支持 `query(i,j)` 和 `update(i, val)`。

### 6.2 动态开点线段树

**问题**：如果区间是 [0, 10⁹]，基本线段树要建 10⁹ 个节点，内存炸了。

**解决**：一开始只有根节点，执行 update 时才按需创建路径上的节点。如果只修改了 k 次，实际创建的节点只有 O(k × log N) 个。

### 6.3 懒更新线段树（Lazy Propagation）

**问题**：如果要把 `nums[i..j]` 的所有元素都加上一个值，基本线段树要一个一个 update，O(N log N)。

**解决**：不立即更新到叶子节点，而是在某个中间节点上打一个「待办标记」（lazy tag）。等到之后 query 访问到这个节点时，才把更新传播下去。

```
rangeAdd(0, 3, +5)：给 nums[0..3] 都加 5

不需要更新 4 个叶子节点！
只需要在 [0,2] 和 [3] 节点上打标记 "待加 5"

以后 query 访问到这些节点时，再把标记往下传播
```

这样区间修改也是 O(log N)。

## 七、线段树 vs 前缀和 vs 树状数组

| 对比 | 前缀和 | 树状数组（BIT） | 线段树 |
|------|--------|---------------|--------|
| 区间查询 | O(1) | O(log N) | O(log N) |
| 单点修改 | O(N) 重建 | O(log N) | O(log N) |
| 区间修改 | 不支持 | O(log N)（差分） | O(log N)（懒更新） |
| 支持的操作 | 只有求和 | 只有求和 | **求和、最值、GCD 等** |
| 代码复杂度 | 最简单 | 中等 | 最复杂 |
| 适用场景 | 数组不变 | 单点修改+区间求和 | **最通用** |

**选择指南**：
- 数组不变，只查询 → 前缀和
- 单点修改 + 区间求和 → 树状数组（代码更短）
- 单点修改 + 区间最值/复杂聚合 → 线段树
- 区间修改 + 区间查询 → 懒更新线段树

## 八、关键概念速查表

| 概念 | 说明 |
|------|------|
| **线段树** | 二叉树，叶子=数组元素，非叶子=区间聚合值 |
| **区间查询** | `query(i,j)` 返回 [i,j] 的聚合值，O(log N) |
| **单点修改** | `update(i,val)` 修改第 i 个元素，O(log N) |
| **merge 函数** | 定义聚合方式（求和、最值等） |
| **树高 O(log N)** | 因为每次从中间切分，树是平衡的 |
| **动态开点** | 按需创建节点，解决区间极大但数据稀疏的问题 |
| **懒更新（Lazy）** | 延迟传播区间修改，使区间修改也是 O(log N) |
| **树状数组** | 线段树的轻量替代，只能处理求和，但代码更短 |

## 九、竞赛中的线段树

### 9.1 常见题型

| 题型 | 用哪种线段树 |
|------|-------------|
| 区间求和 + 单点修改 | 基本线段树（或树状数组） |
| 区间最值 + 单点修改 | 基本线段树 |
| 区间求和 + 区间加值 | 懒更新线段树 |
| 区间赋值 + 区间查询 | 懒更新线段树 |
| 极大区间 + 稀疏操作 | 动态开点线段树 |

### 9.2 学习建议

线段树的代码相对复杂，建议按以下顺序学习：

```
1. 先理解原理（本篇） ← 你在这里
2. 刷完二叉树习题，熟练掌握递归
3. 学习基本线段树的代码实现
4. 学习懒更新线段树
5. 学习动态开点线段树
```
