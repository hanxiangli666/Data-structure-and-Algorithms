# 数据压缩与霍夫曼树 学习笔记

> **前置知识**：二叉树基础  
> **核心认知**：霍夫曼树 = 用二叉树构造**最优变长编码**，高频字符编码短、低频字符编码长，实现最佳压缩率  
> **本篇定位**：理解原理，代码实现后面再学。重点理解「前缀编码」和「WPL 最小化」这两个概念

---

## 一、数据压缩的两大类

| 类型 | 定义 | 例子 | 特点 |
|------|------|------|------|
| **无损压缩** | 压缩后能**完全还原**，不丢失任何信息 | zip 压缩包 | 压缩率有限 |
| **有损压缩** | 压缩后丢失部分信息，但在可接受范围内 | JPEG 图片压缩 | 压缩率更高 |

**类比**：
- 无损压缩就像把一篇文章的重复段落用「参见第 X 段」代替，还原时完全展开就行
- 有损压缩就像把一篇文章改写成精简版，省了很多字但意思差不多

**无损压缩的本质**：编码 + 解码。找到原始数据中的**冗余信息**，用更紧凑的方式表示。

```
原始: hahahahahahaha    （14 个字符）
压缩: ha*7              （4 个字符）
还原: 把 ha 重复 7 次 → hahahahahahaha  ✓ 完全一样
```

霍夫曼编码就是一种**通用的无损压缩算法**。

## 二、定长编码 vs 变长编码

### 2.1 定长编码

每个字符用**相同长度**的二进制表示。

```
字符串 "aaabacc"，只有 a、b、c 三种字符
用 2 bit 就够了（2 bit 能表示 4 种字符）：

a → 00
b → 01
c → 10

"aaabacc" → 00 00 00 01 00 10 10 = 14 bit
```

**优点**：简单，支持随机访问（第 i 个字符从第 2i 位开始）  
**缺点**：没有利用频率信息，压缩率不够好

### 2.2 变长编码

高频字符用**短编码**，低频字符用**长编码**。

```
"aaabacc" 中：a 出现 4 次，b 出现 1 次，c 出现 2 次
a 频率最高 → 给它最短的编码

a → 0      （1 bit）
b → 10     （2 bit）
c → 11     （2 bit）

"aaabacc" → 0 0 0 10 0 11 11 = 10 bit  ← 比定长编码的 14 bit 更短！
```

**类比**：就像摩尔斯电码。英语中最常用的字母 E 的摩尔斯码是一个点 `·`（最短），而不常用的 Q 是 `--·-`（很长）。高频字符用短码，整体长度就短了。

## 三、变长编码的难点：前缀问题

### 3.1 错误示例：编码有歧义

```
a → 1
b → 10
c → 11

"aaabacc" → 1 1 1 10 1 11 11 = 11111011111

但解码时：11 是 c 还是 aa？无法确定！
```

问题出在：`a` 的编码 `1` 是 `b` 编码 `10` 和 `c` 编码 `11` 的**前缀**。

### 3.2 前缀编码规则

> **任意一个字符的编码，都不能是另一个字符编码的前缀。**

满足这个条件的编码叫**前缀编码**（prefix-free code）。

```
✅ 合法的前缀编码：        ❌ 不合法：
  a → 0                     a → 1
  b → 10                    b → 10     ← 1 是 10 的前缀！
  c → 11                    c → 11     ← 1 是 11 的前缀！

解码 "0001001111"：          解码 "11111011111"：
  0 → a                      11 → c 还是 aa？歧义！
  0 → a
  0 → a
  10 → b
  0 → a
  11 → c
  11 → c
  → "aaabacc" ✓
```

**前缀编码的好处**：
- **解码唯一**：不会有歧义
- **解码高效**：从左到右扫描一遍 O(N)，不需要回头探测

## 四、霍夫曼树：用二叉树构造最优前缀编码

### 4.1 核心思路

把编码方案表示成一棵二叉树：

- **左树枝** = 0，**右树枝** = 1
- **叶子节点** = 每个字符
- 从根到叶子的路径 = 该字符的编码

```
         (根)
        /    \
       0      1
      /         \
     a          (内部节点)
               /    \
              0      1
             /        \
            b          c

a → 0（走了 1 步）
b → 10（走了 2 步）
c → 11（走了 2 步）
```

**为什么这样能保证前缀编码？**

因为所有字符都在**叶子节点**上。叶子节点是路径的终点，一个叶子节点的路径不可能是另一个叶子节点路径的前缀（想到叶子就停了，不会继续往下走）。

**类比**：就像一棵决策树。你站在树根，每次向左走代表 0，向右走代表 1。走到某个叶子就得到了一个字符。因为叶子是死胡同，所以不可能有两个字符共享同一条路径。

### 4.2 不同的树形 → 不同的编码方案

同样的 5 个字符，可以构造出不同形状的树：

```
方案 A：                       方案 B：
      (根)                          (根)
     /    \                        /    \
    a    (节点)                 (节点)  (节点)
        /    \                 /    \   /    \
     (节点)  (节点)           a      b  c   (节点)
     /  \    /  \                           /    \
    b    c  d    e                         d      e

方案 A:                        方案 B:
  a → 0                         a → 00
  b → 100                       b → 01
  c → 101                       c → 10
  d → 110                       d → 110
  e → 111                       e → 111
```

哪个方案更好？取决于每个字符的**出现频率**。

### 4.3 WPL：衡量编码方案好坏的指标

**WPL = 带权路径长度 = Σ (字符频率 × 编码长度)**

```
假设频率：a:10, b:8, c:9, d:2, e:5

方案 B 的 WPL:
  10×2 + 8×2 + 9×2 + 2×3 + 5×3
= 20 + 16 + 18 + 6 + 15
= 75

方案 A 的 WPL:
  10×1 + 8×3 + 9×3 + 2×3 + 5×3
= 10 + 24 + 27 + 6 + 15
= 82
```

方案 B 的 WPL = 75 < 方案 A 的 WPL = 82，所以**方案 B 更好**。

> **霍夫曼树 = WPL 最小的前缀编码树**

### 4.4 构造霍夫曼树的算法（贪心策略）

```
输入：每个字符的频率
  a:10, b:8, c:9, d:2, e:5

步骤：
1. 把所有字符看作独立的叶子节点，放入一个最小优先级队列（按频率排序）
   队列：[d:2, e:5, b:8, c:9, a:10]

2. 每次取出频率最小的两个节点，合并成一棵新子树
   取出 d:2 和 e:5 → 合并成新节点 (de):7
   队列：[(de):7, b:8, c:9, a:10]

3. 重复步骤 2
   取出 (de):7 和 b:8 → 合并成 (deb):15
   队列：[c:9, a:10, (deb):15]

4. 继续
   取出 c:9 和 a:10 → 合并成 (ca):19
   队列：[(deb):15, (ca):19]

5. 最后
   取出 (deb):15 和 (ca):19 → 合并成根节点 34
   完成！
```

```
最终的霍夫曼树：
           (34)
          /    \
       (15)   (19)
       /  \   /  \
     (7)   b  c   a
    /   \
   d     e
```

**关键**：每次合并频率最小的两个 → 频率低的字符被推到树的深处（编码长） → 频率高的字符留在浅处（编码短） → WPL 最小。

这就是**贪心策略**：每一步都做局部最优选择（合并最小的两个），最终得到全局最优结果。

### 4.5 需要优先级队列！

每次「取出最小的两个」这个操作，就是优先级队列（小顶堆）的拿手好戏：

```cpp
// 伪码
priority_queue<pair<int,Node*>, vector<...>, greater<...>> pq;

// 把所有字符和频率放入小顶堆
for (char c : chars) pq.push({freq[c], new LeafNode(c)});

// 反复取出最小的两个，合并，放回
while (pq.size() > 1) {
    auto [freq1, node1] = pq.top(); pq.pop();
    auto [freq2, node2] = pq.top(); pq.pop();
    auto merged = new InternalNode(node1, node2);
    pq.push({freq1 + freq2, merged});
}
// 最后队列中剩下的就是霍夫曼树的根节点
```

看到了吗？前面学的**优先级队列**在这里派上用场了！

## 五、完整示例

```
原始字符串: "aaaabbbccd"

字符频率: a:4, b:3, c:2, d:1

构造过程:
  [d:1, c:2, b:3, a:4]
  合并 d:1 + c:2 → (dc):3     队列: [(dc):3, b:3, a:4]
  合并 (dc):3 + b:3 → (dcb):6  队列: [a:4, (dcb):6]
  合并 a:4 + (dcb):6 → 根:10

霍夫曼树:
         (10)
        /    \
       a     (6)
            /   \
          (3)    b
         /   \
        d     c

编码:
  a → 0      （1 bit，出现 4 次）
  b → 11     （2 bit，出现 3 次）
  c → 101    （3 bit，出现 2 次）
  d → 100    （3 bit，出现 1 次）

编码后:
  "aaaabbbccd" → 0 0 0 0 11 11 11 101 101 100
               = 0000111111101101100
               = 19 bit

对比:
  ASCII 编码:   10 × 8 = 80 bit
  定长编码(2bit): 10 × 2 = 20 bit
  霍夫曼编码:     4×1 + 3×2 + 2×3 + 1×3 = 19 bit  ← 最短！
```

## 六、关键概念速查表

| 概念 | 说明 |
|------|------|
| **无损压缩** | 压缩后能完全还原，霍夫曼编码属于此类 |
| **有损压缩** | 丢失部分信息换取更高压缩率（如 JPEG） |
| **定长编码** | 每个字符编码长度相同，简单但压缩率低 |
| **变长编码** | 高频字符短编码、低频长编码，压缩率高 |
| **前缀编码** | 任意编码不是另一个编码的前缀，保证解码唯一 |
| **霍夫曼树** | WPL 最小的前缀编码树 |
| **WPL** | 带权路径长度 = Σ(频率 × 编码长度) |
| **贪心构造** | 每次合并频率最小的两个节点 |
| **优先级队列** | 构造霍夫曼树时用小顶堆取最小的两个节点 |

## 八、知识串联：二叉树的实际应用全景

到这里，你已经见识了二叉树的各种实际应用：

```
二叉树的应用
├── BST / TreeMap / TreeSet → 有序数据操作
│    └── 红黑树 → 自平衡，C++ map/set 底层
├── 二叉堆 / 优先级队列 → 动态排序，取极值
├── Trie 树 → 字符串前缀操作
├── 线段树 → 区间查询 + 动态修改
└── 霍夫曼树 → 数据压缩，最优编码  ← 你在这里
```

这些全是二叉树的「变形」，核心结构都一样。理解了二叉树，这些数据结构就是在不同场景下的自然延伸。
