# 二叉堆与优先级队列 学习笔记

> **前置知识**：二叉树基础、完全二叉树、二叉树遍历  
> **核心认知**：二叉堆 = 能**动态排序**的数据结构，不断加入/删除元素，它始终帮你维护最大值或最小值  
> **主要应用**：优先级队列（Priority Queue）、堆排序（Heap Sort）

---

## 一、二叉堆是什么？

### 1.1 一句话定义

> 二叉堆是一棵特殊的**完全二叉树**，满足：任意节点的值**大于等于**（或小于等于）其所有子节点的值。

- **大顶堆（Max Heap）**：每个节点 ≥ 子节点 → 根节点是**最大值**
- **小顶堆（Min Heap）**：每个节点 ≤ 子节点 → 根节点是**最小值**

你可以先把堆理解成“只保证局部顺序”的结构。它不追求全局完全有序，只保证父子关系满足规则。正是因为这个约束比较弱，维护成本才低，动态操作才快。很多实时取最值的问题都靠它。

> 💡 **老师提醒：** 堆不是排序数组，也不是 BST。别要求它满足“中序有序”这种条件。

```
大顶堆：              小顶堆：
      9                     1
     / \                   / \
    7   8                 3   2
   / \                   / \
  3   5                 7   5
  
根节点 9 是最大值       根节点 1 是最小值
```

### 1.2 类比理解

**大顶堆**像一个公司的组织架构——每个上司的「级别」都比下属高。CEO（根节点）就是级别最高的人。

**小顶堆**像一个「最便宜的商品」排行榜——每个节点都比它下面的子节点便宜。堆顶就是最便宜的那个。

*就像“领奖台选手站位”和“按学号整齐排队”的区别。前者只关心第一名，后者追求全局顺序。*

### 1.3 二叉堆的关键性质

| 性质 | 说明 |
|------|------|
| **完全二叉树** | 节点紧凑靠左排列，可以用**数组**存储 |
| **堆序性** | 父节点 ≥ 子节点（大顶堆）或 ≤ 子节点（小顶堆） |
| **子堆也是堆** | 任意节点的左右子树本身也满足堆的性质 |
| **根节点是极值** | 大顶堆的根是最大值，小顶堆的根是最小值 |

**注意**：二叉堆**不是** BST！BST 要求「左小右大」，堆只要求「父比子大/小」。堆的左右子节点之间**没有大小关系**。

这是初学者最容易混掉的点。BST 强在“范围查询、有序遍历”，堆强在“快速拿极值”。它们服务的目标不同，所以结构约束也不同。会区分这个边界，你选数据结构就不会乱。

> ✅ 你已经进入“按任务选结构”的阶段了，这很重要。

```
这是一个合法的大顶堆：
      9
     / \
    5   7      ← 5 < 7 没关系！堆不要求左右有序
   / \
  3   2

这不是 BST（5 在左边但 5 < 7，左边应该更小？不对，BST 要求左 < 根 < 右）
堆不管这些，只要求父 ≥ 子
```

## 二、为什么需要二叉堆？

### 2.1 动态排序的需求

普通排序算法（快排、归并）是一次性的——给你一个数组，排好序，结束。

但很多场景需要**不断地加入新元素，同时随时取出最大/最小值**：

- 任务调度：不断有新任务到来，每次取优先级最高的任务执行
- 合并有序链表：不断从多个链表中取最小的元素
- 求数据流的中位数：数据不断流入，随时要算中位数
- Dijkstra 最短路算法：不断取距离最小的未处理节点

这就是「动态排序」——二叉堆的核心用途。

### 2.2 为什么不用 BST？

二叉搜索树（TreeMap/TreeSet）也能动态排序，而且功能更强大。但：

| 对比 | 优先级队列（二叉堆） | TreeMap（BST） |
|------|---------------------|---------------|
| 取最大/最小 | O(1) peek | O(log N) |
| 插入 | O(log N) | O(log N) |
| 删除最大/最小 | O(log N) | O(log N) |
| 实现复杂度 | 简单（数组就行） | 复杂（红黑树） |
| 额外功能 | 只能取极值 | 排名、区间查询等 |

> **能用优先级队列解决的问题，没必要用 TreeMap**。杀鸡不用牛刀。

这句话背后是工程思维。只需要“最大/最小”时，优先级队列接口更直接、常数更小、代码更短。TreeMap 能做更多事，但更多功能也意味着更高心智负担。写题时先用够用的工具。

*就像“开瓶器”和“多功能工具钳”的区别。只开瓶就用开瓶器。*

## 三、二叉堆用数组存储

### 3.1 为什么能用数组？

因为二叉堆是**完全二叉树**（节点紧凑靠左），父子节点的索引有固定规律：

```
数组:  [9, 7, 8, 3, 5]
索引:   0  1  2  3  4

对应的树：
        9 (索引0)
       / \
 (1) 7    8 (2)
    / \
(3)3   5(4)
```

**索引公式**（0-based）：

```
父节点索引:     (i - 1) / 2
左子节点索引:    2 * i + 1
右子节点索引:    2 * i + 2
```

**有些实现用 1-based 索引**（数组第 0 位空着不用），公式更简洁：

```
父节点索引:     i / 2
左子节点索引:    2 * i
右子节点索引:    2 * i + 1
```

### 3.2 不需要 TreeNode！

二叉堆虽然逻辑上是一棵树，但存储上**只是一个数组**。不需要 left/right 指针，不需要 new 节点。这就是为什么它比 BST 实现简单得多。

**类比**：完全二叉树用数组存储，就像一栋公寓楼。每间房有固定的房号（索引），你根据房号就能算出它楼上/楼下的房间（父/子节点），不需要额外的门牌指引（指针）。

## 四、核心操作：swim（上浮）和 sink（下沉）

二叉堆的所有操作都围绕这两个基本动作。

### 4.1 swim（上浮）—— 插入新元素时使用

**场景**：新元素加在数组末尾（树的最底层最右边），如果它比父节点大（大顶堆），就违反了堆序性，需要「上浮」。

**过程**：不断和父节点比较，如果比父节点大就交换，直到满足堆序性或到达根节点。

```
大顶堆，插入 10：

步骤 1: 加到末尾
      9
     / \
    7   8
   / \ /
  3  5 10  ← 新元素，比父节点 8 大，上浮！

步骤 2: 和父节点 8 交换
      9
     / \
    7   10  ← 10 上来了，比父节点 9 大，继续上浮！
   / \ /
  3  5 8

步骤 3: 和父节点 9 交换
      10   ← 10 成为新的根节点
     / \
    7   9
   / \ /
  3  5 8

完成！堆序性恢复。
```

### 4.2 sink（下沉）—— 删除堆顶时使用

**场景**：删除堆顶后，把数组最后一个元素放到堆顶，它可能比子节点小（大顶堆），需要「下沉」。

**过程**：不断和**较大的子节点**比较（大顶堆），如果比子节点小就交换，直到满足堆序性或到达底部。

```
大顶堆，删除堆顶 10：

步骤 1: 把最后一个元素 5 放到堆顶
      5   ← 5 比子节点小，下沉！
     / \
    7   9
   / \
  3   8

步骤 2: 和较大的子节点 9 交换（7 vs 9，选 9）
      9
     / \
    7   5  ← 5 比子节点 8 小，继续下沉！
   / \
  3   8

步骤 3: 和子节点 8 交换
      9
     / \
    7   8
   / \
  3   5  ← 到底了，完成！
```

**为什么要和「较大的子节点」交换？** 因为交换上去的那个子节点要成为新的父节点，它必须比另一个子节点大，才能维持堆序性。

### 4.3 操作复杂度

| 操作 | 过程 | 复杂度 |
|------|------|--------|
| **push（插入）** | 加到末尾 + swim 上浮 | O(log N) |
| **pop（删除堆顶）** | 堆顶和末尾交换 + 删末尾 + sink 下沉 | O(log N) |
| **peek（查看堆顶）** | 直接返回数组第一个元素 | O(1) |

上浮和下沉最多走树的高度那么多步，完全二叉树的高度是 log N，所以是 O(log N)。

记复杂度别靠死记。你每次都问“最坏走几层”，自然就能推出 O(log N)。这种推理习惯会帮你在陌生结构里快速判断效率。考试时也更稳。

> 💡 **老师提醒：** 复杂度写出来时，顺便说明“为什么”，得分更完整。

## 五、优先级队列 API

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
int firstInLine = 5;               // 当前优先处理 5 号
cout << firstInLine;               // 看看谁在最前面
firstInLine = 7;                   // 下一个优先处理的人变成 7 号
```

> 上面的例子是“每次只关心当前第一名”，下面的接口定义就是把这个动作抽象成 push/peek/pop。

```cpp
class MyPriorityQueue {
public:
  void push(int x);      // 插入元素，内部会维护堆序性，O(log N)
  int peek();            // 查看堆顶（最大/最小值），O(1)
  int pop();             // 弹出堆顶并重排，O(log N)
  int size();            // 返回元素个数，O(1)
};
```

**为什么叫「队列」？** 因为 API 和队列很像（push 入、pop 出、peek 看队头）。但本质是二叉堆，和队列的 FIFO（先进先出）完全不同——优先级队列是**优先级最高的先出**。

## 六、C++ 标准库中的优先级队列

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
priority_queue<int> q;             // 像“奖牌榜”，分数高的在前
q.push(3);                         // 放入一个分数
cout << q.top();                   // 看当前最高分
```

> 上面的最小例子是 priority_queue 的核心用法，下面的代码是在这个基础上补齐常见操作和大小顶堆写法。

```cpp
#include <queue>

// 默认是大顶堆（最大的在顶部）
priority_queue<int> maxHeap;                  // 默认比较器是 less，形成大顶堆

maxHeap.push(3);                              // 入堆 3
maxHeap.push(1);                              // 入堆 1
maxHeap.push(4);                              // 入堆 4
maxHeap.push(1);                              // 入堆 1
maxHeap.push(5);                              // 入堆 5

maxHeap.top();    // 5（最大值）—— 对应 peek
maxHeap.pop();    // 弹出堆顶（不返回值）
maxHeap.top();    // 4（新的最大值）
maxHeap.size();   // 4（当前元素个数）
maxHeap.empty();  // false（堆不为空）

// 小顶堆（最小的在顶部）—— 竞赛中非常常用！
priority_queue<int, vector<int>, greater<int>> minHeap; // ⚠️ 第三个模板参数是比较器

minHeap.push(3);                              // 入堆 3
minHeap.push(1);                              // 入堆 1
minHeap.push(4);                              // 入堆 4
minHeap.top();    // 1（最小值）
```

**C++ 知识点**：
- `priority_queue<int>`：默认大顶堆
- `priority_queue<int, vector<int>, greater<int>>`：小顶堆。这个写法比较长，需要记住
  - 第一个参数：元素类型
  - 第二个参数：底层容器（固定写 `vector<int>`）
  - 第三个参数：比较器（`greater<int>` 表示小的优先）
- `top()`：查看堆顶（注意不是 `front()`，也不是 `peek()`）
- `push()`：插入
- `pop()`：弹出堆顶（不返回值！要先 `top()` 再 `pop()`）

**⚠️ 常见坑**：C++ 的 `priority_queue::pop()` **不返回值**，这和很多其他语言不一样。必须先 `top()` 拿到值，再 `pop()` 删除。

> ✅ 你只要记住“先 top 再 pop”，这个坑基本就不会再踩。

### 6.1 自定义排序的优先级队列

竞赛中经常需要把自定义结构体放进优先级队列：

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
pair<int, int> job = {3, 101};     // 像“优先级=3，任务号=101”
cout << job.first;                 // 先看优先级
cout << job.second;                // 再看任务编号
```

> 上面的例子是“任务和优先级成对出现”，下面代码就是把这个想法扩展到结构体比较器和 pair 小顶堆。

```cpp
// 方法 1：重载 < 运算符
struct Task {
  int priority;                    // 任务优先级
  string name;                     // 任务名称
    // 大顶堆：优先级高的在前
    bool operator<(const Task& other) const {
    return priority < other.priority;  // ⚠️ 运算符重载：定义 Task 的比较规则
    }
};
priority_queue<Task> pq;             // 用重载后的比较规则建堆

// 方法 2：自定义比较器（小顶堆）
struct Compare {
    bool operator()(const Task& a, const Task& b) {
    return a.priority > b.priority;  // ⚠️ 仿函数比较器：返回 true 表示 a 优先级更低
    }
};
priority_queue<Task, vector<Task>, Compare> pq; // 显式传入比较器类型

// 方法 3：用 pair，按 first 排序
// 小顶堆：距离小的优先
priority_queue<pair<int,int>, vector<pair<int,int>>, greater<pair<int,int>>> pq; // ⚠️ pair 默认先比 first
pq.push({5, 1});   // {距离, 节点编号}
pq.push({3, 2});   // 更小距离会更靠近堆顶
pq.top().second;    // 2（距离 3 最小的那个节点）
```

## 七、堆排序（了解即可）

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
vector<int> box = {3, 1};          // 两个数字像两个待排队盒子
sort(box.begin(), box.end());      // 让它们按从小到大排好
cout << box[0];                    // 先拿最小的那个
```

> 上面是“先排序再取”的思路，下面的代码是“放进小顶堆后反复取最小”，本质上也是得到有序结果。

```cpp
// 堆排序的本质思路（伪码）
vector<int> heapSort(vector<int>& arr) {
  priority_queue<int, vector<int>, greater<int>> minHeap;  // 小顶堆：堆顶始终最小
    
  for (int x : arr) minHeap.push(x);      // 把所有元素压入小顶堆
    
    vector<int> result;
    while (!minHeap.empty()) {
    result.push_back(minHeap.top());     // 每轮拿当前最小值
    minHeap.pop();                       // 删掉已取出的最小值
    }
  return result;    // 返回从小到大的结果数组
}
```

时间复杂度 O(N log N)，空间复杂度 O(N)。

真正的堆排序可以原地进行（O(1) 空间），但那是在数组上直接建堆操作，原理一样但代码更复杂。竞赛中一般直接用 `sort()` 排序，很少手写堆排序。

## 八、关键概念速查表

| 概念 | 说明 |
|------|------|
| **二叉堆** | 满足堆序性的完全二叉树，用数组存储 |
| **大顶堆** | 父 ≥ 子，根节点最大 |
| **小顶堆** | 父 ≤ 子，根节点最小 |
| **swim（上浮）** | 新元素插入后，向上调整位置 |
| **sink（下沉）** | 删除堆顶后，从顶部向下调整位置 |
| **优先级队列** | 二叉堆的主要应用，自动维护极值 |
| `priority_queue<int>` | C++ 大顶堆 |
| `priority_queue<int,vector<int>,greater<int>>` | C++ 小顶堆 |
| `top()` | 查看堆顶，O(1) |
| `push()` / `pop()` | 插入 / 弹出堆顶，O(log N) |
