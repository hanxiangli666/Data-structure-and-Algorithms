# 图结构的代码实现 学习笔记

> **前置知识**：多叉树、图论基本术语（上一篇）  
> **核心认知**：图 ≈ 高级版多叉树，存储方式主要有两种——**邻接表**和**邻接矩阵**  
> **竞赛实用性**：⭐⭐⭐⭐⭐ 建图是所有图论算法的第一步，必须熟练

---

## 一、图的逻辑结构 = 多叉树的升级版

### 1.1 对比：多叉树节点 vs 图节点

```cpp
// 多叉树节点
class TreeNode {
    int val;
    vector<TreeNode*> children;   // 只有父→子的方向
};

// 图节点（逻辑上）
class Vertex {
    int id;
    vector<Vertex*> neighbors;    // 任意方向，可以互相指
};
```

**几乎一模一样！** 区别在于：
- 树：只有父→子的单向关系，不能有环
- 图：节点之间可以互相指向，可以有环

> 所以树的 DFS/BFS 遍历算法，**全部适用于图**（只是图需要额外记录「已访问」防止走回头路）。

### 1.2 但实际中不用 Vertex 类

上面的 Vertex 类只是逻辑结构。实际代码中，我们用**邻接表**或**邻接矩阵**来存图，更高效也更方便。

## 二、邻接表（Adjacency List）

### 2.1 原理

每个节点维护一个列表，记录它的所有邻居。

```
图：
    0 ──→ 1
    ↑     │
    │     ↓
    3 ←── 2

邻接表：
  0: [1]
  1: [2]
  2: [3]
  3: [0]
```

### 2.2 代码实现

```cpp
// 最简单的形式：无权图
vector<vector<int>> graph(n);    // n 个节点
graph[0].push_back(1);           // 0 → 1
graph[1].push_back(2);           // 1 → 2

// 加权图：用结构体存边
struct Edge {
    int to;
    int weight;
};
vector<vector<Edge>> graph(n);
graph[0].push_back({1, 5});      // 0 → 1，权重 5
```

### 2.3 特点

| 操作 | 复杂度 | 说明 |
|------|--------|------|
| 添加边 | O(1) | 直接 push_back |
| 删除边 | O(V) | 需要遍历邻居列表找到目标 |
| 查询是否相邻 | O(V) | 需要遍历邻居列表 |
| 获取所有邻居 | O(1) | 直接返回列表 |
| **空间复杂度** | **O(V + E)** | 只存实际存在的边 |

## 三、邻接矩阵（Adjacency Matrix）

### 3.1 原理

用一个 V×V 的二维数组，`matrix[x][y]` 表示 x 到 y 是否有边（或边的权重）。

```
图：
    0 ──→ 1
    ↑     │
    │     ↓
    3 ←── 2

邻接矩阵：
      0  1  2  3
  0 [ 0, 1, 0, 0 ]    ← 0 → 1
  1 [ 0, 0, 1, 0 ]    ← 1 → 2
  2 [ 0, 0, 0, 1 ]    ← 2 → 3
  3 [ 1, 0, 0, 0 ]    ← 3 → 0

matrix[0][1] = 1 表示有边 0→1
matrix[1][0] = 0 表示没有边 1→0
```

### 3.2 代码实现

```cpp
// 无权图：布尔矩阵
vector<vector<bool>> matrix(n, vector<bool>(n, false));
matrix[0][1] = true;     // 0 → 1

// 加权图：整数矩阵，0 表示无边
vector<vector<int>> matrix(n, vector<int>(n, 0));
matrix[0][1] = 5;        // 0 → 1，权重 5
```

### 3.3 特点

| 操作 | 复杂度 | 说明 |
|------|--------|------|
| 添加边 | O(1) | 直接赋值 |
| 删除边 | O(1) | 直接置 0 |
| 查询是否相邻 | O(1) | 直接查数组 |
| 获取所有邻居 | O(V) | 需要扫描一整行 |
| **空间复杂度** | **O(V²)** | 不管有多少边都占 V² |

## 四、邻接表 vs 邻接矩阵

| 对比 | 邻接表 | 邻接矩阵 |
|------|--------|---------|
| 空间 | O(V+E)，稀疏图更省 | O(V²)，稠密图差不多 |
| 添加边 | O(1) | O(1) |
| 删除边 | O(V) | **O(1)** |
| 判断相邻 | O(V) | **O(1)** |
| 获取邻居 | **O(1)** | O(V) |
| 适合 | **稀疏图**（E << V²） | 稠密图（E ≈ V²） |
| 竞赛中使用频率 | **更高** | 较低 |

**竞赛中绝大多数图是稀疏图，所以默认用邻接表。**

**什么时候用邻接矩阵？**
- 需要频繁判断两个节点是否相邻（O(1) vs O(V)）
- 节点数很少（V < 1000 左右）
- 涉及矩阵运算的高级图论题

## 五、所有图类型的实现关系

```
有向加权图（最通用的形式）
  │
  ├── 有向无权图 = 权重全设为 1
  │
  ├── 无向加权图 = 每条边加两次（双向）
  │     │
  │     └── 无向无权图 = 双向 + 权重全设为 1
  │
  └── 所有图都可以用邻接表或邻接矩阵实现
```

**核心思路**：只要实现了「有向加权图」，其他所有图类型都是它的简化版。

### 5.1 无权图 = 权重设为 1

```cpp
// 有向无权图：复用加权图，权重默认 1
graph.addEdge(from, to, 1);   // 权重写 1 就行
```

### 5.2 无向图 = 双向添加

```cpp
// 无向图：每条边添加两次
void addUndirectedEdge(int u, int v, int w) {
    graph[u].push_back({v, w});   // u → v
    graph[v].push_back({u, w});   // v → u（反向也加一条）
}
```

## 六、竞赛中最常用的建图方式

### 6.1 邻接表建图模板（背下来）

```cpp
#include <vector>
using namespace std;

// ===== 无权图 =====
int n, m;  // n 个节点，m 条边
vector<vector<int>> graph(n);

// 读入 m 条有向边
for (int i = 0; i < m; i++) {
    int u, v;
    cin >> u >> v;
    graph[u].push_back(v);
    // 如果是无向图，加上：graph[v].push_back(u);
}

// 遍历节点 u 的所有邻居
for (int v : graph[u]) {
    // v 是 u 的邻居
}


// ===== 加权图 =====
struct Edge {
    int to, weight;
};
vector<vector<Edge>> graph(n);

// 读入 m 条有向加权边
for (int i = 0; i < m; i++) {
    int u, v, w;
    cin >> u >> v >> w;
    graph[u].push_back({v, w});
    // 如果是无向图，加上：graph[v].push_back({u, w});
}

// 遍历节点 u 的所有邻居及权重
for (auto& [v, w] : graph[u]) {
    // v 是邻居，w 是边权
}
```

**C++ 知识点**：
- `vector<vector<int>> graph(n)`：n 个节点，每个节点有一个邻居列表
- `graph[u].push_back(v)`：给节点 u 添加邻居 v
- `auto& [v, w] : graph[u]`：C++17 的结构化绑定，自动拆开 Edge 的两个字段
- 如果编译器不支持 C++17，用 `for (auto& e : graph[u])` 再 `e.to`, `e.weight`

### 6.2 节点不是整数怎么办？

用哈希表做映射：

```cpp
unordered_map<string, int> idMap;
int nextId = 0;

int getId(string name) {
    if (idMap.find(name) == idMap.end()) {
        idMap[name] = nextId++;
    }
    return idMap[name];
}

// 然后用整数 id 建图
graph[getId("Alice")].push_back(getId("Bob"));
```

## 七、完整代码：有向加权图（邻接表实现）

```cpp
class WeightedDigraph {
public:
    struct Edge {
        int to;
        int weight;
        Edge(int to, int weight) : to(to), weight(weight) {}
    };

private:
    vector<vector<Edge>> graph;

public:
    WeightedDigraph(int n) : graph(n) {}

    // 添加有向边 from → to，权重 weight
    void addEdge(int from, int to, int weight) {
        graph[from].emplace_back(to, weight);
    }

    // 删除有向边 from → to
    void removeEdge(int from, int to) {
        auto& edges = graph[from];
        for (auto it = edges.begin(); it != edges.end(); ++it) {
            if (it->to == to) {
                edges.erase(it);
                break;
            }
        }
    }

    // 判断是否存在边 from → to
    bool hasEdge(int from, int to) {
        for (auto& e : graph[from]) {
            if (e.to == to) return true;
        }
        return false;
    }

    // 获取边 from → to 的权重
    int weight(int from, int to) {
        for (auto& e : graph[from]) {
            if (e.to == to) return e.weight;
        }
        return -1;  // 不存在
    }

    // 获取节点 v 的所有邻居
    const vector<Edge>& neighbors(int v) {
        return graph[v];
    }
};
```

**无向加权图**只需要在 addEdge 和 removeEdge 时操作两个方向：

```cpp
class WeightedUndigraph {
    WeightedDigraph g;
public:
    WeightedUndigraph(int n) : g(n) {}

    void addEdge(int u, int v, int w) {
        g.addEdge(u, v, w);
        g.addEdge(v, u, w);    // 双向
    }

    void removeEdge(int u, int v) {
        g.removeEdge(u, v);
        g.removeEdge(v, u);    // 双向
    }

    // hasEdge, weight, neighbors 直接委托给 g
};
```

## 八、关键概念速查表

| 概念 | 说明 |
|------|------|
| **邻接表** | `graph[u]` = 节点 u 的所有邻居列表，空间 O(V+E) |
| **邻接矩阵** | `matrix[u][v]` = u→v 的边信息，空间 O(V²) |
| **稀疏图用邻接表** | 竞赛中绝大多数情况 |
| **无向图 = 双向添加** | `addEdge(u,v)` + `addEdge(v,u)` |
| **无权图 = 权重为 1** | 复用加权图代码 |
| **`emplace_back`** | 比 `push_back` 少一次拷贝，更高效 |
| **结构化绑定** | `auto& [v, w] : edges`，C++17 语法糖 |

## 九、后续学习路线

```
图的存储（邻接表/邻接矩阵） ← 你在这里
  │
  ├── 图的 DFS / BFS 遍历（和树几乎一样，多了 visited 数组）
  ├── 拓扑排序（有向无环图，用入度）
  ├── 最短路算法
  │    ├── BFS（无权图最短路）
  │    ├── Dijkstra（加权图，无负权）
  │    └── Bellman-Ford / SPFA（有负权）
  ├── 最小生成树（Kruskal / Prim）
  └── 并查集（判断连通性）
```

建图是所有图算法的第一步。第六节的建图模板一定要背下来，后面每道图论题都会用到。
