## 一、图的逻辑结构 = 多叉树的升级版

### 1.1 对比：多叉树节点 vs 图节点

先别急着看完整类定义，你可以先把它想成“联系人列表”。
树像公司组织架构，一个员工通常只汇报给上级那条线往下走；图更像社交网络，彼此都可能互相关联。*就像家谱和微信好友关系的区别*。你先抓住“连接方向是否受限”这个核心，后面 DFS/BFS 就会顺很多。

> 💡 **老师提醒：** 初学时最容易把“树的孩子列表”和“图的邻居列表”当成一回事，真正的差别在于图可能有回边和环。

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
int xiaoming = 1;                 // 把“小明”编号成 1，像给快递贴编号
int xiaohong = 2;                 // 把“小红”编号成 2
vector<int> friends = {xiaohong}; // 小明的好友列表里先放小红
friends.push_back(3);             // 再加一个编号 3 的好友
```

**第二步：过渡说明（1-2 句话）**

> 上面这个小例子就是“一个人连到若干人”的最小模型。下面的类定义做的事情一样，只是把“好友列表”封装成了更规范的节点结构。

**第三步：给原始代码加注释**

```cpp
// 多叉树节点
class TreeNode {
    int val;                      // 当前节点存的值
    vector<TreeNode*> children;   // 只有父→子的方向，不允许随便回指
};

// 图节点（逻辑上）
class Vertex {
    int id;                       // 节点编号，用来唯一标识
    vector<Vertex*> neighbors;    // ⚠️ 邻居可形成回路，所以遍历时要配 visited
};
```

**几乎一模一样！** 区别在于：
- 树：只有父→子的单向关系，不能有环
- 图：节点之间可以互相指向，可以有环

> 所以树的 DFS/BFS 遍历算法，**全部适用于图**（只是图需要额外记录「已访问」防止走回头路）。

> ✅ 你已经抓到图和树最关键的第一层联系了，继续保持这个比较思维。

### 1.2 但实际中不用 Vertex 类

上面的 Vertex 类只是逻辑结构。实际代码中，我们用**邻接表**或**邻接矩阵**来存图，更高效也更方便。

## 二、邻接表（Adjacency List）

### 2.1 原理

每个节点维护一个列表，记录它的所有邻居。

你可以把邻接表想成“每个站点的直达车次表”。为什么它在竞赛里常用？因为大多数题目的边不多，没必要开一整张 $V\times V$ 的大表，按需存边更省空间。你访问某个点的邻居时也很直接，拿出它那一行列表就能遍历。

```
图：
    0 ──→ 1
    ↑     │
    │     ↓
    3 ←── 2

邻接表：
  0: [1]
  1: [2]
  2: [3]
  3: [0]
```

### 2.2 代码实现

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
vector<vector<int>> line(4);      // 4 个人的“可直接联系名单”
line[0].push_back(1);             // 0 可以直接联系 1
line[0].push_back(2);             // 0 还可以直接联系 2
line[2].push_back(3);             // 2 可以直接联系 3
```

**第二步：过渡说明（1-2 句话）**

> 这个小例子已经是邻接表的完整骨架了。下面的真实代码只是进一步区分“有权边”和“无权边”。

**第三步：给原始代码加注释**

```cpp
// 最简单的形式：无权图
vector<vector<int>> graph(n);     // 建立 n 个节点的邻接表，每个点先是空邻居列表
graph[0].push_back(1);            // 添加一条有向边：0 → 1
graph[1].push_back(2);            // 添加一条有向边：1 → 2

// 加权图：用结构体存边
struct Edge {
    int to;                        // 边的终点
    int weight;                    // 边权（距离/花费/时间等）
};
vector<vector<Edge>> graph(n);    // 每个点连出去的是若干条 Edge
graph[0].push_back({1, 5});       // 添加边：0 → 1，权重 5
```

> 💡 **老师提醒：** `graph(n)` 里的 `n` 必须先确定好；很多同学会先写声明再读 n，结果数组大小不对。

### 2.3 特点

| 操作 | 复杂度 | 说明 |
|------|--------|------|
| 添加边 | O(1) | 直接 push_back |
| 删除边 | O(V) | 需要遍历邻居列表找到目标 |
| 查询是否相邻 | O(V) | 需要遍历邻居列表 |
| 获取所有邻居 | O(1) | 直接返回列表 |
| **空间复杂度** | **O(V + E)** | 只存实际存在的边 |

> ✅ 你已经会读懂邻接表复杂度表格了，这是做图题的基本功。

## 三、邻接矩阵（Adjacency Matrix）

### 3.1 原理

用一个 V×V 的二维数组，`matrix[x][y]` 表示 x 到 y 是否有边（或边的权重）。

邻接矩阵的优点是“查得快”，下标一定位就知道有没有边，特别适合频繁判断连边关系的题。它的代价是空间大，不管有没有边都要占位置。*就像整排储物柜都先租下来，找东西很快，但空柜子也要付钱*。

```
图：
    0 ──→ 1
    ↑     │
    │     ↓
    3 ←── 2

邻接矩阵：
      0  1  2  3
  0 [ 0, 1, 0, 0 ]    ← 0 → 1
  1 [ 0, 0, 1, 0 ]    ← 1 → 2
  2 [ 0, 0, 0, 1 ]    ← 2 → 3
  3 [ 1, 0, 0, 0 ]    ← 3 → 0

matrix[0][1] = 1 表示有边 0→1
matrix[1][0] = 0 表示没有边 1→0
```

### 3.2 代码实现

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
vector<vector<bool>> seat(3, vector<bool>(3, false)); // 3 个人之间是否能直达
seat[0][1] = true;                                    // 0 可以直达 1
seat[1][2] = true;                                    // 1 可以直达 2
bool ok = seat[2][0];                                 // 查 2 能不能直达 0
```

**第二步：过渡说明（1-2 句话）**

> 这个例子就是邻接矩阵最核心的“查表”动作。下面代码同样思路，只是再补上权重版本。

**第三步：给原始代码加注释**

```cpp
// 无权图：布尔矩阵
vector<vector<bool>> matrix(n, vector<bool>(n, false));
matrix[0][1] = true;      // 把 0→1 这个位置标成 true，表示有边

// 加权图：整数矩阵，0 表示无边
vector<vector<int>> matrix(n, vector<int>(n, 0));
matrix[0][1] = 5;         // 在 0→1 的格子里填权重 5
```

> 💡 **老师提醒：** 如果题目边权可能为 0，就不能再用 0 代表“无边”，要改用 `INF` 或 `-1` 这类哨兵值。

### 3.3 特点

| 操作 | 复杂度 | 说明 |
|------|--------|------|
| 添加边 | O(1) | 直接赋值 |
| 删除边 | O(1) | 直接置 0 |
| 查询是否相邻 | O(1) | 直接查数组 |
| 获取所有邻居 | O(V) | 需要扫描一整行 |
| **空间复杂度** | **O(V²)** | 不管有多少边都占 V² |

## 四、邻接表 vs 邻接矩阵

这部分你不要死记表格，先记“题目在高频做什么操作”。如果你总在问“这两个点连没连”，邻接矩阵会更顺手；如果你总在“从一个点出发枚举能走去哪”，邻接表更自然。*就像查电话簿和翻通讯录的区别*：一个按位置秒查，一个按人展开关系。

| 对比 | 邻接表 | 邻接矩阵 |
|------|--------|---------|
| 空间 | O(V+E)，稀疏图更省 | O(V²)，稠密图差不多 |
| 添加边 | O(1) | O(1) |
| 删除边 | O(V) | **O(1)** |
| 判断相邻 | O(V) | **O(1)** |
| 获取邻居 | **O(1)** | O(V) |
| 适合 | **稀疏图**（E << V²） | 稠密图（E ≈ V²） |
| 竞赛中使用频率 | **更高** | 较低 |

**竞赛中绝大多数图是稀疏图，所以默认用邻接表。**

**什么时候用邻接矩阵？**
- 需要频繁判断两个节点是否相邻（O(1) vs O(V)）
- 节点数很少（V < 1000 左右）
- 涉及矩阵运算的高级图论题

> ✅ 你会根据操作类型选存储结构，这已经是从“会写代码”走向“会做题”的关键一步。

## 五、所有图类型的实现关系

```
有向加权图（最通用的形式）
  │
  ├── 有向无权图 = 权重全设为 1
  │
  ├── 无向加权图 = 每条边加两次（双向）
  │     │
  │     └── 无向无权图 = 双向 + 权重全设为 1
  │
  └── 所有图都可以用邻接表或邻接矩阵实现
```

**核心思路**：只要实现了「有向加权图」，其他所有图类型都是它的简化版。

这样设计的好处是复用同一套代码骨架，减少你在考场上临时改来改去出错的风险。你可以先写最通用版本，再用“默认权重=1”或“双向加边”快速退化成其他图。思路统一，心里就不慌。

### 5.1 无权图 = 权重设为 1

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
int applePrice = 1;               // 默认每个苹果按 1 元计价
int from = 2;                     // 从 2 号摊位
int to = 5;                       // 到 5 号摊位
// addRoad(from, to, applePrice); // 给这条路一个默认成本
```

**第二步：过渡说明（1-2 句话）**

> 这和图里“无权边当作权重 1”完全同一个想法。下面代码就是把这个默认值传进通用加权接口。

**第三步：给原始代码加注释**

```cpp
// 有向无权图：复用加权图，权重默认 1
graph.addEdge(from, to, 1);    // 复用加权接口，把权重固定成 1
```

### 5.2 无向图 = 双向添加

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
vector<vector<int>> road(6);    // 6 个地点的道路表
road[1].push_back(4);           // 1 到 4 能走
road[4].push_back(1);           // 4 到 1 也能走（双向路）
```

**第二步：过渡说明（1-2 句话）**

> 小例子里你手动写了两次。下面函数就是把这两步封装起来，避免你每次忘记补反向边。

**第三步：给原始代码加注释**

```cpp
// 无向图：每条边添加两次
void addUndirectedEdge(int u, int v, int w) {
    graph[u].push_back({v, w});    // 正向边：u → v
    graph[v].push_back({u, w});    // 反向边：v → u，保证无向
}
```

> 💡 **老师提醒：** 无向图最常见失误就是只加了一边，结果连通性、最短路全错。

## 六、竞赛中最常用的建图方式

### 6.1 邻接表建图模板（背下来）

你背模板不是为了机械记忆，而是为了在比赛里省脑力。模板熟了之后，你就能把注意力放在算法本身，而不是担心建图细节写错。先抄熟，再默写，最后做到看题就能改。

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
int n = 3;                       // 3 个窗口
vector<vector<int>> q(n);        // 每个窗口一条排队队列
q[0].push_back(2);               // 0 号窗口后面排了 2 号人
q[0].push_back(1);               // 0 号窗口后面又排了 1 号人
```

**第二步：过渡说明（1-2 句话）**

> 你可以把“窗口队列”替换成“节点邻居列表”，结构一模一样。下面代码只是把输入、无权/有权两种版本都整理成比赛可直接套用的模板。

**第三步：给原始代码加注释**

```cpp
#include <vector>
using namespace std;

// ===== 无权图 =====
int n, m;  // n 个节点，m 条边
vector<vector<int>> graph(n);  // ⚠️ 实战里要先读入 n，再按 n 初始化

// 读入 m 条有向边
for (int i = 0; i < m; i++) {
    int u, v;                    // 一条边的起点和终点
    cin >> u >> v;               // 从输入读取 u, v
    graph[u].push_back(v);       // 加边 u → v
    // 如果是无向图，加上：graph[v].push_back(u);
}

// 遍历节点 u 的所有邻居
for (int v : graph[u]) {
    // v 是 u 的一个邻居，可以在这里写遍历逻辑
}


// ===== 加权图 =====
struct Edge {
    int to, weight;              // 终点与边权
};
vector<vector<Edge>> graph(n);   // 每个点连出去若干条带权边

// 读入 m 条有向加权边
for (int i = 0; i < m; i++) {
    int u, v, w;                 // 起点、终点、权重
    cin >> u >> v >> w;          // 读入一条边
    graph[u].push_back({v, w});  // 添加 u → v，权重 w
    // 如果是无向图，加上：graph[v].push_back({u, w});
}

// 遍历节点 u 的所有邻居及权重
for (auto& [v, w] : graph[u]) {
    // ⚠️ 结构化绑定：v 对应 to，w 对应 weight（C++17）
}
```

> 💡 **老师提醒：** 模板里变量名别在不同作用域乱复用，考场紧张时很容易把 `v`（节点）和 `v`（终点）看混。

**C++ 知识点**：
- `vector<vector<int>> graph(n)`：n 个节点，每个节点有一个邻居列表
- `graph[u].push_back(v)`：给节点 u 添加邻居 v
- `auto& [v, w] : graph[u]`：C++17 的结构化绑定，自动拆开 Edge 的两个字段
- 如果编译器不支持 C++17，用 `for (auto& e : graph[u])` 再 `e.to`, `e.weight`

> ✅ 你已经接近“建图即本能”的状态了，后面刷题速度会明显提升。

### 6.2 节点不是整数怎么办？

用哈希表做映射：

这是把“字符串世界”翻译成“整数世界”的桥梁。因为数组和向量都喜欢连续下标，先映射再建图会快很多，也更稳。*就像给每位同学发学号后再点名，比每次念全名更高效*。

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
unordered_map<string, int> seat; // 名字 -> 座位号
seat["Alice"] = 0;              // 给 Alice 分配 0 号座位
seat["Bob"] = 1;                // 给 Bob 分配 1 号座位
int bobId = seat["Bob"];        // 之后都用整数编号处理
```

**第二步：过渡说明（1-2 句话）**

> 上面是手动分配编号。下面这段代码把“如果没出现过就自动分配”的逻辑封装成函数，更适合读入大规模数据。

**第三步：给原始代码加注释**

```cpp
unordered_map<string, int> idMap;
int nextId = 0;  // 下一个可分配的整数编号

int getId(string name) {
    if (idMap.find(name) == idMap.end()) {
        idMap[name] = nextId++;  // 新名字第一次出现，分配新编号
    }
    return idMap[name];          // 返回这个名字对应的稳定编号
}

// 然后用整数 id 建图
graph[getId("Alice")].push_back(getId("Bob"));
```

## 七、完整代码：有向加权图（邻接表实现）

到这里你会看到一个“工程化”的版本：把增删查都封装进类。这样做的原因是接口统一，调用时不容易写乱，后面接 Dijkstra、拓扑排序也能直接复用。

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
vector<vector<int>> road(4);      // 4 个地点的路线表
road[0].push_back(2);             // 0 能到 2
bool ok = false;                  // 先假设 0 不能到 1
for (int x : road[0]) if (x == 1) ok = true; // 扫一遍找有没有 1
```

**第二步：过渡说明（1-2 句话）**

> 这个小例子展示了“加边 + 查边”的最基本动作。下面类代码把这些动作系统化，并加入权重和删除功能。

**第三步：给原始代码加注释**

```cpp
class WeightedDigraph {
public:
    struct Edge {
        int to;                                 // 边的终点
        int weight;                             // 边权
        Edge(int to, int weight) : to(to), weight(weight) {} // 构造函数
    };

private:
    vector<vector<Edge>> graph;                 // 邻接表本体

public:
    WeightedDigraph(int n) : graph(n) {}        // 创建 n 个节点

    // 添加有向边 from → to，权重 weight
    void addEdge(int from, int to, int weight) {
        graph[from].emplace_back(to, weight);   // emplace_back 原地构造，减少一次临时对象
    }

    // 删除有向边 from → to
    void removeEdge(int from, int to) {
        auto& edges = graph[from];              // ⚠️ 引用，避免拷贝整条邻居列表
        for (auto it = edges.begin(); it != edges.end(); ++it) {
            if (it->to == to) {
                edges.erase(it);                // 删除匹配到的第一条边
                break;                          // 删完立即退出，避免迭代器继续失效使用
            }
        }
    }

    // 判断是否存在边 from → to
    bool hasEdge(int from, int to) {
        for (auto& e : graph[from]) {
            if (e.to == to) return true;        // 找到就立刻返回
        }
        return false;                           // 扫完没找到
    }

    // 获取边 from → to 的权重
    int weight(int from, int to) {
        for (auto& e : graph[from]) {
            if (e.to == to) return e.weight;    // 找到目标边，返回权重
        }
        return -1;  // 不存在该边
    }

    // 获取节点 v 的所有邻居
    const vector<Edge>& neighbors(int v) {
        return graph[v];                        // 返回只读引用，避免拷贝
    }
};
```

> 💡 **老师提醒：** `erase` 后迭代器会失效，所以这里删完就 `break`，不要继续用原迭代器往后走。

**无向加权图**只需要在 addEdge 和 removeEdge 时操作两个方向：

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
vector<vector<int>> road(5);      // 5 个地点
road[2].push_back(4);             // 去程：2 -> 4
road[4].push_back(2);             // 回程：4 -> 2
```

**第二步：过渡说明（1-2 句话）**

> 小例子体现了无向边的本质：一条边等价于两条方向相反的有向边。下面类就是把这个规则封装进接口。

**第三步：给原始代码加注释**

```cpp
class WeightedUndigraph {
    WeightedDigraph g;                 // 复用有向图实现作为底层
public:
    WeightedUndigraph(int n) : g(n) {} // 初始化 n 个节点

    void addEdge(int u, int v, int w) {
        g.addEdge(u, v, w);     // 正向
        g.addEdge(v, u, w);     // 反向，合起来就是无向边
    }

    void removeEdge(int u, int v) {
        g.removeEdge(u, v);     // 删正向
        g.removeEdge(v, u);     // 删反向
    }

    // hasEdge, weight, neighbors 直接委托给 g
};
```

> ✅ 你已经能把“有向实现”平滑扩展到“无向实现”了，这个抽象能力非常关键。

## 八、关键概念速查表

这一节建议你做成考前 1 分钟速读卡片。每次刷题前先扫一遍，能快速唤醒图论的“操作肌肉记忆”。短句、高频、可回忆，就是这张表的价值。

| 概念 | 说明 |
|------|------|
| **邻接表** | `graph[u]` = 节点 u 的所有邻居列表，空间 O(V+E) |
| **邻接矩阵** | `matrix[u][v]` = u→v 的边信息，空间 O(V²) |
| **稀疏图用邻接表** | 竞赛中绝大多数情况 |
| **无向图 = 双向添加** | `addEdge(u,v)` + `addEdge(v,u)` |
| **无权图 = 权重为 1** | 复用加权图代码 |
| **`emplace_back`** | 比 `push_back` 少一次拷贝，更高效 |
| **结构化绑定** | `auto& [v, w] : edges`，C++17 语法糖 |

## 九、后续学习路线

```
图的存储（邻接表/邻接矩阵） ← 你在这里
  │
  ├── 图的 DFS / BFS 遍历（和树几乎一样，多了 visited 数组）
  ├── 拓扑排序（有向无环图，用入度）
  ├── 最短路算法
  │    ├── BFS（无权图最短路）
  │    ├── Dijkstra（加权图，无负权）
  │    └── Bellman-Ford / SPFA（有负权）
  ├── 最小生成树（Kruskal / Prim）
  └── 并查集（判断连通性）
```

建图是所有图算法的第一步。第六节的建图模板一定要背下来，后面每道图论题都会用到。

> 💡 **老师提醒：** 学路线图时不要贪快并行学太多分支，按“存储→遍历→最短路→最小生成树→并查集”顺序会更稳。
