# 欧拉图与一笔画问题 学习笔记（修正版）

> **前置知识** ：图论术语（度、入度、出度）、图的 DFS 遍历
> **核心能力** ：通过节点的**度数**快速判断一笔画是否可行，以及从哪里开始画
> **竞赛应用** ：欧拉路径/回路问题在 ICPC 中偶尔出现，判定条件必须记住

---

## 一、一笔画游戏的规则

> 从某个节点出发， **每条边恰好经过一次** （不能重复、不能遗漏），把所有边都走完。
> 节点可以重复经过，但边不行。

```
这个图能一笔画吗？

    1 --- 2
    |     |
    3 --- 4

答案：能！比如 1→2→4→3→1（回到起点，这叫欧拉回路）
```

```
这个图呢？

    1 --- 2
    |   / |
    | /   |
    3 --- 4

答案：能！但必须从 2 或 3 出发
```

**小时候全靠运气瞎试，现在有套路了：看度数！**

你会发现这个套路的价值特别高：不用把所有走法都试一遍，只看度数就能先判"有没有解"。这就是算法训练最重要的思维转变，从暴力枚举转向条件判定。先判断，再动手画，效率会高很多。

## 二、核心判定规则（必背）

### 2.1 无向图

| 条件                              | 结论                                                                   |
| --------------------------------- | ---------------------------------------------------------------------- |
| **所有节点度数都是偶数**    | ✅ 存在欧拉**回路** （从任意节点出发，走完所有边回到起点）       |
| **恰好 2 个节点度数为奇数** | ✅ 存在欧拉**路径** （必须从其中一个奇数度节点出发，走到另一个） |
| **奇数度节点超过 2 个**     | ❌ 无法一笔画                                                          |

### 2.2 有向图

| 条件                                                                                    | 结论                      |
| --------------------------------------------------------------------------------------- | ------------------------- |
| **每个节点入度 = 出度**                                                           | ✅ 存在欧拉**回路** |
| **1 个节点出度比入度多 1（起点），1 个节点入度比出度多 1（终点），其余入度=出度** | ✅ 存在欧拉**路径** |
| **其他情况**                                                                      | ❌ 不存在欧拉路径         |

> **前提** ：图必须是连通的（无向图连通，有向图弱连通）。不连通的图即使度数满足也不行。

> 💡 **老师提醒：** 考场上最常见丢分点就是"只看度数不看连通性"，这一步一定要单独检查。

## 三、为什么看度数就能判断？

这部分你一定要真的想通。因为每条边只能走一次，所以一个中间节点每进一次就得出一次，天然成对出现。 *就像进地铁闸机和出闸机的区别* ：除起点终点外，进出次数要配平。理解了这个，你就不是在背结论，而是在推结论。

### 3.1 直觉理解

每条边经过 **恰好一次** ，意味着每次「进入」一个节点，就必须「离开」它。

 **类比** ：想象你在一个城市里走路，每条街道只能走一次。每次你走进一个路口，就必须从另一条街道走出去。所以每个路口的「进」和「出」必须配对。

* **中间节点** ：进一次、出一次，消耗 2 条边 → 度数必须是偶数
* **起点** ：多出去一次（开始时不需要进来） → 可以是奇数
* **终点** ：多进来一次（结束时不需要出去） → 可以是奇数

所以最多只有 2 个奇数度节点（起点和终点）。如果起点=终点（回路），那么 0 个奇数度节点。

### 3.2 七桥问题验证

简化图（线=桥，数字=有几条桥）：

等价为节点间有多少条边（多重图）：

  C：连接 A 有 2 条 + 连接 B 有 1 条 + 连接 D 有 1 条 = 度数 4（偶数）

实际上经典七桥问题的原始连接是：
  A──C（2条边）  A──D（2条边）

  C──D（1条边）

## 四、术语定义

| 术语                                   | 定义                                       |
| -------------------------------------- | ------------------------------------------ |
| **欧拉路径（Eulerian Path）**    | 经过图中**每条边恰好一次**的路径     |
| **欧拉回路（Eulerian Circuit）** | 起点和终点**相同**的欧拉路径         |
| **欧拉图**                       | 存在欧拉**回路**的图                 |
| **半欧拉图**                     | 存在欧拉**路径**但不存在欧拉回路的图 |
| **非欧拉图**                     | 既不存在欧拉路径也不存在欧拉回路           |

### 4.1 三种情况的例子

#### 欧拉图（存在欧拉回路）

```
    1 ──→ 2
    ↑     │
    │     ↓
    4 ←── 3

每个节点入度=出度=1：
  节点1：出度1(→2)，入度1(←4)  ✅
  节点2：出度1(→3)，入度1(←1)  ✅
  节点3：出度1(→4)，入度1(←2)  ✅
  节点4：出度1(→1)，入度1(←3)  ✅

欧拉回路：1→2→3→4→1
```

#### 半欧拉图（只有欧拉路径）【修正版】

原版图有重复节点的 bug，下面是正确的例子：

```
    1 ──→ 2
          │
          ↓
    4 ←── 3
    │
    ↓
    5

每个节点的入度和出度：
  节点1：出度=1(→2)，入度=0      → 出度比入度多1（起点）
  节点2：出度=1(→3)，入度=1(←1) → 入度=出度  ✅
  节点3：出度=1(→4)，入度=1(←2) → 入度=出度  ✅
  节点4：出度=1(→5)，入度=1(←3) → 入度=出度  ✅
  节点5：出度=0，    入度=1(←4)  → 入度比出度多1（终点）

结论：只有1出发、5结束的欧拉路径，不存在回路
欧拉路径：1→2→3→4→5
```

#### 非欧拉图【修正版】

原版度数标注有误（节点2和4中间没有边，但写了度数=2），下面是正确版：

```
    1 ── 2
    |
    3         4 ── 5

节点1：度数=2（连着 2 和 3）
节点2：度数=1（只连着 1）   ← 奇数度
节点3：度数=1（只连着 1）   ← 奇数度
节点4：度数=1（只连着 5）   ← 奇数度（注意：4和1/2/3之间没有边！）
节点5：度数=1（只连着 4）   ← 奇数度

奇数度节点有 4 个，但更关键的问题是：
  {1,2,3} 和 {4,5} 是两个独立的连通分量，图根本不连通！

即使只看度数（4个奇数度节点 > 2），结论已经是不行了。
连通性是更根本的前提，不满足连通性，直接判 ❌。
```

## 五、寻找欧拉路径的算法

### 5.1 暴力思路（能想到就行）

1. 用度数判断起点
2. 从起点出发，用 DFS「遍历所有路径」的框架
3. 穷举所有路径，检查是否每条边都走了

能做到，但复杂度很高（指数级），因为要穷举所有可能的路径。

### 5.2 Hierholzer 算法（高效解法）

 **核心思想** ：DFS 遍历所有边，把**后序遍历**的结果反转，就是欧拉路径。

```
Hierholzer 算法流程：
1. 确定起点（奇数度节点，或任意节点如果是欧拉回路）
2. DFS 遍历，每走过一条边就删除它（或标记为已用）
3. 当一个节点没有可走的边了，把它加入结果
4. 最后把结果反转 → 就是欧拉路径
```

 **时间复杂度** ：O(E)，每条边只走一次。

 **伪码** ：

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
vector<int> done;                    // 记录"走到底后"回填的站点
done.push_back(4);                   // 4 号站没有新路了，先记下来
done.push_back(2);                   // 回到 2 号站也走完了，再记
reverse(done.begin(), done.end());   // 反转得到正向路线
```

**第二步：过渡说明（1-2 句话）**

> Hierholzer 的精髓就是"后序记录、最后反转"。下面伪码正是这个动作，只是把它放到真正的图 DFS 里。

**第三步：给原始代码加注释**

```cpp
vector<int> result;

void dfs(int u) {
    while (graph[u] 还有未走过的边) {
        int v = graph[u] 的下一条未走过的边; // 取一条还能走的边
        标记边 u→v 为已走过;                  // ⚠️ 每条边只能用一次
        dfs(v);                                // 继续深入直到无边可走
    }
    result.push_back(u);   // 后序位置记录，最后需要整体反转
}

// 调用后 reverse(result) 就是欧拉路径
```

> 具体代码实现会在后面的图论算法章节详细讲解，现在理解思路即可。

## 六、欧拉路径 vs 哈密顿路径（易混淆）

这个对比必须牢。题目里只改一个词，解法就可能完全不同。 *就像"把街道都走一遍"和"把景点都打卡一遍"的区别* ：一个按边，一个按点，难度差很多。

| 对比   | 欧拉路径                   | 哈密顿路径                   |
| ------ | -------------------------- | ---------------------------- |
| 要求   | 每条**边**恰好走一次 | 每个**节点**恰好走一次 |
| 判定   | 简单（看度数）             | 困难（NP 完全问题）          |
| 算法   | Hierholzer，O(E)           | 没有多项式时间算法           |
| 竞赛中 | 有套路可循                 | 通常需要暴力或状压 DP        |

> 一个是走遍所有 **边** ，一个是走遍所有 **节点** 。名字很像但完全不同！

> ✅ 你能把这两个概念分开，后面做题就不容易被题面"误导词"带偏。

## 七、关键概念速查表

| 概念                      | 说明                                                          |
| ------------------------- | ------------------------------------------------------------- |
| **欧拉路径**        | 每条边恰好经过一次的路径                                      |
| **欧拉回路**        | 起点=终点的欧拉路径                                           |
| **欧拉图**          | 无向图所有节点度数为偶数 / 有向图所有节点入度=出度            |
| **半欧拉图**        | 无向图恰好 2 个奇数度节点 / 有向图 1 个出度多 1，1 个入度多 1 |
| **Hierholzer 算法** | DFS 后序遍历 + 反转 = 欧拉路径，O(E)                          |
| **哈密顿路径**      | 每个节点恰好经过一次（和欧拉路径不同！）                      |

## 八、快速判定口诀

```
无向图一笔画：
  全偶 → 任意起点，回路
  两奇 → 奇数度起点，路径
  其他 → 不行

有向图一笔画：
  全相等 → 任意起点，回路
  一多一少 → 多出的是起点
  其他 → 不行

别忘了前提：图必须连通！
```
