## 一、什么是生成树？

### 1.1 定义

给定一个**无向连通图** G，它的**生成树**是一个子图，满足：

- ✅ 包含 G 的**所有节点**
- ✅ 是一棵**树**（连通且无环）
- ✅ 恰好有 **V-1 条边**（V 是节点数）

```
原图（5 个节点，7 条边）：
    1 ──3── 2
    |╲      |
   5|  ╲4   |2
    |    ╲  |
    3 ──6── 4
        |
        7
        |
        5

一棵生成树（5 个节点，4 条边）：
    1 ──3── 2
    |       |
   5|      2|
    |       |
    3       4
            |
           7|
            |
            5
保留了所有 5 个节点，用 4 条边把它们连通，且无环。
```

**类比**：原图像一张复杂的蜘蛛网，生成树就是从中挑出最少的丝线，恰好能把所有连接点串起来，同时不形成环。

### 1.2 关键性质

- 一个图可以有**多棵不同的**生成树
- 生成树的边数**固定为 V-1**（少一条就不连通，多一条就有环）
- 生成树是原图的**生成子图**（保留所有节点的子图）

## 二、什么是最小生成树（MST）？

### 2.1 定义

> 在加权无向图中，**边权重总和最小**的生成树就是最小生成树（Minimum Spanning Tree，简称 MST）。

```
加权图：
    A ──5── B
    |╲      |
   3|  ╲8   |2
    |    ╲  |
    C ──7── D

生成树 1：A-B(5), B-D(2), A-C(3)  总权重 = 10
生成树 2：A-C(3), B-D(2), A-B(5)  总权重 = 10（和上面一样）
生成树 3：A-C(3), B-D(2), C-D(7)  总权重 = 12
生成树 4：A-B(5), A-C(3), A-D(8)  总权重 = 16

最小生成树 = 生成树 1（或 2），总权重 = 10
```

### 2.2 实际应用场景

| 场景 | 节点 | 边 | 权重 | 目标 |
|------|------|------|------|------|
| **城市修路** | 城市 | 可修建的道路 | 修路成本 | 最低成本连通所有城市 |
| **网络布线** | 服务器/电脑 | 可铺设的网线 | 布线成本 | 最低成本连通所有设备 |
| **电路设计** | 电子元件 | 可连接的线路 | 线路长度 | 最短总线路连接所有元件 |
| **管道铺设** | 建筑/水源 | 可铺设的管道 | 管道费用 | 最低成本供水所有建筑 |

**共同特点**：要把所有点连起来，且总代价最小。

## 三、两种经典算法

### 3.1 Kruskal 算法——按边排序，逐条加入

**核心思路**：

1. 把所有边按权重**从小到大排序**
2. 从最小的边开始，逐条考虑：
   - 如果加入这条边**不会形成环** → 加入
   - 如果会形成环 → 跳过
3. 直到选够 V-1 条边为止

**关键工具**：**并查集（Union-Find）**用来快速判断「加入这条边是否会形成环」。如果边的两个端点已经在同一个集合中（已连通），加入就会成环。

```
边排序后：B-D(2), A-C(3), A-B(5), C-D(7), A-D(8)

第 1 条：B-D(2)  → B 和 D 不连通，加入 ✓
第 2 条：A-C(3)  → A 和 C 不连通，加入 ✓
第 3 条：A-B(5)  → A 和 B 不连通，加入 ✓
  （已经选了 3 = V-1 条边，结束！）

MST 边：B-D(2), A-C(3), A-B(5)，总权重 = 10
```

**类比**：就像你在折扣季购物。把所有商品按价格排序，从最便宜的开始买，但如果买了会重复（成环），就跳过。

**贪心策略**：每次选最小的边，只要不成环就要。

### 3.2 Prim 算法——从一个点出发，逐步扩展

**核心思路**：

1. 从任意一个节点开始
2. 在所有「已选节点」到「未选节点」的边中，选**权重最小的边**
3. 把这条边和对应的新节点加入生成树
4. 重复步骤 2-3，直到所有节点都被选入

**关键工具**：**优先级队列**（小顶堆）用来快速找到权重最小的边。

```
从 A 开始：

已选: {A}    候选边: A-C(3), A-B(5), A-D(8)
→ 选 A-C(3)

已选: {A,C}  候选边: A-B(5), A-D(8), C-D(7)
→ 选 A-B(5)

已选: {A,C,B} 候选边: A-D(8), C-D(7), B-D(2)
→ 选 B-D(2)

已选: {A,C,B,D} 所有节点都选入，结束！

MST 边：A-C(3), A-B(5), B-D(2)，总权重 = 10
```

**类比**：你从家出发，每次选最近的（最便宜的）邻居城市修路，把它纳入你的网络，然后再看这个扩大的网络周围哪个城市最近。

**与 Dijkstra 的关系**：Prim 算法的代码结构和 Dijkstra 非常像，都用优先级队列逐步扩展。区别在于 Dijkstra 关注「到起点的总距离」，Prim 关注「到已选集合的单条边权重」。

## 四、Kruskal vs Prim 对比

| 对比 | Kruskal | Prim |
|------|---------|------|
| **视角** | 从**边**的角度出发 | 从**节点**的角度出发 |
| **排序方式** | 一次性对所有边排序 | 用优先级队列动态排序 |
| **核心工具** | 并查集（Union-Find） | 优先级队列 |
| **思路** | 选最小的边，不成环就加 | 从一点扩展，选最近的邻居 |
| **适合** | 稀疏图（边少） | 稠密图（边多） |
| **复杂度** | O(E log E) | O(E log V) |

**竞赛中怎么选？**

- 大多数题目两种都能做
- Kruskal 代码通常更简短（排序 + 并查集），竞赛中更常用
- Prim 适合边非常多的稠密图场景

---

## 五、两种算法的共同点——贪心

Kruskal 和 Prim 都基于**贪心思想**：

- Kruskal：每次选全局最小的边
- Prim：每次选距离已选集合最近的边

**为什么贪心在这里是正确的？**

这背后有一个叫做**切割定理（Cut Property）**的数学保证：对于图中任意一个「切割」（把节点分成两组），横跨两组的最小权重边一定属于某棵最小生成树。

你不需要记住证明，只需要知道：MST 问题上，贪心策略是被数学严格证明过的，不是碰运气。

## 六、有趣的应用：随机迷宫生成

MST 算法还可以用来**生成游戏地图**：

- 把地图网格的每个格子看作节点
- 相邻格子之间的墙壁看作边
- 给每条边一个**随机权重**
- 跑一遍 MST 算法 → 被选中的边 = 打通的通道

因为 MST 保证连通且无环，所以生成的迷宫：
- ✅ 起点到终点一定有路径（连通）
- ✅ 没有多余的环路（每条路都是唯一的）
- ✅ 看起来随机自然（权重是随机的）

Kruskal 生成的迷宫特点：**多个区域同时生长**，最后合并  
Prim 生成的迷宫特点：**从一点向外扩展**，像树根蔓延

## 七、关键概念速查表

| 概念 | 说明 |
|------|------|
| **生成树** | 包含所有节点、无环、连通的子图，恰好 V-1 条边 |
| **最小生成树（MST）** | 边权重总和最小的生成树 |
| **Kruskal** | 按边排序 + 并查集判环，贪心选最小边 |
| **Prim** | 从一点扩展 + 优先级队列选最近邻居 |
| **切割定理** | 横跨两组节点的最小边一定属于 MST（贪心正确的数学保证） |
| **并查集** | 快速判断两个节点是否已连通（Kruskal 的核心工具） |
| **V-1 条边** | 生成树的边数固定 = 节点数 - 1 |
