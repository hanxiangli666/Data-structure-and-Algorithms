## 一、最短路径问题的分类

先把分类框架背熟，你写代码时才不会“拿着锤子找钉子”。很多同学不是不会实现，而是第一步选错算法。你只要先判断“单源/多源、是否负权”，后面思路就会很清晰。

```
最短路径问题
├── 单源最短路径（一个起点到所有其他节点）
│   ├── 无负权 → Dijkstra 算法
│   └── 有负权 → Bellman-Ford / SPFA 算法
│
├── 点对点最短路径（一个起点到一个终点）
│   └── 无负权 → A* 算法（Dijkstra 的优化版）
│
└── 多源最短路径（任意两点之间）
    └── Floyd 算法（动态规划）
```

### 1.1 输出格式

| 问题类型 | 输出 | 含义 |
|---------|------|------|
| 单源最短路径 | 一维数组 `distTo[i]` | 起点到节点 i 的最短距离 |
| 多源最短路径 | 二维数组 `dist[i][j]` | 节点 i 到节点 j 的最短距离 |

---

## 二、算法选择速查表（必背）

| 算法 | 解决什么问题 | 能否处理负权 | 本质 | 时间复杂度 |
|------|-------------|-------------|------|-----------|
| **Dijkstra** | 单源最短路 | ❌ 不能 | BFS + 贪心 + 优先队列 | O(E log V) |
| **A*** | 点对点最短路 | ❌ 不能 | Dijkstra + 启发函数 | 优于 Dijkstra |
| **SPFA** | 单源最短路 | ✅ 能 | BFS 变种 | 平均 O(E)，最坏 O(VE) |
| **Bellman-Ford** | 单源最短路 | ✅ 能，可检测负环 | 松弛操作 | O(VE) |
| **Floyd** | 多源最短路 | ✅ 能 | 动态规划 | O(V³) |

**快速判断用哪个**：

*就像你选出行工具：同城短途骑车，跨城坐高铁，国际出行坐飞机*。算法选择也是同样逻辑，场景决定工具。

```
需要多源最短路？ → Floyd
  ↓ 否
有负权边？ → SPFA / Bellman-Ford
  ↓ 否
知道终点？ → A*（如果有好的启发函数）
  ↓ 否
→ Dijkstra（最常用！）
```

## 三、负权重边为什么麻烦？

你需要记住的不是“负权危险”这句口号，而是它破坏了 Dijkstra 的贪心前提。贪心一旦前提失效，局部最优就不再保证全局最优。这个认知会反复出现在后面的很多算法里。

### 3.1 负权边破坏贪心假设

Dijkstra 的贪心逻辑：「当前距离最小的节点，以后不可能找到更短的路径到达它」。

**无负权时，这个假设成立**：

```
    s ──3──→ a
    │
    └──4──→ b ──?──→ a

s→a 的距离 = 3
s→b→...→a 的距离 ≥ 4（因为后面的边权重 ≥ 0）
所以 s→a = 3 就是最短的，可以放心确定 ✓
```

**有负权时，假设被打破**：

```
    s ──3──→ a
    │
    └──4──→ b ──(-10)──→ a

s→a 的距离 = 3
s→b→a 的距离 = 4 + (-10) = -6 ← 比 3 小！
贪心提前确定了 3 是最短的，但其实 -6 才是最短的 ✗
```

**类比**：在没有折扣的商店（无负权），买到的最便宜的东西就是最终最便宜的。但如果有的商店给你「倒贴钱」（负权），那走远一点绕过去反而更划算。

### 3.2 负权环让问题无解

```
    s ──→ a ──→ b
              ↑    │
              │    ↓
              └─(-5)─ c

a→b→c→a 的环权重 = ... + (-5) + ... < 0

你可以在这个环上无限转圈，距离无限减小
→ 最短路径 = -∞，问题无意义
```

Bellman-Ford 算法可以**检测负权环**（如果跑完 V-1 轮松弛后还能继续更新，说明存在负权环）。

> 💡 **老师提醒：** 题目只要出现“可能有负权边”，就先别默认 Dijkstra，先确认是否允许用 Bellman-Ford/SPFA。

## 四、Dijkstra 算法

### 4.1 核心思想

> **BFS + 贪心 + 优先级队列**

标准 BFS 用普通队列，每次取出一个节点，先进先出。Dijkstra 用**优先级队列**（小顶堆），每次取出**距离最小**的节点。

### 4.2 从 BFS 推导出 Dijkstra

| 对比 | 标准 BFS | Dijkstra |
|------|---------|----------|
| 数据结构 | 普通 queue | **priority_queue**（小顶堆） |
| 防重复 | `visited` 布尔数组 | **`distTo` 数组**（同时记录距离） |
| 适用场景 | 无权图最短路 | 加权图最短路（无负权） |
| 优先级 | 先进先出 | **距离小的先出** |

### 4.3 直觉理解

**类比**：你在一张地图上，从起点出发找到所有地方的最短路。

- BFS 就像水波扩散——每一圈扩散一步，所有方向均匀推进
- Dijkstra 就像**水在地形不同的地面扩散**——水会优先流向更低（权重更小）的地方

### 4.4 伪码概览

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
int oldCost = 10;                    // 当前记录的最小花费
int newCost = 7;                     // 新路线花费更低
if (newCost < oldCost) oldCost = newCost; // 发现更优就更新
```

**第二步：过渡说明（1-2 句话）**

> 这个三行逻辑就是“松弛”的本质。下面 Dijkstra 伪码只是在图上不断重复这个更新动作，并用小顶堆优先处理更近的点。

**第三步：给原始代码加注释**

```cpp
vector<int> dijkstra(Graph& graph, int src) {
    vector<int> distTo(n, INT_MAX); // distTo[i]：src 到 i 的当前最短距离
    // 小顶堆：{距离, 节点}
    priority_queue<pair<int,int>, vector<pair<int,int>>, greater<>> pq;
    
    distTo[src] = 0;                // 起点到自己距离为 0
    pq.push({0, src});              // 起点入堆
    
    while (!pq.empty()) {
        auto [dist, cur] = pq.top(); pq.pop(); // ⚠️ 结构化绑定：取最小距离节点
        
        if (dist > distTo[cur]) continue;   // 过期状态，直接跳过
        
        for (auto& [next, weight] : graph[cur]) {
            int newDist = distTo[cur] + weight; // 走 cur->next 的候选新距离
            if (newDist < distTo[next]) {
                distTo[next] = newDist;         // 松弛成功，更新最短路
                pq.push({newDist, next});       // 把新状态入堆
            }
        }
    }
    return distTo;
}
```

> ✅ 你已经抓住 Dijkstra 主循环了，后面只要练熟模板就能稳定拿分。

## 五、A* 算法

### 5.1 核心思想

> **Dijkstra + 启发函数**

Dijkstra 只看「已经走了多远」（g(x)），A* 还看「估计离终点多远」（h(x)）。

```
f(x) = g(x) + h(x)

g(x) = 起点到 x 的实际距离（已知）
h(x) = x 到终点的估计距离（启发函数猜的）
f(x) = 综合评分，越小越优先出队
```

### 5.2 直觉理解

**类比**：你在迷宫里找出口。

- **Dijkstra**：像一个盲人，只知道自己走了多远，均匀地向所有方向探索
- **A***：像一个正常人，知道出口大概在哪个方向，**优先朝出口方向走**

### 5.3 启发函数 h(x) 的要求

h(x) 必须是「乐观的」——估计值 ≤ 实际值（不能高估）。

常见的启发函数：
- 网格地图：曼哈顿距离 `|x1-x2| + |y1-y2|`
- 欧几里得距离：`sqrt((x1-x2)² + (y1-y2)²)`

如果 h(x) = 0，A* 退化为 Dijkstra。

## 六、Bellman-Ford / SPFA 算法

### 6.1 为什么需要它？

Dijkstra 不能处理负权边，但现实中确实存在负权场景（比如某条路有补贴/折扣）。

### 6.2 SPFA（基于队列的 Bellman-Ford）

> **本质是 BFS 的变种**，允许节点被多次入队。

和 BFS 的区别：

| 对比 | 标准 BFS | SPFA |
|------|---------|------|
| visited 机制 | 每个节点只入队一次 | 节点可以**多次入队** |
| 更新距离 | — | 发现更短路径就更新并重新入队 |
| 检测负环 | — | 如果某节点入队超过 V 次 → 存在负环 |

### 6.3 朴素 Bellman-Ford

对所有边做 V-1 轮「松弛」操作：

```
松弛操作：对于边 u→v，权重 w
  如果 distTo[u] + w < distTo[v]，
  则 distTo[v] = distTo[u] + w
```

做 V-1 轮后如果还能松弛 → 存在负权环。

## 七、Floyd 算法

### 7.1 核心思想

> **动态规划**：i 到 j 的最短路径，要么直接走，要么经过某个中间节点 k。

```
dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])

对所有可能的中间节点 k，都试一遍
```

### 7.2 代码极简（只有 3 行核心逻辑）

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
int direct = 9;                      // i 到 j 的直达成本
int viaK = 4 + 3;                    // 经过中转站 k 的成本
direct = min(direct, viaK);          // 选更便宜的一条
```

**第二步：过渡说明（1-2 句话）**

> Floyd 就是把这件事对所有 i、j、k 反复做一遍。下面三重循环正是在系统地“尝试所有中转点”。

**第三步：给原始代码加注释**

```cpp
// Floyd 算法核心
for (int k = 0; k < n; k++)            // 枚举“是否经过 k”这个中转点
    for (int i = 0; i < n; i++)        // 枚举起点 i
        for (int j = 0; j < n; j++)    // 枚举终点 j
            dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j]); // 走直达或经 k 二选一
```

### 7.3 适用场景

- 节点数少（V ≤ 500 左右，因为 O(V³)）
- 需要任意两点之间的最短路
- 图可能有负权边（但不能有负权环）

## 八、各算法的联系

```
二叉树层序遍历（BFS）
    │
    ├── 图的 BFS 遍历（加 visited 数组）
    │       │
    │       ├── Dijkstra（queue → priority_queue，贪心）
    │       │       │
    │       │       └── A*（加启发函数 h(x)）
    │       │
    │       └── SPFA（允许多次入队，处理负权）
    │
    └── 这些算法全是 BFS 的变种！

动态规划
    │
    └── Floyd 算法（完全不同的思路，用 DP 解多源最短路）
```

> **关键认知**：Dijkstra、A*、SPFA 全是从 BFS 演变来的。你把 BFS 搞透了，这些算法就是改两行代码的事。

> 💡 **老师提醒：** “改两行代码”建立在你真的理解状态定义和更新规则上，别只背表面模板。

## 九、关键概念速查表

| 概念 | 说明 |
|------|------|
| **单源最短路** | 一个起点到所有节点的最短路 |
| **多源最短路** | 任意两节点之间的最短路 |
| **负权边** | 边的权重为负数，会破坏 Dijkstra 的贪心假设 |
| **负权环** | 环上权重和为负，最短路无解（可无限减小） |
| **松弛（Relaxation）** | 发现更短路径就更新 distTo，是所有最短路算法的核心操作 |
| **启发函数 h(x)** | A* 算法中估计到终点距离的函数，必须不高估 |
| **distTo 数组** | 记录起点到各节点的当前最短距离 |
| **priority_queue** | Dijkstra 的核心，让距离最小的节点先处理 |

## 十、竞赛实战建议

1. **默认先想 Dijkstra**——大部分最短路题目没有负权
2. **看到负权 → SPFA 或 Bellman-Ford**
3. **节点数很少（V ≤ 500）且要多源最短路 → Floyd**
4. **网格地图上的点对点 → 考虑 A***
5. **无权图最短路 → 直接 BFS**（不需要 Dijkstra）

```
无权图？ → BFS
有权无负？ → Dijkstra
有负权？ → SPFA
多源？ → Floyd
点对点+网格？ → A*
```
