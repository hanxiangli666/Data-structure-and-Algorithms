# 图论基本术语 学习笔记

> **前置知识**：二叉树、多叉树（图可以看作多叉树的泛化）  
> **核心认知**：图 = 节点 + 边，是最通用的数据结构。树其实是一种特殊的图（无环连通图）  
> **本篇定位**：掌握图论中的基本术语，为后续图算法（BFS、DFS、最短路、最小生成树等）打好基础

---

## 一、图的基本组成

图由两个要素构成：

- **节点（Vertex / Node）**：图中的一个点，有唯一 ID
- **边（Edge）**：连接两个节点的线

```
节点: 1, 2, 3, 4
边:   1--2, 2--3, 3--4, 1--4

图示：
    1 --- 2
    |     |
    4 --- 3
```

---

## 二、图的四种分类

根据边是否有方向和是否有权重，图分为四种：

### 2.1 有向图 vs 无向图

#### 有向图（Directed Graph）

边有方向，用箭头表示。`1 → 3` 表示能从 1 到 3，但不能从 3 到 1。

```
    1 ──→ 2
    │     ↑
    ↓     │
    3 ──→ 4
    
从 1 可以到 3（有箭头）
从 3 不能直接到 1（没有反向箭头）
```

#### 无向图（Undirected Graph）

边没有方向，双向都能走。`1 -- 3` 表示 1 和 3 互相可达。

```
    1 --- 2
    |     |
    3 --- 4

1 和 3 之间有边 → 1 到 3 ✓，3 到 1 ✓
```

> **无向图 = 双向图**。代码实现时，无向边 `1 -- 3` 会存成两条有向边 `1 → 3` 和 `3 → 1`。

### 2.2 加权图 vs 无权图

#### 无权图（Unweighted Graph）

边上没有数值，只表示「连通」。

```
    1 --- 2
    |     |
    3 --- 4
```

#### 加权图（Weighted Graph）

边上有权重（数值），表示距离、费用、时间等。

```
有向加权图：              无向加权图：
    1 ─5→ 2                 1 ──5── 2
    │     ↑                 |       |
   3↓    2│                 3       2
    │     │                 |       |
    3 ─7→ 4                 3 ──7── 4
```

**实际应用**：
- 地图导航：节点=地点，边的权重=距离
- 网络：节点=路由器，边的权重=延迟
- 物流：节点=仓库，边的权重=运输成本

### 2.3 四种组合总结

| 类型 | 方向 | 权重 | 例子 |
|------|------|------|------|
| 无向无权图 | ✗ | ✗ | 社交网络（互相关注） |
| 无向加权图 | ✗ | ✓ | 地图（城市之间的距离） |
| 有向无权图 | ✓ | ✗ | 微博（单向关注） |
| 有向加权图 | ✓ | ✓ | 航班（有方向、有票价） |

## 三、度（Degree）

### 3.1 无向图中的度

节点的**度** = 与该节点相连的边数。

```
    1 --- 2
    |   / |
    | /   |
    3 --- 4 --- 5

节点 1 的度 = 2（连着 2、3）
节点 3 的度 = 3（连着 1、2、4）
节点 4 的度 = 3（连着 2、3、5）
节点 5 的度 = 1（只连着 4）
```

### 3.2 有向图中的度

有向图的边有方向，所以度被分成两种：

- **入度（In-degree）**：指向该节点的边数（有多少箭头指向我）
- **出度（Out-degree）**：从该节点出发的边数（我有多少箭头指向别人）

```
    1 ──→ 2
    │     ↑
    ↓     │
    3 ──→ 4

节点 3：
  入度 = 1（1 → 3，有 1 条边指向 3）
  出度 = 1（3 → 4，3 有 1 条边指出去）

节点 2：
  入度 = 1（1 → 2）
  出度 = 0（没有从 2 出发的边）

节点 4：
  入度 = 1（3 → 4）
  出度 = 1（4 → 2）
```

**类比**：
- 入度就像你收到的信件数
- 出度就像你寄出的信件数

**竞赛应用**：入度为 0 的节点在**拓扑排序**中很重要（它没有前置依赖，可以最先处理）。

## 四、边和节点的数量关系

假设图有 V 个节点（Vertex）、E 条边（Edge）。

### 4.1 简单图

我们通常讨论的是**简单图**——没有以下两种边：

```
自环（Self Loop）：         多重边（Multiple Edges）：
    ┌──┐                      1 ═══ 2
    │  │                    （两条边连接同一对节点）
    └→ 1
 （自己指向自己）
```

### 4.2 边数的范围

| 情况 | 边数 | 说明 |
|------|------|------|
| 最少 | E = 0 | 只有孤立的节点，没有边 |
| 最多（无向图） | E = V(V-1)/2 | 每两个节点之间都有一条边 |
| 最多（有向图） | E = V(V-1) | 每两个节点之间有两条方向相反的边 |

### 4.3 稠密图 vs 稀疏图

| 类型 | 条件 | 特点 |
|------|------|------|
| **稠密图（Dense）** | E 接近 V² | 节点之间大多都相连 |
| **稀疏图（Sparse）** | E 远小于 V² | 只有少数节点相连 |

**为什么要区分稠密和稀疏？** 因为不同的图存储方式适合不同的密度：

- 稠密图 → **邻接矩阵**更合适（后面会学）
- 稀疏图 → **邻接表**更合适（后面会学）

**类比**：
- 稠密图像一个小型公司，每个人都认识其他人
- 稀疏图像一个大城市，每个人只认识几个朋友

## 五、子图（Subgraph）

### 5.1 基本概念

从原图中**删掉一些节点和/或边**，剩下的就是子图。

```
原图 G：                    一个子图 G'：
    1 --- 2                     1 --- 2
    |   / |                           |
    | /   |                           |
    3 --- 4                     3 --- 4
```

G' 的所有节点和边都在 G 中 → G' 是 G 的子图。

### 5.2 两种特殊子图

#### 生成子图（Spanning Subgraph）

**保留所有节点**，只删掉一些边。

```
原图 G：                    生成子图：
    1 --- 2                     1 --- 2
    |   / |                     |     |
    | /   |                     |     |
    3 --- 4                     3     4
                            （删掉了 2-3 和 3-4 的边）
```

所有 4 个节点都保留了，只是少了几条边。

**竞赛应用**：**最小生成树**就是在寻找一个总权重最小的生成子图（保留所有节点，选最少的边使图连通，且总权重最小）。

#### 导出子图（Induced Subgraph）

**选一部分节点**，保留这些节点之间在原图中的**所有边**。

```
原图 G：                    节点 {1,2,3} 的导出子图：
    1 --- 2                     1 --- 2
    |   / |                     |   /
    | /   |                     | /
    3 --- 4                     3
                            （选了 1,2,3，保留它们之间的所有边）
```

## 六、连通性（Connectivity）

### 6.1 无向图的连通性

#### 连通图（Connected Graph）

任意两个节点之间都**存在路径**（不要求直接相连，可以绕路）。

```
连通图：                    非连通图：
    1 --- 2                     1 --- 2     5 --- 6
    |     |                     |     |
    3 --- 4                     3 --- 4

从任意节点都能到其他所有节点    节点 5、6 和 1~4 之间没有路径
```

#### 连通分量（Connected Component）

非连通图可以拆成若干个**连通分量**——每个连通分量内部是连通的，分量之间不连通。

```
这幅图有 3 个连通分量：

    1 --- 2        5 --- 6        8
    |     |        |
    3 --- 4        7

  分量 1: {1,2,3,4}    分量 2: {5,6,7}    分量 3: {8}
```

**竞赛应用**：很多题目要求「判断图是否连通」或「求连通分量的个数」，用 BFS/DFS 或**并查集**都可以解决。

### 6.2 有向图的连通性

有向图因为边有方向，连通性更复杂：

#### 强连通图（Strongly Connected）

任意两个节点之间都有**有向路径**（正着能到，反着也能到）。

```
强连通图：
    1 ──→ 2
    ↑     │
    │     ↓
    4 ←── 3

1→2→3→4→1 形成环路
从任意节点出发都能到达其他所有节点 ✓
```

#### 弱连通图（Weakly Connected）

**忽略方向后**图是连通的，但考虑方向则不一定每对节点互相可达。

```
弱连通图（不是强连通）：
    1 ──→ 2 ──→ 3 ──→ 4

忽略方向后是连通的 ✓
但 4 无法到达 1（有向路径不存在）✗
所以不是强连通，只是弱连通
```

#### 强连通分量（SCC - Strongly Connected Component）

有向图中**最大的**强连通子图。

```
    1 ──→ 2        4 ──→ 5
    ↑     │        ↑     │
    │     ↓        │     ↓
    3 ←── 2        6 ←── 5
    
    wait，我重新画一个更清楚的：

    1 ←→ 2 ←→ 3  ──→  4 ←→ 5 ←→ 6

   SCC 1: {1,2,3}       SCC 2: {4,5,6}
   （内部互相可达）       （内部互相可达）
   但 SCC 1 到 SCC 2 是单向的
```

**竞赛应用**：SCC 在高级图论题中有用（Tarjan 算法、Kosaraju 算法），目前了解概念即可。

## 七、图 vs 树的关系

| 对比 | 树 | 图 |
|------|------|------|
| 方向 | 有根时有方向（父→子） | 有向或无向 |
| 环 | **无环** | **可以有环** |
| 连通性 | 连通（所有节点可达） | 不一定连通 |
| 每个节点的父节点 | 最多 1 个 | 没有限制 |
| 节点关系 | 层级关系（父子） | 平等关系 |

> **树 = 无环连通图**。如果一个图既连通又无环，它就是一棵树。

**类比**：
- 树像一个公司组织架构——每个人只有一个直属上司，从上到下没有循环
- 图像一个社交网络——任意两个人之间都可能有关系，也可能形成小圈子（环）

## 八、关键概念速查表

| 概念 | 说明 |
|------|------|
| **节点（Vertex）** | 图中的点 |
| **边（Edge）** | 连接两个节点的线 |
| **有向图** | 边有方向（箭头） |
| **无向图** | 边无方向（双向） |
| **加权图** | 边有权重（数值） |
| **度（Degree）** | 无向图中与节点相连的边数 |
| **入度 / 出度** | 有向图中指向/指出某节点的边数 |
| **稠密图 / 稀疏图** | E ≈ V² 为稠密，E << V² 为稀疏 |
| **简单图** | 没有自环和多重边 |
| **子图** | 从原图中删掉部分节点/边后的图 |
| **生成子图** | 保留所有节点的子图 |
| **导出子图** | 选部分节点，保留它们之间所有边 |
| **连通图** | 无向图中任意两节点可达 |
| **连通分量** | 非连通图中的最大连通子图 |
| **强连通** | 有向图中任意两节点互相可达 |
| **弱连通** | 忽略方向后连通 |
| **SCC** | 强连通分量 |
