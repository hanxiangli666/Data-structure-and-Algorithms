## 1. 什么是数组

「数组」分为两大类：

| 类型               | 说明                                                                  |
| ------------------ | --------------------------------------------------------------------- |
| **静态数组** | 一块连续的内存空间，创建时就要确定大小，这是数组的原始形态            |
| **动态数组** | 在静态数组基础上封装了常用 API（如 push、insert、remove），使用更方便 |

> 理解了静态数组的底层原理，才能真正明白 vector、list 等数据结构为什么这样设计。

你现在看到的两类数组，不只是“写法不同”，它们背后是两种思维方式。静态数组强调“提前规划”，一开始就把空间定死；动态数组强调“按需增长”，先用起来，不够再扩。*就像租房时一次签一年合同和先按月租再续租的区别*，两种方式都能住，但管理成本和灵活性完全不一样。

> 💡 **老师提醒：** 初学时最容易只记 API，不理解底层代价。你只要盯住“是否连续内存、是否要搬移数据”这两点，复杂度就不会记混。

## 2. 静态数组

### 2.1 基本概念

静态数组的本质：**一块连续的内存空间**。

**第一步：最简单的例子（用最日常的比喻）**

```cpp
int box[3] = {0, 0, 0};      // 准备3个小盒子，先都放0
box[0] = 7;                   // 把数字7放进第1个盒子
box[1] = 9;                   // 把数字9放进第2个盒子
int take = box[0];            // 从第1个盒子取出数字
```

> 上面这个例子是在“按位置放东西、按位置取东西”；下面真实代码做的是同一件事，只是换成了更规范的数组初始化和访问写法。

```cpp
// 定义一个大小为 10 的静态数组
int arr[10];

// 用 memset 初始化为 0（否则内容不确定）
memset(arr, 0, sizeof(arr));

// 赋值
arr[0] = 1;
arr[1] = 2;

// 取值
int a = arr[0];
```

### 2.2 底层发生了什么？

`int arr[10]` 这一行代码做了两件事：

1. 在内存中开辟了一段**连续的内存空间**，大小是 `10 × sizeof(int) = 40 字节`
2. 定义了名为 `arr` 的**数组指针**，指向这段内存的首地址

`arr[1] = 2` 又做了两件事：

1. 计算地址：`arr 首地址 + 1 × sizeof(int)`，找到第二个元素的位置
2. 向该地址写入整数 `2`

### 2.3 三个重要推论

**① 为什么数组索引从 0 开始？**

方便计算地址：

```
arr[i] 的地址 = arr 首地址 + i × sizeof(元素类型)
```

当 `i = 0` 时，就是首地址本身，最自然。

**② 数组名就是指针**

**第一步：最简单的例子（用最日常的比喻）**

```cpp
int snacks[3] = {5, 6, 7};    // 3个零食格子
int* p = snacks;              // p 指向第1个格子
int first = *p;               // 通过指针拿到第1个格子的值
```

> 上面的例子是在说“数组名能当首地址用”；下面代码就是同一个概念的直接表达：`arr` 和 `*arr` 的关系。

```cpp
int arr[10];
// arr 就是指向这块内存首地址的指针
// 所以 *arr 和 arr[0] 是完全等价的
*arr == arr[0]   // 第一个元素的值
```

**③ 未初始化的数组内容不确定**

`int arr[10]` 只是向操作系统申请了一块内存，里面可能存着别人用过的"二手数据"，所以要用 `memset` 初始化。

> Java、Go 等语言会自动初始化为 0，C/C++ 需要手动处理。

### 2.4 数组的超能力：随机访问 O(1)

因为：

- 知道首地址
- 知道每个元素的大小
- 内存连续

所以任意给定索引 `i`，都能**直接计算出**目标元素的地址，不需要遍历，时间复杂度 O(1)。

这句话很关键。因为数组地址是可计算的，所以访问第 1 个和第 10000 个元素，在底层都是“算一次地址再读内存”。你不用一个个走过去，这就是它快的根源。你后面学哈希表、堆、DP 表时，都会反复利用这个特性。

> ✅ 你已经抓住数组最核心的优势了：按索引访问就是快，而且是稳定的快。

## 3. 增删查改详解

### 3.1 查（Read）— O(1)

给定索引，直接计算地址，返回值：

**第一步：最简单的例子（用最日常的比喻）**

```cpp
int ticket[3] = {11, 22, 33}; // 三张号码牌
int num = ticket[1];           // 直接拿第2张号码牌
cout << num << endl;           // 打印22
```

> 上面是“按编号直接拿”；下面代码和它完全一样，只是换了下标数字。

```cpp
int a = arr[3];   // 直接取索引3的值，O(1)
```

> ⚠️ 注意：**给定索引查值是 O(1)**，但如果给定值找索引，只能遍历，是 **O(N)**。

这两个“查”看起来只差两个字，底层成本却差很多。按索引查值时，索引就是地址线索；按值找索引时，数组并不知道值在哪，只能从头到尾比对。*就像“按座位号找人”和“只知道名字在全场找人”的区别*，信息量不同，速度自然不同。

> 💡 **老师提醒：** 题目里如果写“频繁按值查位置”，你就要警惕：纯数组方案大概率会超时。

### 3.2 改（Update）— O(1)

给定索引，直接计算地址，写入值：

**第一步：最简单的例子（用最日常的比喻）**

```cpp
int shelf[3] = {1, 2, 3};     // 三个格子先放好数字
shelf[1] = 99;                // 把第2个格子改成99
cout << shelf[1] << endl;     // 看到更新后的99
```

> 上面的例子是“按位置改值”；下面代码就是同一个操作在原数组上的写法。

```cpp
arr[3] = 999;   // 直接修改索引3的值，O(1)
```

### 3.3 增（Insert）— 分情况

#### 情况一：末尾追加 — O(1)

数组末尾有空位，直接在对应索引赋值：

**第一步：最简单的例子（用最日常的比喻）**

```cpp
int line[5] = {10, 20, 30, 0, 0}; // 前3个位置有人，后2个空位
line[3] = 40;                      // 在下一个空位加一个人
line[4] = 50;                      // 再加一个人
```

> 上面的例子是“队尾补人”；下面代码同理，只是用 for 先放入前4个元素再继续追加。

```cpp
// 数组已有 4 个元素（索引 0~3），末尾追加
int arr[10];
for (int i = 0; i < 4; i++) arr[i] = i; // 先把前4个位置填上0~3

arr[4] = 4;   // 直接赋值，O(1)
arr[5] = 5;   // 再追加一个元素
```

#### 情况二：中间插入 — O(N)

需要把插入位置及之后的元素**全部往后移一位**，才能腾出空间：

**第一步：最简单的例子（用最日常的比喻）**

```cpp
int q[5] = {1, 2, 3, 0, 0};    // 队伍里先有3个人
for (int i = 3; i > 1; i--) {  // 从后往前挪位置
    q[i] = q[i - 1];            // 给中间腾空位
}
q[1] = 99;                      // 把新同学插到中间
```

> 上面的例子是“先挪开，再插入”；下面代码做的是同一件事，目标索引换成了 2。

```cpp
// 在索引 2 处插入元素 666
// ⚠️ 必须从后往前移，否则会覆盖数据
for (int i = 4; i > 2; i--) {
    arr[i] = arr[i - 1]; // 每次把前一个元素拷到后一个位置
}
arr[2] = 666;            // 腾出的位置放入新元素
```

图示：

```
插入前：[0] [1] [2] [3] [_] [_]
                            ↑ 空位

往后移：[0] [1] [_] [2] [3] [_]
                ↑ 腾出空间

插入后：[0] [1] [666] [2] [3] [_]
```

涉及数据搬移，最坏情况移动 N 个元素，时间复杂度 **O(N)**。

你可以把“中间插入慢”当作数组的结构性代价，而不是实现问题。只要是连续存储，就必须给新元素腾位置；想腾位置，就得搬移后面的数据。这个结论不依赖语言，C/C++/Java 都一样。

> ✅ 看到“中间插入”，你能立刻想到 O(N)，这就是算法直觉在建立。

#### 情况三：数组已满 — 需要扩容 O(N)

静态数组大小固定，满了之后**无法原地扩展**（因为连续内存必须一次性分配，后面的空间可能已被占用）。

解决方案：**重新申请更大的内存，复制数据**：

**第一步：最简单的例子（用最日常的比喻）**

```cpp
int small[3] = {1, 2, 3};        // 小盒子装满了
int big[6] = {0, 0, 0, 0, 0, 0}; // 换一个更大的盒子
for (int i = 0; i < 3; i++) big[i] = small[i]; // 旧东西搬过去
big[3] = 4;                       // 再放新东西
```

> 上面的例子就是“换更大的盒子再搬家”；下面代码同理，只是容量从 10 扩到 20。

```cpp
// 原数组满了，扩容到 2 倍
int newArr[20];
for (int i = 0; i < 10; i++) {
    newArr[i] = arr[i];   // 复制数据
}
// 释放旧数组，使用新数组
newArr[10] = 10;           // 插入新元素
```

扩容涉及开辟新空间 + 复制 N 个元素，时间复杂度 **O(N)**。

为什么不能原地长大？因为数组要求整段空间连续，而你后面的内存很可能已经被别的数据占了，系统没法保证“就地追加”。所以扩容本质是“搬家”而不是“加一层”。*就像书架满了不能把墙往后推，只能换更大的书架再一本本搬过去*。

> 💡 **老师提醒：** 面试或笔试里一问到扩容，优先回答“重新申请 + 复制 + 释放旧空间”，这三个动作缺一不可。

### 3.4 删（Delete）— 分情况

#### 情况一：删除末尾元素 — O(1)

直接标记末尾元素为"已删除"（动态数组中会用 size-- 实现）：

**第一步：最简单的例子（用最日常的比喻）**

```cpp
int seats[5] = {8, 9, 10, 11, 12}; // 5个座位都有人
seats[4] = -1;                      // 把最后一个座位标记为空
cout << seats[4] << endl;           // 看到 -1 代表已删除
```

> 上面的例子是在“尾部打空标记”；下面代码就是同一操作，复杂度是 O(1)。

```cpp
arr[4] = -1;   // 用 -1 标记已删除，O(1)
```

#### 情况二：删除中间元素 — O(N)

需要把被删元素后面的所有元素**往前移一位**，保持数组连续性：

**第一步：最简单的例子（用最日常的比喻）**

```cpp
int q[5] = {1, 9, 2, 3, 4};      // 要删除中间的9
for (int i = 1; i < 4; i++) {    // 从前往后挪
    q[i] = q[i + 1];              // 后面的补到前面
}
q[4] = -1;                        // 最后一个位置清空标记
```

> 上面的例子是“后面的人往前补位”；下面代码做的是同一个动作，删除的是 `arr[1]`。

```cpp
// 删除 arr[1]
// ⚠️ 必须从前往后移，否则会覆盖数据
for (int i = 1; i < 4; i++) {
    arr[i] = arr[i + 1]; // 让后一个元素覆盖当前元素
}
arr[4] = -1;   // 最后一个元素标记为已删除
```

图示：

```
删除前：[0] [1] [2] [3] [4]
            ↑ 要删除 arr[1]

往前移：[0] [2] [3] [4] [_]

删除后：[0] [2] [3] [4] [-1]
```

涉及数据搬移，时间复杂度 **O(N)**。

删除中间元素和插入中间元素是一对镜像操作。插入是给某个位置“腾空”，删除是给某个位置“补位”，都需要移动一段连续元素，所以复杂度同阶。你把这对关系记住，很多题可以不用推导就直接判断。

> ✅ 你现在已经能用“是否搬移数据”这条主线统一解释增删复杂度了。

## 4. 动态数组

### 4.1 动态数组是什么

动态数组**底层仍然是静态数组**，只是额外做了两件事：

1. **自动扩缩容**：数组满了自动扩大，元素很少时自动缩小
2. **封装 API**：把增删查改操作封装好，直接调用就行

> 动态数组**无法解决**中间插入/删除 O(N) 的问题，这是数组连续内存结构决定的，无法改变。

这句话你一定要吃透。动态数组解决的是“容量管理麻烦”，不是“中间操作慢”。也就是说，它帮你省掉了手工扩容的工作，但不会改变数组的物理结构。*就像自动挡汽车让你换挡更省心，但不会把红灯等待时间变没*。

> 💡 **老师提醒：** 很多人把“动态”误解成“所有操作都快”。动态只代表容量可变，不代表中间增删是 O(1)。

### 4.2 C++ 中的动态数组：vector

**第一步：最简单的例子（用最日常的比喻）**

```cpp
vector<int> bag;               // 准备一个会自动变大的背包
bag.push_back(3);              // 往背包末尾放一个数字3
bag.push_back(5);              // 再放一个数字5
cout << bag[0] << endl;        // 取出第1个放进去的数字
```

> 上面演示了 vector 的核心：自动扩容 + 按索引访问；下面代码是在这个基础上把常见增删查改都展示出来。

```cpp
#include <vector>

// 创建动态数组（不用指定大小）
vector<int> arr;

// 末尾追加，O(1)
for (int i = 0; i < 10; i++) {
    arr.push_back(i);
}

// 中间插入，O(N)
arr.insert(arr.begin() + 2, 666);  // 在索引2处插入666

// 头部插入，O(N)
arr.insert(arr.begin(), -1);

// 删除末尾，O(1)
arr.pop_back();

// 删除中间，O(N)
arr.erase(arr.begin() + 2);  // 删除索引2处的元素

// 按索引查询，O(1)
int a = arr[0];

// 按索引修改，O(1)
arr[0] = 100;

// 按值查找索引，O(N)
int index = find(arr.begin(), arr.end(), 666) - arr.begin();
```

### 4.3 扩容的均摊复杂度

你可能会问：扩容是 O(N)，为什么 `push_back` 说是 O(1)？

因为**不是每次 push_back 都会触发扩容**。一般动态数组满了才扩容到 2 倍：

```
容量：1 → 2 → 4 → 8 → 16 → ...

前1次操作：触发扩容，复制1个元素
前2次操作：触发扩容，复制2个元素
前4次操作：触发扩容，复制4个元素
...
```

N 次 push_back，总共复制元素次数约为 `1 + 2 + 4 + ... = 2N`，**平均每次是 O(1)**。

这就是「**均摊时间复杂度**」的概念：偶尔的高代价操作，分摊到每次操作后，平均代价仍然是 O(1)。

你可以这样理解均摊：我们不否认某一次会慢，但看长期平均每次花费。动态数组的扩容是“少数几次重活 + 大量轻活”的组合，所以长期看仍然便宜。这个思想在并查集、哈希表再散列里也会出现，是算法里的高频模型。

> ✅ 均摊复杂度你已经入门了，这会帮你读懂很多看起来“偶尔很慢但总体很快”的数据结构。

>
> 理解均摊复杂度（Amortized Complexity）最直观的方法是 **“存钱罐”模型** 。
>
> 之所以觉得 O(1) 难以理解，是因为我们的直觉往往停留在“单次操作”上。但实际上，动态数组（如 C++ 的 `std::vector` 或 Python 的 `list`）是通过“一次性透支，多次享受”来维持效率的。
>
> ---
>
> ### 1. 存钱罐模型（会计法）
>
> 想象你每次执行 `push_back` 操作时，除了支付当前操作本身的 1 元成本，你额外多存 2 元到你的“扩容存钱罐”里。
>
> * **普通插入（不扩容）：** 实际消耗 1 元，你存入 2 元。此时你的总支出是 3 元（常数级 **$O(1)$**）。
> * **触发扩容：** 当数组满了，需要搬运 **$N$** 个元素时。这时你不需要额外掏钱，直接打开 **存钱罐** 。
> * **奇迹发生了：** 因为你之前每次插入都存了 2 元，当你需要搬运 **$N$** 个元素时，存钱罐里的钱正好够支付这 **$N$** 次搬运的成本。
>
> ---
>
> ### 2. 数学推导（聚合分析）
>
> 如果我们不看单次，而是看**连续 **$N$** 次**操作的总成本：
>
> 假设我们要插入 **$N$** 个元素（假设 **$N$** 正好是 2 的幂，比如 8）：
>
> 1. **直接插入的成本：** 每次插入本身都要写一次内存，共 **$N$** 次。
> 2. **扩容搬运的成本：**
>    * 当容量从 1 变 2 时，搬运 1 个元素。
>    * 当容量从 2 变 4 时，搬运 2 个元素。
>    * 当容量从 4 变 8 时，搬运 4 个元素。
>    * ... 依此类推。
>
> 总搬运次数 = **$1 + 2 + 4 + 8 + \dots + \frac{N}{2} = N - 1$**。
>
> **总成本** = 直接插入成本 (**$N$**) + 扩容搬运成本 (**$N-1$**) **$\approx 2N$**。
>
> $$
> \text{平均成本} = \frac{\text{总成本}}{\text{操作次数}} = \frac{2N}{N} = 2
> $$
>
> 在算法分析中，常数 2 依然属于 **$O(1)$**。
>
> ---
>
> ### 3. 为什么是 2 倍扩容？
>
> 如果你每次只增加 1 个固定大小（比如每次只加 10 个空位），均摊复杂度会退化成 **$O(N)$**。
>
> 只有 **几何级数（倍增）** 扩容，才能保证搬运的总和永远不超过总操作次数的 2 倍，从而维持 **$O(1)$** 的均摊效率。
>
>> **一句话总结：**
>>
>> 均摊复杂度就是把“偶尔的大爆发”平摊到“长期的平淡”中。虽然某一次扩容很慢，但它换来了后面极多次的极速操作。
>>

## 5. 时间复杂度总结

### 静态/动态数组的增删查改

| 操作         | 场景           |   时间复杂度   | 原因            |
| ------------ | -------------- | :------------: | --------------- |
| **查** | 给定索引查值   | **O(1)** | 直接计算地址    |
| **查** | 给定值找索引   | **O(N)** | 需要遍历        |
| **改** | 给定索引修改值 | **O(1)** | 直接计算地址    |
| **增** | 末尾追加       | **O(1)** | 直接赋值        |
| **增** | 中间插入       | **O(N)** | 需要搬移数据    |
| **增** | 触发扩容       | **O(N)** | 申请新空间+复制 |
| **删** | 删除末尾       | **O(1)** | 直接标记        |
| **删** | 删除中间       | **O(N)** | 需要搬移数据    |

### 核心规律

```
数组的超能力：随机访问（按索引）→ O(1)
数组的弱点：中间增删 → O(N)，因为要搬数据

记住这个本质，就能推导出所有复杂度，不需要死记硬背。

当你能用“结构决定代价”去解释复杂度时，学习就从记忆型变成理解型了。后面你遇到链表、堆、哈希表，也继续问同样的问题：内存怎么放？操作会不会搬移？需不需要额外维护信息？沿着这三问走，几乎不会迷路。

> 💡 **老师提醒：** 做题卡住时，先别急着写代码。先把“我要做的操作”列出来，再对照复杂度选结构，成功率会明显提升。
```

---

> 📌 **一句话总结**
>
> 静态数组是一块连续内存，随机访问 O(1) 是它的超能力，中间增删 O(N) 是它无法逃脱的代价。动态数组（vector）只是在它基础上加了自动扩缩容和封装 API，本质不变。
