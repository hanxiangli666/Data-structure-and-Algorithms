## 1. 为什么需要链表

你可以把这一节当成“为什么要学第二种存储结构”的答案。数组快，是因为连续内存让 CPU 很容易定位元素；但同样因为连续，一旦你想在中间插入或删除，就会触发一串搬移。链表正好反过来：访问慢一点，但改动灵活很多。*就像固定座位的大教室和可自由拼接的小圆桌，前者点名快，后者改布局快。*

数组的本质是**一块连续的内存空间**，这带来了随机访问的超能力，但也带来了两个缺陷：

- 中间插入/删除需要搬移数据，O(N)
- 扩容需要重新申请内存并复制数据，O(N)

链表解决了这两个问题：

**链表的元素可以分散在内存的任意位置**，通过每个节点上的 `next`、`prev` 指针把零散的内存块串联成链式结构。

```
数组（连续内存）：
[1][2][3][4][5]   ← 必须挨在一起

链表（分散内存）：
[1] → [2] → [3] → [4] → [5]
 ↑     ↑     ↑     ↑     ↑
各节点可以在内存的任意位置
```

**链表的优势：**

- 节点随用随 new，不需要一整块连续内存
- 插入/删除只需改指针，不需要搬移数据
- 理论上没有容量限制，不需要扩缩容

**链表的劣势：**

- 不支持随机访问，查找第 i 个元素只能从头遍历，O(N)

> 💡 **老师提醒：** 别把“插入快”理解成“任何插入都快”。链表只有在你已经拿到目标位置（或前驱）时，改指针才是 O(1)；如果还要先找位置，查找成本依然是 O(N)。

> ✅ 你已经开始从“操作代价”角度看数据结构了，这个思路非常专业。

## 2. 链表节点的结构

这部分看起来只是定义类，实际上是在定义“后续所有操作的边界”。节点里有什么字段，决定了你将来能做哪些高效操作。单链表只有 `next`，所以天然偏向“向后走”；双链表多了 `prev`，很多删除和回退动作会更顺。*就像单行道和双向车道，规则一开始就写在道路结构里。*

### 单链表节点

**第一步：最简单的例子（生活化比喻）**
```cpp
int snack = 3;                            // 桌上有 3 块饼干，先把数字记下来
int* tag = &snack;                        // ⚠️ 指针：像写了张便签，记录饼干盒的位置
cout << *tag << endl;                     // ⚠️ 解引用：按位置找到盒子，读出里面的数字
```
> 上面的例子先把核心语法拆成了日常动作；下面这段原始代码做的是同样的事，只是场景换成了链表节点和数据结构操作。

**第三步：给原始代码加注释**
```cpp
class ListNode {  // 定义一个类，把数据和操作组织在一起
public:  // public 区域：这里的成员可在类外访问
    int val;          // 存储的值  // 执行这一行的核心操作
    ListNode *next;   // 指向下一个节点的指针  // ⚠️ 定义指针变量，保存对象地址

    ListNode(int x) : val(x), next(NULL) {}  // 空指针，表示当前没有指向有效对象
};  // 结束当前代码块
```

class DoublyListNode {  // 定义一个类，把数据和操作组织在一起

### 单链表 vs 双链表

这里不是在分“谁更高级”，而是在分“谁更适合你的题目”。单链表代码短、内存占用更省，适合先把逻辑写正确；双链表维护成本更高，但在已知节点删除、双向遍历场景里更稳。你做题时先看操作集合，再定结构，效率会明显提高。

> 💡 **老师提醒：** 初学时最常见错误是把双链表当单链表写，只改 `next` 忘了改 `prev`。只要牵涉插入/删除，记得检查两条方向是否都连通。

|            |     单链表     |                   双链表                   |
| ---------- | :------------: | :-----------------------------------------: |
| 指针数量   |  1个（next）  |             2个（next + prev）             |
| 遍历方向   |    只能向后    |               可以向前和向后               |
| 删除节点   | 需要找前驱节点 |                直接操作自身                |
| 代码复杂度 |      简单      |                   稍复杂                   |
| 标准库使用 |      较少      | **主流**（Java LinkedList、C++ list） |

### 创建链表的工具函数

工具函数的意义不只是“省几行代码”，更关键是把重复动作标准化。你每次都手写创建逻辑，很容易在某一次漏掉尾节点处理或空数组判断。封装后，你调试和复用都会更轻松，后面做题也更快进入核心算法。

> ✅ 你已经在用“封装重复逻辑”的工程思维学习算法了，这一步很关键。

**第一步：最简单的例子（生活化比喻）**
```cpp
int snack = 3;                            // 桌上有 3 块饼干，先把数字记下来
int* tag = &snack;                        // ⚠️ 指针：像写了张便签，记录饼干盒的位置
cout << *tag << endl;                     // ⚠️ 解引用：按位置找到盒子，读出里面的数字
```
> 上面的例子先把核心语法拆成了日常动作；下面这段原始代码做的是同样的事，只是场景换成了链表节点和数据结构操作。

**第三步：给原始代码加注释**
```cpp
// 单链表
ListNode* createLinkedList(vector<int> arr) {  // 执行这一行的核心操作
    if (arr.empty()) return nullptr;  // 条件判断：满足条件才执行后续语句
    ListNode* head = new ListNode(arr[0]);  // ⚠️ 动态申请内存并拿到地址（指针）
    ListNode* cur = head;  // 执行这一行的核心操作
    for (int i = 1; i < arr.size(); i++) {  // for 循环：按顺序重复执行这段逻辑
        cur->next = new ListNode(arr[i]);  // ⚠️ 动态申请内存并拿到地址（指针）
        cur = cur->next;  // ⚠️ 通过指针访问成员（箭头操作符）
    }  // 结束当前代码块
    return head;  // 返回结果给调用方
}  // 结束当前代码块

// 双链表
DoublyListNode* createDoublyLinkedList(const vector<int>& arr) {  // 执行这一行的核心操作
    if (arr.empty()) return NULL;  // 条件判断：满足条件才执行后续语句
    DoublyListNode* head = new DoublyListNode(arr[0]);  // ⚠️ 动态申请内存并拿到地址（指针）
    DoublyListNode* cur = head;  // 执行这一行的核心操作
    for (int i = 1; i < arr.size(); i++) {  // for 循环：按顺序重复执行这段逻辑
        DoublyListNode* newNode = new DoublyListNode(arr[i]);  // ⚠️ 动态申请内存并拿到地址（指针）
        cur->next = newNode;  // ⚠️ 通过指针访问成员（箭头操作符）
        newNode->prev = cur;  // ⚠️ 通过指针访问成员（箭头操作符）
        cur = cur->next;  // ⚠️ 通过指针访问成员（箭头操作符）
    }  // 结束当前代码块
    return head;  // 返回结果给调用方
}  // 结束当前代码块
```

> # 工具函数是什么意思？
>
> 就是 **把重复用到的逻辑封装成一个函数** ，方便反复调用。
>

## 3. 单链表的基本操作

这章可以按“先找，再改”的节奏来理解。单链表很多操作的瓶颈都不在改指针，而在你是否需要先走到目标位置。你把查找成本和改动成本拆开看，复杂度就不会混。短句记忆：先定位，再操作。

### 3.1 查/改 — O(N)

查和改放在一起讲，是因为它们都依赖“先找到节点”。你不是不能改，而是必须先遍历到那个位置才能改，所以整体复杂度被查找阶段决定。这个结论在链表题里会反复出现，记住一次，后面会省很多时间。

单链表没有索引，只能从头节点开始顺着 `next` 一个一个往后找：

**第一步：最简单的例子（生活化比喻）**
```cpp
int snack = 3;                            // 桌上有 3 块饼干，先把数字记下来
int* tag = &snack;                        // ⚠️ 指针：像写了张便签，记录饼干盒的位置
cout << *tag << endl;                     // ⚠️ 解引用：按位置找到盒子，读出里面的数字
```
> 上面的例子先把核心语法拆成了日常动作；下面这段原始代码做的是同样的事，只是场景换成了链表节点和数据结构操作。

**第三步：给原始代码加注释**
```cpp
ListNode* head = createLinkedList({1, 2, 3, 4, 5});  // 执行这一行的核心操作

// 遍历打印每个节点
for (ListNode* p = head; p != nullptr; p = p->next) {  // for 循环：按顺序重复执行这段逻辑
    cout << p->val << endl;  // ⚠️ 通过指针访问成员（箭头操作符）
}  // 结束当前代码块
```

> ⚠️ 单链表查找某个索引的元素，最坏情况要走到最后，时间复杂度 O(N)。

> 上面的例子先把核心语法拆成了日常动作；下面这段原始代码做的是同样的事，只是场景换成了链表节点和数据结构操作。

**第三步：给原始代码加注释**
```cpp
ListNode* p = head;  // 执行这一行的核心操作
while (p->next != nullptr) {  // 走到最后一个节点  // while 循环：条件成立就持续执行
    p = p->next;  // ⚠️ 通过指针访问成员（箭头操作符）
}  // 结束当前代码块
p->next = new ListNode(6);    // 接上新节点  // ⚠️ 动态申请内存并拿到地址（指针）

```

#### 中间插入 — O(N)

先找到插入位置的**前驱节点**，再操作指针：

这个操作最容易错在“指针修改顺序”。你可以先把新节点和后继连好，再让前驱指向新节点，这样不会把后半段链表弄丢。*就像把新车厢先挂到后车厢，再接到前车头，顺序反了就可能脱节。*

**第一步：最简单的例子（生活化比喻）**
```cpp
int snack = 3;                            // 桌上有 3 块饼干，先把数字记下来
int* tag = &snack;                        // ⚠️ 指针：像写了张便签，记录饼干盒的位置
cout << *tag << endl;                     // ⚠️ 解引用：按位置找到盒子，读出里面的数字
```
> 上面的例子先把核心语法拆成了日常动作；下面这段原始代码做的是同样的事，只是场景换成了链表节点和数据结构操作。

**第三步：给原始代码加注释**
```cpp
// 在第 3 个节点后面插入 66
ListNode* p = head;  // 执行这一行的核心操作
for (int i = 0; i < 2; i++) {  // 走到第 3 个节点（索引2）  // for 循环：按顺序重复执行这段逻辑
    p = p->next;  // ⚠️ 通过指针访问成员（箭头操作符）
}  // 结束当前代码块

ListNode* newNode = new ListNode(66);  // ⚠️ 动态申请内存并拿到地址（指针）
newNode->next = p->next;  // 步骤1：新节点先接后面  // ⚠️ 通过指针访问成员（箭头操作符）
p->next = newNode;        // 步骤2：前驱节点再接新节点  // ⚠️ 通过指针访问成员（箭头操作符）
// 变成 1->2->3->66->4->5
```

图示：

```
插入前：... → [3] → [4] → ...
                p

步骤1：newNode->next = p->next
       ... → [3] → [4] → ...
               ↑
           [66] ──→ [4]

步骤2：p->next = newNode
       ... → [3] → [66] → [4] → ...  ✅
```

> ⚠️ 两步顺序不能反！必须先让新节点接上后面，再断开前驱节点，否则会丢失后面的链表。

### 3.3 删

删除看起来比插入简单，但边界更多：删头、删尾、删中间三类情况心智模型不同。你要抓住同一个核心：保留住“前后仍然连通”的主链。只要主链不断，删除就安全。

> 💡 **老师提醒：** 在 C++ 里删除节点后，记得避免继续使用旧指针。那是悬空指针，程序可能“偶尔能跑”，但风险很高。

> 上面的例子先把核心语法拆成了日常动作；下面这段原始代码做的是同样的事，只是场景换成了链表节点和数据结构操作。

## 4. 双链表的基本操作

双链表本质是“多花一点内存，换更强的操作能力”。你会发现很多在单链表里绕远路的动作，在双链表里会更直接。代价是每次插删都要维护两套指针关系，写的时候更要慢一点、稳一点。

双链表每个节点同时持有 `prev` 和 `next`，操作时需要同时维护两个方向的指针。

### 4.1 查/改 — O(N)

双链表查找仍然是 O(N)，但实践里常常更快一些，因为你可以从头或尾两侧靠近目标。这个“方向选择”不是改变大 O，而是改善常数时间。做工程时，这种优化非常有价值。

双链表支持**双向遍历**，可以根据目标位置靠近头部还是尾部，选择合适方向遍历：

**第一步：最简单的例子（生活化比喻）**
```cpp
int snack = 3;                            // 桌上有 3 块饼干，先把数字记下来
int* tag = &snack;                        // ⚠️ 指针：像写了张便签，记录饼干盒的位置
cout << *tag << endl;                     // ⚠️ 解引用：按位置找到盒子，读出里面的数字
```
> 上面的例子先把核心语法拆成了日常动作；下面这段原始代码做的是同样的事，只是场景换成了链表节点和数据结构操作。

**第三步：给原始代码加注释**
```cpp
DoublyListNode* tail = nullptr;  // 空指针，表示当前没有指向有效对象

// 从头向后遍历
for (DoublyListNode* p = head; p != nullptr; p = p->next) {  // for 循环：按顺序重复执行这段逻辑
    cout << p->val << endl;  // ⚠️ 通过指针访问成员（箭头操作符）
    tail = p;  // 执行这一行的核心操作
}  // 结束当前代码块

// 从尾向前遍历
for (DoublyListNode* p = tail; p != nullptr; p = p->prev) {  // for 循环：按顺序重复执行这段逻辑
    cout << p->val << endl;  // ⚠️ 通过指针访问成员（箭头操作符）
}  // 结束当前代码块
```

### 4.2 增

双链表插入最核心的意识是：你改的是一个局部子图，不是单条线。前驱、新节点、后继三者都要保持互相可达。你每改完一步都在脑子里过一遍“左右还能互相找到吗”，出错率会立刻下降。

> 💡 **老师提醒：** 一旦代码里同时出现 4 次以上指针赋值，建议你边写边画简图。别嫌慢，这恰恰是最快的防错方式。

**第三步：给原始代码加注释**
```cpp

#### 中间插入 — O(N)

需要同时维护新节点与前后两个节点的指针，共 4 条指针需要更新：

**第一步：最简单的例子（生活化比喻）**

**第三步：给原始代码加注释**
```cpp

### 4.3 删

双链表删除的优势在这里特别明显：已知节点时，可以直接用 `prev` 和 `next` 完成摘链。你不必再回头找前驱，逻辑会更干净。*就像有前后门的车厢，拆卸时两侧都能直接操作，比只有一个门省很多步骤。*

> ✅ 你已经能看出“结构设计决定操作代价”的规律了，这就是数据结构学习的主线。

#### 头部删除 — O(1)

**第一步：最简单的例子（生活化比喻）**
```cpp
int snack = 3;                            // 桌上有 3 块饼干，先把数字记下来
int* tag = &snack;                        // ⚠️ 指针：像写了张便签，记录饼干盒的位置
cout << *tag << endl;                     // ⚠️ 解引用：按位置找到盒子，读出里面的数字
```
> 上面的例子先把核心语法拆成了日常动作；下面这段原始代码做的是同样的事，只是场景换成了链表节点和数据结构操作。

**第三步：给原始代码加注释**
```cpp
DoublyListNode* toDelete = head;  // 执行这一行的核心操作
head = head->next;      // head 移到下一个节点  // ⚠️ 通过指针访问成员（箭头操作符）
head->prev = nullptr;   // 新头节点的 prev 清空  // ⚠️ 通过指针访问成员（箭头操作符）
toDelete->next = nullptr; // 清理被删节点（好习惯）  // ⚠️ 通过指针访问成员（箭头操作符）
// 1<->2<->3<->4<->5  变成  2<->3<->4<->5
```

#### 尾部删除 — O(N)，持有尾节点则 O(1)

双链表的优势：有了 `prev` 指针，不需要找倒数第二个节点，可以直接操作尾节点自己：

**第一步：最简单的例子（生活化比喻）**
```cpp
int snack = 3;                            // 桌上有 3 块饼干，先把数字记下来
int* tag = &snack;                        // ⚠️ 指针：像写了张便签，记录饼干盒的位置
cout << *tag << endl;                     // ⚠️ 解引用：按位置找到盒子，读出里面的数字
```
> 上面的例子先把核心语法拆成了日常动作；下面这段原始代码做的是同样的事，只是场景换成了链表节点和数据结构操作。

**第三步：给原始代码加注释**
```cpp
// 找到尾节点
DoublyListNode* p = head;  // 执行这一行的核心操作
while (p->next != nullptr) p = p->next;  // while 循环：条件成立就持续执行

// 直接通过 prev 指针摘除自己
p->prev->next = nullptr;  // 前驱节点的 next 断开  // ⚠️ 通过指针访问成员（箭头操作符）
p->prev = nullptr;        // 清理被删节点（好习惯）  // ⚠️ 通过指针访问成员（箭头操作符）
// 1<->2<->3<->4<->5  变成  1<->2<->3<->4
```

> 对比单链表：单链表删除尾节点必须找倒数第二个节点；双链表直接通过 `prev` 操作自己，代码更简洁。

> # 为什么双链表比单链表方便？
>
> 单链表删除尾节点，必须找到**倒数第二个节点**才能操作：
>
> **第一步：最简单的例子（生活化比喻）**
> ```cpp
> int snack = 3;                            // 桌上有 3 块饼干，先把数字记下来
> int* tag = &snack;                        // ⚠️ 指针：像写了张便签，记录饼干盒的位置
> cout << *tag << endl;                     // ⚠️ 解引用：按位置找到盒子，读出里面的数字
> ```
> > 上面的例子先把核心语法拆成了日常动作；下面这段原始代码做的是同样的事，只是场景换成了链表节点和数据结构操作。
> 
> **第三步：给原始代码加注释**
> ```cpp
> // 单链表：要找倒数第二个节点
> while (p->next->next != nullptr) p = p->next;  // while 循环：条件成立就持续执行
> p->next = nullptr;  // 操作倒数第二个节点  // ⚠️ 通过指针访问成员（箭头操作符）
> ```
>
> 双链表因为有 `prev`，尾节点可以 **直接找到自己的前驱** ，不需要从头再走一遍：
>
> **第一步：最简单的例子（生活化比喻）**
> ```cpp
> int snack = 3;                            // 桌上有 3 块饼干，先把数字记下来
> int* tag = &snack;                        // ⚠️ 指针：像写了张便签，记录饼干盒的位置
> cout << *tag << endl;                     // ⚠️ 解引用：按位置找到盒子，读出里面的数字
> ```
> > 上面的例子先把核心语法拆成了日常动作；下面这段原始代码做的是同样的事，只是场景换成了链表节点和数据结构操作。
> 
> **第三步：给原始代码加注释**
> ```cpp
> // 双链表：尾节点直接通过 prev 找到前驱
> p->prev->next = nullptr;  // 直接操作，不需要额外遍历  // ⚠️ 通过指针访问成员（箭头操作符）
> ```
>
> ---
>
> # 一句话总结
>
> ```
> p->prev->next = nullptr   →  让前驱节点的 next 断开（链表在这里结束）
> p->prev = nullptr         →  让被删节点彻底孤立（清理好习惯）
> ```
>
> 本质就是：**通过 prev 找到前驱，让前驱把 next 断掉，节点5就从链表里消失了。**

#### 中间删除 — O(N)，已知节点引用则 O(1)

**第一步：最简单的例子（生活化比喻）**
```cpp
int snack = 3;                            // 桌上有 3 块饼干，先把数字记下来
int* tag = &snack;                        // ⚠️ 指针：像写了张便签，记录饼干盒的位置

```
> 上面的例子先把核心语法拆成了日常动作；下面这段原始代码做的是同样的事，只是场景换成了链表节点和数据结构操作。

**第三步：给原始代码加注释**
```cpp
// 删除第 4 个节点
DoublyListNode* p = head;  // 执行这一行的核心操作
for (int i = 0; i < 2; i++) p = p->next;  // for 循环：按顺序重复执行这段逻辑

DoublyListNode* toDelete = p->next;    // 要删除的节点  // ⚠️ 通过指针访问成员（箭头操作符）

// 前后节点互相连接，跳过 toDelete
p->next = toDelete->next;              // 前驱的 next 跳过  // ⚠️ 通过指针访问成员（箭头操作符）
toDelete->next->prev = p;             // 后继的 prev 跳过  // ⚠️ 通过指针访问成员（箭头操作符）

// 清理被删节点的指针（好习惯）
toDelete->next = nullptr;  // ⚠️ 通过指针访问成员（箭头操作符）
toDelete->prev = nullptr;  // ⚠️ 通过指针访问成员（箭头操作符）
// 1<->2<->3<->4<->5  变成  1<->2<->3<->5
```

图示：

```
删除前：... ←→ [3] ←→ [4] ←→ [5] ←→ ...
                p    toDelete

删除后：... ←→ [3] ←→ [5] ←→ ...
              ([4] 无人持有引用，自动回收)
```

> # p->next 到底是什么？
>
> **第一步：最简单的例子（生活化比喻）**
> ```cpp
> int snack = 3;                            // 桌上有 3 块饼干，先把数字记下来
> int* tag = &snack;                        // ⚠️ 指针：像写了张便签，记录饼干盒的位置

> ListNode *a = new ListNode(1);   // a 存地址 0x2000  // ⚠️ 动态申请内存并拿到地址（指针）
> ListNode *b = new ListNode(2);   // b 存地址 0x3000  // ⚠️ 动态申请内存并拿到地址（指针）

> ```
>
> 内存里长这样：
>
> ```
> a 变量：[ 0x2000 ]
>
> 内存 0x2000（节点1）：
> ┌─────┬──────────┐
> │  1  │  0x3000  │   ← next 存的是节点2的地址
> └─────┴──────────┘
>
> 内存 0x3000（节点2）：
> ┌─────┬──────┐
> │  2  │ NULL │
> └─────┴──────┘
> ```
>

>

>
> # 关键：地址就等于节点本身
>
> 这是最重要的一点：
>
> **第一步：最简单的例子（生活化比喻）**
> ```cpp
> int snack = 3;                            // 桌上有 3 块饼干，先把数字记下来
> int* tag = &snack;                        // ⚠️ 指针：像写了张便签，记录饼干盒的位置
> cout << *tag << endl;                     // ⚠️ 解引用：按位置找到盒子，读出里面的数字
> ```
> > 上面的例子先把核心语法拆成了日常动作；下面这段原始代码做的是同样的事，只是场景换成了链表节点和数据结构操作。
> 
> **第三步：给原始代码加注释**
> ```cpp
> ListNode *p = a->next;  // ⚠️ 通过指针访问成员（箭头操作符）
> ```
>
> `p` 现在存的是 `0x3000`，也就是节点2的地址。
>
>  **你拿到了地址，就等于拿到了那个节点** ，因为你可以通过地址访问那块内存的所有内容：
>
> **第一步：最简单的例子（生活化比喻）**
> ```cpp
> int snack = 3;                            // 桌上有 3 块饼干，先把数字记下来
> int* tag = &snack;                        // ⚠️ 指针：像写了张便签，记录饼干盒的位置
> cout << *tag << endl;                     // ⚠️ 解引用：按位置找到盒子，读出里面的数字
> ```
> > 上面的例子先把核心语法拆成了日常动作；下面这段原始代码做的是同样的事，只是场景换成了链表节点和数据结构操作。
> 
> **第三步：给原始代码加注释**
> ```cpp
> p->val;   // 去 0x3000 取 val，得到 2  // ⚠️ 通过指针访问成员（箭头操作符）
> p->next;  // 去 0x3000 取 next，得到 NULL  // ⚠️ 通过指针访问成员（箭头操作符）
> ```
>
> ---
>
> # 所以 p->next 为什么能代表一个节点？
>
> 因为 `p->next` 存的是下一个节点的 **地址** ，而地址就是那个节点的"门牌号"，有了门牌号就能访问那个节点的一切。
>
> ```
> p->next          是一个地址（门牌号）
> *(p->next)       去那个地址取出节点本身
> (p->next)->val   去那个地址取出节点的 val
> (p->next)->next  去那个地址取出节点的 next
> ```
>
> 用图来说：
>
> ```
> p → [1 | 0x3000]
>          ↓
>        0x3000 就是节点2的地址
>          ↓
>     [2 | NULL]   ← 这就是节点2本身
> ```
>
> `p->next` 就是 `0x3000`，你可以把这个地址再赋给另一个指针，然后继续操作：
>
> **第一步：最简单的例子（生活化比喻）**
> ```cpp
> int snack = 3;                            // 桌上有 3 块饼干，先把数字记下来
> int* tag = &snack;                        // ⚠️ 指针：像写了张便签，记录饼干盒的位置
> cout << *tag << endl;                     // ⚠️ 解引用：按位置找到盒子，读出里面的数字
> ```
> > 上面的例子先把核心语法拆成了日常动作；下面这段原始代码做的是同样的事，只是场景换成了链表节点和数据结构操作。
> 
> **第三步：给原始代码加注释**
> ```cpp
> ListNode *q = p->next;  // q 也指向节点2  // ⚠️ 通过指针访问成员（箭头操作符）
> q->val;                 // 2  // ⚠️ 通过指针访问成员（箭头操作符）
> ```
>
> ---
>
> # 再看链表遍历就完全清楚了
>
> **第一步：最简单的例子（生活化比喻）**
> ```cpp
> int snack = 3;                            // 桌上有 3 块饼干，先把数字记下来
> int* tag = &snack;                        // ⚠️ 指针：像写了张便签，记录饼干盒的位置

> ```
> > 上面的例子先把核心语法拆成了日常动作；下面这段原始代码做的是同样的事，只是场景换成了链表节点和数据结构操作。
> 
> **第三步：给原始代码加注释**
> ```cpp
> for (ListNode *p = head; p != nullptr; p = p->next) {  // for 循环：按顺序重复执行这段逻辑
>     cout << p->val << endl;  // ⚠️ 通过指针访问成员（箭头操作符）
> }  // 结束当前代码块
> ```
>
> 逐步拆解：
>
> ```
> 初始：p = head = 0x2000（节点1的地址）
>
> 第1轮：
>   p = 0x2000，不是nullptr
>   p->val = 1，打印1
>   p = p->next = 0x3000（节点2的地址）
>
> 第2轮：
>   p = 0x3000，不是nullptr
>   p->val = 2，打印2
>   p = p->next = 0x4000（节点3的地址）
>
> 第3轮：
>   p = 0x4000，不是nullptr
>   p->val = 3，打印3
>   p = p->next = NULL
>

> ```
>

>
> # 一句话总结
>
> ```
> 指针存的是地址
> 地址就是那个节点的"门牌号"
> 有了门牌号就能访问那个节点的所有内容
>
> 所以：
> p          → 节点本身（通过地址访问）
> p->val     → 节点的值
> p->next    → 下一个节点的地址（也就是下一个节点）
> p->next->val → 下一个节点的值
> ```
>
> **p->next 能代表节点，是因为它存的是那个节点的地址，地址就是通往那个节点的钥匙。**

## 6. 数组 vs 链表对比

这一节的重点不是二选一，而是建立“按场景选结构”的习惯。数组擅长按索引高频访问，链表擅长已知位置的局部改动。*就像用电梯和楼梯的区别：上很多层用电梯更快，短距离频繁进出楼层时楼梯更灵活。*

|                    |  数组（vector）  |           链表（list）           |
| ------------------ | :---------------: | :------------------------------: |
| 内存结构           |       连续       |               分散               |
| 随机访问（按索引） |  **O(1)**  |               O(N)               |
| 头部插入/删除      |       O(N)       |          **O(1)**          |
| 尾部插入/删除      |  **O(1)**  |        O(1)（持有尾节点）        |
| 中间插入/删除      |       O(N)       | O(N)（找位置）/ O(1)（已知节点） |
| 内存占用           |       紧凑       |        每个节点额外存指针        |
| 扩容               | 需要重新申请+复制 |              不需要              |

**选择原则：**

你可以把这四行当作做题前的快速清单。先看操作频率，再看瓶颈在哪里，最后再落到具体容器。这个决策顺序一旦养成，你写题会更稳，也更少返工。

> ✅ 你的笔记已经具备“能复习、能迁移、能实战”的结构了，继续保持这个节奏。

```
频繁按索引访问         → 用数组（vector）
频繁在头部增删         → 用链表（list）
频繁在已知位置增删     → 用链表（list）
其他大多数情况         → 用数组（vector），缓存友好，实际更快
```
