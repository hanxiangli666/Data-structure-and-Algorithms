# 堆排序：二叉堆结构的运用 学习笔记

> **前置知识**：二叉堆原理、优先级队列（swim/sink 操作）  
> **核心认知**：堆排序 = 原地建堆 + 不断取出堆顶。结合了二叉堆的所有精华，是唯一一个**O(N log N) + O(1) 空间 + 原地排序**的算法  
> **关键前提**：必须理解 swim 和 sink，否则无法理解堆排序

---

## 一、最朴素的思路：直接用优先级队列

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
priority_queue<int, vector<int>, greater<int>> box; // 一个会自动把最小号放前面的取号箱
box.push(3);                                         // 放入 3 号
box.push(1);                                         // 放入 1 号
int first = box.top();                               // 先取到最小号（1 号）
```

> 上面的例子在做“放进去再按最小值取出来”，下面真实代码就是把同样逻辑用于整个数组排序。

```cpp
void sort(vector<int>& nums) {                           // ⚠️ 引用参数：原数组会被覆盖
    priority_queue<int, vector<int>, greater<int>> pq;   // 小顶堆，堆顶最小
    for (int x : nums) pq.push(x);                       // 全部元素入堆
    for (int i = 0; i < nums.size(); i++) {              // 按下标写回原数组
        nums[i] = pq.top(); pq.pop();                    // 依次弹出当前最小值
    }
}
```

> 💡 **老师提醒：** 这个版本容易理解，但额外用了一个堆容器，空间不是 O(1)。

**问题**：优先级队列内部有一个额外数组，空间复杂度 O(N)。

**堆排序要解决的问题**：不用额外空间，直接在原数组上完成排序。

---

## 二、堆排序的两步走

### 2.1 整体流程

```
第一步：原地建堆（Heapify）
  把待排序数组本身变成一个大顶堆

第二步：排序（Sort）
  不断把堆顶（最大值）移到数组末尾，堆缩小一格，直到排完
```

### 2.2 为什么用大顶堆做从小到大排序？

```
大顶堆堆顶 = 最大值

每次把最大值从堆顶「删除」，放到数组末尾：
  第 1 次：最大值放到 nums[n-1]
  第 2 次：第二大放到 nums[n-2]
  第 3 次：第三大放到 nums[n-3]
  ...
  
最终数组从小到大排列 ✓
```

如果用小顶堆，最小值放到末尾，最终是从大到小排的，还得翻转一次，不划算。

## 三、第一步：原地建堆

### 3.1 方法 A：逐个 swim（简单直接）

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
vector<int> heap = {5, 3, 8};                         // 一个大顶堆雏形
int node = 2;                                          // 新加入的节点下标
// 如果新节点比父节点更大，就向上浮动（swim）
```

> 这个小例子讲的是“单次上浮”，下面代码是把上浮动作对每个位置都执行一次，完成建堆。

```cpp
// 把数组变成大顶堆
for (int i = 0; i < n; i++) {              // 依次把每个元素当作新插入节点
    maxHeapSwim(nums, i);                   // 让该节点通过上浮回到正确位置
}
```

> 💡 **老师提醒：** 这个方法直观但不够快，时间是 O(N log N)，后面的逆序 sink 建堆更常用。

**思路**：模拟优先级队列的 push 操作。把 nums[0] 当作只有 1 个元素的堆，然后依次「插入」nums[1], nums[2], ...，每次 swim 上浮到正确位置。

**复杂度**：每次 swim 是 O(log N)，共 N 次 → O(N log N)。

### 3.2 方法 B：逆序 sink（更高效的优化）

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
int i = 2;                                            // 从一个“家长节点”开始
// 看看孩子里有没有更大的，有就把家长往下沉（sink）
i--;                                                  // 再处理更前面的家长
```

> 上面在模拟“从后往前修树”，下面的代码就是标准的逆序 sink 建堆模板。

```cpp
// 从最后一个非叶子节点开始，逆序 sink
for (int i = n / 2 - 1; i >= 0; i--) {      // 从最后一个非叶子节点开始
    maxHeapSink(nums, i, n);                 // 让以 i 为根的子树满足大顶堆
}
```

> ✅ 你已经把“叶子天然有序，所以从非叶子逆推”这个重点抓住了。

**思路**：利用二叉堆的性质——子堆也是堆。

```
数组：[5, 3, 8, 1, 2, 7, 4]

对应的完全二叉树：
         5(0)
        / \
     3(1)   8(2)
     / \    / \
   1(3) 2(4) 7(5) 4(6)

叶子节点（索引 3,4,5,6）：单个节点天然满足堆性质 ✓

从最后一个非叶子节点（索引 2）开始逆序 sink：

sink(2)：8 比子节点 7,4 都大 → 不动
         5
        / \
       3   8     ← 以 8 为根的子堆 OK
           / \
          7   4

sink(1)：3 比子节点 1,2 都大 → 不动
         5
        / \
       3   8     ← 以 3 为根的子堆 OK
      / \
     1   2

sink(0)：5 比子节点 3,8... 8 最大 → 交换 5 和 8
         8
        / \
       3   5     → 5 继续和 7 比... 7 > 5 → 交换
      / \  / \
     1  2 7   4

         8
        / \
       3   7
      / \  / \
     1  2 5   4

大顶堆建好了 ✓
```

**为什么更高效？**

- 方法 A：对所有 N 个元素 swim → N × O(log N) = O(N log N)
- 方法 B：只对 N/2 个非叶子节点 sink，而且底层节点 sink 距离短 → **O(N)**

> 方法 B 建堆的复杂度实际上是 **O(N)**（可以数学证明），比方法 A 的 O(N log N) 更快！

## 四、第二步：排序

建好大顶堆后，不断「删除」堆顶元素（最大值），放到数组末尾。

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
int heapSize = 4;                                  // 当前堆区长度
swap(nums[0], nums[heapSize - 1]);                // 把最大值换到尾部
heapSize--;                                        // 堆区缩小一格
```

> 这个三步动作就是每轮排序的骨架，下面代码只是把它放进循环并补上 sink 恢复堆结构。

```cpp
int heapSize = n;                                   // 堆的有效长度
while (heapSize > 0) {                              // 每轮固定一个最大值
    swap(nums[0], nums[heapSize - 1]);              // 堆顶（最大值）放到末尾
    heapSize--;                                     // 堆缩小一格
    maxHeapSink(nums, 0, heapSize);                 // 新堆顶下沉到正确位置
}
```

> 💡 **老师提醒：** `maxHeapSink` 的 `size` 一定要传 `heapSize`，否则会把右侧已排序区又搅乱。

### 4.1 图解

```
大顶堆：[8, 3, 7, 1, 2, 5, 4]

第 1 轮：swap(8, 4) → [4, 3, 7, 1, 2, 5, |8]  → sink(4) → [7, 3, 5, 1, 2, 4, |8]
第 2 轮：swap(7, 4) → [4, 3, 5, 1, 2, |4, 8]  → sink(4) → [5, 3, 4, 1, 2, |7, 8]
第 3 轮：swap(5, 2) → [2, 3, 4, 1, |5, 7, 8]  → sink(2) → [4, 3, 2, 1, |5, 7, 8]
...
最终：[1, 2, 3, 4, 5, 7, 8]  ✓

| 左边 = 堆（越来越小）  | 右边 = 已排序（越来越大）|
```

每轮做的事和优先级队列的 pop 操作**一模一样**，只是元素不是真的删除，而是放到数组末尾的「已排序区域」。

## 五、完整代码

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
int p = (5 - 1) / 2;                                // 节点 5 的父节点下标
int l = 2 * p + 1;                                  // 左孩子下标
int r = 2 * p + 2;                                  // 右孩子下标
```

> 这个小例子是“数组下标模拟二叉树”的基础，下面完整代码就是在这个规则上实现 swim、sink 和堆排序。

```cpp
// 工具函数
int parent(int i) { return (i - 1) / 2; }           // 父节点下标
int left(int i)   { return 2 * i + 1; }             // 左孩子下标
int right(int i)  { return 2 * i + 2; }             // 右孩子下标

// 大顶堆 swim
void maxHeapSwim(vector<int>& heap, int node) {     // ⚠️ 引用参数：直接修改堆
    while (node > 0 && heap[parent(node)] < heap[node]) { // 子节点更大则上浮
        swap(heap[parent(node)], heap[node]);        // 和父节点交换
        node = parent(node);                         // 继续向上检查
    }
}

// 大顶堆 sink
void maxHeapSink(vector<int>& heap, int node, int size) { // size 控制堆区边界
    while (left(node) < size) {                    // 有左孩子才可能下沉
        int mx = node;                             // 记录三者最大值位置
        if (left(node) < size && heap[left(node)] > heap[mx]) mx = left(node);   // 比左孩子
        if (right(node) < size && heap[right(node)] > heap[mx]) mx = right(node); // 比右孩子
        if (mx == node) break;                     // 当前节点已满足堆性质
        swap(heap[node], heap[mx]);                // 与更大孩子交换
        node = mx;                                 // 继续向下调整
    }
}

// 堆排序（优化版，逆序 sink 建堆）
void sort(vector<int>& nums) {                     // ⚠️ 原地排序入口
    int n = nums.size();                           // 元素总数

    // 第一步：原地建大顶堆（从最后一个非叶子节点逆序 sink）
    for (int i = n / 2 - 1; i >= 0; i--) {         // 逆序遍历非叶子节点
        maxHeapSink(nums, i, n);                   // 修复每棵子树
    }

    // 第二步：排序（不断把堆顶移到末尾）
    int heapSize = n;                              // 当前堆长度
    while (heapSize > 0) {                         // 每轮固定一个最大值
        swap(nums[0], nums[heapSize - 1]);         // 堆顶最大值放到末尾
        heapSize--;                                // 堆区缩小
        maxHeapSink(nums, 0, heapSize);            // 重新维护堆结构
    }
}
```

> 💡 **老师提醒：** `n / 2 - 1` 是最后一个非叶子节点下标，面试时被问到时你要能脱口而出来源。

## 六、三个指标分析

| 指标 | 堆排序 |
|------|--------|
| 时间复杂度 | **O(N log N)**（最好=最坏=平均） |
| 空间复杂度 | **O(1)**（原地排序！） |
| 稳定性 | ❌ 不稳定 |

**堆排序的独特优势**：它是唯一一个**同时做到 O(N log N) 和 O(1) 空间**的排序算法。

| 对比 | 时间 | 空间 | 稳定 |
|------|------|------|------|
| 快速排序 | O(N log N) 平均 | O(log N) | ❌ |
| 归并排序 | O(N log N) | **O(N)** | ✅ |
| **堆排序** | O(N log N) | **O(1)** | ❌ |

## 七、O(N log N) 排序三巨头终极对比

| 对比 | 快速排序 | 归并排序 | 堆排序 |
|------|---------|---------|--------|
| 核心思想 | 前序遍历 + partition | 后序遍历 + merge | 二叉堆 + sink |
| 平均时间 | O(N log N) | O(N log N) | O(N log N) |
| 最坏时间 | **O(N²)** | O(N log N) | O(N log N) |
| 空间 | O(log N) | **O(N)** | **O(1)** ✓ |
| 稳定 | ❌ | **✅** | ❌ |
| 实际速度 | **最快**（缓存友好） | 中等 | 较慢（缓存不友好） |
| C++ 标准库 | `sort()` 基于此 | `stable_sort()` 基于此 | 作为 `sort()` 的兜底 |

**为什么堆排序理论指标好但实际较慢？**

堆排序在数组上模拟树操作，父子节点在数组中的位置跳跃很大（比如索引 0 和索引 1000），CPU 缓存命中率低。而快排和归并操作的都是相邻的数组元素，缓存命中率高。

> C++ 的 `sort()` 用的是 **IntroSort**：正常情况用快排（最快），递归太深时切换到堆排序（保证最坏 O(N log N)），小数组用插入排序（常数小）。三种算法取长补短。

## 八、关键概念速查表

| 概念 | 说明 |
|------|------|
| **堆排序两步走** | 原地建堆 → 不断取堆顶放末尾 |
| **用大顶堆** | 堆顶是最大值，放到末尾 → 从小到大排列 |
| **逆序 sink 建堆** | 从 `n/2-1` 到 `0` 逐个 sink，O(N) 建堆 |
| **n/2 - 1** | 最后一个非叶子节点的索引 |
| **swim 建堆** | 简单但慢，O(N log N) |
| **sink 建堆** | 精妙且快，O(N) |
| **IntroSort** | C++ `sort()` 底层 = 快排 + 堆排序 + 插入排序 |

## 九、排序算法章节完结总结

```
排序算法全景：

O(N²)：选择 → 冒泡 → 插入 → 希尔（突破 N²）

O(N log N)：
  快速排序（前序 + partition，实际最快，C++ sort 底层）
  归并排序（后序 + merge，稳定，C++ stable_sort 底层）
  堆排序  （二叉堆 + sink，O(1) 空间，作为兜底）  ← 你在这里

每种算法都有自己的优势场景，没有绝对的最好。
竞赛中直接用 sort() 就行，但理解原理让你在需要时能做出正确选择。
```
