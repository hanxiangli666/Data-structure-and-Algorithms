# 归并排序：妙用二叉树后序位置 学习笔记

> **前置知识**：二叉树遍历（后序位置）、快速排序思路  
> **核心认知**：归并排序 = 二叉树的后序遍历。**先递归排好左右两半，再在后序位置合并两个有序数组**  
> **和快排的关系**：快排在前序位置做事（先 partition 再递归），归并在后序位置做事（先递归再 merge）

---

## 一、快排 vs 归并：前序 vs 后序

### 1.1 代码框架对比

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
vector<int> left = {1, 4};                        // 左边两位同学已经排好
vector<int> right = {2, 3};                       // 右边两位同学也排好
// 先分别排好左右队伍，再把两队按号码合并成一队（这就是 merge 思想）
```

> 这个例子强调“先分开处理，再统一合并”，下面代码就是把这个流程写成递归框架，并和快排的前序框架做对照。

```cpp
// 快速排序 —— 前序位置做事
void quickSort(int nums[], int lo, int hi) {       // ⚠️ 使用下标区间 [lo, hi]
  if (lo >= hi) return;                           // 终止条件：区间长度 <= 1
    // 【前序位置】先把 pivot 排好序
  int p = partition(nums, lo, hi);                // 先做切分，拿到 pivot 最终位置
  quickSort(nums, lo, p - 1);                     // 递归排序左区间
  quickSort(nums, p + 1, hi);                     // 递归排序右区间
}

// 归并排序 —— 后序位置做事
void mergeSort(int nums[], int lo, int hi) {       // ⚠️ 也是区间递归
  if (lo >= hi) return;                           // 终止条件同样是区间不可再分
  int mid = (lo + hi) / 2;                        // 取中点，拆成左右两半
  mergeSort(nums, lo, mid);                       // 先递归排左半
  mergeSort(nums, mid + 1, hi);                   // 再递归排右半
    // 【后序位置】左右都排好了，合并两个有序数组
  merge(nums, lo, mid, hi);                       // 最后执行合并
}
```

> 💡 **老师提醒：** `mid = (lo + hi) / 2` 在极大数据范围里可能溢出，工程里常写成 `lo + (hi - lo) / 2` 更稳妥。

> ✅ 你已经能清楚区分“前序做事”和“后序做事”，这就是分治理解的分水岭。

| 对比 | 快速排序 | 归并排序 |
|------|---------|---------|
| 做事时机 | **前序位置**（先处理，再递归） | **后序位置**（先递归，再处理） |
| 核心操作 | partition（把 1 个元素排好序） | merge（合并两个有序数组） |
| 思路 | 先排好 1 个，再递归排剩下的 | 先递归排好两半，再合并 |
| 对应遍历 | 前序遍历 | 后序遍历 |

### 1.2 直觉对比

**快排**像一个自上而下的**指挥官**：先做决策（partition 分左右），再让下属去执行（递归排序左右）。

**归并**像一个自下而上的**组装工**：先让各零部件做好（递归排左右），最后自己来组装（merge 合并）。

## 二、归并排序的核心思路

### 2.1 分治三步走

```
第一步：分（Divide）
  把数组从中间切成两半

第二步：治（Conquer）
  递归地对两半分别排序

第三步：合（Combine）
  用 merge 合并两个已排序的子数组
```

### 2.2 图解完整过程

```
原数组：[5, 3, 1, 4, 2, 8, 7, 6]

======= 分（递归拆分到只剩 1 个元素）=======

             [5, 3, 1, 4, 2, 8, 7, 6]
            /                         \
      [5, 3, 1, 4]              [2, 8, 7, 6]
       /        \                /        \
   [5, 3]    [1, 4]         [2, 8]    [7, 6]
    / \        / \            / \        / \
  [5] [3]   [1] [4]        [2] [8]   [7] [6]
  
  ↑ 1 个元素，天然有序，不需要排序！

======= 合（从底向上，合并两个有序数组）=======

  [5] [3]   [1] [4]        [2] [8]   [7] [6]
    \ /        \ /            \ /        \ /
   [3, 5]    [1, 4]         [2, 8]    [6, 7]
       \        /                \        /
      [1, 3, 4, 5]              [2, 6, 7, 8]
            \                         /
             [1, 2, 3, 4, 5, 6, 7, 8]  ✓
```

### 2.3 「分无可分」的智慧

> 你不需要知道怎么排序 1 万个元素。  
> 你只需要知道：**1 个元素天然有序，两个有序数组可以合并成一个有序数组**。  
> 剩下的，递归会帮你搞定。

```
1 万个元素 → 拆成 2 个 5000
5000 → 拆成 2 个 2500
...
2 → 拆成 2 个 1
1 个元素 = 已有序 ✓

然后反向合并：
1 + 1 → 2 个有序
2 + 2 → 4 个有序
4 + 4 → 8 个有序
...
5000 + 5000 → 1 万个有序 ✓
```

**类比（老子的话）**：「一尺之捶，日取其半，万世不竭。」反过来理解——无穷小可以合成无穷大。1 个元素开始合并，最终整个数组有序。

## 三、merge 函数是什么？

merge 是归并排序的核心——把两个**已排序**的子数组合并成一个有序数组。

```
输入：[1, 3, 5] 和 [2, 4, 6]（两个有序数组）
输出：[1, 2, 3, 4, 5, 6]（合并后的有序数组）

过程（双指针）：
  i → [1, 3, 5]
  j → [2, 4, 6]
  
  比较 1 和 2 → 取 1，i 右移
  比较 3 和 2 → 取 2，j 右移
  比较 3 和 4 → 取 3，i 右移
  比较 5 和 4 → 取 4，j 右移
  比较 5 和 6 → 取 5，i 右移
  剩下 6 → 取 6
  
  结果：[1, 2, 3, 4, 5, 6] ✓
```

merge 需要一个**额外的临时数组**来存放合并结果（这就是归并排序不是原地排序的原因）。

具体实现用**双指针**技巧，后面会学。

## 四、时间复杂度：O(N log N)

和快排的分析方法一样——看递归树每层的总工作量：

```
              每一层的元素总数 = N
              ┌─────────────────────────────────┐
第 1 层：     [5, 3, 1, 4, 2, 8, 7, 6]           merge 处理 N 个元素
              ┌────────────┴────────────┐
第 2 层：  [5, 3, 1, 4]        [2, 8, 7, 6]      两次 merge，共 N 个元素
              ┌──┴──┐           ┌──┴──┐
第 3 层：  [5,3] [1,4]        [2,8] [7,6]        四次 merge，共 N 个元素

层数 = log₂(N)（每次减半）
```

> **总复杂度 = 每层 O(N) × 层数 O(log N) = O(N log N)**

**和快排不同的是**：归并排序**没有最坏情况退化**！因为每次都是从正中间切分，递归树永远是平衡的。

| 情况 | 快速排序 | 归并排序 |
|------|---------|---------|
| 最好 | O(N log N) | O(N log N) |
| 平均 | O(N log N) | O(N log N) |
| **最坏** | **O(N²)** | **O(N log N)** ← 永远稳定！ |

## 五、三个指标分析

| 指标 | 归并排序 |
|------|---------|
| 时间复杂度 | **O(N log N)**（最好=最坏=平均） |
| 空间复杂度 | **O(N)**（merge 需要临时数组） |
| 原地排序 | ❌ 不是（需要 O(N) 额外空间） |
| 稳定性 | **✅ 稳定**（merge 时保持相等元素的相对顺序） |

## 六、快排 vs 归并 终极对比

| 对比 | 快速排序 | 归并排序 |
|------|---------|---------|
| 核心操作 | partition（前序） | merge（后序） |
| 平均时间 | O(N log N) | O(N log N) |
| 最坏时间 | O(N²) | **O(N log N)** ✓ |
| 空间 | O(log N) | **O(N)** ✗ |
| 原地排序 | ✅ | ❌ |
| 稳定 | ❌ | **✅** |
| 实际速度 | **通常更快**（常数小） | 稍慢（需要额外内存拷贝） |

**选择建议**：
- 需要稳定排序 → 归并排序（C++ `stable_sort()` 底层）
- 追求最快速度 → 快速排序（C++ `sort()` 底层）
- 两者不确定 → 直接用 `sort()`，让标准库帮你选

## 七、关键概念速查表

| 概念 | 说明 |
|------|------|
| **归并排序** | 分成两半 → 递归排左右 → 后序位置 merge 合并 |
| **merge** | 合并两个有序数组为一个有序数组（双指针） |
| **后序位置** | 左右子数组排好序之后，再做 merge |
| **分治** | 把大问题拆成小问题，小到不可分（1 个元素）就自然解决 |
| **永远 O(N log N)** | 每次从中间切，递归树永远平衡，没有退化风险 |
| **需要 O(N) 额外空间** | merge 需要临时数组，是归并排序的唯一缺点 |
| `stable_sort()` | C++ 稳定排序，底层基于归并排序 |

## 八、排序算法全景回顾

```
排序算法进化史：

O(N²) 时代：
  选择排序 → 冒泡排序 → 插入排序 → 希尔排序（突破 N²）

O(N log N) 时代：
  快速排序（前序遍历 + partition）
  归并排序（后序遍历 + merge）  ← 你在这里
  堆排序（二叉堆 + sink）       ← 下一篇

核心思想的演变：
  线性扫描 → 利用有序度 → 大步长预处理 → 分治（前序/后序）
```
