# 希尔排序：突破 O(N²) 学习笔记

> **前置知识**：选择排序、插入排序  
> **核心认知**：插入排序对「接近有序」的数组极快（接近 O(N)）。希尔排序的策略是**先让数组变得「大致有序」，再用插入排序扫尾**  
> **历史意义**：第一个突破 O(N²) 的排序算法！

---

## 一、插入排序的痛点

回顾插入排序的特点：

| 输入情况 | 复杂度 | 原因 |
|---------|--------|------|
| 已有序 | O(N) | 内层循环直接 break |
| 接近有序 | 接近 O(N) | break 很早触发 |
| **完全随机** | **O(N²)** | **每次要往左走很远** |

插入排序的问题：如果一个很小的元素在数组的最右边，它需要一步步交换到最左边，每步只移动 1 个位置，太慢了。

**类比**：如果你要把一张 A 牌从手牌最右边挪到最左边，但每次只能和旁边的牌交换一次位置，那要交换好多次。如果能让它一次跳好几步，岂不是快多了？

> **希尔排序的核心思想**：让元素能够「跳着走」，先大步跳，再小步调整。

## 二、h 有序数组

### 2.1 定义

> 一个数组是 **h 有序的**，意思是：每隔 h 个位置取出的元素，构成一个有序的子序列。

### 2.2 图解

```
h = 3 时，一个 3-有序数组：

nums = [1, 2, 4, 3, 5, 7, 8, 6, 10, 9, 12, 11]

把它按间隔 3 分成三组：
  组 1（索引 0,3,6,9）:  [1, 3, 8, 9]    ← 有序 ✓
  组 2（索引 1,4,7,10）: [2, 5, 6, 12]   ← 有序 ✓
  组 3（索引 2,5,8,11）: [4, 7, 10, 11]  ← 有序 ✓
```

整个数组看起来是乱的，但每隔 3 个一组都是有序的。

### 2.3 关键理解

```
h-有序 → 数组「大致有序」（有序度较高）
h 越小 → 越接近完全有序
h = 1 → 完全有序（就是普通的排好序的数组）
```

---

## 三、希尔排序的策略

### 3.1 思路

```
不直接排序（1-有序），而是分步骤：

  乱序数组
     ↓  先排成 13-有序
  13-有序数组（大致有序了）
     ↓  再排成 4-有序
  4-有序数组（更有序了）
     ↓  再排成 1-有序
  完全有序数组 ✓
```

每一步都在用**插入排序**，但步长从大到小逐渐递减。

### 3.2 为什么这样更快？

- **大步长阶段**（h 很大）：元素能一次跳很远，快速消除大量逆序，让数组变得大致有序
- **小步长阶段**（h 很小）：数组已经接近有序了，插入排序此时效率极高（接近 O(N)）

**类比**：整理一间极乱的房间。
- 先「大扫除」：把东西大致分区归位（大步长，快速消除混乱）
- 再「精细整理」：在每个区域内部仔细排列（小步长，微调细节）

如果上来就一件件精细整理（普通插入排序），每件东西可能要搬很远，很慢。

## 四、代码实现

### 4.1 和插入排序的对比

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
vector<int> seats = {9, 1, 5};                // 三个座位上的数字
int gap = 2;                                   // 先按“隔一个座位”比较
if (seats[2] < seats[0]) swap(seats[2], seats[0]); // 让远处较小值先跳到前面
```

> 这个例子展示了“按间隔比较”的核心感觉，下面的真实代码就是把 gap 从大到小变化，逐步逼近普通插入排序。

```cpp
// 插入排序（步长固定为 1）
void insertionSort(vector<int>& nums) {             // ⚠️ 引用参数：会原地修改
    int n = nums.size();                            // 数组长度
    for (int i = 1; i < n; i++) {                  // 从第 2 个元素开始插入
        for (int j = i; j > 0; j -= 1) {           // 步长 = 1
            if (nums[j] < nums[j - 1])             // 比较相邻元素
                swap(nums[j], nums[j - 1]);        // 逆序就交换
            else break;                            // 已到正确位置，提前结束
        }
    }
}

// 希尔排序（步长从大到小变化）
void shellSort(vector<int>& nums) {                // ⚠️ 引用参数：原地排序
    int n = nums.size();                            // 数组长度
    int h = 1;                                      // 初始步长
    while (h < n / 3) h = 3 * h + 1;               // 计算初始步长序列

    while (h >= 1) {                                // 外层控制步长
        for (int i = h; i < n; i++) {              // 从第 h 位开始做“间隔插入”
            for (int j = i; j >= h; j -= h) {      // 步长 = h
                if (nums[j] < nums[j - h])         // 比较间隔 h 的两个元素
                    swap(nums[j], nums[j - h]);    // 逆序就交换
                else break;                         // 当前元素已到合适位置
            }
        }
        h /= 3;                                      // 缩小步长，逐步逼近 1
    }
}
```

> 💡 **老师提醒：** 希尔排序对步长序列非常敏感，写对代码只是第一步，选合适的 `h` 才能体现性能优势。

> ✅ 你已经从“相邻交换”升级到“跨步交换”了，这是一个很大的理解跃迁。

**只有 4 处改动**：

| 改动 | 插入排序 | 希尔排序 |
|------|---------|---------|
| 外层 | 无 | 加了 `while (h >= 1)` 循环 |
| sortedIndex 初始值 | 1 | **h** |
| 内层步长 | `j -= 1` | `j -= h` |
| 比较对象 | `nums[j-1]` | `nums[j-h]` |

### 4.2 推荐的递增函数

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
int gap = 1;                                      // 先从小步开始
gap = 3 * gap + 1;                               // 步长放大：1 -> 4
gap = 3 * gap + 1;                               // 再放大：4 -> 13
```

> 上面的例子是在“生成步长表”，下面的代码就是把这个表和数组长度联系起来，自动找到合适的起始步长。

```cpp
// 递增函数：(3^k - 1) / 2
// 序列：1, 4, 13, 40, 121, 364, ...
int h = 1;                                         // 初始步长
while (h < n / 3) {                               // 保证 h 不会过大
    h = 3 * h + 1;    // 1 → 4 → 13 → 40 → 121 → ...
}
// 使用时：h /= 3 来缩小
```

> 💡 **老师提醒：** `h < n / 3` 这个条件别随手改，不然可能一上来 gap 太大，循环几乎不起作用。

### 4.3 完整代码（推荐版本）

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
vector<int> shelf = {8, 3, 6, 1};                // 书架上的书高低不齐
int gap = 2;                                       // 先隔一本比较
if (shelf[2] < shelf[0]) swap(shelf[2], shelf[0]); // 让远处小书先跳到前面
gap = 1;                                           // 最后再做相邻微调
```

> 这个例子就是希尔排序的缩影：先“粗排”再“细排”。下面的完整代码把这个流程写成了通用版本。

```cpp
void sort(vector<int>& nums) {                    // ⚠️ 引用参数：原地修改原数组
    int n = nums.size();                          // 数组长度
    
    // 计算初始步长 h（递增函数 (3^k-1)/2）
    int h = 1;                                   // 初始步长
    while (h < n / 3) {                          // 预先放大步长
        h = 3 * h + 1;    // 1, 4, 13, 40, 121, 364...
    }

    while (h >= 1) {                             // gap 从大到小逐轮缩小
        // 对每个间隔为 h 的子数组进行插入排序
        for (int i = h; i < n; i++) {            // 遍历每个待插入位置
            for (int j = i; j >= h; j -= h) {    // 按间隔 h 向左寻找位置
                if (nums[j] < nums[j - h]) {     // 间隔 h 的逆序
                    swap(nums[j], nums[j - h]);  // 交换纠正
                } else {
                    break;                        // 已到位，结束当前插入
                }
            }
        }
        h /= 3;    // 缩小步长
    }
}
```

> 💡 **老师提醒：** 希尔排序不是稳定排序，面试时如果题目明确要求“稳定”，你要主动说明不能选它。

> 这个版本可以通过 LeetCode 912！这是前面的选择排序、冒泡排序、插入排序都做不到的。

## 五、图解执行过程

```
原数组：[8, 5, 1, 9, 3, 7, 2, 6, 4]   n=9

初始 h：h=1 → 4 → 13（13 > 9/3，停）→ h=4

=== h=4 阶段（4-有序排序）===
把数组按间隔 4 分组：
  组 1（索引 0,4,8）: [8, 3, 4] → 排成 [3, 4, 8]
  组 2（索引 1,5）:   [5, 7]   → 已有序
  组 3（索引 2,6）:   [1, 2]   → 已有序
  组 4（索引 3,7）:   [9, 6]   → 排成 [6, 9]

排完后：[3, 5, 1, 6, 4, 7, 2, 9, 8]   ← 4-有序

=== h=1 阶段（普通插入排序）===
数组已经大致有序了，插入排序很快完成
排完后：[1, 2, 3, 4, 5, 6, 7, 8, 9]   ✓
```

注意 h=4 阶段把 8 从最左边直接「跳」到了后面，把 3 从中间「跳」到了最前面。如果用普通插入排序，这些元素要一步步挪过去。

## 六、三个指标分析

### 6.1 时间复杂度

希尔排序的精确复杂度分析非常困难（至今没有完整的数学证明），取决于递增函数的选择：

| 递增函数 | 复杂度估计 |
|---------|-----------|
| 2^(k-1)：1, 2, 4, 8, 16... | O(N^(3/2)) 左右 |
| **(3^k-1)/2：1, 4, 13, 40...** | **O(N^(4/3)) 左右** |

> 关键结论：**小于 O(N²)**，这是第一次突破！

### 6.2 空间复杂度

O(1)，原地排序。只修改了步长 h 和几个临时变量。

### 6.3 稳定性

❌ **不稳定**。

```
[2', 2'', 1]，h=2

h=2 阶段：比较索引 0 和索引 2
  nums[0]=2' 和 nums[2]=1 → 1 < 2'，交换
  → [1, 2'', 2']

2' 和 2'' 的相对顺序被打乱了 ❌
```

步长 > 1 时的交换可能跨越相同元素，破坏稳定性。

## 八、关键概念速查表

| 概念 | 说明 |
|------|------|
| **h-有序数组** | 每隔 h 个位置取出的元素构成有序子序列 |
| **递增函数** | 定义 h 从大到小的变化序列 |
| **推荐递增函数** | `(3^k-1)/2`：1, 4, 13, 40, 121... |
| **h 初始化** | `while (h < n/3) h = 3*h+1` |
| **h 缩小** | `h /= 3` |
| **本质** | 步长可变的插入排序 |
| **大步长** | 快速消除大量逆序，让数组大致有序 |
| **小步长** | 利用插入排序对接近有序数组的高效性 |

## 九、阶段总结：O(N²) 排序到此结束

```
O(N²) 排序算法的进化路径：

选择排序（最朴素）
  ↓ 获得稳定性
冒泡排序
  ↓ 逆向思维，利用有序度
插入排序
  ↓ 大步长预处理 + 插入排序
希尔排序（突破 O(N²)！）

接下来：归并排序、快速排序（O(N log N) 的世界）
```

从下一篇开始，排序算法将引入**分治思想**——把大问题拆成小问题分别解决再合并，一举把复杂度降到 O(N log N)。
