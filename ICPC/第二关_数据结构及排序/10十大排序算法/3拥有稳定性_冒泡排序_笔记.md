# 冒泡排序：让选择排序重获稳定性 学习笔记

> **前置知识**：选择排序及其问题  
> **核心认知**：冒泡排序不是凭空出现的，它是**选择排序经过两步优化**后得到的结果  
> **进化路径**：选择排序 → 用插入代替交换（获得稳定性）→ 合并两个 for 循环（冒泡排序）→ 加提前终止

---

## 一、回顾：选择排序的三个缺陷

| 缺陷 | 说明 |
|------|------|
| ❌ 不稳定 | 交换操作会打乱相同元素的相对顺序 |
| ❌ 时间复杂度和数据有序度无关 | 即使数组已经有序，还是要跑 N²/2 次 |
| ❌ O(N²) 无法降低 | 存在冗余计算，但因为数组会变，预计算方案行不通 |

本篇解决前两个问题，第三个问题后面的排序算法来解决。

---

## 二、第一步优化：用插入代替交换 → 获得稳定性

### 2.1 选择排序为什么不稳定？

```
[2', 2''', 2'', 1]

选择排序：找到最小值 1，和 2' 交换
  → [1, 2''', 2'', 2']
                    ↑ 2' 被甩到了最后面！

2 的原始顺序 2'→2'''→2'' 变成了 2'''→2''→2'  ❌
```

关键：把 `nums[minIndex]=1` 搬到前面是对的，但把 `nums[sortedIndex]=2'` 直接甩到 `minIndex` 位置，跨越了其他的 2，这才破坏了稳定性。

### 2.2 修复方法：整体后移，腾出位置

不直接交换，而是把 `[sortedIndex, minIndex)` 的元素**整体后移一位**，然后把最小值插到 `sortedIndex`。

```
[2', 2''', 2'', 1, 1']
 ↑ sortedIndex   ↑ minIndex

第一步：取出最小值 1
[2', 2''', 2'', _, 1']

第二步：整体后移
[_, 2', 2''', 2'', 1']

第三步：最小值放到 sortedIndex
[1, 2', 2''', 2'', 1']

2', 2''', 2'' 的相对顺序没变 ✓
1, 1' 的相对顺序也没变 ✓
```

### 2.3 代码

```cpp
void sort(vector<int>& nums) {
    int n = nums.size();
    int sortedIndex = 0;
    while (sortedIndex < n) {
        // 找最小值
        int minIndex = sortedIndex;
        for (int i = sortedIndex + 1; i < n; i++) {
            if (nums[i] < nums[minIndex]) minIndex = i;
        }

        // 不交换！改用插入：整体后移 + 放到正确位置
        int minVal = nums[minIndex];
        for (int i = minIndex; i > sortedIndex; i--) {
            nums[i] = nums[i - 1];     // 元素后移
        }
        nums[sortedIndex] = minVal;

        sortedIndex++;
    }
}
```

**问题**：稳定了，但多了一个 for 循环做数据搬移，实际执行次数变多了（虽然 Big O 还是 O(N²)）。

## 三、第二步优化：合并两个 for 循环 → 冒泡排序诞生

### 3.1 思路

上面的代码 while 内部有两个 for 循环：

1. 找最小值
2. 把最小值搬到正确位置

**能不能合二为一？** 在找最小值的**过程中**，顺便把它挪到正确位置？

### 3.2 巧妙的方法：倒序遍历 + 交换相邻逆序对

```cpp
void sort(vector<int>& nums) {
    int n = nums.size();
    int sortedIndex = 0;
    while (sortedIndex < n) {
        // 从右向左扫描，遇到相邻逆序对就交换
        for (int i = n - 1; i > sortedIndex; i--) {
            if (nums[i] < nums[i - 1]) {
                swap(nums[i], nums[i - 1]);
            }
        }
        sortedIndex++;
    }
}
```

### 3.3 图解执行过程

```
原数组：[5, 3, 1, 4, 2]，sortedIndex = 0

从右向左扫描，交换相邻逆序对：
  i=4: nums[4]=2 < nums[3]=4 → 交换 → [5, 3, 1, 2, 4]
  i=3: nums[3]=2 < nums[2]=1? 不是 → 跳过
       等等，1 < 2，不是逆序，跳过
  i=2: nums[2]=1 < nums[1]=3 → 交换 → [5, 1, 3, 2, 4]
  i=1: nums[1]=1 < nums[0]=5 → 交换 → [1, 5, 3, 2, 4]

一轮结束：最小值 1 已经「冒泡」到了 sortedIndex=0 的位置！
  [1, 5, 3, 2, 4]
   ✓
```

**像气泡一样浮上来**：最小的元素通过一连串相邻交换，从底部一步步「冒」到了顶部。这就是「冒泡排序」名字的由来。

### 3.4 为什么这样是稳定的？

因为**只交换相邻的逆序对**。

```
如果 nums[i] == nums[i-1]，不满足 nums[i] < nums[i-1]，不会交换
→ 相同元素永远不会被交换 → 相对顺序不变 ✓
```

关键在于判断条件是**严格小于** `<`，不是 `<=`。如果写成 `<=` 就不稳定了！

### 3.5 时间复杂度

```
外层：n 轮
内层：第 1 轮 n-1 次，第 2 轮 n-2 次，...

总次数 = (n-1) + (n-2) + ... + 1 = n(n-1)/2 ≈ n²/2
```

和选择排序一样，还是 **O(N²)**。但至少稳定了。

## 四、第三步优化：提前终止

### 4.1 思路

如果某一轮扫描中，**一次交换都没发生**，说明数组已经完全有序了，可以直接结束。

### 4.2 代码

```cpp
void sort(vector<int>& nums) {
    int n = nums.size();
    int sortedIndex = 0;
    while (sortedIndex < n) {
        bool swapped = false;     // 记录本轮是否发生过交换
        
        for (int i = n - 1; i > sortedIndex; i--) {
            if (nums[i] < nums[i - 1]) {
                swap(nums[i], nums[i - 1]);
                swapped = true;
            }
        }
        
        if (!swapped) break;      // 没交换 = 已有序，提前结束！
        sortedIndex++;
    }
}
```

### 4.3 效果

| 输入情况 | 选择排序 | 冒泡排序（带提前终止） |
|---------|---------|---------------------|
| 已经有序 | O(N²)（无视有序度） | **O(N)**（一轮就结束） |
| 接近有序 | O(N²) | 接近 O(N) |
| 完全逆序 | O(N²) | O(N²) |
| 随机数据 | O(N²) | O(N²) |

> **最好情况从 O(N²) 改善到了 O(N)**！这就是利用了数据的有序度。

## 五、完整的演化路径

```
选择排序
  │ 缺陷：不稳定、不利用有序度
  │
  ├── 优化 1：交换改为插入（整体后移）
  │   └── 获得稳定性 ✓，但多了一个 for 循环
  │
  ├── 优化 2：合并两个 for（倒序交换相邻逆序对）
  │   └── 冒泡排序诞生！稳定 ✓，效率和选择排序相当
  │
  └── 优化 3：加 swapped 标记提前终止
      └── 最好情况 O(N) ✓
```

## 六、冒泡排序的完整指标

| 指标 | 冒泡排序 | 对比选择排序 |
|------|---------|------------|
| 最好时间 | **O(N)**（已有序） | O(N²) |
| 最坏时间 | O(N²)（完全逆序） | O(N²) |
| 平均时间 | O(N²) | O(N²) |
| 空间 | O(1)（原地） | O(1)（原地） |
| **稳定性** | **✅ 稳定** | ❌ 不稳定 |
| 利用有序度 | ✅ 是 | ❌ 否 |

**冒泡排序相比选择排序**：稳定了，能利用有序度了，但平均/最坏复杂度没变。

## 七、关键概念速查表

| 概念 | 说明 |
|------|------|
| **冒泡排序** | 倒序遍历，交换相邻逆序对，最小值「冒泡」到前面 |
| **逆序对** | `nums[i] > nums[i+1]`，即相邻两个元素顺序反了 |
| **稳定的关键** | 只交换**相邻**元素，且用**严格小于** `<` 判断 |
| **提前终止** | 一轮没有交换 = 已有序，`break` |
| **swapped 标记** | 布尔变量记录本轮是否发生交换 |
| **最好情况 O(N)** | 数组已有序时，一轮扫描无交换，直接结束 |
