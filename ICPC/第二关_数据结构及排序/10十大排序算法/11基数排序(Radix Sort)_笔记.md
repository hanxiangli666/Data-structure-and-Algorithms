# 基数排序（Radix Sort）学习笔记

> **前置知识**：计数排序（必须先掌握）  
> **核心认知**：基数排序 = 对数字的**每一位**依次做计数排序。利用计数排序的稳定性，逐位排完后整个数组就有序了  
> **解决的问题**：计数排序在元素范围 R 很大时空间爆炸，基数排序把 R 拆成每位只有 0~9，每次计数排序只需 O(N+10) = O(N)

---

## 一、基数排序的核心思路

### 1.1 一句话描述

> 从个位开始，逐位对所有数字做计数排序，每位排完后整个数组就有序了。

### 1.2 完整示例

```
原数组：[329, 457, 839, 439, 720, 355, 350]

第 1 轮：按个位排序（稳定排序！）
  720  350  355  457  329  839  439
                                  ^  个位

第 2 轮：按十位排序（稳定排序！）
  720  329  839  439  350  355  457
                               ^   十位

第 3 轮：按百位排序（稳定排序！）
  329  350  355  439  457  720  839
                             ^     百位

排序完成！✓
```

**关键**：每轮排序都必须是**稳定排序**。这就是为什么用计数排序——它是稳定的。

### 1.3 为什么每位必须用稳定排序？

```
考虑这两个数：56, 57

个位排序后：56, 57（正确）

现在对十位排序，十位都是 5。
  稳定排序：56, 57（保持原来的顺序）✓
  不稳定排序：可能变成 57, 56 ✗
```

如果某一位上的排序不稳定，之前低位排好的顺序就可能被打乱，最终结果就错了。

---

## 二、基数排序解决了什么问题？

### 2.1 计数排序的痛点

计数排序需要 `count` 数组大小 = 元素范围 R = max - min。

```
nums = [1, 1000000000]
R = 10⁹ → count 数组需要 10⁹ 个位置 → 内存爆炸！💥
```

### 2.2 基数排序的解决方案

不看整个数字（范围可能极大），只看**每一位**（范围只有 0~9）。

```
每一位的计数排序：count 数组大小 = 10（0~9）
做几轮？= 最大数字的位数 m

空间：O(N + 10) = O(N)   ← 无论数字多大都行！
时间：O(m × N)            ← m 通常很小（整数最多 10 位）
```

**类比**：要按身份证号排序 1 亿人。直接计数排序？身份证号范围太大了。基数排序的方法：先按最后一位排，再按倒数第二位排……一共排 18 轮（身份证 18 位），每轮都很快。

## 三、关键问题解答

### 3.1 位数不同怎么办？

逻辑上高位补 0 即可。公式 `(num / 10^k) % 10` 天然处理了这个问题：

```
num = 5, k = 2（取百位）
(5 / 100) % 10 = 0 % 10 = 0  ← 自动得到 0，相当于 005
```

### 3.2 有负数怎么办？

和计数排序一样，给所有数字加上偏移量使其变成非负数，排完再减回来。

```
nums = [-3, 4, -1, 2]
offset = 3（= -min）
→ [0, 7, 2, 5]   排序   → [0, 2, 5, 7]   减 offset   → [-3, -1, 2, 4] ✓
```

### 3.3 必须从低位到高位（LSD）吗？

**整数排序：必须 LSD（低位到高位）**

```
反例：如果从高位到低位
  90, 18
  先排十位：18, 90 ✓
  再排个位：90, 18 ✗（0 < 8，所以 90 被排到前面了）
```

高位排好后被低位排序破坏了。LSD 则相反——低位先排好，高位后排，高位的排序结果才是最终结果，不会被覆盖。

### 3.4 LSD vs MSD

| 对比 | LSD（低位到高位） | MSD（高位到低位） |
|------|------------------|------------------|
| 适用 | **整数排序** | 字典序排序（字符串） |
| 实现 | 简单，直接循环 | 复杂，需要递归分组 |
| 竞赛常用度 | **更常用** | 较少 |

MSD 用于字典序排序（如 "1" < "10" < "2"），需要按高位分组后递归排序，实现复杂。本篇只讲 LSD。

## 四、取第 k 位数字的公式

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
int number = 329;                                   // 一个三位号码
int ones = number % 10;                             // 个位：像看门牌最后一位
int tens = (number / 10) % 10;                      // 十位：先去掉个位再取最后一位
```

> 上面的例子是“拆出每一位”的基础动作，下面代码就是把这个动作写成通用的第 k 位提取公式。

```cpp
// 取正整数 num 从右往左第 k 位（k 从 0 开始）
int digit = (num / (int)pow(10, k)) % 10;          // ⚠️ pow 返回浮点数，这里强转为 int

// 举例：num = 329
// k=0（个位）: (329 / 1) % 10 = 9
// k=1（十位）: (329 / 10) % 10 = 2
// k=2（百位）: (329 / 100) % 10 = 3
```

> 💡 **老师提醒：** `pow` 是浮点运算，数据很大时可能有精度问题，工程里常用循环乘 10 的整数写法更稳。

> ✅ 你已经把“按位提取”这个核心技巧掌握了，后面的基数排序就顺了。

如果 num 的位数不够 k，自动得到 0（比如 num=5, k=2 → 0）。

## 五、LSD 基数排序代码

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
vector<int> id = {21, 13, 32};                      // 三个号码
int k = 0;                                           // 先按个位排
int d = (id[0] / 1) % 10;                            // 取 21 的个位
```

> 这个例子展示了“先取某一位再分组”，下面完整代码就是把这个流程扩展成逐位稳定排序。

```cpp
// 对第 k 位做计数排序（nums 中的元素都是非负数）
void countSortByDigit(vector<int>& nums, int k) {      // ⚠️ 引用参数：直接修改 nums
  vector<int> count(10, 0);                          // 每位只有 0~9

  int divisor = (int)pow(10, k);                     // 当前位对应除数（1,10,100...）

    // 统计第 k 位的出现次数
  for (int num : nums) {                             // 遍历原数组
    int digit = (num / divisor) % 10;              // 提取第 k 位
    count[digit]++;                                // 统计该位频次
    }

    // 累加 count
  for (int i = 1; i < 10; i++) {                     // 变成前缀和
    count[i] += count[i - 1];                      // 转成“结束位置”
    }

    // 倒序遍历，保证稳定性
  vector<int> sorted(nums.size());                   // 临时结果数组
  for (int i = nums.size() - 1; i >= 0; i--) {      // 从后往前回填
    int digit = (nums[i] / divisor) % 10;          // 当前元素第 k 位
    sorted[count[digit] - 1] = nums[i];            // 放到该位应在的位置
    count[digit]--;                                 // 同位下一个位置前移
    }

  nums = sorted;                                     // 本轮按第 k 位排序完成
}

// LSD 基数排序
void radixSort(vector<int>& nums) {                    // ⚠️ 低位到高位排序入口
  if (nums.empty()) return;                          // 空数组直接返回

    // 处理负数：加偏移量
  int minVal = *min_element(nums.begin(), nums.end()); // 最小值
  int offset = -minVal;                                // 偏移量
  for (int& x : nums) x += offset;                     // ⚠️ 引用遍历：直接改每个元素

    // 找最大值，确定位数
  int maxVal = *max_element(nums.begin(), nums.end()); // 偏移后的最大值
  int maxDigits = 0;                                   // 需要排序的总位数
  while (maxVal > 0) {                                 // 不断除 10 统计位数
    maxVal /= 10;
    maxDigits++;
    }
    if (maxDigits == 0) maxDigits = 1;  // 全是 0 的情况

    // 从低位到高位，逐位计数排序
  for (int k = 0; k < maxDigits; k++) {               // k=0,1,2... 从个位到高位
    countSortByDigit(nums, k);                      // 每轮稳定地按一位排序
    }

    // 减回偏移量
  for (int& x : nums) x -= offset;                    // 还原原始数值范围
}
```

> 💡 **老师提醒：** 基数排序每一轮必须稳定，*就像先按楼层排队再按房号排队*，后一步不能把前一步规则破坏掉。

## 六、三个指标分析

设 N = 数组长度，m = 最大数字的位数。

| 指标 | 基数排序（LSD） |
|------|----------------|
| 时间复杂度 | **O(m × N)** |
| 空间复杂度 | **O(N)**（sorted 数组 + count 数组） |
| 稳定性 | **✅ 稳定** |
| 原地排序 | ❌ |

**m 通常很小**：32 位整数最多 10 位十进制数，所以 m ≤ 10，实际上接近 O(N)。

---

## 七、基数排序 vs 计数排序 vs 桶排序

| 对比 | 计数排序 | 基数排序 | 桶排序 |
|------|---------|---------|--------|
| 核心思想 | 统计每个值出现次数 | 逐位做计数排序 | 分桶 + 桶内排序 + 合并 |
| 时间 | O(N+R) | O(m×N) | O(N+k) 均摊 |
| 空间 | O(N+R) | O(N) | O(N+k) |
| 元素范围大时 | **空间爆炸** | **依然 O(N)** ✓ | 取决于桶数 |
| 稳定 | ✅ | ✅ | ✅（取决于桶内排序） |
| 关系 | 基数排序的子程序 | 计数排序的扩展 | 独立思路 |

> **基数排序 = 对计数排序「空间爆炸」问题的优雅解决方案**。把大范围拆成每位 0~9 的小范围，多做几轮计数排序即可。

## 八、关键概念速查表

| 概念 | 说明 |
|------|------|
| **基数（Radix）** | 进制，十进制的基数是 10 |
| **LSD** | Least Significant Digit，从低位到高位排序，用于整数排序 |
| **MSD** | Most Significant Digit，从高位到低位排序，用于字典序排序 |
| **取第 k 位** | `(num / 10^k) % 10` |
| **稳定性是关键** | 每位排序必须稳定，否则低位的排序结果会被破坏 |
| **计数排序是子程序** | 基数排序每一轮调用计数排序 |
| **偏移量处理负数** | `offset = -min`，排完再减回来 |
| **O(m×N)** | m = 最大数字位数，通常 ≤ 10 |

## 九、非比较排序的完整版图

```
非比较排序三兄弟：

计数排序
  ├── 核心：统计每个值出现次数
  ├── 限制：元素范围 R 不能太大
  └── O(N+R)

基数排序 ← 你在这里
  ├── 核心：逐位做计数排序
  ├── 解决：计数排序 R 太大的问题
  └── O(m×N)，m 通常很小

桶排序
  ├── 核心：分桶 + 桶内排序 + 合并
  ├── 灵活：桶内可以用任意排序算法
  └── O(N+k) 均摊

三者的关系：
  计数排序 = 桶排序的特例（每桶 1 个值）
  基数排序 = 计数排序的扩展（逐位拆解大范围）
```

到这里，**排序算法章节真正全部完成**！从 O(N²) 的选择排序到线性的基数排序，你已经掌握了所有主流排序算法的原理。
