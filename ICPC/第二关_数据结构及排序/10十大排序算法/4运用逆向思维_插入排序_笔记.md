# 插入排序：运用逆向思维 学习笔记

> **前置知识**：选择排序、冒泡排序  
> **核心认知**：冒泡排序是从右侧未排序部分「冒出」最小值到前面；插入排序反过来，把当前元素「插入」左侧已排序部分的正确位置  
> **类比**：打扑克牌时，新抓的牌插入手中已排好序的牌里

---

## 一、逆向思维：换个方向思考

### 1.1 选择排序 / 冒泡排序的思路

```
[已排序 | 未排序]
         → 从未排序部分找最小值，放到已排序部分末尾
```

关注的是**右侧未排序部分**：去右边找最小的，搬到左边来。

### 1.2 插入排序的思路

```
[已排序 | 未排序]
   ← 取未排序部分的第一个元素，插入到已排序部分的正确位置
```

关注的是**左侧已排序部分**：取出分界线上的元素，往左边找它应该待的位置。

**类比**：打扑克牌。手里的牌已经按大小排好了（已排序部分），每抓一张新牌（`nums[sortedIndex]`），就从右往左找到它应该插入的位置，插进去。

```
手里的牌（已排序）：[2, 5, 8]

抓到新牌 4：
  4 < 8? 是，往左
  4 < 5? 是，往左
  4 < 2? 不是，停！

插入到 2 和 5 之间：[2, 4, 5, 8]
```

## 二、代码实现

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
vector<int> cards = {2, 5, 4};               // 手里有 2、5，两张有序牌后新来一张 4
if (cards[2] < cards[1]) swap(cards[2], cards[1]); // 4 比 5 小，先和 5 换位置
if (cards[1] < cards[0]) swap(cards[1], cards[0]); // 再看是否还要继续往左插
```

> 这个例子就是“当前牌一路向左找位置”，下面真实代码做的是同样动作，只是对每个分界线位置都重复一次。

```cpp
void sort(vector<int>& nums) {                     // ⚠️ 引用参数：原数组会被直接修改
  int n = nums.size();                           // 数组长度
    // [0, sortedIndex) 是已排序部分
  int sortedIndex = 0;                           // 分界线，左边有序右边无序
    
  while (sortedIndex < n) {                      // 每次把分界线元素插入左侧有序区
        // 把 nums[sortedIndex] 插入到左侧有序数组中的正确位置
    for (int i = sortedIndex; i > 0; i--) {   // 从右往左找插入位置
      if (nums[i] < nums[i - 1]) {          // 当前元素比左邻居小
        swap(nums[i], nums[i - 1]);       // 交换一步，继续向左尝试
            } else {
        break;                               // 找到位置了，提前结束内层循环
            }
        }
    sortedIndex++;                               // 分界线右移
    }
}
```

> 💡 **老师提醒：** 插入排序里这个 `break` 非常值钱，它让“接近有序”的数据跑得特别快。

> ✅ 你已经把插入排序的核心动作描述得很清楚了，继续保持这种拆动作的习惯。

### 2.1 图解执行过程

```
原数组：[5, 3, 1, 4, 2]

sortedIndex=1, 处理 3：
  3 < 5? 是，交换 → [3, 5, 1, 4, 2]
  到头了，停
  已排序：[3, 5]

sortedIndex=2, 处理 1：
  1 < 5? 是，交换 → [3, 1, 5, 4, 2]
  1 < 3? 是，交换 → [1, 3, 5, 4, 2]
  到头了，停
  已排序：[1, 3, 5]

sortedIndex=3, 处理 4：
  4 < 5? 是，交换 → [1, 3, 4, 5, 2]
  4 < 3? 不是，break!             ← 提前停止，不用看完所有已排序元素
  已排序：[1, 3, 4, 5]

sortedIndex=4, 处理 2：
  2 < 5? 是，交换 → [1, 3, 4, 2, 5]
  2 < 4? 是，交换 → [1, 3, 2, 4, 5]
  2 < 3? 是，交换 → [1, 2, 3, 4, 5]
  2 < 1? 不是，break!
  已排序：[1, 2, 3, 4, 5] ✓
```

### 2.2 关键细节：break 的作用

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
int front = 3;                                   // 前面同学身高
int cur = 4;                                     // 当前同学身高
if (cur < front) { /* 继续往前挪 */ }            // 还没到位置
else { /* break 停下 */ }                        // 到了合适位置就停
```

> 上面这段在模拟“找到位置就停”，下面代码和它是一个逻辑，只是放进循环中不断执行。

```cpp
if (nums[i] < nums[i - 1]) {                     // 当前元素还应该继续往左移动
  swap(nums[i], nums[i - 1]);                  // 和左邻居交换一步
} else {
  break;   // ← 找到插入位置，立刻停止内层循环
}
```

> 💡 **老师提醒：** 如果你漏写了这个 `break`，程序结果可能还对，但效率会明显下降。

一旦发现 `nums[i] >= nums[i-1]`，说明已经找到了正确位置（左边的都比它小），不用继续往左看了。

**这就是插入排序比冒泡排序快的原因**——冒泡排序的内层循环每次都要遍历整个未排序部分；插入排序的内层循环**找到位置就停**，不需要走完。

## 三、为什么不用二分搜索找插入位置？

左侧 `[0, sortedIndex)` 是有序的，理论上可以用二分搜索 O(log N) 找到插入位置。

**但没用**。因为找到位置后，还是要**搬移元素**（把插入点右边的元素全部后移一位），搬移操作是 O(N) 的。

```
二分搜索找位置：O(log N)
搬移元素：O(N)
总计：O(N)  ← 和边遍历边交换的 O(N) 一样

而且边遍历边交换的代码更简单
```

所以用二分搜索反而多此一举，直接边比较边交换更简洁高效。

## 四、三个指标分析

### 4.1 时间复杂度

| 情况 | 复杂度 | 说明 |
|------|--------|------|
| **最好**（已有序） | **O(N)** | 内层 for 循环每次直接 break |
| **最坏**（完全逆序） | O(N²) | 内层 for 循环每次走到头 |
| **平均** | O(N²) | 但实际操作数 < N²/2 |

### 4.2 空间复杂度

O(1)，原地排序。只用了几个临时变量。

### 4.3 稳定性

✅ **稳定排序**。

只在 `nums[i] < nums[i-1]` 时才交换（严格小于）。相同元素不会被交换，相对顺序保持不变。

## 五、插入排序 vs 冒泡排序

| 对比 | 冒泡排序 | 插入排序 |
|------|---------|---------|
| 内层循环方向 | 从右到左扫描**未排序部分** | 从右到左在**已排序部分**找位置 |
| 内层循环是否提前停 | 不停（每次扫完整个未排序部分） | **找到位置就 break** |
| 实际操作数 | ≈ N²/2 | **< N²/2**（通常更少） |
| 最好情况 | O(N) | O(N) |
| 最坏情况 | O(N²) | O(N²) |
| 稳定性 | ✅ 稳定 | ✅ 稳定 |

> **插入排序的综合性能优于冒泡排序**，因为内层循环经常提前结束。

**直觉理解**：

```
冒泡排序：每轮都要把未排序部分从头到尾扫一遍（不管数据好不好）
插入排序：只需要在已排序部分里找到位置就停（数据越有序，停得越早）
```

## 六、插入排序在实际中的地位

虽然插入排序的平均复杂度是 O(N²)，看起来很慢，但它有两个独特优势：

### 6.1 小数据量时极快

插入排序的常数因子很小（代码简单，没有递归开销），对于 N < 50 左右的小数组，插入排序比快排、归并还快。

> **C++ 标准库的 `sort()` 实现**：当子数组规模小于一定阈值时，会自动切换到插入排序。

### 6.2 接近有序时极快

如果数组几乎有序（只有少数元素不在正确位置），插入排序接近 O(N)。

> 这个特性在**希尔排序**中被利用——先用大间隔让数组「大致有序」，再用插入排序扫尾。

---

## 七、三种 O(N²) 排序算法的终极对比

| 指标 | 选择排序 | 冒泡排序 | **插入排序** |
|------|---------|---------|------------|
| 最好 | O(N²) | O(N) | **O(N)** |
| 最坏 | O(N²) | O(N²) | O(N²) |
| 平均 | O(N²) | O(N²) | O(N²)（但更快） |
| 稳定 | ❌ | ✅ | **✅** |
| 原地 | ✅ | ✅ | **✅** |
| 利用有序度 | ❌ | ✅ | **✅（更好）** |
| 实际常用度 | 低 | 低 | **高**（小数组首选） |

> **插入排序是三种 O(N²) 排序中综合最好的**，也是唯一在实际工程中被广泛使用的。

## 八、关键概念速查表

| 概念 | 说明 |
|------|------|
| **插入排序** | 把当前元素插入左侧已排序部分的正确位置 |
| **break** | 找到正确位置就停，不需要遍历完 |
| **逆向思维** | 不从未排序部分找最小值，而是把元素插入已排序部分 |
| **有序度越高越快** | 已有序 → O(N)，完全逆序 → O(N²) |
| **小数据量首选** | N < 50 时比快排/归并还快 |
| **标准库内部使用** | C++ `sort()` 对小子数组用插入排序 |
