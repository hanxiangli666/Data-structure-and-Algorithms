# 选择排序及其面临的问题 学习笔记

> **前置知识**：排序算法的关键指标（时空复杂度、稳定性、原地排序）  
> **核心认知**：选择排序是最直觉的排序方法，但时间复杂度 O(N²) 无法优化。它的价值在于引出优化思路  
> **思维方法**：遇到新问题先自己想，找到冗余计算的位置，尝试消除它

---

## 一、选择排序的思路

### 1.1 最直觉的排序方法

> 每次从未排序部分找到**最小的元素**，放到已排序部分的末尾。

```
原始数组：[5, 3, 1, 4, 2]

第 1 轮：找到最小值 1，和第 1 个位置交换
  [1, 3, 5, 4, 2]
   ✓

第 2 轮：在 [3, 5, 4, 2] 中找到最小值 2，和第 2 个位置交换
  [1, 2, 5, 4, 3]
   ✓  ✓

第 3 轮：在 [5, 4, 3] 中找到最小值 3，和第 3 个位置交换
  [1, 2, 3, 4, 5]
   ✓  ✓  ✓

第 4 轮：在 [4, 5] 中找到最小值 4，已经在正确位置
  [1, 2, 3, 4, 5]
   ✓  ✓  ✓  ✓

完成！
```

**类比**：就像你整理一排书架。每次扫一遍剩下的书，找出最矮的那本，放到最左边已经排好的位置旁边。

### 1.2 代码实现

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
vector<int> books = {5, 3, 4};             // 书架上三本书的高度
int minPos = 1;                            // 先假设第 2 本最矮
swap(books[0], books[minPos]);             // 把最矮的先放到最左边
```

> 这个小例子做的是“先找最小再换到前面”，下面的真实代码就是把这个动作重复很多轮，直到整个数组有序。

```cpp
void sort(vector<int>& nums) {                    // ⚠️ 引用参数：函数会直接改动原数组
  int n = nums.size();                          // 数组长度
  // sortedIndex 是分割线：左边已排序，右边未排序
  int sortedIndex = 0;                          // 从第 0 位开始放置最小值
    
  while (sortedIndex < n) {                     // 逐步扩大左侧有序区间
    // 在未排序部分 [sortedIndex, n) 中找最小值
    int minIndex = sortedIndex;               // 先把分界线位置当作最小值
    for (int i = sortedIndex + 1; i < n; i++) { // 扫描右侧未排序区间
      if (nums[i] < nums[minIndex]) {       // 找到更小的元素
        minIndex = i;                     // 更新最小值下标
            }
        }
        // 最小值和 sortedIndex 位置交换
    swap(nums[sortedIndex], nums[minIndex]);  // 把本轮最小值放到正确位置
    sortedIndex++;                            // 分界线右移，进入下一轮
    }
}
```

> 💡 **老师提醒：** 选择排序里 `swap` 每轮最多一次，所以交换次数少，但比较次数并不会减少，时间复杂度依然是 O(N²)。

> ✅ 你已经能把“分界线”这个概念说清楚了，这对后面所有排序都很关键。

**变量理解**：
- `sortedIndex`：分割线。左边 `[0, sortedIndex)` 已排好，右边 `[sortedIndex, n)` 还没排
- `minIndex`：记录当前未排序部分中最小元素的位置

## 二、三个指标分析

### 2.1 是否原地排序？

✅ **是**。只用了几个变量（`sortedIndex`、`minIndex`、`tmp`），空间复杂度 O(1)。

### 2.2 时间复杂度

```
外层循环：n 次
内层循环：第 1 次 n-1 次，第 2 次 n-2 次，...，第 n 次 0 次

总次数 = (n-1) + (n-2) + ... + 1 = n(n-1)/2 ≈ n²/2
```

**时间复杂度：O(N²)**

而且有个特点：**即使数组已经有序，还是要跑 N²/2 次**。原始数据的有序度对选择排序没有任何帮助。

### 2.3 是否稳定排序？

❌ **不稳定**。

```
原数组：[2', 2''', 2'', 1]   （三个 2 用标记区分）

第 1 轮：找到最小值 1，和 2' 交换
  → [1, 2''', 2'', 2']

2 的原始顺序是 2' → 2''' → 2''
排序后变成了 2''' → 2'' → 2'
顺序被打乱了！❌
```

**问题出在哪？** 是「交换」操作导致的。把 2' 和远处的 1 交换时，2' 直接跳到了末尾，跨过了其他的 2，打乱了相对顺序。

### 2.4 总结

| 指标 | 选择排序 |
|------|---------|
| 时间复杂度 | O(N²)（最好=最坏=平均） |
| 空间复杂度 | O(1)（原地排序） |
| 稳定性 | ❌ 不稳定 |
| 特点 | 数据有序度不影响性能 |

## 三、能不能优化？（重要的思考过程）

### 3.1 找到冗余计算

```
第 1 轮：遍历 nums[0..] 找最小值
第 2 轮：遍历 nums[1..] 找最小值
第 3 轮：遍历 nums[2..] 找最小值
...
```

第 1 轮遍历 `nums[0..]` 时，其实已经**路过了** `nums[1..]` 和 `nums[2..]` 的所有元素。能不能顺便把后面的最小值也记住？

### 3.2 尝试：预计算 suffixMin 数组

```
suffixMin[i] = nums[i..] 中的最小值

关键：倒着计算！
  suffixMin[n-1] = nums[n-1]
  suffixMin[i] = min(nums[i], suffixMin[i+1])
```

```cpp
// O(N) 时间预计算
vector<int> suffixMin(n);                        // suffixMin[i] 表示区间 [i..n-1] 最小值
suffixMin[n-1] = nums[n-1];                     // 最右端区间只有一个元素
for (int i = n-2; i >= 0; i--) {               // 倒序推导每个位置的后缀最小值
  suffixMin[i] = min(nums[i], suffixMin[i+1]); // 当前后缀最小 = min(自己, 右侧后缀最小)
}
// 现在 suffixMin[i] 就是 nums[i..] 的最小值，O(1) 查询
```

> 💡 **老师提醒：** `suffixMin` 这种预处理只适合“原数组不再变化”的场景，和会频繁交换的排序过程一起用时要特别小心。

**为什么要倒着算？**

- 正着：知道 `min(nums[0..])` **推不出** `min(nums[1..])`（信息不足）
- 倒着：知道 `min(nums[1..])` **能推出** `min(nums[0..])` = `min(nums[0], min(nums[1..]))`

> 这个「倒着推」的思路就是**前缀和/后缀和技巧**的核心，后面会反复用到。

### 3.3 为什么最终还是不行？

有了 `suffixMin` 数组，我 O(1) 就知道 `nums[i..]` 的最小值了。但问题是：

> **排序会修改 nums 数组！** 一旦做了交换操作，`nums` 变了，之前算好的 `suffixMin` 就全废了。

```
原数组：[5, 3, 1, 4, 2]
suffixMin：[1, 1, 1, 2, 2]

第 1 轮用 suffixMin[0]=1，找到最小值 1，和 5 交换
nums 变成 [1, 3, 5, 4, 2] ← nums 变了！

但 suffixMin 还是基于旧的 nums 算的，已经不准了 ❌
```

每次交换后都要重新算 suffixMin → 又变回 O(N²) 了。

### 3.4 结论

> **选择排序的 O(N²) 复杂度无法优化。**

但这个思考过程非常有价值：

| 尝试的方法 | 遇到的问题 | 后面会学到的解决方案 |
|-----------|-----------|-------------------|
| 预计算 suffixMin | nums 变化后 suffixMin 失效 | **线段树**可以 O(logN) 动态维护 |
| 倒着计算 suffixMin | 本身是个有价值的技巧 | **前缀和/后缀和**技巧 |

**核心心法**：优化算法的方法就是**找到冗余计算，尝试消除它**。这个思路在任何算法题中都适用。

## 四、选择排序的不稳定性能修复吗？

问题出在「交换」——最小值和前面的元素交换时，可能跨越相同值的元素。

一种修复思路：不用交换，改用**插入**。把最小值从原位置「拔出来」，然后「插入」到正确位置，其他元素依次后移。

```
[2', 2''', 2'', 1]

找到最小值 1，不交换，而是把 1 拔出来，前面的元素依次后移：
  [_, 2', 2''', 2'']  → 把 1 放到最前面
  [1, 2', 2''', 2'']  

2 的相对顺序保持了 ✓
```

但这样每次插入要移动 O(N) 个元素，总复杂度还是 O(N²)。不过至少变稳定了。

> 这其实就引出了**插入排序**的思路——后面会学到。

## 五、关键概念速查表

| 概念 | 说明 |
|------|------|
| **选择排序** | 每次选最小的放到前面，O(N²)，不稳定 |
| **sortedIndex** | 已排序和未排序的分割线 |
| **冗余计算** | 优化算法的关键——找到重复做的工作 |
| **suffixMin** | 后缀最小值数组，倒着计算 |
| **数组修改导致预计算失效** | 选择排序无法优化的根本原因 |
| **交换导致不稳定** | 跨越相同元素的交换会打乱相对顺序 |

## 六、后面的排序算法要带着的问题

学后面的每种排序算法时，思考这两个问题：

1. **它和选择排序的本质区别是什么？** 选择排序每轮只确定一个元素的位置，有没有办法一轮做更多的事？
2. **它凭什么能把复杂度降到 O(N²) 以下？** 它避免了选择排序中的哪些冗余计算？

带着这两个问题学，理解会深刻得多。
