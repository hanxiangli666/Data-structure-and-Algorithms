# 计数排序：全新的排序原理 学习笔记

> **前置知识**：排序算法关键指标  
> **核心认知**：计数排序**不比较元素大小**，而是利用**数组索引的有序性**完成排序。是一种线性时间的非比较排序  
> **适用条件**：元素是整数，且元素的取值范围（max - min）不能太大

---

## 一、最简单的理解

> 统计每种元素出现的次数，然后按顺序填回去。

```
nums = [2, 0, 2, 1, 1, 0]

统计：0 出现 2 次，1 出现 2 次，2 出现 2 次

按顺序填：
  填 2 个 0 → [0, 0, ...]
  填 2 个 1 → [0, 0, 1, 1, ...]
  填 2 个 2 → [0, 0, 1, 1, 2, 2]

排序完成！
```

**类比**：你有一堆硬币（1 元、5 角、1 角），要按面值排序。最简单的方法不是一个个比较，而是分三堆数一数，然后按「1 角→5 角→1 元」的顺序排好。

## 二、简单版代码（LeetCode 75）

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
vector<int> coins = {0, 2, 1};                    // 三枚硬币面值
vector<int> cnt(3, 0);                            // 三个盒子分别统计 0/1/2
for (int c : coins) cnt[c]++;                     // 把硬币放进对应盒子计数
```

> 这个小例子是“按值计数”，下面代码就是把计数结果重新写回原数组，完成排序。

```cpp
// nums 只包含 0, 1, 2 三种元素
void sortColors(vector<int>& nums) {                    // ⚠️ 引用参数：原数组会被直接改写
  vector<int> count(3, 0);                            // count[i] = 元素 i 出现的次数
  for (int x : nums) count[x]++;                      // 统计每个值出现次数

  int idx = 0;                                        // 写回数组的位置指针
  for (int val = 0; val < 3; val++) {                // 按值从小到大填充
    for (int i = 0; i < count[val]; i++) {         // 连续填 count[val] 次
      nums[idx++] = val;                          // 写回当前值
        }
    }
}
```

> 💡 **老师提醒：** 这个版本依赖“值域固定且很小”，一旦值域变大就要切到通用版写法。

简单直观，但这个简化版有两个问题：
1. 不能处理负数
2. 不是稳定排序（相同元素的原始相对位置丢失了）

## 三、通用计数排序的三个关键技巧

### 3.1 技巧一：偏移量处理负数

```
nums = [-3, 4, -1, 2, 1]
最小值 = -3

映射函数：f(x) = x - min = x + 3
  -3 → 0
  -1 → 2
   1 → 4
   2 → 5
   4 → 7

现在所有索引都是非负的了 ✓
```

**关键要求**：映射函数必须**保持大小关系**。`x + 3` 满足：如果 a > b，则 a+3 > b+3 ✓。

### 3.2 技巧二：累加 count 数组确定最终位置

简单版直接按计数填充，无法保持稳定性。通用版用**累加 count 数组**来精确定位每个元素的位置。

```
nums = [2, 3, 1, 0, 2, 2, 3, 0]

步骤 1：统计
  count = [2, 1, 3, 2]
  ← 0 出现 2 次，1 出现 1 次，2 出现 3 次，3 出现 2 次

步骤 2：累加
  count = [2, 3, 6, 8]

步骤 3：累加后的 count 的含义——
  count[0] = 2 → 元素 0 在排序后占据索引 [0, 2)
  count[1] = 3 → 元素 1 在排序后占据索引 [2, 3)
  count[2] = 6 → 元素 2 在排序后占据索引 [3, 6)
  count[3] = 8 → 元素 3 在排序后占据索引 [6, 8)

验证排序结果：[0, 0, 1, 2, 2, 2, 3, 3]  ✓
```

**类比**：就像排队取号。count 累加后告诉你「元素 2 排在第 4 到第 6 号位」，你直接把它们放到对应位置就行了。

### 3.3 技巧三：倒序遍历保证稳定性

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
for (int i = 2; i >= 0; i--) {                    // 从队尾往前点名
  // 把同号码同学放到该号码最后一个空位
}
```

> 上面的例子强调“从后往前放”，下面代码就是稳定计数排序里最关键的倒序填充步骤。

```cpp
// 从后往前遍历原数组
for (int i = n - 1; i >= 0; i--) {
  int idx = nums[i] + offset;                   // 映射到 count 下标
  sorted[count[idx] - 1] = nums[i];             // 放到该值当前“最后空位”
  count[idx]--;                                 // 下一个同值元素向前放
}
```

> 💡 **老师提醒：** 这里若改成正序遍历，很多场景下相同元素顺序会被打乱，稳定性就没了。

> ✅ 你已经抓住“倒序遍历保稳定”这个高频考点了。

**为什么倒序？**

累加后的 count 存的是**末尾位置**，从后往前遍历 + 从末尾向前填充 = 相同元素的相对顺序保持不变 = **稳定排序**。

```
nums = [2a, 3, 2b]   ← 两个 2 标记为 a 和 b

累加后 count[2] = 2（元素 2 占据索引 [0, 2)）

倒序遍历：
  i=2: nums[2]=2b → sorted[count[2]-1] = sorted[1] = 2b, count[2]--
  i=1: nums[1]=3  → sorted[count[3]-1] = sorted[2] = 3,  count[3]--
  i=0: nums[0]=2a → sorted[count[2]-1] = sorted[0] = 2a, count[2]--

sorted = [2a, 2b, 3]  ← 2a 在 2b 前面，顺序保持 ✓
```

## 四、完整通用代码

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
int minVal = -2;                                   // 最小号码
int offset = -minVal;                              // 偏移后最小值映射到 0
int idx = (-1) + offset;                           // -1 这个号码映射到非负下标
```

> 这个例子对应“负数映射到非负下标”，下面完整代码在此基础上补齐计数、前缀和、稳定回填全过程。

```cpp
void countingSort(vector<int>& nums) {                  // ⚠️ 引用参数：会直接修改原数组
  if (nums.empty()) return;                           // 边界：空数组直接返回
    
    // 1. 找最大最小值，计算偏移量
  int minVal = *min_element(nums.begin(), nums.end()); // 最小值
  int maxVal = *max_element(nums.begin(), nums.end()); // 最大值
  int offset = -minVal;                                // 偏移量，处理负数
  int range = maxVal - minVal + 1;                    // 值域长度

    // 2. 统计每个元素出现的次数
  vector<int> count(range, 0);                        // 计数数组
  for (int x : nums) {                                // 遍历原数组计数
    count[x + offset]++;                            // 映射后累加频次
    }

    // 3. 累加 count 数组
  for (int i = 1; i < range; i++) {                   // 做前缀和
    count[i] += count[i - 1];                       // 转成“结束位置”信息
    }

    // 4. 倒序遍历原数组，填入 sorted 数组（保证稳定性）
  vector<int> sorted(nums.size());                    // 临时有序数组
  for (int i = nums.size() - 1; i >= 0; i--) {       // 倒序回填，保证稳定
    int idx = nums[i] + offset;                    // 当前值映射下标
    sorted[count[idx] - 1] = nums[i];              // 放到目标位置
    count[idx]--;                                   // 同值下一个位置前移
    }

    // 5. 复制回原数组
  nums = sorted;                                      // 最终结果写回
}
```

> 💡 **老师提醒：** `range` 很大时会占用大量内存，做题前先估算值域，再决定是否用计数排序。

**C++ 知识点**：
- `*min_element(begin, end)`：返回范围内的最小值
- `*max_element(begin, end)`：返回范围内的最大值

## 五、时空复杂度

设 N = 数组长度，R = max - min（元素范围）。

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | **O(N + R)** | 遍历 nums O(N) + 遍历 count O(R) |
| 空间 | **O(N + R)** | sorted 数组 O(N) + count 数组 O(R) |
| 原地 | ❌ | 需要 sorted 和 count 两个辅助数组 |
| 稳定 | **✅** | 倒序遍历保证 |

**什么时候用计数排序？**

- ✅ 元素范围 R 不大（比如成绩 0~100，颜色 0~2）
- ✅ 元素是整数（或可以映射成整数）
- ❌ 元素范围 R 极大（比如 [0, 10⁹]）→ count 数组太大，内存爆炸
- ❌ 元素是浮点数或复杂对象且无法合理映射

## 六、非比较排序的本质

### 6.1 什么是非比较排序？

之前学的所有排序（选择、冒泡、插入、快排、归并、堆排序）都有 `if (nums[i] > nums[j])` 这样的**比较操作**。

计数排序的代码中**没有比较任何两个元素的大小**。

### 6.2 那它靠什么排序？

> **靠数组索引的天然有序性。**

把元素映射成 count 数组的索引后，`count[0], count[1], count[2], ...` 这些索引**天然从小到大**。按索引顺序处理，就等于按大小顺序排列了。

**类比**：你有一堆信件要按邮编排序。你不需要两两比较邮编大小，只要准备好编号 00000~99999 的格子，把每封信扔到对应格子里，然后从格子 00000 开始依次取出来，就排好序了。

### 6.3 为什么不能用哈希表代替 count 数组？

哈希表的键**没有顺序**（`unordered_map`），你无法按键的大小顺序遍历。而 count 数组的索引**天然有序**（0, 1, 2, 3, ...），这正是计数排序能工作的根基。

### 6.4 比较排序 vs 非比较排序

| 对比 | 比较排序 | 非比较排序（计数排序） |
|------|---------|---------------------|
| 核心操作 | 比较两个元素的大小 | 把元素映射到有序的索引 |
| 理论下界 | **O(N log N)**（不可能更快） | **O(N)**（线性！） |
| 通用性 | 任何可比较的数据 | 只适用于可映射成整数索引的数据 |
| 代表算法 | 快排、归并、堆排序 | 计数排序、基数排序、桶排序 |

> 比较排序的理论下界是 O(N log N)，这是信息论证明的极限。非比较排序绕过了这个限制，所以能达到 O(N)。

## 七、关键概念速查表

| 概念 | 说明 |
|------|------|
| **计数排序** | 统计每个元素出现次数 → 累加确定位置 → 填充有序数组 |
| **count 数组** | `count[i]` = 值为 i 的元素出现次数 |
| **累加 count** | 累加后 `count[i]` = 值 ≤ i 的元素总数 = 值为 i 的结束位置 |
| **偏移量 offset** | `offset = -min`，把负数映射成非负索引 |
| **倒序遍历** | 从后往前填充 sorted 数组，保证**稳定性** |
| **非比较排序** | 不比较元素大小，利用索引有序性排序 |
| **O(N + R)** | N = 数组长度，R = max - min = 元素范围 |

## 八、排序算法完整全景图

```
排序算法
├── 比较排序（理论下界 O(N log N)）
│   ├── O(N²)：选择、冒泡、插入
│   ├── 突破 N²：希尔排序
│   └── O(N log N)：快排（前序）、归并（后序）、堆排序（二叉堆）
│
└── 非比较排序（可以突破 O(N log N)）
    ├── 计数排序  O(N+R)  ← 你在这里
    ├── 基数排序  O(N×k)  ← 后面可能会学
    └── 桶排序    O(N+k)  ← 后面可能会学
```
