# 快速排序：妙用二叉树前序位置 学习笔记

> **核心认知**：快速排序 = 二叉树的前序遍历。**先把一个元素排好序（partition），再递归排剩下的**  
> **本篇定位**：理解整体思路和复杂度分析，具体代码实现后面再学

## 一、快排的核心思路

### 1.1 一句话描述

> **选一个元素（pivot），把它放到正确位置，然后递归搞定左右两部分。**

### 1.2 分步图解

```
第 1 步：选 4 作为 pivot
  [4, 1, 7, 2, 8, 5, 3, 6, 9]
   ↑
  pivot

第 2 步：partition——把小于 4 的放左边，大于 4 的放右边
  [3, 1, 2,| 4 |, 8, 5, 7, 6, 9]
   < 4      pivot     > 4
  4 已经在正确位置了！✓

第 3 步：递归处理左半部分 [3, 1, 2] 和右半部分 [8, 5, 7, 6, 9]

左半部分选 3 做 pivot：
  [2, 1,| 3 |]
  3 归位 ✓

右半部分选 8 做 pivot：
  [5, 7, 6,| 8 |, 9]
  8 归位 ✓

继续递归... 直到所有元素都归位

最终：[1, 2, 3, 4, 5, 6, 7, 8, 9] ✓
```

### 1.3 和选择排序的对比

| 选择排序 | 快速排序 |
|---------|---------|
| 每轮找到最小值，放到最左边 | 每轮选一个 pivot，放到正确位置 |
| **只搞定 1 个位置** | **搞定 1 个位置，同时把剩余元素分成两组** |
| 线性扫描 O(N) × N 轮 = O(N²) | 每层 O(N) × log N 层 = O(N log N) |

快排的精髓在于：partition 操作不仅把 pivot 排好序，还**顺便把数组分成了两半**——左边都小于 pivot，右边都大于 pivot。这为后续递归排序提供了极大的便利。

## 二、快排 = 二叉树的前序遍历

### 2.1 代码框架对比

```cpp
// 二叉树前序遍历
void traverse(TreeNode* root) {
    if (root == nullptr) return;
    // 【前序位置】—— 处理当前节点
    traverse(root->left);
    traverse(root->right);
}

// 快速排序
void sort(int nums[], int lo, int hi) {
    if (lo >= hi) return;
    // 【前序位置】—— 把 nums[p] 排好序
    int p = partition(nums, lo, hi);
    sort(nums, lo, p - 1);    // 递归排左半部分
    sort(nums, p + 1, hi);    // 递归排右半部分
}
```

**完全一样的结构！**

- `traverse(root)` ↔ `sort(nums, lo, hi)`
- 前序位置处理当前节点 ↔ 前序位置做 partition
- `traverse(left)` ↔ `sort(lo, p-1)`
- `traverse(right)` ↔ `sort(p+1, hi)`

### 2.2 递归树的样子

```
              sort([4,1,7,2,8,5,3,6,9])
              partition → p=4 归位
             /                          \
    sort([3,1,2])                sort([8,5,7,6,9])
    partition → 3 归位           partition → 8 归位
     /       \                   /            \
 sort([2,1]) sort([])     sort([5,7,6])   sort([9])
 2 归位                    5 归位
  /    \                   /     \
sort([1]) sort([])    sort([]) sort([7,6])
                               6 归位
                               /    \
                          sort([]) sort([7])
```

每个树节点 = 一次 partition 操作 = 一个元素归位。当所有节点处理完毕，整个数组就排好序了。

## 三、partition 函数是什么？

partition 是快排的**核心**——把 pivot 放到正确位置，使得左边都 ≤ pivot，右边都 > pivot。

```
输入：[4, 1, 7, 2, 8, 5, 3, 6, 9]，pivot = 4
输出：[3, 1, 2, 4, 8, 5, 7, 6, 9]，返回 p = 3（pivot 在索引 3）
```

具体实现用**双指针**技巧，现在不需要掌握细节，后面学完双指针技巧后再看会非常简单。

## 四、时间复杂度

### 4.1 理想情况：O(N log N)

```
              每一层的元素总数 ≈ N
              ┌───────────────────────────┐
第 1 层：     [4, 1, 7, 2, 8, 5, 3, 6, 9]    N 个元素
              ┌─────────────┴─────────────┐
第 2 层：  [3, 1, 2]              [8, 5, 7, 6, 9]   还是 ≈ N 个
              ┌──┴──┐            ┌────┴────┐
第 3 层：  [2,1]  []          [5,7,6]      [9]      还是 ≈ N 个
           ...
```

**关键观察**：
- 每一层的数组元素总数 ≈ N（因为只是被分成了不同的组）
- 理想情况下每次 pivot 恰好在中间，树高 = log N

> **总复杂度 = 每层 O(N) × 层数 O(log N) = O(N log N)**

### 4.2 最坏情况：O(N²)

如果每次 pivot 恰好是最小/最大元素（比如数组已经有序），那么每次分组都极度不平衡：

```
第 1 层：[1, 2, 3, 4, 5]   pivot=1，左边 0 个，右边 4 个
第 2 层：   [2, 3, 4, 5]   pivot=2，左边 0 个，右边 3 个
第 3 层：      [3, 4, 5]   pivot=3 ...
第 4 层：         [4, 5]
第 5 层：            [5]

树高 = N → 总复杂度 O(N²)
```

这棵递归树退化成了链表！和二叉搜索树按顺序插入退化成链表一模一样。

**解决方案**：**随机选择 pivot**（而不是固定选第一个），退化的概率极低。具体方法后面学。

### 4.3 复杂度总结

| 情况 | 复杂度 | 发生条件 |
|------|--------|---------|
| **最好** | O(N log N) | 每次 pivot 在中间 |
| **平均** | **O(N log N)** | 随机数据 |
| **最坏** | O(N²) | pivot 总是最小/最大（可用随机化避免） |

## 五、空间复杂度

快排是**原地排序**，不需要额外数组。

但递归调用会占用**函数调用栈**：

| 情况 | 栈深度 |
|------|--------|
| 理想（平衡树） | O(log N) |
| 最坏（退化链表） | O(N) |

---

## 六、稳定性

❌ **不稳定**。

partition 过程中会交换不相邻的元素，可能打乱相同元素的相对顺序。

```
[4, 4', 1]  pivot = 4

partition 后可能变成：
[1, 4', 4]  → 4 和 4' 的顺序被交换了 ❌
```

---

## 七、快排的完整指标

| 指标 | 快速排序 |
|------|---------|
| **平均时间** | **O(N log N)** |
| 最坏时间 | O(N²)（随机化后极少发生） |
| 空间 | O(log N)（递归栈） |
| 原地排序 | ✅ |
| 稳定 | ❌ |
| **实际性能** | **常数因子小，通常是最快的通用排序** |

> C++ 标准库的 `sort()` 底层就是基于快排的（结合了插入排序和堆排序的混合策略，叫 IntroSort）。

---

## 八、为什么快排比 O(N²) 排序快？

回答上一篇留下的问题：「O(N²) 排序每轮只搞定 1 个元素，凭什么快排能降到 O(N log N)？」

> **因为快排的 partition 不仅把 1 个元素排好序，还把整个数组分成了两个更小的子问题。**

O(N²) 排序：每轮遍历 N 个元素，只搞定 1 个 → 需要 N 轮 → N × N = N²

快排：每轮遍历 N 个元素（partition），但搞定 1 个元素的同时把问题**规模减半** → log N 层 × 每层 N = N log N

**类比**：
- O(N²) 排序像是在一本乱序的书里，每次翻完整本书只找到 1 页放对位置
- 快排像是每次翻完书后，不仅放对 1 页，还顺手把书分成了「前半本」和「后半本」，后续只需要分别整理

## 九、关键概念速查表

| 概念 | 说明 |
|------|------|
| **快速排序** | 选 pivot → partition → 递归左右 |
| **pivot** | 切分元素，partition 后它在正确位置 |
| **partition** | 把小于 pivot 的放左边，大于的放右边 |
| **前序位置** | partition 发生在递归之前（先处理当前，再递归子问题） |
| **随机化 pivot** | 避免最坏情况退化 |
| **O(N log N)** | 每层 O(N) × 树高 O(log N) |
| **IntroSort** | C++ `sort()` 的底层，快排+插入排序+堆排序的混合 |
