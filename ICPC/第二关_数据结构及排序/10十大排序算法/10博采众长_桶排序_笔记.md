# 桶排序：博采众长 学习笔记

> **前置知识**：计数排序、插入排序、归并排序  
> **核心认知**：桶排序 = 分桶（类似归并的拆分）+ 桶内排序（任意算法）+ 合并（依次拼接）。是计数排序的泛化版本  
> **精髓**：把大数组拆成小数组分别排序，总工作量比直接排整个数组更少（数学保证）

---

## 一、桶排序的三步走

```
第一步：分桶
  把元素按照映射函数分配到 k 个桶中

第二步：桶内排序
  对每个桶内的元素分别排序（用插入排序或递归桶排序）

第三步：合并
  把所有桶的元素按顺序拼接起来 → 排序完成
```

```
原数组：[29, 25, 3, 49, 9, 37, 21, 43]

分成 5 个桶（按 nums[i]/10 分桶）：
  桶 0: [3, 9]
  桶 1: []
  桶 2: [29, 25, 21]
  桶 3: [37]
  桶 4: [49, 43]

桶内排序：
  桶 0: [3, 9]
  桶 2: [21, 25, 29]
  桶 3: [37]
  桶 4: [43, 49]

依次拼接：[3, 9, 21, 25, 29, 37, 43, 49]  ✓
```

## 二、为什么分开排比整体排更快？

### 2.1 数学直觉

假设数组长度 N = 100，用选择排序（O(N²)）：

- 整体排序：100² = **10000** 次操作
- 分成 10 个桶，每桶 10 个元素：10 × 10² = **1000** 次操作

**分开排只需要 1/10 的工作量！**

### 2.2 严格数学关系

把 N 分成 k 份：N = n₁ + n₂ + ... + nₖ

```
整体排：N² 
分开排：n₁² + n₂² + ... + nₖ²

数学保证：N² ≥ n₁² + n₂² + ... + nₖ²（当 k > 1 时严格大于）
```

**类比**：一个大正方形的面积，一定大于沿边排列的若干小正方形面积之和。

### 2.3 极端情况

| 桶数 k | 每桶元素数 | 效果 |
|--------|-----------|------|
| k = 1 | N 个 | 退化为直接排序，O(N²) |
| k = N | 1 个 | 每桶只有 1 个元素 = 计数排序，O(N) |
| 1 < k < N | N/k 个 | 介于两者之间 |

> 桶越多，效率越高（趋近 O(N)）。桶排序是计数排序的泛化：计数排序 = 桶数等于元素范围的桶排序。

## 三、关键问题：用除法分桶，不能用取模！

### 3.1 错误方式：取模

```
nums = [0, 1, 2, ..., 99]，k = 10

取模分桶（nums[i] % 10）：
  桶 0: [0, 10, 20, 30, ...]
  桶 1: [1, 11, 21, 31, ...]
  桶 2: [2, 12, 22, 32, ...]
  ...

桶内排序后：每个桶有序 ✓
但桶之间的元素交错了！不能简单拼接 ✗
合并需要用归并算法 → O(N log k)，失去了 O(N) 合并的优势
```

### 3.2 正确方式：除法（向下取整）

```
除法分桶（nums[i] / 10）：
  桶 0: [0, 1, 2, ..., 9]      ← 0~9
  桶 1: [10, 11, 12, ..., 19]  ← 10~19
  桶 2: [20, 21, 22, ..., 29]  ← 20~29
  ...

桶内排序后：每个桶有序 ✓
且桶 0 的所有元素 < 桶 1 的所有元素 < 桶 2 的 ...
直接依次拼接就行！O(N) ✓
```

**关键区别**：除法保证了「桶之间有序」（桶 0 的所有元素 ≤ 桶 1 的所有元素），取模做不到这一点。

## 四、代码实现（插入排序版）

```cpp
// 插入排序（复用之前学过的）
void insertSort(vector<int>& nums) {
    for (int i = 1; i < nums.size(); i++) {
        for (int j = i; j > 0 && nums[j] < nums[j-1]; j--) {
            swap(nums[j], nums[j-1]);
        }
    }
}

// 桶排序
void bucketSort(vector<int>& nums, int bucketCount) {
    if (nums.size() <= 1) return;

    // 1. 找最大最小值，计算偏移量和桶大小
    int minVal = *min_element(nums.begin(), nums.end());
    int maxVal = *max_element(nums.begin(), nums.end());
    int offset = -minVal;
    int bucketSize = (maxVal - minVal) / bucketCount + 1;

    // 2. 初始化桶
    vector<vector<int>> buckets(bucketCount);

    // 3. 分配元素到桶中（除法映射）
    for (int num : nums) {
        int index = (num + offset) / bucketSize;
        buckets[index].push_back(num);
    }

    // 4. 对每个桶排序
    for (auto& bucket : buckets) {
        insertSort(bucket);
    }

    // 5. 合并所有桶（直接依次拼接）
    int idx = 0;
    for (auto& bucket : buckets) {
        for (int num : bucket) {
            nums[idx++] = num;
        }
    }
}
```

**`bucketSize` 的含义**：每个桶负责的值域范围大小。比如值域 [0, 99]，10 个桶，`bucketSize = 10`，桶 0 负责 [0,9]，桶 1 负责 [10,19]，...

## 五、代码实现（递归版）

把 `insertSort(bucket)` 换成 `bucketSort(bucket, bucketCount)`，递归地对每个桶再做桶排序。

```cpp
void bucketSort(vector<int>& nums, int bucketCount) {
    // base case：检查是否已经有序
    bool sorted = true;
    for (int i = 1; i < nums.size(); i++) {
        if (nums[i] < nums[i-1]) { sorted = false; break; }
    }
    if (sorted) return;   // 已有序，结束递归

    // 后续逻辑和插入排序版完全一样，只是第 4 步改为递归
    // ...
    for (auto& bucket : buckets) {
        bucketSort(bucket, bucketCount);   // 递归！
    }
    // ...
}
```

**注意 base case**：必须检查数组是否已有序，否则如果所有元素都相同（全进同一个桶），会无限递归。

## 六、桶排序 = 归并 + 计数的融合

```
桶排序的思想中同时包含了：

归并排序的影子：
  把大数组拆成小数组 → 分别排序 → 合并
  
计数排序的影子：
  桶数 = 元素范围时 → 每桶只有 1 个元素 → 就是计数排序
  
插入排序的影子：
  桶内排序用的就是插入排序
```

## 七、三个指标分析

| 指标 | 桶排序 |
|------|--------|
| **均摊时间** | **O(N + k)**（k = 桶数） |
| **最坏时间** | O(N²)（所有元素进同一个桶 → 退化为插入排序） |
| **空间** | O(N + k)（桶数组 + 所有桶内的元素） |
| **原地排序** | ❌ |
| **稳定性** | ✅（如果桶内排序算法是稳定的） |

## 八、排序算法终极全景图

```
排序算法大家族

比较排序（理论下界 O(N log N)）：
  ├── O(N²)：选择 → 冒泡 → 插入
  ├── 突破 N²：希尔排序
  └── O(N log N)：快排（前序）、归并（后序）、堆排序（二叉堆）

非比较排序（可达线性 O(N)）：
  ├── 计数排序：每个值一个槽位，O(N+R)
  └── 桶排序：分桶+桶内排序+合并，O(N+k)  ← 你在这里
      └── 计数排序是桶排序的特例（每桶 1 个元素）

C++ 标准库：
  sort()         → IntroSort（快排+堆排序+插入排序的混合）
  stable_sort()  → 基于归并排序
  
竞赛中：99% 直接用 sort()，偶尔用计数排序/桶排序处理特殊场景
```

## 九、关键概念速查表

| 概念 | 说明 |
|------|------|
| **桶排序** | 分桶 → 桶内排序 → 合并 |
| **除法分桶** | `(num + offset) / bucketSize`，保证桶间有序 |
| **不能用取模** | 取模会打乱桶间顺序，无法 O(N) 合并 |
| **bucketSize** | `(max - min) / k + 1`，每个桶负责的值域范围 |
| **桶数越多越快** | k 越大 → 每桶元素越少 → 越接近 O(N) |
| **k = N** | 退化为计数排序，O(N) |
| **k = 1** | 退化为直接排序，O(N²) |
| **递归版 base case** | 必须检查是否已有序，防止无限递归 |
