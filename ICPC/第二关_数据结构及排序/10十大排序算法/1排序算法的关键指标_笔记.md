# 排序算法的关键指标 学习笔记

> **前置知识**：数组基础、时空复杂度  
> **核心认知**：评价一个排序算法好不好，看三个指标——**时空复杂度**、**稳定性**、**是否原地排序**  
> **本篇定位**：后面学每种排序算法时，都会用这三个指标来分析它的优劣

---

## 一、为什么要学排序算法？

竞赛和面试中很少直接让你「手写快排」，但排序的思想渗透在大量算法题中。而且理解不同排序算法的特点，能帮你在需要排序时选对工具。

C++ 中直接用 `sort()` 就能排序，但你得知道：

- 它的复杂度是多少？
- 它是稳定排序吗？（不是！C++ 的 `sort` 是不稳定的，`stable_sort` 才是稳定的）
- 什么场景下需要稳定排序？

---

## 二、指标一：时空复杂度

### 2.1 时间复杂度

排序算法的时间复杂度通常分三种情况讨论：

| 情况 | 含义 | 举例 |
|------|------|------|
| **最好情况** | 输入数据对算法最友好 | 数组已经有序 |
| **最坏情况** | 输入数据对算法最不友好 | 数组完全逆序 |
| **平均情况** | 随机输入的期望复杂度 | 通常最有参考价值 |

常见排序算法的时间复杂度范围：

```
O(N²)    ← 慢：冒泡、选择、插入排序
O(N logN) ← 快：归并、快排、堆排序
O(N)     ← 最快：计数排序、桶排序（有条件限制）
```

### 2.2 空间复杂度

排序过程中需要多少额外内存。

- O(1)：原地排序，不需要额外数组
- O(N)：需要一个和原数组一样大的辅助数组
- O(log N)：需要递归栈空间

> 时间和空间都越小越好，但往往需要**权衡**——有些算法时间快但空间大，有些省空间但时间慢。

## 三、指标二：排序稳定性

### 3.1 定义

> **稳定排序**：相同值的元素，排序后**相对顺序不变**。  
> **不稳定排序**：相同值的元素，排序后相对顺序**可能改变**。

### 3.2 用例子理解

假设有三张牌：`5♠ 3♥ 5♦`，按数值排序。

```
稳定排序结果：  3♥  5♠  5♦    ← 两个 5 的相对顺序没变（♠ 还在 ♦ 前面）
不稳定排序可能：3♥  5♦  5♠    ← 两个 5 的顺序被交换了
```

如果只看数值，两种结果都是「排好序的」。但如果你关心相同数值元素的原始顺序，就需要稳定排序。

### 3.3 稳定性什么时候重要？

**类比**：你有一份订单表，已经按**日期排好序**了：

```
日期          用户ID
2020-01-01    1002
2020-01-02    1001
2020-01-03    1002
2020-02-01    1001
2020-02-02    1002
2020-02-03    1001
```

现在你想**按用户 ID 再排一次**，让同一个用户的订单聚在一起。

#### 用稳定排序：

```
日期          用户ID
2020-01-02    1001    ← 1001 的三条记录
2020-02-01    1001       内部仍然按日期有序 ✓
2020-02-03    1001

2020-01-01    1002    ← 1002 的三条记录
2020-01-03    1002       内部仍然按日期有序 ✓
2020-02-02    1002
```

因为稳定排序保持了相同 ID 记录的原始相对位置，所以之前按日期排的序**不会被破坏**。

#### 用不稳定排序：

```
日期          用户ID
2020-02-03    1001    ← 1001 的记录聚在一起了
2020-01-02    1001       但内部日期顺序被打乱了 ✗
2020-02-01    1001

...
```

之前按日期排的序**白费了**。

### 3.4 C++ 中的注意事项

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
#include <algorithm>                        // 引入排序相关工具
vector<int> queueNum = {3, 1, 3};          // 想象成 3 号、1 号、3 号同学排队
sort(queueNum.begin(), queueNum.end());    // 按号码重新排队（默认不保证同号先后）
stable_sort(queueNum.begin(), queueNum.end()); // 稳定排队：同号同学保持原先顺序
```

> 上面的例子是在“叫号排队”场景里比较 `sort` 和 `stable_sort`，下面这段真实代码做的是同一件事，只是数据换成了常见的整数数组。

```cpp
#include <algorithm>                       // 提供 sort / stable_sort

vector<int> arr = {5, 3, 5, 1, 5};        // 待排序数组，包含重复元素 5

sort(arr.begin(), arr.end());              // ❌ 不稳定排序：相同元素相对顺序可能变化
stable_sort(arr.begin(), arr.end());       // ✅ 稳定排序：相同元素相对顺序保持不变
```

> 💡 **老师提醒：** 你在做“多关键字排序”时（比如先按日期再按用户），第二次排序要优先考虑稳定排序，否则第一次排好的信息会被打乱。

如果你只是排 `int` 数组，稳定不稳定无所谓。但如果排的是结构体/pair，且关心相同键的元素顺序，就要用 `stable_sort`。

## 四、指标三：是否原地排序

### 4.1 定义

> **原地排序（In-place）**：排序过程中只使用 **O(1)** 额外空间，直接在原数组上操作。  
> **非原地排序**：需要 **O(N)** 或更多的额外空间（比如一个辅助数组）。

### 4.2 代码对比

**第一步：最简单的例子（用你能想到的最日常的比喻）**

```cpp
vector<int> box = {4, 2, 3};               // 一个装数字的小盒子
vector<int> helper = box;                  // 另外拿一个同样大小的盒子做辅助
swap(box[0], box[1]);                      // 也可以直接在原盒子里交换位置
```

> 上面这个小例子在对比“借助新盒子”和“直接原地调整”，下面的代码就是把这个思路放到排序函数层面来理解原地与非原地。

```cpp
// 非原地排序：需要额外数组
void sort(vector<int>& nums) {             // ⚠️ 引用参数：直接修改外部传入的数组
    vector<int> tmp(nums.size());          // O(N) 额外空间，开一个辅助数组
    // ... 排序过程用 tmp 辅助 ...        // 用空间换实现便利
}

// 原地排序：只在 nums 上操作
void sort(vector<int>& nums) {             // ⚠️ 仍是引用参数，操作的就是原数组
    // 只用几个临时变量（O(1) 额外空间）
    // ... 直接交换 nums 中的元素 ...      // 不申请同规模辅助数组
}
```

> 💡 **老师提醒：** 看到“原地排序”别只记定义，先问自己一句：有没有额外开和原数组同级别大小的容器。

> ✅ 你已经能用“空间是否额外占用”来判断排序风格了，这个判断非常专业。

### 4.3 原地排序的优势

数据量很大时，额外分配一个同样大的数组会很费内存。原地排序不需要额外数组，内存友好。

## 五、三个指标速查表

| 指标 | 好的标准 | 说明 |
|------|---------|------|
| **时间复杂度** | O(N log N) 或更低 | O(N²) 只适合小数据 |
| **空间复杂度** | O(1)（原地排序） | O(N) 需要额外数组 |
| **稳定性** | 稳定 | 保持相同元素的原始相对顺序 |

没有哪种排序算法在三个指标上都是最优的，只能根据具体场景选择：

- 要求速度快 → O(N log N) 的算法（快排、归并、堆排序）
- 要求稳定 → 归并排序、插入排序
- 要求省空间 → 堆排序、插入排序（原地）
- 数据量小 → 插入排序（常数小，实际很快）

## 六、后续排序算法预览

后面会学到的排序算法，以及它们在三个指标上的表现：

| 排序算法 | 平均时间 | 空间 | 稳定 | 特点 |
|---------|---------|------|------|------|
| 冒泡排序 | O(N²) | O(1) | ✅ | 最简单，但最慢 |
| 选择排序 | O(N²) | O(1) | ❌ | 简单，交换次数少 |
| 插入排序 | O(N²) | O(1) | ✅ | 小数据量或近乎有序时很快 |
| 归并排序 | O(N logN) | O(N) | ✅ | 稳定且快，但不原地 |
| 快速排序 | O(N logN) | O(logN) | ❌ | 实践中最快，但不稳定 |
| 堆排序 | O(N logN) | O(1) | ❌ | 原地且快，但不稳定 |
| 计数排序 | O(N+K) | O(K) | ✅ | 数据范围小时极快 |

> **竞赛中 99% 的情况直接用 `sort()`**，但理解这些算法的原理和特性对你理解更复杂的算法非常有帮助。

## 七、关键概念速查表

| 概念 | 说明 |
|------|------|
| **稳定排序** | 相同值元素的相对顺序不变 |
| **不稳定排序** | 相同值元素的相对顺序可能改变 |
| **原地排序** | O(1) 额外空间，直接操作原数组 |
| **非原地排序** | 需要 O(N) 或更多额外空间 |
| `sort()` | C++ 标准库排序，O(N logN)，**不稳定** |
| `stable_sort()` | C++ 稳定排序，O(N logN)，**稳定** |
| **O(N²)** | 冒泡、选择、插入（慢，适合小数据） |
| **O(N logN)** | 归并、快排、堆排（快，通用） |
