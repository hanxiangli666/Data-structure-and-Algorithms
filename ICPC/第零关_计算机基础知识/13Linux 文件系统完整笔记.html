01

告别盘符：大都会的中央广场

为什么 Linux 里没有 C盘 和 D盘？

如果你习惯了 Windows，你可能会觉得电脑是由一个个独立运作的“城堡”（C盘、D盘、E盘）组成的。但在 Linux 的世界里，这种概念完全不存在。

Linux 是一个被**高度统一规划的超级大都会**。这座城市只有一个绝对的中心起点——被称为 根目录 ( / ) 的中央广场。

在这个城市里，所有的街区（目录）和外部物资（外接硬盘、U盘），都必须通过一条马路，老老实实地接入到这个中央广场的辐射网络中。这就是 Linux 最核心的哲学：“万物皆文件，一树连所有”。

💡
直觉转换

不要再问“我的 D 盘去哪了”。在 Linux 里，你的第二块硬盘可能只是被规划成了城市东南角的一个叫 /data 的新开发区而已。

02

大都会区划指南

当服务崩溃或找不到命令时，你应该去哪个区？

作为市长，你不需要记住每一条街道的名字，但你必须清楚这座城市的**功能区划（Zoning）**。当你遇到问题时，直奔对应的区域：

🏛️
/etc (市政厅与档案馆)

存放着整个城市**所有的配置文件**。想要修改网络 IP、调整 Nginx 服务器的规则、改密码策略？来这里找文本文件。

🧰
/bin & /sbin (公共事务局与警卫处)

存放着城市运转的基础**命令与工具**（比如 ls, cp）。带 s 的 /sbin 存放的是只有高级管理员（sudo）才能动用的高危命令（如配网、重启）。

🏭
/var (工业区与垃圾处理厂)

Var 代表 Variable（可变的）。这里存放着城市里**不断变化的数据**。比如所有的排查错误必备的运行日志 /var/log，以及数据库文件。

🏡
/home (居民生活区)

普通市民（用户）的私人住宅区。比如 /home/alice。每个人的私人物品和个性化配置（如 ~/.config）都存在这里。

物流港口的货物交接

什么是“挂载 (Mount)”？

如果有外部的物资运到了城市（比如你插上了一个 1TB 的外接硬盘，在系统里显示为 /dev/sdb1），它不能直接在城市里使用。

你必须在城市里划出一块空地（通常是在 /mnt 或自动在 /media 下），然后建立一条连接。这个“建立连接，让外部硬盘变成城市里一个普通目录”的动作，就叫做 挂载 (Mount)。

04

建筑安保门禁法则

读懂天书般的 rwx 和 755。

在 Linux 城市里，每一栋建筑（文件/目录）都有极其严格的门禁系统。当你输入命令遇到 `Permission Denied`（权限被拒绝）时，就需要动用 chmod（改权限）和 chown（改归属）了。

门禁权限分三层，每层用三个字母 rwx 表示（r=读/看, w=写/改, x=执行/进入）：

  • 第一层（拥有者）：房主本人能干啥。
  • 第二层（群组）：房主的家人/同组人能干啥。
  • 第三层（其他人）：街上的陌生人能干啥。
权限变更指令 
# 1. 改变权限 (r=4, w=2, x=1)
# 用户=4+2+1(7), 组=4+0+1(5), 其他=4+0+1(5)
chmod 755 script.sh

# 2. 改变文件的归属者和归属组
chown alice:dev file.txt
大白话翻译

给 script.sh 这个脚本重新设置门禁密码：

房主拥有 7 (读4+写2+执行1) 的绝对权力；

同组人和陌生人只有 5 (读4+无0+执行1) 的权力，也就是**只能看和运行，绝对不准修改它！**


把 file.txt 这个文件的房产证，正式过户给叫 alice 的人，并划归到 dev 这个家族群组里。

🧑‍💻
普通开发者

市长！我刚往 /opt/myapp 里部署了代码，但程序启动时报错：日志写入失败，Permission Denied！

👑
首席规划师 (Root)

因为 /opt 这个地段是系统公产（拥有者是 root），你的服务进程没有写入（w）权限。你应该用 chown -R 你的服务名:你的组名 /opt/myapp 把那栋楼的产权交给你的程序！

05

任意门与传送牌

搞懂硬链接 (Hard Link) 与软链接 (Symlink)。

在规划城市时，你可能希望从不同的街区都能快速到达同一个设施。Linux 提供了两种魔法：

硬链接 (Hard Link)

相当于给同一个物理房间（底层数据区 inode）开了**两扇平等的真门**。无论你炸掉（删除）哪扇门，只要还有一扇门在，房间里的东西就完好无损。

局限：无法跨越不同的文件系统（不能建一扇门直通外接硬盘），且通常不能对目录使用。

软链接 (Symbolic Link)

生成命令加了 -s 参数。它就像在街上立了一块**“传送路牌（快捷方式）”**。路牌上写着“目标在 5 号街区”。

注意：它可以跨系统、跨目录！但如果原本那栋楼被强拆了，这块路牌就会变成“悬挂链接（死链）”，你点进去会报错说找不到文件。

市长的危机决策

当城市面临空间枯竭，你该如何排查？

服务器磁盘报警是日常最常见的危机之一。拯救城市，你需要熟练运用这两个空间雷达命令：df -h（看整体分区剩余）和 du -sh *（看具体目录谁最占地方）。

紧急突发：凌晨 3 点，系统疯狂报警显示“根目录 `/` 空间已满 100%”，导致所有数据库服务集体崩溃。作为紧急上线的首席规划师，你的第一反应排查动作应该是什么？
