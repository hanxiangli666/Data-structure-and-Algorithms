## 二、总体结构

Linux 文件系统以 `/`（根目录）为起点，所有文件和目录都挂在这棵树下。

```text
/
├── bin        基础命令
├── sbin       系统管理命令
├── boot       启动文件
├── dev        设备文件
├── etc        配置文件
├── home       普通用户家目录
├── lib        共享库
├── media      自动挂载点
├── mnt        手动挂载点
├── opt        可选软件
├── proc       进程/内核信息（虚拟）
├── root       root 用户家目录
├── run        运行时数据（内存）
├── srv        服务数据
├── sys        硬件与内核信息（虚拟）
├── tmp        临时文件
├── usr        用户程序和资源
└── var        可变数据（日志等）
```

## 三、目录详解（含典型使用场景）

### 3.1 /bin 和 /sbin

| 目录 | 说明 | 你最可能在什么情况下用到它 |
|------|------|----------------------------|
| /bin | 基础命令（ls/cp/cat/chmod） | 在终端做日常文件操作 |
| /sbin | 系统管理命令（多需 sudo） | 配网、改防火墙、系统维护 |

提示：很多现代发行版里 `/bin` 可能是指向 `/usr/bin` 的符号链接（后文会讲）。

### 3.2 /boot

- 作用：系统启动相关文件（内核、引导器）。
- 场景：排查启动问题、管理内核版本。

### 3.3 /dev

- 作用：设备文件入口（磁盘、终端等）。
- 场景：挂载新磁盘、查看块设备（如 `/dev/sda1`）。

### 3.4 /etc

- 作用：系统和服务的主配置目录。
- 场景：改 SSH、Nginx、DNS、本地 hosts。

### 3.5 /lib /lib64

- 作用：共享库，命令运行依赖。
- 场景：程序报“找不到库文件”时排查。

### 3.6 /media 和 /mnt

| 目录 | 作用 | 你最可能在什么情况下用到它 |
|------|------|----------------------------|
| /media | 自动挂载位置 | 插 U 盘后找文件 |
| /mnt | 手动挂载位置 | 运维手动 mount 测试盘/网络盘 |

### 3.7 /opt

- 作用：第三方可选软件安装目录。
- 场景：手工安装 IDE、商业软件或厂商包。

### 3.8 /proc

- 作用：内核动态暴露的进程/系统信息（虚拟文件系统）。
- 场景：查 CPU、内核版本、进程参数。

常用命令：

```bash
cat /proc/version
cat /proc/cpuinfo
```

### 3.9 /root

- 作用：root 用户家目录（不是根目录 `/`）。
- 场景：以 root 身份运维时的默认工作目录。

### 3.10 /run 和 /sys

| 目录 | 作用 | 场景 |
|------|------|------|
| /run | 运行时临时状态 | 排查服务运行时 pid/socket |
| /sys | 硬件与内核接口 | 查看或调节设备参数 |

提示：它们很多内容在内存中，重启后会变化或消失。

### 3.11 /srv

- 作用：服务对外提供的数据目录。
- 场景：自建 FTP/静态资源服务时集中放数据。

### 3.12 /tmp

- 作用：临时文件目录。
- 场景：程序中转文件、排查崩溃残留。

### 3.13 /usr

- 作用：用户空间程序与资源主目录。
- 场景：找已安装命令、桌面启动器、共享资源。

常见子目录：

```text
/usr/bin
/usr/sbin
/usr/local/bin
/usr/share
/usr/share/applications
```

### 3.14 /var

- 作用：可变数据（日志、缓存、队列）。
- 场景：排查服务异常、清理日志占用。

常看：

```text
/var/log/
/var/log/syslog
/var/log/nginx/
```

### 3.15 /home

- 作用：普通用户个人目录。
- 场景：个人文件、用户级配置、用户级命令。

常见结构：

```text
/home/alice/
├── .config/
├── .cache/
└── .local/bin/
```

## 四、挂载（Mount）到底是什么

### 4.1 什么是挂载

挂载就是“把一个存储设备（或分区、网络文件系统）的文件树，接到当前目录树的某个目录上”。

例如把 `/dev/sdb1` 挂到 `/mnt/data` 后，访问 `/mnt/data` 就是在访问这块盘的内容。

### 4.2 为什么 Linux 没有 C:/D: 盘符

Windows 用盘符分割空间（C:, D:）。Linux 用“单一目录树 + 挂载点”：

- 所有存储都归入 `/` 这棵树
- 不同设备通过 mount 挂到不同目录

这就是理解 `/media` 和 `/mnt` 的基础。

## 五、文件权限必备基础

### 5.1 rwxr-xr-- 是什么意思

以 `rwxr-xr--` 为例，共 10 位里常看后 9 位：

```text
rwx | r-x | r--
用户  组    其他人
```

含义：

- `r` 可读（4）
- `w` 可写（2）
- `x` 可执行（1）

所以：

- 用户 `rwx` = 4+2+1 = 7
- 组 `r-x` = 4+0+1 = 5
- 其他 `r--` = 4+0+0 = 4

即对应数字权限 `754`。

### 5.2 chmod 755 怎么算

`755 = 7|5|5`：

- 第 1 位（用户）7 = rwx
- 第 2 位（组）5 = r-x
- 第 3 位（其他）5 = r-x

命令示例：

```bash
chmod 755 script.sh
```

### 5.3 chown 是干什么的

`chown` 用于改文件所有者和所属组。

```bash
chown alice file.txt
chown alice:dev file.txt
chown -R alice:dev /opt/myapp
```

常见场景：

- 部署后文件归属不对导致“权限拒绝”
- 服务进程需要拥有目录写权限

## 六、软链接 vs 硬链接

### 6.1 ln 和 ln -s

- `ln 源 目标`：硬链接（hard link）
- `ln -s 源 目标`：软链接（symbolic link）

### 6.2 核心区别

| 对比项 | 硬链接 | 软链接 |
|--------|--------|--------|
| 本质 | 同一 inode 的另一个名字 | 指向路径的“快捷方式” |
| 跨文件系统 | 不支持 | 支持 |
| 指向目录 | 一般不允许 | 允许 |
| 源文件删除后 | 仍可访问数据（只要还有硬链接） | 链接失效（悬挂链接） |

### 6.3 为什么 /bin 常是 /usr/bin 的符号链接

很多现代 Linux 采用 `/usr` 合并（usr merge）：

- 把历史上的 `/bin`、`/sbin` 等与 `/usr/bin`、`/usr/sbin` 统一
- 用符号链接保持兼容

所以你看到 `/bin -> /usr/bin` 是正常且常见的。

## 七、磁盘空间管理（高频操作）

这部分是日常 Linux 使用最常用的命令之一。

### 7.1 看分区使用情况：df -h

```bash
df -h
```

你会看到每个挂载点的总量、已用、可用、使用率。

常见排查：

- 某分区使用率 100%
- 根分区 `/` 爆满导致系统异常

### 7.2 看目录谁最占空间：du -sh *

```bash
du -sh *
```

作用：查看当前目录下各子目录大小，快速定位“大户”。

常用组合：

```bash
cd /var
du -sh *
```

可快速发现是日志、缓存还是临时文件占满磁盘。
