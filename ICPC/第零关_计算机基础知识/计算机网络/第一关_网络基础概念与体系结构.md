# 第一关：网络基础概念与体系结构

---

## 一、为什么需要"协议"？

### 1.1 从人类沟通说起

想象你和一个只会说法语的人面对面站着，你想告诉他"明天下午三点在咖啡厅见面"。

你说中文，他说法语，完全聊不起来。

要解决这个问题，你们必须**事先约定一套规则**：

- 用什么语言？
- 怎么表达时间？
- 先挥手表示"我要开始说话了"，说完点头表示"我说完了"

**这套事先约定好的规则，就是"协议"（Protocol）。**

### 1.2 计算机之间也一样

两台电脑想交流，同样需要事先约定好：

| 需要约定的事 | 类比 |
|-------------|------|
| 数据用什么格式？ | 说中文还是英文？ |
| 谁先说话？ | 打电话时，接听方先说"喂" |
| 怎么知道对方收到了？ | 你说完一句话，对方点头表示听到了 |
| 出错了怎么办？ | 听不清就说"请再说一遍" |

### 1.3 核心结论

> **网络协议 = 计算机之间通信必须遵守的规则集合。**
>
> 没有协议，两台电脑就像说不同语言的两个人，完全无法沟通。

互联网上有很多不同的协议（IP、TCP、HTTP、DNS 等），各管各的事。**协议是网络世界的"共同语言"。**

## 二、为什么要"分层"？

### 2.1 问题：要约定的事太多了

你在浏览器里打开一个网页，背后需要解决一连串问题：

1. 你的电脑网线/WiFi 信号怎么传出去？（电信号怎么走？）
2. 数据怎么从你家路由器跳到隔壁小区的路由器，再跳到服务器？（路怎么找？）
3. 数据万一丢了怎么办？顺序乱了怎么办？（可靠性怎么保证？）
4. 服务器怎么知道你要的是网页而不是邮件？（内容怎么解读？）

如果把所有规则写成**一个巨大的协议**，就像一个人同时当厨师、服务员、收银员、采购员、清洁工——什么都干，什么都管，改任何一件小事都可能把其他事搞崩。

### 2.2 解决思路：分工

> **分层 = 把复杂的大问题拆成几个独立的小问题，每层只负责一件事。**

**类比：快递寄信**

```
你（寄件人）
  写好信的内容，装进信封           ← 你只管"写什么"

快递公司前台
  贴上收件地址、寄件地址           ← 只管"寄给谁"

分拣中心
  决定走空运还是陆运、经过哪些中转站  ← 只管"走哪条路"

运输车/飞机
  把包裹从A点物理搬到B点            ← 只管"怎么搬"
```

你写信的时候，需要知道快递公司内部怎么分拣吗？**不需要。**
分拣员需要知道你信里写了什么吗？**也不需要。**

**每一层只做自己的事，只跟相邻层打交道。**

### 2.3 分层最大的好处

> **改动一层不影响其他层。**

比如你从有线网换成 WiFi（最底层变了），但你打开网页的方式完全不用变（上面几层完全无感）。

## 三、五层模型——网络的"楼层结构"

### 3.1 为什么用五层？

你可能听过"OSI 七层模型"和"TCP/IP 四层模型"。实际学习中最常用的是**五层模型**——它取了 OSI 的清晰分层，又贴合 TCP/IP 的实际实现，最适合学习。

### 3.2 五层结构一览

```
第5层  应用层        ← 你写的信件内容（HTTP、DNS、SMTP…）
第4层  传输层        ← 快递的"服务等级"：挂号信 or 普通信（TCP、UDP）
第3层  网络层        ← 快递公司规划从北京到纽约走哪条路（IP）
第2层  数据链路层    ← 同一条街上快递小哥门对门送（以太网、WiFi）
第1层  物理层        ← 实际的公路、铁路、航线（网线、光纤、无线电波）
```

### 3.3 每层干什么？

**第5层 · 应用层——"说什么"**

用户能直接感知到的层。浏览器请求网页用 HTTP，查域名用 DNS，发邮件用 SMTP。解决的是"这次通信的具体内容和目的是什么"。

**第4层 · 传输层——"怎么保证送到"**

数据可能很大，需要拆成小块发出去。传输层负责拆分、编号，确保对方按顺序完整收到。TCP 像挂号信（丢了会重发），UDP 像撒传单（快但不保证送达）。

**第3层 · 网络层——"走哪条路"**

互联网上有无数台设备，数据从你的电脑到目标服务器可能经过十几个路由器。网络层负责规划路径，核心协议是 IP。IP 地址就是每台设备在网络上的"门牌号"。

**第2层 · 数据链路层——"隔壁邻居怎么递"**

网络层规划了整条路线，但每一小段（比如从你的电脑到你家路由器）怎么传？这是数据链路层的事。它管的是"在同一个局域网内，数据怎么从一个设备到相邻的下一个设备"。

**第1层 · 物理层——"信号怎么传"**

最底层。数据最终要变成电信号、光信号或无线电波才能传出去。网线里的电脉冲、光纤里的光、WiFi 的无线电波，都是物理层的事。

### 3.4 与 OSI 七层、TCP/IP 四层的对照

```
OSI 七层          五层模型         TCP/IP 四层
─────────        ─────────       ─────────
应用层  ┐
表示层  ├──→     应用层      ←──  应用层
会话层  ┘
传输层  ───→     传输层      ←──  传输层
网络层  ───→     网络层      ←──  网络层（网际层）
数据链路层 ─→    数据链路层   ┐
物理层  ───→     物理层      ┘←── 网络接口层
```

- OSI 把应用层细分成了三层（应用、表示、会话）
- TCP/IP 把最下面两层合并成一层（网络接口层）
- 五层模型取了中间值，**学习时记住五层就够了**

## 四、封装与解封装——数据的"套信封"之旅

### 4.1 核心问题

你在浏览器里输入了一个网址，这个请求是怎么从第5层一路跑到第1层发出去，到了对方又怎么一路还原回来的？

答案：**封装**（发送端从上往下，逐层套信封）和 **解封装**（接收端从下往上，逐层拆信封）。

### 4.2 封装过程：发送端（从上往下）

**类比：俄罗斯套娃式寄信**

```
第1步（应用层）：你写好信的内容
       "周六下午3点视频聊天"
       这就是你的原始数据

第2步（传输层）：把信装进一个信封，写上"这是第1封/共1封"
       加上端口号、序列号等信息
       → [传输层信封 [你的信]]

第3步（网络层）：再装进一个大信封，写上寄件地址和收件地址
       加上源 IP 和目标 IP
       → [网络层信封 [传输层信封 [你的信]]]

第4步（数据链路层）：最外面贴上面单，写上"从你家MAC到路由器MAC"
       加上 MAC 地址等信息
       → [链路层信封 [网络层信封 [传输层信封 [你的信]]]]

第5步（物理层）：包裹被放上卡车，变成实际在路上跑的物理信号
       变成电信号/光信号发出去
```

> **每经过一层，就在外面多套一层"信封"，每层的信封里写的都是该层需要的控制信息。**

### 4.3 解封装过程：接收端（从下往上）

```
物理层收到信号       → 还原成比特流
数据链路层拆掉面单   → 取出里面的包裹
网络层拆掉地址信封   → 取出里面的信封
传输层拆掉序号信封   → 取出里面的信
应用层读到信的内容   → "周六下午3点视频聊天"
```

> **一层一层拆信封，每层只看自己那层的信息，拆完就往上传。**

### 4.4 每层数据的专业名称

每一层给数据加上的"信封"，专业名称叫**头部（Header）**。加上头部后的数据在每层有不同的叫法：

```
应用层产出：                            消息（Message）
传输层加上传输头部后叫：                 段（Segment）
网络层加上网络头部后叫：                 包（Packet）
数据链路层加上链路头部和尾部后叫：       帧（Frame）
物理层转换成：                          比特流（Bits）
```

> 不需要现在死记这些名字，后面每一层深入学的时候都会再碰到。

### 4.5 核心结论

> **发送数据 = 从上到下逐层套信封。**
> **接收数据 = 从下到上逐层拆信封。**
> **每层只关心自己那层信封上的信息。**

这也解释了为什么分层设计好用——每层只需要读自己的"信封"，完全不用管里面嵌套了什么东西。

## 五、第一关知识总结

```
计算机网络基础概念
├── 协议（Protocol）
│   └── 计算机之间通信必须遵守的规则集合
│       类比：两个人交流时约定好的共同语言
│
├── 分层（Layering）
│   └── 把复杂通信问题拆成独立的小问题，每层各管一件事
│       好处：改动一层不影响其他层
│       类比：快递系统的分工（写信、填地址、分拣、运输各管各的）
│
├── 五层模型
│   ├── 第5层 应用层    → "说什么"（HTTP、DNS、SMTP）
│   ├── 第4层 传输层    → "怎么保证送到"（TCP、UDP）
│   ├── 第3层 网络层    → "走哪条路"（IP）
│   ├── 第2层 数据链路层 → "隔壁邻居怎么递"（以太网、WiFi）
│   └── 第1层 物理层    → "信号怎么传"（电信号、光信号、无线电波）
│
└── 封装与解封装
    ├── 封装：发送时从上往下逐层套信封（加头部）
    ├── 解封装：接收时从下往上逐层拆信封（去头部）
    └── 每层数据名称：消息 → 段 → 包 → 帧 → 比特流
```
