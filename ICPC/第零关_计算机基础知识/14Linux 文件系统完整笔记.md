# Linux 文件系统完整笔记

---

## 一、总体结构

Linux 文件系统以 `/`（根目录）为起点，所有文件和目录都从这里展开。无论是哪个 Linux 发行版（Ubuntu、CentOS、Debian 等），根目录下的结构基本一致。

> **类比理解：**
>
> Linux 文件系统就像一棵倒置的树——根（`/`）在最顶端，所有的"枝干"（目录）和"叶子"（文件）都从根生长出来。

```
/
├── bin        可执行命令（基础）
├── sbin       系统管理命令
├── boot       启动文件
├── dev        设备文件
├── etc        配置文件
├── home       普通用户家目录
├── lib        共享库文件
├── media      自动挂载设备
├── mnt        手动挂载设备
├── opt        可选软件包
├── proc       进程和内核信息（内存虚拟）
├── root       root 用户家目录
├── run        运行时信息（内存）
├── srv        服务数据
├── sys        系统硬件信息（内存）
├── tmp        临时文件
├── usr        用户程序和资源
└── var        日志等可变数据
```

---

## 二、各目录详解

### 2.1 `/bin` 和 `/sbin` — 可执行命令

| 目录      | 全称          | 说明                                | 典型命令                           |
| --------- | ------------- | ----------------------------------- | ---------------------------------- |
| `/bin`  | Binary        | 所有用户都能用的基础命令            | `cp`、`cat`、`chmod`、`ls` |
| `/sbin` | System Binary | 系统管理命令，修改配置需要 `sudo` | `ifconfig`、`iptables`         |

> ⚠️ **注意：** Linux 不根据文件后缀判断文件类型，`/bin` 中的文件虽然没有 `.exe` 后缀，但都是可执行的二进制文件。

**举例：**

```bash
ifconfig          # 普通用户可用（查看网卡状态）
sudo ifconfig ... # 需要授权（修改网卡配置）
```

---

### 2.2 `/boot` — 系统启动文件

存放系统启动所需的文件，包括常见的开机引导程序 `grub`。

> ⚠️ **不要随意修改这里的文件，否则系统可能无法启动。**

---

### 2.3 `/dev` — 设备文件

Linux 的核心哲学之一： **一切皆文件** ，包括硬件设备。

```
/dev/sda     → 第一块硬盘
/dev/sda1    → 第一块硬盘的第一个分区
/dev/sdb     → 第二块硬盘
```

> **类比理解：**
>
> 就像 Windows 中硬盘显示为 `C:` 盘、`D:` 盘，Linux 把硬盘显示为 `/dev/sda`、`/dev/sdb`，鼠标、键盘等设备也在这里以文件形式存在。

---

### 2.4 `/etc` — 配置文件 ⭐（常用）

存放几乎所有系统程序和服务的配置文件。

```
/etc/apt/          → apt 包管理器的配置（如镜像源列表）
/etc/hosts         → 本地 DNS 映射
/etc/ssh/          → SSH 服务配置
/etc/nginx/        → Nginx Web 服务器配置
```

> 💡 **记住：** 想修改某个系统程序的配置，十有八九在 `/etc` 下找。

---

### 2.5 `/lib` — 共享库文件

类似 Windows 中的 `.dll` 文件，是 `/bin` 和 `/sbin` 中命令运行所依赖的库文件。

* `/lib` → 32 位或通用库
* `/lib32` → 32 位库
* `/lib64` → 64 位库

---

### 2.6 `/media` 和 `/mnt` — 设备挂载

| 目录       | 说明                                                      |
| ---------- | --------------------------------------------------------- |
| `/media` | 系统**自动**挂载设备的地方（如 U 盘插入后自动出现） |
| `/mnt`   | 用户**手动**挂载设备的地方，通常为空                |

**举例：**

```
插入 U 盘后，系统自动挂载到：
/media/your_username/USB_NAME/
```

> **类比理解：**
>
> `/media` 像是机场的自动传送带——设备插入后自动送到你面前；`/mnt` 像是手推车——需要你自己去推。

---

### 2.7 `/opt` — 可选软件包

用于存放自行下载安装的软件（非包管理器安装的）。

```
/opt/google/chrome/    → Chrome 浏览器
/opt/netease/          → 网易云音乐
/opt/clion/            → CLion IDE
```

---

### 2.8 `/proc` — 进程和内核信息（虚拟文件系统）

存放所有**正在运行的进程**的状态信息，以及系统和内核信息。

> ⚠️ 这里的文件 **并不是真实存储在磁盘上的文件** ，而是内核动态生成的，是程序和内核交流的接口。

```
/proc/1234/        → PID 为 1234 的进程信息
/proc/version      → 当前操作系统版本
/proc/cpuinfo      → CPU 信息
```

**举例：**

```bash
cat /proc/version    # 查看系统版本
cat /proc/cpuinfo    # 查看 CPU 状态
```

---

### 2.9 `/root` — root 用户的家目录

这是超级用户 `root` 专属的家目录，普通用户需要授权才能访问。

> 注意区分：`/root` 是 root 用户的家目录，`/` 才是根目录，两者不同。

---

### 2.10 `/run` 和 `/sys` — 运行时信息

| 目录     | 说明                     |
| -------- | ------------------------ |
| `/run` | 程序运行时产生的临时信息 |
| `/sys` | 系统硬件相关信息         |

> ⚠️ **重要：** 这两个目录的数据存储在**内存**中，重启后数据全部丢失，不要在这里存放文件。

**有趣的例子：** 修改屏幕亮度

```bash
# 文件里存着一个数字，就是当前亮度值
sudo vim /sys/devices/pci0000:00/0000:00:02.0/drm/card0/card0-eDP-1/intel_backlight/brightness
```

---

### 2.11 `/srv` — 服务数据

`srv` = service，存放 Web 服务或 FTP 服务的数据文件。

* 桌面版 Linux：通常为空
* 服务器：Web 服务、FTP 服务的资源可放在这里

---

### 2.12 `/tmp` — 临时文件

程序运行时产生的临时文件存放于此。

> 💡 **实用小技巧：** 如果文本编辑器意外崩溃，可以去 `/tmp` 里找找临时文件，也许能抢救回数据。

> ⚠️ 系统重启后 `/tmp` 会被自动清空。如果重启后没有清空，可能需要手动删除。

---

### 2.13 `/usr` — 用户程序和资源

`usr` = Universal System Resource，存放非系统必须的用户程序和资源。

```
/usr/bin/          → 用户安装的应用程序命令（如 Chrome、vim）
/usr/sbin/         → 用户级系统管理命令
/usr/local/        → 本地安装的软件（优先级高于 /usr）
/usr/local/bin/    → 本地安装的命令
/usr/share/        → 共享数据（文档、图标等）
/usr/share/applications/  → 桌面图标配置文件（.desktop 文件）
```

> **类比理解：**
>
> `/bin` 是出厂预装的系统工具（类似手机预装 App），`/usr/bin` 是用户后来安装的软件（类似自己下载的 App）。

**找不到桌面快捷方式？**

```bash
# 在这里手动创建 .desktop 文件
/usr/share/applications/
```

---

### 2.14 `/var` — 可变数据（主要是日志）

`var` = variable，最主要的用途是 **存储日志信息** 。

```
/var/log/          → 日志文件
/var/log/syslog    → 系统日志
/var/log/nginx/    → Nginx 访问日志
```

> ⚠️ 日志文件不会自动删除，随着使用时间增长，`/var` 会越来越大，需要适时清理。

---

### 2.15 `/home` — 普通用户家目录 ⭐（常用）

每个普通用户都有一个 `/home/用户名/` 目录，存放个人文件。

```
/home/alice/
├── 下载/
├── 桌面/
├── .cache/          → 应用缓存
├── .config/         → 应用配置（如 Chrome 配置）
├── .local/
│   └── bin/         → 该用户专属的可执行文件
└── .IntelliJIdea/   → 部分应用自建的隐藏配置目录
```

> 💡 Linux 中以 `.` 开头的文件/目录是 **隐藏文件** ，使用 `ls -a` 才能看到。

#### 关于 `~/.local/bin` 的特别说明

一个重要但容易被忽视的细节：存放在 `~/.local/bin` 中的命令只对该用户可用。

**举例：** 用 `pip` 安装的命令行工具通常装在 `~/.local/bin/`

```bash
普通用户：pip_tool    ✅ 可以直接用（在 PATH 中）
sudo 执行：sudo pip_tool  ❌ 找不到命令

解决方法：写全绝对路径
sudo /home/alice/.local/bin/pip_tool
```

## 三、快速查找指南

### 找配置文件

```
系统程序的配置   → /etc/程序名/
用户程序的配置   → ~/.config/程序名/ 或 ~/.程序名/
```

### 找可执行命令

命令的可执行文件可能在以下位置之一：

```
/bin                    系统基础命令
/sbin                   系统管理命令
/usr/bin                用户安装的命令
/usr/sbin               用户级系统管理命令
/usr/local/bin          本地安装的命令
/usr/local/sbin         本地安装的系统命令
~/.local/bin            当前用户专属命令
~/.local/sbin           当前用户专属系统命令
```

> 💡 **让自己的脚本全局可用：** 把脚本复制到上述任意目录，或在 PATH 中添加脚本所在目录。

### 排查程序问题

```
程序崩溃、报错信息  → /var/log/
临时文件残留        → /tmp/
进程运行状态        → /proc/
```

### 访问外部设备

```
U 盘/移动硬盘（自动挂载）  → /media/用户名/
手动挂载设备              → /mnt/
所有设备文件              → /dev/
```
