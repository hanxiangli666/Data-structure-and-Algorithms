队列 & 优先队列
01

什么是队列？

从地铁安检口到内存里的数据结构——队列的核心思想只有四个字。

你每天都在用队列

想象地铁安检口：先到的人先通过，后到的人排在后面等。先进先出，绝不插队。

计算机里的队列完全相同——它是一条有序的通道，数据从一端进入，从另一端离开。

🚇
安检口
最先排队的人最先过安检，不管你多着急。
🖨️
打印机
第一个发送的文件第一个被打印，后来的文件等着。
💻
BFS 搜索
ICPC 里最常用的场景：广度优先搜索每一层都靠队列维护顺序。
📡
网络数据包
路由器把每个数据包按到达顺序排成队列，依次转发。

队列 vs 栈：方向不同，用途不同

队列和栈是算法里最常见的两对搭档，区别只在于「谁先出来」。

数据结构 规则 C++ 类型 典型用途
queue 先进先出 FIFO queue<T> BFS、任务调度
stack 后进先出 LIFO stack<T> DFS、括号匹配
priority_queue 按优先级出队 priority_queue<T> Dijkstra、贪心
💡 学这两个文件能干什么？

queue.cpp 演示了普通队列的全部操作。priority_queue.cpp 演示了优先队列（最大堆和最小堆）。这两个数据结构覆盖了 ICPC 里绝大多数需要有序队列的题型。

02

queue 的全部操作

六个函数，一个陷阱——把 queue.cpp 的每一行读懂。

代码逐行翻译

STL 的 queue 有且只有六个常用操作。来看真实代码：

CODE 
queue<int> q;

q.push(10);
q.push(20);
q.push(30);
// 队列：10, 20, 30

cout << q.size();   // 3
cout << q.empty();  // 0 (false)
cout << q.front();  // 10
cout << q.back();   // 30
PLAIN ENGLISH

创建一个存整数的空队列，起名叫 q。

把 10 加到队列的末尾（排队）。

再把 20 排进去，紧跟在 10 后面。

再把 30 排进去——队列现在是 10→20→30。

问：现在有多少人在排队？答：3 个。

问：队伍是空的吗？答：0 = false = 不空。

看队伍最前面是谁（不让他走），答：10。

看队伍最后面是谁（不动他），答：30。

⚠️ 最容易踩的坑：pop() 不返回值

C++ 的 pop() 只负责删除，不返回被删的元素。这和 Python 的 popleft() 完全不同。

CODE 
// ❌ 错误写法（会直接丢失数据）
int val = q.pop();  // 编译报错！

// ✅ 正确写法
int val = q.front();  // 先取出 10
q.pop();              // 再删除
cout << val;            // 10
cout << q.front();    // 20（10 已走）
PLAIN ENGLISH

试图用 pop() 取值——这行代码根本无法编译，因为 pop() 没有返回值。

————

正确做法第一步：先用 front() 把队头的值 10 存进 val。

第二步：再调用 pop() 把 10 从队列里移除。

现在 val 里存着 10，可以安全使用。

queue 的下一个队头变成了 20。

🤔 为什么这样设计？

历史原因：如果 pop() 在抛出异常时同时返回值，会导致数据丢失。分开成两步（先 front() 读，再 pop() 删）在异常安全上更可靠。在竞赛里记住：先 front()，再 pop()，两步走。

清空队列：没有 clear()，只能 while 循环

CODE 
// queue 没有 clear() 方法！
while (!q.empty()) {
    q.pop();
}

if (q.empty()) {
    cout << "queue 已空\n";
}
PLAIN ENGLISH

C++ 的 queue 没有一键清空的 clear()——这是故意的设计选择。

用 while 循环：只要队列不空，就一直 pop()。

直到每个元素都被弹出，循环自然结束。

——

验证：如果 empty() 返回 true，说明清空成功。

打印确认信息。

✅ 更快的清空技巧（竞赛常用）

与其 while 循环，更常见的竞赛写法是直接用赋值覆盖：q = queue<int>();——用一个新的空队列替换旧的，O(1) 时间，一行搞定。

03

优先队列：有特权的队列

不是先来先得，而是「最重要的先处理」——这就是堆的哲学。

急诊室，不是地铁站

普通队列就像地铁安检：先来先过，公平无歧视。但急诊室不同——不管你几点挂号，病情最危急的人先被处理。

优先队列就是急诊室：每次 pop，出来的都是当前「最重要」的元素（默认是最大值）。

🏥
急诊优先
危重病人优先处理，对应「最大堆」——数值最大的先出。
🗓️
任务调度
截止时间最近的任务先处理，对应「最小堆」——数值最小的先出。
🗺️
Dijkstra 算法
每次从距离最小的节点出发，用最小堆高效实现。
🤑
贪心策略
总是选当前最优的选项——用最大堆快速找到「当前最大」。

堆长什么样？

堆是 priority_queue 的底层结构。它是一棵完全二叉树，规则是：父节点的值永远 ≥ 子节点（最大堆）。

priority_queue 代码翻译

CODE 
// 最大堆（默认）：值最大的先出
priority_queue<int> pq;

pq.push(30);
pq.push(10);
pq.push(50);
pq.push(20);

cout << pq.top();   // 50

int val = pq.top();
pq.pop();
cout << pq.top();  // 30
PLAIN ENGLISH

创建一个最大堆。不写额外参数，默认就是最大堆。

依次加入 30、10、50、20 四个数。

堆内部自动重排，把最大值 50 放到顶部。

进入的顺序无所谓——每次都是最大值在上面。

——

top() 查看堆顶，答：50（最大值），不删除。

先用 top() 把 50 存到 val，再 pop() 删除它。

50 走了，下一个最大值 30 浮到顶部。

💡 queue vs priority_queue：接口差异

注意：queue 用 front() 看队头；priority_queue 用 top() 看堆顶。别混了！两者都用 push() 和 pop()，这是相同的。

最小堆：反转优先级

三个模板参数，一行代码反转世界——以及什么时候该用最小堆。

那三个参数是什么意思？

最小堆的声明方式是竞赛里公认的「背下来」写法：

CODE 
// 最小堆：值最小的先出
priority_queue<
  int,
  vector<int>,
  greater<int>
> pq2;

pq2.push(30);
pq2.push(10);
pq2.push(50);
pq2.push(20);

cout << pq2.top();   // 10！最小的
pq2.pop();
cout << pq2.top();   // 20
PLAIN ENGLISH

声明一个最小堆，需要写三个模板参数：

第 1 个：int — 存什么类型的数据。

第 2 个：vector<int> — 底层用什么容器存（固定写 vector<T>）。

第 3 个：greater<int> — 比较器，告诉堆「较大的优先级较低」，从而反转成最小堆。

——

加入 30、10、50、20，堆把 10 推到顶部。

top() 返回 10（最小值）。pop() 删除后，20 变成堆顶。

最大堆 vs 最小堆：直观对比

同样加入 {30, 10, 50, 20}，两种堆的出队顺序完全相反：

最大堆
priority_queue<int>
1 先出：50
2 其次：30
3 然后：20
4 最后：10
最小堆
priority_queue<int,vector<int>,greater<int>>
1 先出：10
2 其次：20
3 然后：30
4 最后：50
🎯 竞赛速记口诀

默认 = 最大堆（大的先出）。
加 greater<T> = 最小堆（小的先出）。
Dijkstra、最短路 → 最小堆。贪心取最大 → 最大堆。

看题目，选结构

?
题目要求「每次取最大值」
→ 直接用 priority_queue<int>，默认最大堆，top() 就是最大值。
?
题目要求「每次取最小值」
→ 用 priority_queue<int, vector<int>, greater<int>>，最小堆。
?
BFS（广度优先搜索）
→ 用普通 queue，不需要优先级，只需要 FIFO 顺序。
?
Dijkstra 最短路
→ 用最小堆，每次弹出距离最小的节点扩展，O((V+E) log V)。
